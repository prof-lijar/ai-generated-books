# Chapter 12: File I/O — ဖိုင်များနှင့် အလုပ်လုပ်ခြင်း

## ၁။ နိဒါန်းနှင့် အရေးပါပုံ (Introduction and Necessity)

ကွန်ပျူတာပရိုဂရမ်တစ်ခုကို ရေးသားပြီး အဲ့ဒီပရိုဂရမ်ကို run ပြီးတာနဲ့ အဲ့ဒီပရိုဂရမ်က လုပ်ဆောင်ခဲ့တဲ့ အချက်အလက်တွေ၊ အခြေအနေတွေဟာ ကွန်ပျူတာရဲ့ RAM (Random Access Memory) ထဲမှာပဲ ခေတ္တနေပြီး အဲ့ဒီပရိုဂရမ်ကို ပိတ်လိုက်တာနဲ့ အဲ့ဒီအချက်အလက်တွေဟာ ပျောက်ကွယ်သွားတတ်ပါတယ်။ ဒါကြောင့် ကျွန်ုပ်တို့ရဲ့ ပရိုဂရမ်တွေဟာ အချိန်ကြာရှည်စွာ အကျိုးရှိစေဖို့၊ အချက်အလက်တွေကို ထိန်းသိမ်းနိုင်ဖို့ လိုအပ်လာပါတယ်။ ဒီနေရာမှာ File I/O (File Input/Output) ဆိုတာ အလွန်အရေးကြီးတဲ့ အခန်းကဏ္ဍကနေ ပါဝင်လာပါတယ်။ File I/O ဆိုသည်မှာ ကွန်ပျူတာရဲ့ Storage device များဖြစ်တဲ့ Hard Disk, SSD စတာတွေပေါ်ရှိ ဖိုင် (File) တွေနဲ့ ပရိုဂရမ်တွေကြားမှာ အချက်အလက်တွေကို ဖလှယ်ခြင်း၊ သိမ်းဆည်းခြင်းနဲ့ ထုတ်ပေးခြင်းဆိုင်ရာ လုပ်ငန်းစဉ်များလို့ ဆိုနိုင်ပါတယ်။

### File I/O လိုအပ်ရခြင်း အကြောင်းရင်း

File I/O လိုအပ်ရခြင်းရဲ့ အဓိက အကြောင်းရင်းတွေကတော့ အချက်အလက်တွေကို ထိန်းသိမ်းနိုင်ဖို့နဲ့ လိုအပ်တဲ့အခါ ပြန်လည်ရယူနိုင်ဖို့ ဖြစ်ပါတယ်။

ပထမအချက်ကတော့ **Data Persistence (အချက်အလက် တည်မြဲမှု)** ဖြစ်ပါတယ်။ ပရိုဂရမ်တစ်ခုကို run လုပ်ပြီး အဲ့ဒီပရိုဂရမ်ကို ပိတ်လိုက်တဲ့အခါ RAM ထဲက အချက်အလက်တွေဟာ ပျောက်သွားပါတယ်။ ဒါပေမယ့် ကျွန်ုပ်တို့ရဲ့ အရေးကြီးတဲ့ အချက်အလက်တွေ (ဥပမာ- ကျောင်းသားမှတ်တမ်းများ၊ စာရင်းအင်းများ) ကို ကွန်ပျူတာကို ပိတ်လိုက်ရင်တောင် သိမ်းဆည်းထားနိုင်ဖို့ လိုအပ်ပါတယ်။ File I/O ဟာ ဒီလို အချက်အလက်တွေကို Disk ပေါ်မှာ အမြဲတမ်း သိမ်းဆည်းနိုင်တဲ့ နည်းလမ်းကို ပေးစွမ်းပါတယ်။

ဒုတိယအချက်ကတော့ **အချိန်ကြာရှည်စွာ သိမ်းဆည်းခြင်းနှင့် ပြန်လည်ရယူခြင်း** ဖြစ်ပါတယ်။ Data တွေကို ကွန်ပျူတာရဲ့ RAM ထက် ပိုမိုကြီးမားတဲ့ Storage နေရာတွေမှာ သိမ်းဆည်းပြီး၊ လိုအပ်တဲ့အချိန်မှာ ပြန်လည်ဖတ်ယူနိုင်ဖို့ File System တွေက အထောက်အကူပြုပါတယ်။ ဒါဟာ ကျွန်ုပ်တို့ရဲ့ ဒေတာတွေကို အချိန်အကြာကြီး ထိန်းသိမ်းနိုင်ဖို့ အခြေခံကျတဲ့ နည်းလမ်းဖြစ်ပါတယ်။

တတိယအချက်ကတော့ **အသုံးပြုသူများထံသို့ အချက်အလက်များ ထုတ်ပေးခြင်းနှင့် လက်ခံခြင်း** ဖြစ်ပါတယ်။ ကျွန်ုပ်တို့ရဲ့ ပရိုဂရမ်တွေဟာ Input (ထည့်သွင်းမှု) နဲ့ Output (ထုတ်ပေးမှု) တွေ လုပ်ဆောင်ရပါတယ်။ စာသားတွေ၊ ဂဏန်းတွေ၊ ပုံတွေ စတဲ့ အချက်အလက်တွေကို ဖန်တီးပြီး ဖိုင်ထဲသို့ ရေးတာ (Output)၊ ဖိုင်ထဲမှ အချက်အလက်တွေကို ဖတ်တာ (Input) တွေဟာ File I/O ရဲ့ အဓိက လုပ်ဆောင်ချက်တွေထဲမှာ ပါဝင်ပါတယ်။

## ၂။ File Handling ၏ အခြေခံများ (Fundamentals of File Handling)

File I/O ကို C Language မှာ အသုံးပြုတဲ့အခါ အခြေခံကျတဲ့ အယူအဆတွေကို နားလည်ဖို့ လိုအပ်ပါတယ်။ အဲ့ဒီအခြေခံတွေကတော့ File Pointer နဲ့ File ကို ဖွင့်ခြင်း၊ ပိတ်ခြင်းတို့ပဲ ဖြစ်ပါတယ်။

### File Pointer (FILE *) နှင့် fopen() စတင်ခြင်း

C Language မှာ File I/O ကို လုပ်ဆောင်ဖို့အတွက် အထူးသဖြင့် `stdio.h` (Standard Input/Output) header file ထဲက အချက်အလက်တွေကို အသုံးပြုရပါတယ်။ File I/O အတွက် အဓိက အသုံးပြုတဲ့ အရာကတော့ `FILE *` (File Pointer) ဆိုတဲ့ data type ဖြစ်ပါတယ်။ File Pointer ဆိုတာ ဖိုင်တစ်ခုကို ရည်ညွှန်းပေးတဲ့ အမှတ်အသား (Address) တစ်ခုလိုမျိုး ဖြစ်ပါတယ်။ ကျွန်ုပ်တို့ရဲ့ Memory ကို ဗီရိုအံဆွဲနဲ့ နှိုင်းယှဉ်ရင် File Pointer ဟာ ဖိုင်ထဲက အချက်အလက်တွေ ဘယ်နေရာမှာ ရှိတယ်ဆိုတဲ့ လိပ်စာကို သိမ်းထားတဲ့ Pointer တစ်ခုနဲ့ ဆင်တူပါတယ်။

File ကို စတင်ဖွင့်ဖို့အတွက် `fopen()` function ကို အသုံးပြုရပါတယ်။ ဒီ function က ဖိုင်တစ်ခုကို ဖွင့်ပြီး အဲ့ဒီဖိုင်ကို ရည်ညွှန်းပေးတဲ့ `FILE *` pointer ကို ပြန်ပေးပါတယ်။

`fopen()` function ရဲ့ အရေးပါပုံကတော့ ဖိုင်အမျိုးအစားနဲ့ အသုံးပြုမယ့် လုပ်ဆောင်ချက်ပေါ် မူတည်ပြီး ဖိုင်ကို ဘယ်လိုဖွင့်မလဲဆိုတာကို ဆုံးဖြတ်ပေးပါတယ်။ `fopen()` function ကို အသုံးပြုတဲ့အခါ ဖိုင်ရဲ့ လမ်းကြောင်း (filename) နဲ့ ဖွင့်မယ့် Mode (ပုံစံ) ကို ထည့်သွင်းပေးရပါတယ်။

#### File Mode များ (Modes): `r`, `w`, `a`, `rb`, `wb` တို့၏ ကွာခြားချက်များနှင့် လက်တွေ့အသုံးပြုပုံ

`fopen()` function မှာ အသုံးပြုမယ့် Mode တွေက File I/O ရဲ့ အခြေခံအကျဆုံး အစိတ်အပိုင်းတွေ ဖြစ်ပါတယ်။ ဒီ Mode တွေဟာ ဖိုင်ကို ဘယ်လိုပုံစံနဲ့ အသုံးပြုမလဲဆိုတာကို သတ်မှတ်ပေးပါတယ်။

၁။ **`r` (read - ဖတ်ရန်)**: ဒီ mode ကို အသုံးပြုတဲ့အခါ ဖိုင်ကို ဖတ်ဖို့ (reading) ရည်ရွယ်ပါတယ်။ ဖိုင်ထဲက အချက်အလက်တွေကို စာသားအဖြစ် ဖတ်နိုင်ပါတယ်။ ဖိုင်မရှိရင် ဒါမှမဟုတ် ဖိုင်ကို ဖတ်ဖို့သက်သက်သာ လိုရင် ဒီ mode ကို သုံးပါတယ်။ ဖိုင်မရှိရင် ဒါမှမဟုတ် ဖိုင်အစကို ပြန်စဖို့အတွက် သင့်တော်ပါတယ်။

၂။ **`w` (write - ရေးရန်)**: ဒီ mode ကို အသုံးပြုတဲ့အခါ ဖိုင်ထဲကို စာသားအဖြစ် ရေးဖို့ (writing) ရည်ရွယ်ပါတယ်။ သို့သော် `w` mode ကို သုံးလိုက်တဲ့အခါ ဖိုင်ထဲမှာ ရှိပြီးသား အချက်အလက်အားလုံးကို **ဖျက်ပစ်ပြီး** အသစ်ပြန်ရေးပါလိမ့်မယ်။ ဖိုင်မရှိရင်တော့ ဖန်တီးပေးပါလိမ့်မယ်။

၃။ **`a` (append - ထပ်ထည့်ရန်)**: ဒီ mode ကို အသုံးပြုတဲ့အခါ ဖိုင်ရဲ့ အဆုံးမှာ အချက်အလက်အသစ်တွေကို ထပ်ထည့်ဖို့ (appending) ရည်ရွယ်ပါတယ်။ ဖိုင်ထဲမှာ ရှိပြီးသား အချက်အလက်တွေကို မဖျက်ဘဲ နောက်ဆုံးမှာ ထပ်ထည့်ပေးပါတယ်။

၄။ **`rb` (read binary - Binary ဖတ်ရန်)**: ဒီ mode ကို Binary Data တွေကို ဖတ်ဖို့ (reading) ရည်ရွယ်ပါတယ်။ Binary Data ဆိုတာက စာသားမဟုတ်ဘဲ byte အတိုင်း သိမ်းဆည်းထားတဲ့ Data တွေ ဖြစ်ပါတယ်။ ဒီ mode ကို သုံးတဲ့အခါ `fread()` လို Binary I/O functions တွေနဲ့ အသုံးပြုရပါမယ်။

၅။ **`wb` (write binary - Binary ရေးရန်)**: ဒီ mode ကို အသုံးပြုတဲ့အခါ Binary Data တွေကို ဖိုင်ထဲကို ရေးဖို့ (writing) ရည်ရွယ်ပါတယ်။ `w` mode နဲ့ ဆင်တူပေမယ့် ဒီမှာတော့ Binary Data တွေကို ရေးပါတယ်။ ဖိုင်ကို ဖျက်ပြီး အသစ်ပြန်ရေးတဲ့ အဓိပ္ပာယ် ရှိပါတယ်။

ဒီ Mode တွေရဲ့ အဓိက ကွာခြားချက်ကတော့ ဖိုင်ကို ဘယ်လိုအခြေအနေနဲ့ ကိုင်တွယ်မလဲဆိုတဲ့ ပုံစံပေါ် မူတည်ပါတယ်။ စာသား (Text) ကို ကိုင်တွယ်မလား၊ Binary Data ကို ကိုင်တွယ်မလား၊ ဖျက်ပစ်မလား၊ ထပ်ထည့်မလား ဆိုတာကို သတ်မှတ်ပေးတာ ဖြစ်ပါတယ်။

### File ကို ပိတ်ခြင်း (Closing the File)

File I/O လုပ်ဆောင်ပြီးတဲ့အခါမှာ `fclose()` function ကို အသုံးပြုပြီး ဖိုင်ကို ပိတ်ဖို့ အလွန်အရေးကြီးပါတယ်။

`fclose()` function ဟာ `fopen()` နဲ့ ဖွင့်ထားတဲ့ `FILE *` pointer ကို ဖိုင်ကို စနစ်တကျ ပိတ်ပြီး System ထဲက Resource တွေကို ပြန်လည်လွှဲပြောင်းပေးပါတယ်။

File ကို မပိတ်ဘဲ ထားခြင်းရဲ့ အကျိုးဆက်တွေကတော့ အရေးကြီးပါတယ်။ ဖိုင်ကို ပိတ်ခြင်းမရှိဘဲ ထားလိုက်ရင် ပရိုဂရမ်က အဲ့ဒီဖိုင်ကို အသုံးပြုနေတဲ့အချိန်မှာ Memory နဲ့ System Resource တွေက အဲ့ဒီဖိုင်အတွက် လိုအပ်တဲ့ နေရာတွေကို သိမ်းဆည်းထားဆဲ ဖြစ်နေပါလိမ့်မယ်။ ဒါကို **Memory/Resource Leakage** လို့ ခေါ်ပါတယ်။ အထူးသဖြင့် ဖိုင်ကို ပိတ်ဖို့မေ့နေရင် ဖိုင်ထဲကို ရေးလိုက်တဲ့ အချက်အလက်တွေဟာ Disk ပေါ်မှာ မှန်ကန်စွာ သိမ်းဆည်းမရဘဲ Data Corruption ဖြစ်နိုင်ခြေ ရှိပါတယ်။ ဒါ့အပြင် အခြားပရိုဂရမ်တွေက အဲ့ဒီဖိုင်ကို ဝင်ရောက်အသုံးပြုဖို့ အခက်အခဲတွေ ဖြစ်ပေါ်စေနိုင်ပါတယ်။ ဒါကြောင့် File I/O လုပ်ငန်းစဉ်တိုင်းမှာ `fclose()` ကို အမြဲတမ်း အသုံးပြုဖို့ လိုအပ်ပါတယ်။

## ၃။ Text File များဖြင့် အချက်အလက်များ ဖလှယ်ခြင်း (Handling Text Files)

Text File တွေဟာ စာသား (Characters) အခြေခံနဲ့ အချက်အလက်တွေကို သိမ်းဆည်းတဲ့ ဖိုင်အမျိုးအစား ဖြစ်ပါတယ်။ ဒီဖိုင်တွေဟာ လူတွေဖတ်နိုင်တဲ့ ပုံစံနဲ့ ရေးသားထားတာ ဖြစ်ပါတယ်။

### Text File ဖတ်ခြင်းနှင့် ရေးခြင်း

Text File တွေနဲ့ အလုပ်လုပ်တဲ့အခါမှာ C Language မှာ အသုံးပြုတဲ့ function တွေက စာသားတွေကို ဖော်မတ်နဲ့အညီ ထည့်သွင်းတာနဲ့ ဖတ်ယူတာကို လုပ်ဆောင်ပေးပါတယ်။

၁။ **`fprintf()` နှင့် `fscanf()`**: ဒီ function နှစ်ခုဟာ ဖိုင်ထဲကို စာသားတွေကို ရေးဖို့နဲ့ ဖတ်ဖို့အတွက် အသုံးများပါတယ်။
    *   `fprintf(file_pointer, format, ...)`: ဒီ function ဟာ `printf()` နဲ့ ဆင်တူပါတယ်။ ပထမအချက်က `file_pointer` (ဖိုင် pointer) ဖြစ်ပြီး၊ ဒုတိယအချက်က ရေးမယ့် ဖိုင် ဖြစ်ပါတယ်။ တတိယအချက်က ရေးမယ့် ပုံစံ (format string) ဖြစ်ပြီး၊ နောက်ဆက်တွဲမှာ `printf()` မှာ သုံးတဲ့အတိုင်း variable တွေကို ထည့်သွင်းပေးရပါတယ်။ ဥပမာအားဖြင့်၊ `fprintf(fp, "Name: %s, Age: %d\n", name, age);` ဆိုတာက `name` နဲ့ `age` ဆိုတဲ့ variable တွေကို ဖိုင်ထဲကို စာသားပုံစံနဲ့ ရေးပေးပါတယ်။
    *   `fscanf(file_pointer, format, ...)`: ဒီ function ဟာ `scanf()` နဲ့ ဆင်တူပါတယ်။ ဖိုင်ထဲကနေ စာသားတွေကို ဖတ်ပြီး ပေးထားတဲ့ format နဲ့ ကိုက်ညီအောင် variable တွေထဲကို ထည့်သွင်းပေးပါတယ်။ ဥပမာအားဖြင့်၊ `fscanf(fp, "Name: %s, Age: %d\n", name, &age);` ဆိုတာက ဖိုင်ထဲက စာသားတွေကို ဖတ်ပြီး `name` နဲ့ `age` variable တွေထဲကို ထည့်သွင်းပေးပါတယ်။

၂။ **`fgets()` နှင့် `fputs()`**: စာကြောင်း (Line) အလိုက် ဖတ်ခြင်းနဲ့ ရေးခြင်းအတွက် ဒီ function တွေက အလွန်အသုံးဝင်ပါတယ်။
    *   `fgets(buffer, size, file_pointer)`: ဒီ function ဟာ ဖိုင်ထဲကနေ စာကြောင်းတစ်ခုလုံးကို ဖတ်ပြီး `buffer` (character array) ထဲမှာ သိမ်းဆည်းပေးပါတယ်။ စာကြောင်းအလိုက် ဖတ်တဲ့အခါမှာ အလွန်အသုံးဝင်ပါတယ်။
    *   `fputs(string, file_pointer)`: ဒီ function ဟာ `puts()` နဲ့ ဆင်တူပါတယ်။ `string` ဆိုတဲ့ string ကို ဖိုင်ထဲကို ရေးပေးပါတယ်။

Text File တွေရဲ့ အားသာချက်ကတော့ လူတွေအတွက် ဖတ်ရလွယ်ကူပြီး နားလည်ရလွယ်ကူပါတယ်။ ဒါပေမယ့် အားနည်းချက်ကတော့ Data အမျိုးအစား (Data Type) တွေကို တိုက်ရိုက်သိမ်းဆည်းတာမျိုး မလုပ်နိုင်ဘဲ စာသား (String) အဖြစ်သာ ကိုင်တွယ်ရတဲ့အတွက် ဂဏန်းတွေ၊ ပုံစံတွေနဲ့ တိုက်ရိုက်လုပ်ဆောင်ဖို့ အခက်အခဲရှိနိုင်ပါတယ်။

### Standard Streams နှင့် Command Line Arguments

File I/O နဲ့ ပတ်သက်ပြီး System Level မှာ အရေးကြီးတဲ့ အချက်တွေက Standard Streams တွေနဲ့ Command Line Arguments တွေ ဖြစ်ပါတယ်။

Standard Streams ဆိုတာကတော့ ကွန်ပျူတာရဲ့ Input, Output, Error တွေကို စနစ်တကျ စီမံခန့်ခွဲပေးတဲ့ စံနှုန်းတွေ ဖြစ်ပါတယ်။

*   **`stdin` (Standard Input)**: ပရိုဂရမ်တစ်ခုကို Input ထည့်သွင်းဖို့အတွက် အသုံးပြုပါတယ်။ ပုံမှန်အားဖြင့် Keyboard ကနေ Input တွေ ရောက်လာတာကို `stdin` ကနေ လက်ခံပါတယ်။
*   **`stdout` (Standard Output)**: ပရိုဂရမ်က ထုတ်ပေးမယ့် အချက်အလက်တွေကို ထုတ်ပေးတဲ့အတွက် အသုံးပြုပါတယ်။ ပုံမှန်အားဖြင့် Terminal (Screen) ကို ထုတ်ပေးပါတယ်။
*   **`stderr` (Standard Error)**: ပရိုဂရမ်မှာ ဖြစ်ပေါ်လာတဲ့ Error တွေ ဒါမှမဟုတ် သတိပေးချက် (Warning) တွေကို ထုတ်ပေးဖို့အတွက် အသုံးပြုပါတယ်။ `stdout` နဲ့ လမ်းကြောင်းက မတူဘဲ သီးခြားစီ ထုတ်ပေးပါတယ်။

ဒီ Standard Streams တွေဟာ Application Level မှာ လုပ်ဆောင်တာတွေနဲ့ System Level မှာ လုပ်ဆောင်တာတွေကြားက ကွာခြားချက်ကို နားလည်ဖို့ အရေးကြီးပါတယ်။

Command Line Arguments တွေကိုလည်း File I/O နဲ့ ဆက်စပ်ပြီး နားလည်ဖို့ လိုအပ်ပါတယ်။ ပရိုဂရမ်ကို Run တဲ့အခါ Terminal မှာ Command Line ကနေ အချက်အလက်တွေ ပေးနိုင်ပါတယ်။

*   **`argc` (argument count)**: Command Line မှာ ပေးလိုက်တဲ့ Argument တွေရဲ့ အရေအတွက်ကို သိရှိပေးပါတယ်။ ပထမအချက်ကတော့ Program name ကို ကိုယ်စားပြုပါတယ်။
*   **`argv[]` (argument values)**: Command Line မှာ ပေးလိုက်တဲ့ Argument တွေရဲ့ တန်ဖိုးတွေကို Array တစ်ခုအဖြစ် သိမ်းဆည်းပေးပါတယ်။ `argv[0]` က Program name ဖြစ်ပြီး၊ ကျန်တဲ့ `argv[1]`, `argv[2]`, ... တို့က ပေးလိုက်တဲ့ တန်ဖိုးတွေ ဖြစ်ပါတယ်။

ဒီအချက်အလက်တွေကို File ထဲကို ရေးတာ၊ ဖတ်တာတွေ လုပ်တဲ့အခါ Command Line ကနေ အချက်အလက်တွေကို ထည့်သွင်းဖို့အတွက်လည်း ဒီအချက်အလက်တွေကို အသုံးပြုနိုင်ပါတယ်။

## ၄။ Binary File များဖြင့် Data Structure များ သိမ်းဆည်းခြင်း (Handling Binary Files)

Text File တွေက စာသားတွေကို သိမ်းဆည်းတာဖြစ်ပြီး၊ Binary File တွေကတော့ Data တွေကို byte အတိုင်း တိုက်ရိုက် သိမ်းဆည်းတာ ဖြစ်ပါတယ်။

### Binary File များ၏ လိုအပ်ချက်

Text File နဲ့ Binary File များရဲ့ အဓိက ကွာခြားချက်ကတော့ Data ကို သိမ်းဆည်းတဲ့ ပုံစံပဲ ဖြစ်ပါတယ်။

Text File တွေက စာသားတွေကို ASCII (သို့မဟုတ်) Unicode ပုံစံနဲ့ သိမ်းဆည်းတာ ဖြစ်ပြီး၊ ကွန်ပျူတာက အဲ့ဒီစာသားတွေကို byte တွေအဖြစ် ပြောင်းလဲပြီး သိမ်းဆည်းပါတယ်။

Binary File တွေမှာတော့ Data တွေကို byte အတိုင်း တိုက်ရိုက် သိမ်းဆည်းပါတယ်။ ဒါကြောင့် Number တွေ၊ Floating-point နံပါတ်တွေ၊ Struct တွေလို Data Structure တွေကို သိမ်းဆည်းတဲ့အခါ Binary File တွေက ပိုမိုထိရောက်ပြီး Memory အသုံးပြုမှုကိုလည်း ထိန်းချုပ်နိုင်ပါတယ်။

`struct` Data များကို File ထဲသို့ သိမ်းဆည်းခြင်းရဲ့ အရေးပါပုံကတော့ Struct ဟာ Data အမျိုးမျိုးကို ပေါင်းစပ်ထားတဲ့ Data Structure တစ်မျိုး ဖြစ်ပါတယ်။ ဒီလို Struct တွေကို Binary File ထဲကို သိမ်းဆည်းလိုက်တဲ့အခါမှာ အဲ့ဒီ Struct ရဲ့ အစိတ်အပိုင်းအားလုံးကို byte အတိုင်း ဖိုင်ထဲမှာ သိမ်းဆည်းပေးပါတယ်။ ဒါကြောင့် Binary File တွေက Data ကို ပိုမိုတိကျစွာနဲ့ ထိန်းသိမ်းနိုင်ပါတယ်။

### Binary I/O Functions

Binary File တွေနဲ့ အလုပ်လုပ်ဖို့အတွက် `fwrite()` နဲ့ `fread()` function တွေကို အသုံးပြုရပါတယ်။

၁။ **`fwrite()`**: ဒီ function ဟာ Memory ထဲက Data တွေကို ဖိုင်ထဲကို ရေးဖို့ (writing) အသုံးပြုပါတယ်။ `fwrite(data_ptr, size, count, file_pointer)` ဆိုတဲ့ syntax ကို အသုံးပြုရပါတယ်။
    *   `data_ptr`: ရေးမယ့် Data တွေရဲ့ ပထမ Address (Pointer) ဖြစ်ပါတယ်။
    *   `size`: ရေးမယ့် Data တစ်ခုရဲ့ အရွယ်အစား (size) ဖြစ်ပါတယ်။ ဒါက byte အတိုင်း သိမ်းဆည်းတာ ဖြစ်တဲ့အတွက် `sizeof()` ကို အသုံးပြုပြီး ရရှိရတဲ့ byte အရေအတွက်ကို ထည့်ပေးရပါမယ်။
    *   `count`: ရေးမယ့် Data တွေရဲ့ အရေအတွက် (count) ဖြစ်ပါတယ်။
    *   `file_pointer`: ရေးမယ့် ဖိုင်ကို ရည်ညွှန်းတဲ့ `FILE *` pointer ဖြစ်ပါတယ်။

၂။ **`fread()`**: ဒီ function ဟာ ဖိုင်ထဲက Data တွေကို ဖတ်ယူဖို့ (reading) အသုံးပြုပါတယ်။ `fread(buffer, size, count, file_pointer)` ဆိုတဲ့ syntax ကို အသုံးပြုရပါတယ်။
    *   `buffer`: ဖတ်ယူမယ့် Data တွေ သိမ်းမယ့် Memory နေရာ (Buffer) ဖြစ်ပါတယ်။
    *   `size`: ဖတ်ယူမယ့် Data တစ်ခုရဲ့ အရွယ်အစား (size) ဖြစ်ပါတယ်။
    *   `count`: ဖတ်ယူမယ့် Data တွေရဲ့ အရေအတွက် (count) ဖြစ်ပါတယ်။
    *   `file_pointer`: ဖတ်မယ့် ဖိုင်ကို ရည်ညွှန်းတဲ့ `FILE *` pointer ဖြစ်ပါတယ်။

Data ကို သိမ်းဆည်းရာမှာ `sizeof()` နဲ့ `byte` အခြေခံကို နားလည်ဖို့ အလွန်အရေးကြီးပါတယ်။ `sizeof(struct_name)` ကို အသုံးပြုပြီး Struct တစ်ခုရဲ့ အတိအကျ byte အရေအတွက်ကို သိရှိနိုင်ပါတယ်။ ဒါကို `fwrite()` နဲ့ `fread()` မှာ အသုံးပြုတဲ့အခါ အဲ့ဒီ byte အရေအတွက်ကို ထည့်ပေးဖို့ လိုအပ်ပါတယ်။

## ၅။ File Pointer နှင့် Random Access (နေရာရွှေ့ခြင်း)

File I/O မှာ အချက်အလက်တွေရဲ့ တည်နေရာကို စီမံခန့်ခွဲဖို့အတွက် File Pointer ဟာ အဓိကကျပါတယ်။

### File Position Management

File Pointer ဟာ ဖိုင်ထဲက ဘယ်နေရာမှာ ရှိနေလဲဆိုတာကို ဖော်ပြပေးပါတယ်။

၁။ **`fseek()`**: ဒီ function ဟာ File Pointer ကို လိုချင်တဲ့ နေရာတစ်ခုဆီကို ရွှေ့ဖို့ အသုံးပြုပါတယ်။ `fseek()` ရဲ့ syntax က `fseek(file_pointer, offset, origin)` ဖြစ်ပါတယ်။
    *   `file_pointer`: ရွှေ့မယ့် ဖိုင်ကို ရည်ညွှန်းတဲ့ `FILE *` pointer ဖြစ်ပါတယ်။
    *   `offset`: ရွှေ့မယ့် အကွာအဝေး (offset) ဖြစ်ပါတယ်။ ဒါက byte အခြေခံနဲ့ ရွှေ့တာ ဖြစ်ပါတယ်။ အပေါင်း (+) သို့မဟုတ် အနုတ် (-) တန်ဖိုးကို အသုံးပြုနိုင်ပါတယ်။
    *   `origin`: ရွှေ့မယ့် အခြေခံနေရာ (origin) ကို သတ်မှတ်ပေးပါတယ်။
        *   `SEEK_SET`: အစ (Beginning) ကနေ ရွှေ့တာ။
        *   `SEEK_CUR`: လက်ရှိနေရာ (Current position) ကနေ ရွှေ့တာ။
        *   `SEEK_END`: ဖိုင်ရဲ့ အဆုံး (End) ကနေ ရွှေ့တာ။

၂။ **`ftell()`**: ဒီ function ဟာ လက်ရှိ File Pointer ရဲ့ တည်နေရာကို ပြန်ပေးပါတယ်။ `ftell(file_pointer)` ကို အသုံးပြုပြီး လက်ရှိ File Pointer ရဲ့ byte အခြေခံ တည်နေရာကို သိရှိနိုင်ပါတယ်။

၃။ **`rewind()`**: ဒီ function ဟာ File Pointer ကို ဖိုင်ရဲ့ အစ (Beginning) ကို ပြန်လည်ရောက်ရှိစေဖို့ အသုံးပြုပါတယ်။ `rewind(file_pointer)` ကို သုံးလိုက်တဲ့အခါ File Pointer ဟာ ဖိုင်ရဲ့ ပထမဆုံး byte (အမှတ် 0) ကို ပြန်ရောက်သွားပါလိမ့်မယ်။

### Random Access (ကျပန်းဝင်ရောက်ခြင်း)

File I/O မှာ အရေးကြီးတဲ့ အချက်တစ်ခုကတော့ **Random Access (ကျပန်းဝင်ရောက်ခြင်း)** ဖြစ်ပါတယ်။

Random Access ဆိုတာက File ရဲ့ အစကနေ မည်သည့်နေရာကိုမဆို တိုက်ရိုက်ဝင်ရောက်ပြီး ဖတ်တာ (Read) ဒါမှမဟုတ် ရေးတာ (Write) ကို ဆိုလိုပါတယ်။ Text File တွေမှာတော့ စာကြောင်းအလိုက် ဖတ်တာမျိုး (Sequential Access) ကို အဓိကထားပေမယ့် Binary File တွေမှာတော့ `fseek()` ကို အသုံးပြုပြီး လိုချင်တဲ့ နေရာကို တိုက်ရိုက်သွားရောက်ပြီး ဖတ်တာ၊ ရေးတာတွေ လုပ်နိုင်ပါတယ်။

Random Access ရဲ့ အကျိုးကျေးဇူးကတော့ ဖိုင်ရဲ့ အစကနေ စပြီး တစ်ဆင့်ချင်းစီ ဖတ်ရမယ့် အချိန်ကို မစောင့်ရဘဲ လိုအပ်တဲ့ နေရာကို တိုက်ရိုက်သွားရောက်နိုင်တာ ဖြစ်ပါတယ်။ ဒါဟာ ပရိုဂရမ်ရဲ့ စွမ်းဆောင်ရည်ကို မြှင့်တင်ပေးပြီး အချိန်ကုန်သက်သာစေပါတယ်။ ဒါပေမယ့် Random Access ကို အသုံးပြုတဲ့အခါ File Pointer ကို မှန်ကန်စွာ စီမံခန့်ခွဲဖို့နဲ့ `fseek()` ရဲ့ အသုံးပြုပုံကို သတိထားဖို့ လိုအပ်ပါတယ်။ မှားယွင်းစွာ အသုံးပြုမိရင် Data Corruption ဖြစ်နိုင်ပါတယ်။

## ၆။ Error Handling နှင့် စနစ်တကျ လုပ်ဆောင်ခြင်း (Error Handling and System Streams)

File I/O လုပ်ငန်းစဉ်တွေဟာ အမှားတွေ ဖြစ်ပေါ်လာနိုင်တဲ့အတွက် Error Handling ဟာ အလွန်အရေးကြီးပါတယ်။

### Error Checking (အမှားစစ်ဆေးခြင်း)

File I/O လုပ်ဆောင်တဲ့အခါမှာ အမှားတွေ ဖြစ်ပေါ်နိုင်ခြေများပါတယ်။ အဓိက အမှားအမျိုးအစားတွေကတော့ ဖိုင်ကို ဖွင့်မရခြင်း (e.g., permission error)၊ ဖိုင်ကို ရေး/ဖတ်ရာမှာ ပြဿနာရှိခြင်း စတာတွေ ဖြစ်ပါတယ်။

*   **`fopen()` မှ ပြန်ပေးသော Pointer ကို စစ်ဆေးခြင်း**: `fopen()` function ဟာ ဖိုင်ကို ဖွင့်ဖို့ အဆင်မပြေတဲ့အခါ `NULL` value ကို ပြန်ပေးပါတယ်။ ဒါကြောင့် `fopen()` ရဲ့ အကျိုးရလဒ်ကို အမြဲတမ်း စစ်ဆေးဖို့ လိုအပ်ပါတယ်။
    ```c
    FILE *fp = fopen("data.txt", "r");
    if (fp == NULL) {
        printf("Error: Could not open the file!\n");
        // Error handling လုပ်ဆောင်ခြင်း
        return 1;
    }
    // ဖိုင်ဖွင့်ပြီးမှသာ လုပ်ဆောင်ရန်
    ```
    ဒီလို စစ်ဆေးခြင်းက ပရိုဂရမ်ရဲ့ တည်ငြိမ်မှုကို အာမခံပေးပါတယ်။

*   **File I/O လုပ်ငန်းစဉ်များတွင် ဖြစ်ပေါ်လာနိုင်သော အမှားများကို ကိုင်တွယ်ဖြေရှင်းခြင်း**: ဖိုင်ကို ရေးတဲ့အခါ Disk နေရာမလုံလောက်တာ၊ ဖိုင်ကို ဖတ်တဲ့အခါ ဖိုင်ရဲ့ အရွယ်အစားထက် ပိုမိုဖတ်ဖို့ ကြိုးစားတာ စတဲ့ အမှားတွေ ဖြစ်နိုင်ပါတယ်။ ဒီလိုအခြေအနေတွေကို `feof()` (End Of File) function ဒါမှမဟုတ် `ferror()` function တွေကို အသုံးပြုပြီး စစ်ဆေးနိုင်ပါတယ်။

### Standard Streams နှင့် System Interaction

အပေါ်မှာ ဖော်ပြခဲ့တဲ့ `stdin`, `stdout`, `stderr` တို့ဟာ Application Level မှာ လုပ်ဆောင်တာတွေနဲ့ System Level မှာ လုပ်ဆောင်တာတွေကြားက ကွာခြားချက်ကို နားလည်ဖို့ အရေးကြီးပါတယ်။

*   **`stdin` (Standard Input)**: အသုံးပြုသူဆီကနေ Input တွေ လက်ခံပါတယ်။
*   **`stdout` (Standard Output)**: ပရိုဂရမ်က ထုတ်ပေးမယ့် အချက်အလက်တွေကို ထုတ်ပေးပါတယ်။
*   **`stderr` (Standard Error)**: အမှား (Error) တွေကို ထုတ်ပေးဖို့ အသုံးပြုပါတယ်။

System Level မှာ လုပ်ဆောင်တာတွေဟာ Terminal ဒါမှမဟုတ် Command Line နဲ့ တိုက်ရိုက်ဆက်သွယ်တာမျိုး ဖြစ်ပြီး၊ Application Level မှာ လုပ်ဆောင်တာတွေကတော့ File I/O လိုမျိုး File System နဲ့ ဆက်စပ်တဲ့ လုပ်ဆောင်ချက်တွေ ဖြစ်ပါတယ်။ ဒီနှစ်ခုကြားက ကွာခြားချက်ကို နားလည်ခြင်းက ပရိုဂရမ်တွေက စနစ်နဲ့ မည်သို့ ဆက်သွယ်နေတယ်ဆိုတာကို ပိုမိုနက်ရှိုင်းစွာ နားလည်စေပါတယ်။

## ၇။ လက်တွေ့ လေ့ကျင့်ခန်း (Practical Implementation)

အထက်ပါ အခြေခံသဘောတရားတွေကို လက်တွေ့ကျကျ အသုံးချကြည့်ဖို့အတွက် ကျောင်းသားမှတ်တမ်း (Student Records) တွေကို File ထဲသို့ သိမ်းဆည်းပြီး ပြန်လည် Load လုပ်တဲ့ လေ့ကျင့်ခန်းကို လုပ်ဆောင်ကြည့်ရအောင်။ ဒီလေ့ကျင့်ခန်းမှာ Text File နဲ့ Binary File နှစ်မျိုးလုံးကို အသုံးပြုပြီး `fseek()`, `fwrite()`, `fscanf()` တို့ကို အသုံးပြုပုံကို အဓိကထားပြီး ရေးသားသွားမှာ ဖြစ်ပါတယ်။

### ကျောင်းသား မှတ်တမ်းများကို File ထဲသို့ Save လုပ်ခြင်း (Text/Binary ပုံစံဖြင့်)

ဒီလေ့ကျင့်ခန်းမှာ ကျွန်ုပ်တို့ရဲ့ ကျောင်းသားအချက်အလက်တွေကို သိမ်းဆည်းဖို့အတွက် အချက်အလက်အမျိုးမျိုးကို ထည့်သွင်းရပါမယ်။ Text File နဲ့ Binary File နှစ်မျိုးလုံးကို နှိုင်းယှဉ်ပြီး ဘယ်ပုံစံက ဘယ်လို အကျိုးပြုလဲဆိုတာကို လေ့လာရမှာ ဖြစ်ပါတယ်။

**Text File ဖြင့် Save လုပ်ခြင်း (fprintf/fscanf အသုံးပြု၍)**:
Text File မှာတော့ စာသားအဖြစ် သိမ်းဆည်းတာ ဖြစ်တဲ့အတွက် `fprintf()` နဲ့ `fscanf()` ကို အသုံးပြုပြီး စာကြောင်းအလိုက် (Line by Line) မှတ်တမ်းတွေကို သိမ်းဆည်းမှာ ဖြစ်ပါတယ်။ ဒါက လူတွေဖတ်ဖို့နဲ့ နားလည်ဖို့ အလွန်လွယ်ကူပါတယ်။

**Binary File ဖြင့် Save လုပ်ခြင်း (fwrite/fread အသုံးပြု၍)**:
Binary File မှာတော့ `struct` ကို အသုံးပြုပြီး Data တွေကို byte အတိုင်း တိုက်ရိုက် သိမ်းဆည်းမှာ ဖြစ်ပါတယ်။ ဒါက Data ကို ပိုမိုထိရောက်စွာ သိမ်းဆည်းပေးပြီး ဖတ်ယူတာ ပိုမြန်ဆန်ပါတယ်။

### Save လုပ်ထားသော မှတ်တမ်းများကို ပြန်လည် Load လုပ်ခြင်း

Save လုပ်ပြီးတဲ့ မှတ်တမ်းတွေကို ပြန်လည် Load လုပ်ဖို့အတွက် `fscanf()` (Text File အတွက်) ဒါမှမဟုတ် `fread()` (Binary File အတွက်) ကို အသုံးပြုရမှာ ဖြစ်ပါတယ်။

File ကို ပြန်လည် Load လုပ်တဲ့အခါမှာလည်း အမှားစစ်ဆေးခြင်း (Error Checking) ကို မမေ့သင့်ပါဘူး။ ဖိုင်ကို ဖွင့်မရတာ၊ ဖတ်/ရေးရာမှာ အမှားတွေဖြစ်ပေါ်လာတာတွေကို စနစ်တကျ ကိုင်တွယ်ဖြေရှင်းဖို့ လိုအပ်ပါတယ်။

### အခန်း၏ အနှစ်ချုပ် (Chapter Summary)

File I/O ဟာ C Programming မှာ အချက်အလက်တွေကို ထိန်းသိမ်းဖို့နဲ့ ဖလှယ်ဖို့အတွက် မရှိမဖြစ်လိုအပ်တဲ့ အခန်းကဏ္ဍတစ်ခု ဖြစ်ပါတယ်။ ကျွန်ုပ်တို့ဟာ `FILE *` pointer နဲ့ `fopen()` ကို အသုံးပြုပြီး ဖိုင်တွေကို ဖွင့်ခြင်း၊ `fclose()` နဲ့ ပိတ်ခြင်းကို မှန်ကန်စွာ လုပ်ဆောင်ရမှာ ဖြစ်ပါတယ်။ ဖိုင်ကို ဘယ်လို Mode နဲ့ ဖွင့်မလဲဆိုတာကို `r`, `w`, `a`, `rb`, `wb` စတဲ့ Modes တွေက သတ်မှတ်ပေးပါတယ်။ Text Data တွေကို `fprintf()`, `fscanf()`, `fgets()`, `fputs()` တို့နဲ့၊ Binary Data (Structs) တွေကို `fwrite()` နဲ့ `fread()` တို့နဲ့ သိမ်းဆည်းနိုင်ပါတယ်။

အချက်အလက်တွေကို လိုအပ်တဲ့ နေရာကို တိုက်ရိုက်သွားရောက်ဖတ်ဖို့အတွက် `fseek()`, `ftell()`, `rewind()` စတဲ့ Random Access လုပ်ဆောင်ချက်တွေဟာ Data ကို ပိုမိုထိရောက်စွာ စီမံခန့်ခွဲနိုင်စေပါတယ်။ အမှားတွေကို ကိုင်တွယ်ဖို့အတွက် `fopen()` ရဲ့ အကျိုးရလဒ်ကို စစ်ဆေးတာ (NULL check) နဲ့ `fclose()` ကို အမြဲတမ်း အသုံးပြုတာဟာ အရေးကြီးပါတယ်။ နောက်ဆုံးအနေနဲ့ `stdin`, `stdout`, `stderr` စတဲ့ Standard Streams တွေကို နားလည်ခြင်းက ပရိုဂရမ်တွေက System နဲ့ မည်သို့ ဆက်သွယ်နေတယ်ဆိုတာကို သိရှိစေပါတယ်။

File I/O ဟာ ကွန်ပျူတာစနစ်ရဲ့ အခြေခံအုတ်မြစ်တစ်ခုဖြစ်ပြီး၊ ကျွန်ုပ်တို့ရေးသားမယ့် လက်တွေ့ပရိုဂရမ်တွေဟာ အချက်အလက်တွေကို ထိန်းသိမ်းနိုင်ပြီး အသုံးပြုသူတွေနဲ့ ဆက်သွယ်နိုင်ဖို့အတွက် မရှိမဖြစ် လိုအပ်တဲ့ စွမ်းဆောင်ရည်တစ်ခု ဖြစ်ပါတယ်။ ဒီအခြေခံတွေကို နားလည်ခြင်းဖြင့် ကျွန်ုပ်တို့သည် ပိုမိုရှုပ်ထွေးပြီး စွမ်းဆောင်ရည်မြင့်မားတဲ့ ပရိုဂရမ်တွေကို တည်ဆောက်နိုင်မှာ ဖြစ်ပါတယ်။
