# Chapter 15: Stack နှင့် Queue — ဒေတာ စီစဉ်ခြင်းနည်းလမ်းများ

ကွန်ပျူတာ သိပ္ပံနှင့် ပရိုဂရမ်မင်းနယ်ပယ်တွင် ဒေတာများကို စနစ်တကျ စီမံခန့်ခွဲခြင်း (Data Organization) သည် အလွန်အရေးကြီးသော အခြေခံကျသော ကျွမ်းကျင်မှုတစ်ခု ဖြစ်သည်။ ဒေတာများကို မှန်ကန်စွာ စီစဉ်ထားခြင်းကသာ ရှုပ်ထွေးသော Algorithm များ၊ Operating System များ၊ နှင့် ရှုပ်ထွေးသော ကွန်ပျူတာစနစ်များကို တည်ဆောက်ရာတွင် အထောက်အကူ ဖြစ်စေသည်။ ဤအခြေခံအုတ်မြစ်များထဲမှ Stack နှင့် Queue တို့သည် ဒေတာများကို ထိန်းသိမ်းပြီး စီမံခန့်ခွဲရန် အသုံးအများဆုံးနှင့် အခြေခံကျသော Data Structure များ ဖြစ်ကြသည်။ ဤအခန်းတွင် ကျွန်ုပ်တို့သည် Stack နှင့် Queue တို့၏ အခြေခံသဘောတရားများ၊ ၎င်းတို့၏ အကောင်အထည်ဖော်မှုများ၊ လက်တွေ့အသုံးချမှုများနှင့် အချင်းချင်း ဆက်စပ်ပုံများကို အသေးစိတ် လေ့လာသွားကြမည် ဖြစ်သည်။

Stack နှင့် Queue တို့သည် ကွန်ပျူတာ၏ မှတ်ဉာဏ် (Memory) အတွင်းရှိ ဒေတာများကို မည်သို့ စီစဉ်ထားသည်ကို နားလည်ရန် အရေးကြီးသည်။ ၎င်းတို့၏ အခြေခံသဘောတရားများကို နားလည်ခြင်းသည် ပိုမိုရှုပ်ထွေးသော Algorithm များဖြစ်သည့် Recursion (ပြန်လည်ခေါ်ဆိုမှု) များ၊ Expression Evaluation များ၊ နှင့် Operating System များ၏ Process Management များအထိ နက်ရှိုင်းစွာ လေ့လာနိုင်စေမည် ဖြစ်သည်။ နေ့စဉ်ဘဝတွင် ကျွန်ုပ်တို့သည် အရာဝတ္ထုများကို စီစဉ်သိမ်းဆည်းပုံနှင့် စောင့်ဆိုင်းပုံတို့ကို အခြေခံအားဖြင့် လိုက်နာနေကြသည်။ Stack နှင့် Queue တို့၏ လုပ်ဆောင်ပုံများကို နေ့စဉ်ဘဝနှင့် နှိုင်းယှဉ်ခြင်းဖြင့် ၎င်းတို့၏ သဘောတရားများကို ပိုမိုလွယ်ကူစွာ နားလည်နိုင်မည် ဖြစ်သည်။

## Stack (အထပ်) အကြောင်း အသေးစိတ် လေ့လာခြင်း

Stack ဆိုသည်မှာ ဒေတာများကို စီစဉ်သိမ်းဆည်းရန်အတွက် အသုံးပြုသော အထပ်ပုံစံ (Linear structure) တစ်မျိုး ဖြစ်သည်။ Stack ၏ အဓိက အထူးခြားဆုံး သဘောတရားမှာ LIFO (Last In, First Out) သဘောတရားပင် ဖြစ်သည်။ ဤသဘောတရားကို နားလည်ရန်အတွက် ကျွန်ုပ်တို့သည် ပန်းကန်ပုံးများနှင့် နှိုင်းယှဉ်ကြည့်ရမည်။ ပန်းကန်ပုံးများကို တစ်ခုပြီးတစ်ခု အပေါ်မှ ထည့်သွင်းလိုက်သည့်အခါ အပေါ်ဆုံးမှ ထည့်လိုက်သော ပန်းကန်ပုံးကိုသာ အရင်ထုတ်ယူရမည် ဖြစ်သည်။ ဤသည်မှာ Stack ၏ အခြေခံလုပ်ဆောင်ပုံပင် ဖြစ်သည်။

Stack ၏ အခြေခံလုပ်ဆောင်ချက်များကို ပိုမိုရှင်းလင်းစွာ နားလည်ရန်အတွက် အဓိက လုပ်ဆောင်ချက်လေးမျိုးကို သိရှိထားရမည်။ ပထမဆုံး လုပ်ဆောင်ချက်မှာ `push` ဖြစ်ပြီး ၎င်းသည် Stack ထဲသို့ အချက်အလက်အသစ်ကို ထည့်သွင်းခြင်းကို ဆိုလိုသည်။ ဒုတိယအနေဖြင့် `pop` လုပ်ဆောင်ချက်သည် Stack ထိပ်မှ အချက်အလက်ကို ထုတ်ယူခြင်း ဖြစ်သည်။ ထို့အပြင် `peek` သို့မဟုတ် `top` လုပ်ဆောင်ချက်က Stack ထိပ်ရှိ အချက်အလက်ကို ထုတ်ယူခြင်းမပြုဘဲ ကြည့်ရှုရုံသာ ဖြစ်သည်။ နောက်ထပ် အရေးကြီးသော အခြေအနေနှစ်ခုမှာ `isEmpty` နှင့် `isFull` ဖြစ်ပြီး ၎င်းတို့သည် Stack ၏ လက်ရှိအခြေအနေကို စစ်ဆေးရန် အသုံးပြုသည်။

Stack ကို ကွန်ပျူတာတွင် အကောင်အထည်ဖော်ပုံများ (Implementations) နှစ်မျိုးရှိသည်။ ပထမတစ်ခုမှာ Array ကို အသုံးပြု၍ Stack ကို အကောင်အထည်ဖော်ခြင်း ဖြစ်သည်။ Array သည် ဒေတာများကို သတ်မှတ်ထားသော အစီအစဉ်အတိုင်း ဆက်တိုက် သိမ်းဆည်းထားသည့် နေရာများ ဖြစ်သည်။ Array တွင် Stack ကို အကောင်အထည်ဖော်ရာတွင် Stack ၏ အရွယ်အစား (`size`) နှင့် လက်ရှိ အထပ်အပေါ်ဆုံးနေရာ (`top` index) တို့ကို ထိန်းသိမ်းရန် လိုအပ်သည်။ ဒုတိယတစ်ခုမှာ Linked List ကို အသုံးပြု၍ Stack ကို အကောင်အထည်ဖော်ခြင်း ဖြစ်သည်။ Linked List သည် Node များဖြင့် ဆက်စပ်နေသော စာရင်းအမျိုးအစား ဖြစ်သည်။ Stack ကို Linked List ဖြင့် အကောင်အထည်ဖော်ရာတွင် Node တစ်ခုချင်းစီကို အသုံးပြု၍ `push` လုပ်သည့်အခါ အသစ်ထည့်သော Node ကို အရင်ဆုံး ထိပ်သို့ ချိတ်ဆက်ပြီး `pop` လုပ်သည့်အခါ ထိပ်ရှိ Node ကို ဖယ်ရှားခြင်းဖြင့် လုပ်ဆောင်ရသည်။

Stack ၏ လက်တွေ့အသုံးချမှုများမှာ အလွန်ကျယ်ပြန့်သည်။ ပထမဆုံးနှင့် အရေးကြီးဆုံးမှာ Function Call Stack ဖြစ်သည်။ ပရိုဂရမ်များတွင် Function တစ်ခုမှ အခြား Function တစ်ခုကို ခေါ်ဆိုသည့်အခါ၊ ထို Function များ၏ အချက်အလက်များ၊ Return Address များ စသည်တို့ကို Stack တွင် စီစဉ်ထားသည်။ Function တစ်ခု ပြီးဆုံးသွားသောအခါ Stack မှ အချက်အလက်များကို ပြန်လည်ရယူပြီး မည်သည့်နေရာသို့ ပြန်သွားရမည်ကို သိရှိနိုင်စေသည်။ ဒုတိယအချက်မှာ Undo Operation ဖြစ်သည်။ ပရိုဂရမ်များတွင် အရင်လုပ်ဆောင်ခဲ့သော အစီအစဉ်ကို ပြန်လည်ရယူရန် (Undo) လုပ်ဆောင်သည့်အခါ၊ အလုပ်များကို Stack တွင် မှတ်သားထားခြင်းဖြင့် နောက်ဆုံးလုပ်ဆောင်ချက်ကို အရင် ပြန်ရယူနိုင်ပါသည်။ တတိယအချက်မှာ Expression Evaluation ဖြစ်သည်။ သင်္ချာတွက်ချက်မှုများ (ဥပမာ: Infix မှ Postfix သို့ ပြောင်းလဲခြင်း) ပြုလုပ်ရာတွင် Stack သည် Operator များ၏ အစီအစဉ်ကို စနစ်တကျ ထိန်းသိမ်းရန် အဓိကကျသည်။

## Queue (တန်းစီစနစ်) အကြောင်း အသေးစိတ် လေ့လာခြင်း

Stack ၏ LIFO သဘောတရားကို နားလည်ပြီးနောက်၊ ဒေတာများကို စီစဉ်ပုံနှင့် အစီအစဉ်တကျ ထည့်သွင်းပုံ (FIFO) ကို လေ့လာရန် ကူးပြောင်းလိုက်ရမည်။ Queue ဆိုသည်မှာ ဒေတာများကို စီစဉ်သိမ်းဆည်းရန် အသုံးပြုသော အခြားသော အထပ်ပုံစံ (Linear structure) တစ်မျိုး ဖြစ်သည်။ Queue ၏ အဓိက သဘောတရားမှာ FIFO (First In, First Out) သဘောတရားပင် ဖြစ်သည်။ လူတန်းစီခြင်း၊ စောင့်ဆိုင်းစင်များ (Waiting lines) သို့မဟုတ် အဝေးပြေးလမ်းမများတွင် လူများ စီတန်းပုံကို မြင်တွေ့ရသည့်အတိုင်းပင် ဖြစ်သည်။ အရင်ဆုံးရောက်လာသောသူသည် အရင်ဆုံး ထွက်ခွာရမည် ဖြစ်သည်။

Queue ၏ အခြေခံလုပ်ဆောင်ချက်များကို နားလည်ရန်အတွက် အဓိက လုပ်ဆောင်ချက်လေးမျိုးကို သိရှိထားရမည်။ ပထမဆုံး လုပ်ဆောင်ချက်မှာ `enqueue` ဖြစ်ပြီး ၎င်းသည် Queue ထဲသို့ အချက်အလက်အသစ်ကို ထည့်သွင်းခြင်းကို ဆိုလိုသည်။ ဒုတိယအနေဖြင့် `dequeue` လုပ်ဆောင်ချက်သည် Queue ထိပ်မှ အချက်အလက်ကို ထုတ်ယူခြင်း ဖြစ်သည်။ ထို့အပြင် `front` နှင့် `rear` ဆိုသော အမှတ်အသားများသည် Queue ၏ ရှေ့ဆုံး (အစ) နှင့် နောက်ဆုံး (အဆုံး) နေရာများကို သိရှိရန် အသုံးပြုသည်။ နောက်ထပ် အခြေအနေနှစ်ခုမှာ `isEmpty` နှင့် `isFull` ဖြစ်ပြီး ၎င်းတို့သည် Queue ၏ လက်ရှိအခြေအနေကို စစ်ဆေးရန် အသုံးပြုသည်။

Queue ကို အကောင်အထည်ဖော်ပုံများတွင် Array ကို အသုံးပြု၍ Queue ကို အကောင်အထည်ဖော်ခြင်း ဖြစ်သည်။ Array ကို အသုံးပြုသောအခါ `front` နှင့် `rear` index များကို ထိန်းသိမ်းရန် လိုအပ်သည်။ Array ၏ အစတွင် အချက်အလက် ထည့်သွင်းခြင်းနှင့် အဆုံးတွင် ထုတ်ယူခြင်းတို့ကို လုပ်ဆောင်ရမည် ဖြစ်သည်။ ထို့အပြင် Array တွင် အလဟသနေရာ (Wasted Space) ဖြစ်ပေါ်လာနိုင်သောကြောင့် Circular Queue (စက်ဝိုင်းပုံ Queue) ကို အသုံးပြုခြင်းသည် အလွန်အကျိုးရှိသည်။ Circular Queue တွင် Index များအား ပြန်လည်အသုံးပြုပုံကို အသုံးပြု၍ Queue ၏ အဆုံးနှင့် အစကို ပေါင်းစပ်အသုံးပြုခြင်းဖြင့် အလဟသနေရာကို ရှောင်ရှားနိုင်သည်။ ဒုတိယနည်းလမ်းမှာ Linked List ကို အသုံးပြု၍ Queue ကို အကောင်အထည်ဖော်ခြင်း ဖြစ်သည်။ Linked List ဖြင့် Queue ကို အကောင်အထည်ဖော်ရာတွင် `enqueue` လုပ်သည့်အခါ အသစ်ထည့်သော Node ကို နောက်ဆုံးသို့ ထည့်ပြီး `dequeue` လုပ်သည့်အခါ ရှေ့ဆုံးရှိ Node ကို ဖယ်ရှားခြင်းဖြင့် လုပ်ဆောင်ရသည်။

Queue နှင့် Stack တို့သည် အပြန်အလှန် ဆက်စပ်မှုရှိသည်။ Stack သည် LIFO ကို အခြေခံပြီး အချက်အလက်များကို အထပ်လိုက် ထိန်းသိမ်းရာတွင် အသုံးဝင်သည်။ Queue သည် FIFO ကို အခြေခံပြီး အချက်အလက်များကို အစီအစဉ်တကျ စောင့်ဆိုင်းစေရန်နှင့် စီမံခန့်ခွဲရန် အသုံးဝင်သည်။ Stack သည် အပေါ်မှ ထည့်ပြီး အပေါ်မှ ထုတ်ယူခြင်းကို အလေးပေးပြီး Queue သည် အရင်ထည့်ပြီး အရင်ထုတ်ယူခြင်းကို အလေးပေးသည်။ ပရိုဂရမ်တစ်ခု၏ လိုအပ်ချက်ပေါ်မူတည်၍ မည်သည့် Data Structure ကို ရွေးချယ်မည်ကို ဆုံးဖြတ်ရမည် ဖြစ်သည်။ Stack သည် ပြန်လည်ရယူခြင်း (Reversal/Tracking) နှင့် စနစ်တကျ ပြန်လည်ရယူခြင်း (LIFO) အတွက် ပိုမိုသင့်တော်ပြီး Queue သည် စောင့်ဆိုင်းခြင်း (Scheduling/Buffering) နှင့် အစီအစဉ်တကျ ထည့်သွင်းခြင်း (FIFO) အတွက် ပိုမိုသင့်တော်သည်။

## Stack နှင့် Queue တို့၏ ဆက်စပ်မှုနှင့် ရွေးချယ်မှု (Relationship and Selection)

Stack နှင့် Queue တို့သည် ဒေတာ စီစဉ်ခြင်း၏ အခြေခံကျသော မူဘောင်များ ဖြစ်ကြသည်။ Stack သည် LIFO (Last In, First Out) သဘောတရားကို အခြေခံပြီး အချက်အလက်များကို အထပ်လိုက် ထိန်းသိမ်းခြင်းကို အဓိကထားသည်။ Queue သည် FIFO (First In, First Out) သဘောတရားကို အခြေခံပြီး အချက်အလက်များကို အစီအစဉ်တကျ စောင့်ဆိုင်းစေခြင်းကို အဓိကထားသည်။ ဤနှစ်ခုသည် ဆန့်ကျင်ဘက် သဘောတရားများ ဖြစ်ကြပြီး၊ ၎င်းတို့၏ အသုံးချမှုများသည် မတူညီကြပေ။ Stack သည် အစီအစဉ်တကျ ပြန်လည်ရယူခြင်း (Reversal/Tracking)၊ Function Call Stack များ၊ နှင့် Expression Evaluation များတွင် အလွန်အသုံးဝင်သည်။ Queue သည် အစီအစဉ်တကျ စောင့်ဆိုင်းခြင်း (Scheduling/Buffering)၊ Data Stream များ၊ နှင့် Process Scheduling များတွင် အလွန်အသုံးဝင်သည်။

ပရိုဂရမ်တစ်ခုကို ရေးသားရာတွင် Stack သို့မဟုတ် Queue ကို မည်သို့ ရွေးချယ်မည်ကို ဆုံးဖြတ်ရမည်။ ပထမဆုံး အခြေအနေမှာ ဒေတာများကို တစ်ခုပြီးတစ်ခု ပြန်လည်ရယူရန် လိုအပ်ပါက Stack ကို ရွေးချယ်သင့်သည်။ ဥပမာအားဖြင့်၊ အဖွင့်ကွင်း (opening parenthesis) နှင့် ပိတ်ကွင်း (closing parenthesis) များ၏ မှန်ကန်မှုကို စစ်ဆေးသည့် Parenthesis Matching Checker များတွင် Stack သည် အကောင်းဆုံး ရွေးချယ်မှု ဖြစ်သည်။ ဒုတိယအခြေအနေမှာ အလုပ်များကို အစဉ်လိုက် စောင့်ဆိုင်းပြီး ဆောင်ရွက်ရန် လိုအပ်ပါက Queue ကို ရွေးချယ်ရမည်။ ဥပမာအားဖြင့်၊ ကွန်ပျူတာစနစ်များတွင် Process Scheduling များ၌ Task များအား အစဉ်လိုက် ဆောင်ရွက်ရန် Queue ကို အသုံးပြုသည်။ ထို့ကြောင့် ပရိုဂရမ်၏ ရည်ရွယ်ချက်ပေါ် မူတည်၍ Stack သို့မဟုတ် Queue ကို ရွေးချယ်အသုံးပြုရန် လိုအပ်သည်။

## လက်တွေ့ လေ့ကျင့်ခန်းများ (Practical Exercises)

ဤအခြေခံသဘောတရားများကို လက်တွေ့တွင် အသုံးချနိုင်ရန်အတွက် လက်တွေ့လေ့ကျင့်ခန်းများ ပြုလုပ်ရန် အရေးကြီးပါသည်။ Stack နှင့် Queue ၏ လုပ်ဆောင်ပုံများကို နားလည်ပြီးနောက် ၎င်းတို့ကို C Programming ဘာသာဖြင့် လက်တွေ့ Code ရေးသားပုံကို လေ့လာရမည် ဖြစ်သည်။

ပထမဆုံး လေ့ကျင့်ခန်းမှာ Stack ကို အသုံးပြု၍ Parenthesis Matching Checker (ကွင်း বন্ধ စစ်ဆေးခြင်း) ကို အကောင်အထည်ဖော်ခြင်း ဖြစ်သည်။ ဤလေ့ကျင့်ခန်းတွင် ပွင့်သောကွင်း (opening parenthesis) နှင့် ပိတ်သောကွင်း (closing parenthesis) များ၏ မှန်ကန်မှုကို စစ်ဆေးရန်အတွက် Stack ကို အသုံးပြုရမည်။ Stack တွင် ပွင့်သောကွင်းများကို `push` လုပ်ပြီး ပိတ်သောကွင်းများကို `pop` လုပ်၍ တိုက်ဆိုင်မှုရှိမရှိ စစ်ဆေးရမည် ဖြစ်သည်။ ဤလေ့ကျင့်ခန်းသည် Stack ၏ LIFO သဘောတရားကို နားလည်ရန် အလွန်အထောက်အကူပြုသည်။

ဒုတိယလေ့ကျင့်ခန်းမှာ Queue ကို အသုံးပြု၍ Task Scheduler (လုပ်ငန်းတာဝန် စီစဉ်ပေးသူ) တစ်ခုကို ရေးသားခြင်း ဖြစ်သည်။ ဤလေ့ကျင့်ခန်းတွင် လုပ်ငန်းတာဝန်များကို အစဉ်လိုက် ဆောင်ရွက်ရန်အတွက် Queue ကို အသုံးပြုရမည်။ လုပ်ငန်းတာဝန်များကို `enqueue` လုပ်ပြီး ထို့နောက် `dequeue` လုပ်၍ အစဉ်လိုက် ဆောင်ရွက်ပုံကို ဖော်ပြရမည် ဖြစ်သည်။ ဤလေ့ကျင့်ခန်းသည် Queue ၏ FIFO သဘောတရားကို လက်တွေ့တွင် အသုံးချပုံကို ပြသပေးမည် ဖြစ်သည်။

ဤလေ့ကျင့်ခန်းများကို C Programming ဘာသာဖြင့် လက်တွေ့ Code ရေးသားပုံကိုလည်း လေ့လာရမည်။ Array ကို အသုံးပြု၍ Stack နှင့် Queue တို့ကို အကောင်အထည်ဖော်ပုံကို ဖော်ပြရမည်။ Stack ကို Array ဖြင့် အကောင်အထည်ဖော်ရာတွင် `size` နှင့် `top` index များကို ထိန်းသိမ်းပုံကို ထည့်သွင်းစဉ်းစားရမည်။ Queue ကို Array ဖြင့် အကောင်အထည်ဖော်ရာတွင် `front` နှင့် `rear` index များကို ထိန်းသိမ်းပုံနှင့် Circular Queue ၏ အကျိုးကျေးဇူးကိုလည်း ထည့်သွင်းစဉ်းစားရမည်။ Linked List ကို အသုံးပြု၍ Stack နှင့် Queue တို့ကို အကောင်အထည်ဖော်ပုံကိုလည်း လေ့လာရမည်။ Node များကို အသုံးပြု၍ `push` နှင့် `pop` လုပ်ဆောင်မှုများကို မည်သို့ အကောင်အထည်ဖော်ပုံကို နားလည်ရမည် ဖြစ်သည်။

## အခန်း၏ အနှစ်ချုပ် (Chapter Summary)

ဤအခန်းတွင် Stack နှင့် Queue တို့သည် ဒေတာများကို စီစဉ်သိမ်းဆည်းရန်အတွက် အခြေခံကျသော Data Structure များအဖြစ် မိတ်ဆက်ပေးခဲ့ပါသည်။ Stack သည် LIFO (Last In, First Out) သဘောတရားကို အခြေခံပြီး ဒေတာများကို အထပ်လိုက် ထိန်းသိမ်းခြင်းကို အဓိကထားသည်။ Stack ၏ အခြေခံလုပ်ဆောင်ချက်များမှာ `push`, `pop`, `peek`/`top`, `isEmpty`, နှင့် `isFull` တို့ ဖြစ်သည်။ Stack ကို Array နှင့် Linked List တို့ဖြင့် အကောင်အထည်ဖော်နိုင်ကြောင်း လေ့လာခဲ့ပြီး၊ Function Call Stack၊ Undo Operation နှင့် Expression Evaluation ကဲ့သို့သော လက်တွေ့အသုံးချမှုများကိုလည်း မိတ်ဆက်ခဲ့သည်။

Queue နှင့် ပတ်သက်၍မူ FIFO (First In, First Out) သဘောတရားကို အခြေခံပြီး ဒေတာများကို အစီအစဉ်တကျ စောင့်ဆိုင်းစေခြင်းကို အဓိကထားသည်။ Queue ၏ အခြေခံလုပ်ဆောင်ချက်များမှာ `enqueue`, `dequeue`, `front`, `rear`, `isEmpty`, နှင့် `isFull` တို့ ဖြစ်သည်။ Array ကို အသုံးပြု၍ Queue ကို အကောင်အထည်ဖော်ခြင်းနှင့် Space Utilization ပြဿနာကို ဖြေရှင်းရန် Circular Queue ကို မိတ်ဆက်ပေးခဲ့သည်။

Stack နှင့် Queue တို့သည် အပြန်အလှန် ဆက်စပ်မှုရှိပြီး၊ မည်သည့်အခါတွင် မည်သည့်နည်းလမ်းကို ရွေးချယ်ရမည်ကို ဆုံးဖြတ်ရန် လိုအပ်သည်။ Stack သည် ပြန်လည်ရယူခြင်းနှင့် အထပ်လိုက် စီစဉ်မှုများအတွက်၊ Queue သည် စောင့်ဆိုင်းခြင်းနှင့် အစီအစဉ်တကျ ထည့်သွင်းမှုများအတွက် ပိုမိုသင့်တော်သည်။ ပရိုဂရမ်တစ်ခု၏ လိုအပ်ချက်ပေါ်မူတည်၍ မှန်ကန်သော Data Structure ကို ရွေးချယ်အသုံးပြုနိုင်ရန် အရေးကြီးသည်။

ကျွန်ုပ်တို့သည် Stack နှင့် Queue တို့၏ အခြေခံသဘောတရားများကို နားလည်ပြီးနောက် Data Structure များ၏ အခြားအမျိုးအစားများဖြစ်သည့် Tree နှင့် Graph များအကြောင်းနှင့် ၎င်းတို့၏ အဆင့် (Big O Notation) များအကြောင်းကို နောက်အခန်းများတွင် ဆက်လက်လေ့လာသွားကြမည် ဖြစ်သည်။ Stack နှင့် Queue များသည် ကွန်ပျူတာ သိပ္ပံ၏ အခြေခံကျသော မူဘောင်များဖြစ်ပြီး ၎င်းတို့ကို ကျွမ်းကျင်စွာ အသုံးပြုတတ်ရန်မှာ ပရိုဂရမ်မင်း၏ အခြေခံကျသော ကျွမ်းကျင်မှုများထဲမှ တစ်ခုဖြစ်ကြောင်း အလေးပေးပြောကြားလိုပါသည်။ ဤအခြေခံများအား ကျွမ်းကျင်စွာ အသုံးပြုတတ်ခြင်းဖြင့် ကျွန်ုပ်တို့သည် ပိုမိုရှုပ်ထွေးသော ကွန်ပျူတာစနစ်များနှင့် Algorithm များကို အောင်မြင်စွာ တည်ဆောက်နိုင်မည် ဖြစ်ပါသည်။
