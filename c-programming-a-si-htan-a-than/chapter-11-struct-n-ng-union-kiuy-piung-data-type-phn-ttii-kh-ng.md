# Chapter 11: Struct နှင့် Union — ကိုယ်ပိုင် Data Type ဖန်တီးခြင်း

## ၁။ အဖွင့်အစနှင့် မိတ်ဆက် (Introduction and Hook)

ပရိုဂရမ်ရေးသားရာတွင် ကျွန်ုပ်တို့သည် မကြာခဏဆိုသလို ရှုပ်ထွေးသော Data အမျိုးအစားများနှင့် စုစည်းမှုများကို ကိုင်တွယ်ရလေ့ရှိသည်။ ဥပမာအားဖြင့်၊ ကျောင်းသားတစ်ဦး၏ အမည်၊ အသက်၊ ကျား/မ အခြေအနေနှင့် စာမေးပွဲ အမှတ် စသည့် အချက်အလက်များကို တစ်စုတည်း စီမံခန့်ခွဲရန် လိုအပ်သည်။ ထိုအချက်အလက်များကို သီးခြား `int`, `char`, `float` စသည့် Data အမျိုးအစားများအဖြစ် သီးခြားသတ်မှတ်ထားပါက၊ ၎င်းတို့ကို ပေါင်းစပ်၍ စီမံခန့်ခွဲရာတွင် အလွန်ရှုပ်ထွေးလာပြီး စနစ်မကျတော့ပေ။ ထို့ကြောင့် C Programming တွင် ကျွန်ုပ်တို့သည် Data အမျိုးအစားများကို ကိုယ်ပိုင်ပုံစံများ ဖန်တီးနိုင်သည့် စွမ်းရည်ကို ရရှိသည်။ ဤအခန်းတွင် ကျွန်ုပ်တို့သည် ထိုစွမ်းရည်ကို အခြေခံပေးမည့် `struct` (အဆောက်အအုံ) နှင့် `union` (ပေါင်းစပ်မှု) တို့အကြောင်းကို လေ့လာသွားမည် ဖြစ်သည်။

`struct` နှင့် `union` များသည် ပရိုဂရမ်ရေးသားရာတွင် ရှုပ်ထွေးသော Data များကို စနစ်တကျ စီမံခန့်ခွဲရန် အလွန်အသုံးဝင်သည်။ `struct` သည် မတူညီသော Data အမျိုးအစားများ (ဥပမာ - integer, character, float) ကို တစ်ခုတည်းသော Data Type အဖြစ် ပေါင်းစည်းခြင်းကို ဆိုလိုသည်။ ၎င်းသည် လက်တွေ့ဘဝရှိ အရာဝတ္ထုများကို ကိုယ်စားပြုရန် အလွန်ကောင်းမွန်သော နည်းလမ်းဖြစ်သည်။ ထို့အပြင် `union` သည် Memory Sharing (အမှတ်နေရာ မျှဝေခြင်း) ဟူသော အခြေခံသဘောတရားကို နားလည်စေရန် ကူညီပေးသည်။

ကျွန်ုပ်တို့သည် ဤအခန်းတွင် Data ကို တစ်စုတည်း ထုပ်ပိုးခြင်း (Grouping) နှင့် Memory Sharing တို့၏ အခြေခံသဘောတရားများကို နားလည်သွားကြမည် ဖြစ်သည်။ ဤအခြေခံသဘောတရားများကို နားလည်ခြင်းဖြင့် ကျွန်ုပ်တို့သည် ပိုမိုကျယ်ပြန့်သော Data Structure များကို တည်ဆောက်နိုင်ပြီး ပိုမိုထိရောက်သော ပရိုဂရမ်များကို ရေးသားနိုင်မည် ဖြစ်သည်။ ကျောင်းသားမှတ်တမ်းများ၊ ပစ္စည်းစာရင်းများ စသည့် လက်တွေ့အခြေအနေများကို ကိုယ်ပိုင် Data Type များဖြင့် မည်သို့ပုံဖော်နိုင်ပုံကို လက်တွေ့ဥပမာများဖြင့် လေ့လာသွားကြမည် ဖြစ်သည်။

## ၂။ Struct ၏ အခြေခံများ (Fundamentals of Struct)

### Struct ဆိုသည်မှာ အဘယ်နည်း

`struct` (အဆောက်အအုံ) ဆိုသည်မှာ မတူညီသော Data အမျိုးအစားများ (ဥပမာ - `char`, `int`, `float`) ကို တစ်ခုတည်းသော Data Type အဖြစ် ပေါင်းစည်းခြင်း ဖြစ်သည်။ ၎င်းသည် သတ်မှတ်ထားသော အချက်အလက်များအား တစ်စုတည်း ထုပ်ပိုးထားသော အရာဝတ္ထုတစ်ခုအဖြစ် သတ်မှတ်ခြင်းပင် ဖြစ်သည်။ ဥပမာအားဖြင့်၊ ကျောင်းသားတစ်ဦး၏ အချက်အလက်များကို ကိုယ်စားပြုရန်အတွက် အမည် (string), အသက် (integer), အမှတ် (float) စသည်တို့ကို တစ်နေရာတည်းတွင် သိမ်းဆည်းနိုင်ရန် `struct` ကို အသုံးပြုသည်။ ဤနည်းလမ်းသည် Data များကို ပိုမိုရှင်းလင်းပြီး စီမံခန့်ခွဲရလွယ်ကူစေသည်။

### Struct Declaration နှင့် Initialization

`struct` တစ်ခုကို ပရိုဂရမ်ထဲတွင် အသုံးပြုရန်အတွက် ဦးစွာ `struct` ကို ကြေညာရန် လိုအပ်သည်။ ဤကြေညာချက်တွင် ထို `struct` ထဲတွင် မည်သည့် Data အမျိုးအစားများ ပါဝင်မည်ကို သတ်မှတ်ပေးရသည်။ ထို့နောက် ထို `struct` ၏ အစိတ်အပိုင်းများ (members) ကို အမည်ပေးရမည်။

ဥပမာအားဖြင့်၊ ကျောင်းသားအချက်အလက်များအတွက် `struct` တစ်ခုကို ကြေညာပုံမှာ အောက်ပါအတိုင်း ဖြစ်သည်။

```c
struct Student {
    char name[50];  // Student ၏ အမည် (အက္ခရာများ)
    int age;         // Student ၏ အသက် (ကျ数値)
    float marks;     // Student ၏ အမှတ် (အမြောက်အမြား)
};
```

ဤကြေညာချက်သည် `struct Student` ဟု အမည်ပေးထားသော Data အမျိုးအစားအသစ်တစ်ခုကို ဖန်တီးသည်။ ထို့နောက် ဤ `struct` ကို အသုံးပြုရန်အတွက် ကျွန်ုပ်တို့သည် ၎င်း၏ အခြေအနေများကို သတ်မှတ်ရန် `initialization` (တန်ဖိုးများ ထည့်သွင်းခြင်း) ပြုလုပ်ရမည်။

```c
struct Student student1; // Student အရာဝတ္ထုတစ်ခုကို ကြေညာခြင်း (Declaration)

// Initialization ပြုလုပ်ခြင်း (တန်ဖိုးများ ထည့်သွင်းခြင်း)
strcpy(student1.name, "Aung Myint"); // string ကို ထည့်ရန် strcpy ကို အသုံးပြုရမည်
student1.age = 20;
student1.marks = 85.5;
```

ဤနေရာတွင် `strcpy` (string copy) function ကို အသုံးပြုရခြင်းက စာကြောင်း (string) များကို စနစ်တကျ သိမ်းဆည်းရန် လိုအပ်ကြောင်း ပြသသည်။ ဤကဲ့သို့ `initialization` ပြုလုပ်ခြင်းဖြင့် ကျွန်ုပ်တို့သည် `student1` ဟူသော `struct` အရာဝတ္ထုကို သတ်မှတ်ထားသော တန်ဖိုးများဖြင့် စတင်အသုံးပြုနိုင်ပြီ ဖြစ်သည်။

### Struct ၏ အစိတ်အပိုင်းများသို့ ဝင်ရောက်ခြင်း (Accessing Members)

`struct` တစ်ခု၏ အတွင်းရှိ Data အစိတ်အပိုင်းများသို့ ဝင်ရောက်၍ တန်ဖိုးများကို ဖတ်ရှုခြင်း သို့မဟုတ် ပြောင်းလဲခြင်းတို့ကို `dot operator` (`.`) ကို အသုံးပြု၍ လုပ်ဆောင်နိုင်သည်။ ဤသည်မှာ `struct` ၏ အဓိက အသုံးပြုပုံ ဖြစ်သည်။

ယခင်က ဖော်ပြခဲ့သော `student1` struct မှ အသက်ကို ဖတ်ရှုရန်အတွက် အောက်ပါအတိုင်း ရယူနိုင်သည်။

```c
printf("Student Name: %s\n", student1.name);
printf("Student Age: %d\n", student1.age);
printf("Student Marks: %.2f\n", student1.marks);
```

ဤနည်းလမ်းသည် `struct` ထဲရှိ သီးခြား Data အမျိုးအစားများကို ၎င်းတို့၏ အမည်ဖြင့် တိုက်ရိုက် ရည်ညွှန်း၍ အသုံးပြုနိုင်စေသည်။ ဤသို့ဖြင့် Data များကို စနစ်တကျ စီမံခန့်ခွဲရာတွင် အလွန်လွယ်ကူလာသည်။

### Struct နှင့် Pointer (အညွှန်းပြချက်)

`struct` အရာဝတ္ထုများကို Pointer (ညွှန်ပြချက်) နှင့် ပေါင်းစပ်အသုံးပြုခြင်းသည် C Programming တွင် အလွန်အရေးကြီးသော အခန်းကဏ္ဍမှ ပါဝင်သည်။ `struct` တစ်ခုလုံးကို Pointer တစ်ခုဖြင့် သိမ်းဆည်းခြင်းဖြင့် ကျွန်ုပ်တို့သည် ထိုအရာဝတ္ထု၏ Memory Address ကို သိရှိပြီး ထိုနေရာမှတဆင့် တိုက်ရိုက်ဝင်ရောက်၍ Data များကို အသုံးပြုနိုင်သည်။

`struct` ကို Pointer နှင့် အသုံးပြုပုံကို ရှင်းပြရမည်ဆိုလျှင် `arrow operator` (`->`) ကို အသုံးပြုရမည် ဖြစ်သည်။ `->` operator သည် `(*ptr).member` ဟူသော ပုံစံနှင့် အတူတူပင် ဖြစ်သည်။ ဤ operator သည် Pointer တစ်ခုမှ ထို Pointer ညွှန်ပြနေသော `struct` ၏ အစိတ်အပိုင်းများသို့ တိုက်ရိုက်ဝင်ရောက်ရန် အထူးရည်ရွယ်သည်။

ဥပမာအားဖြင့်၊ `student1` struct ကို Pointer တစ်ခုဖြင့် သိမ်းဆည်းပြီး ၎င်းကို `studentPtr` ဟု အမည်ပေးလိုက်ရအောင်။

```c
struct Student student1;
struct Student *studentPtr; // Student struct ၏ လိပ်စာကို သိမ်းဆည်းမည့် Pointer

studentPtr = &student1; // student1 ၏ လိပ်စာကို studentPtr ထဲသို့ ထည့်သွင်းခြင်း

// Arrow operator (->) ကို အသုံးပြု၍ တန်ဖိုးများသို့ ဝင်ရောက်ခြင်း
printf("Student Name (via Pointer): %s\n", studentPtr->name);
printf("Student Age (via Pointer): %d\n", studentPtr->age);
```

ဤနည်းလမ်းသည် `struct` များကို အကြီးအကျယ် စီမံခန့်ခွဲရာတွင် အလွန်ထိရောက်သည်။ အထူးသဖြင့် `Array of Structs` များကဲ့သို့သော ပုံစံများတွင် Memory Address များကို အသုံးပြု၍ အချက်အလက်များကို အလွယ်တကူ ရယူနိုင်သောကြောင့် ပရိုဂရမ်၏ စွမ်းဆောင်ရည်ကို မြှင့်တင်ပေးသည်။

## ၃။ Struct နှင့် Function များ၊ Array များ (Structs with Functions and Arrays)

### Struct ကို Function များထဲသို့ ပေးပို့ခြင်း (Passing Structs to Functions)

`struct` အရာဝတ္ထုများကို `function` များအတွင်းသို့ ပေးပို့သည့်အခါ မည်သည့်နည်းလမ်းကို အသုံးပြုမည်ကို ဆုံးဖြတ်ရန် လိုအပ်သည်။ ဤနေရာတွင် `call by value` နှင့် `call by reference` (သို့မဟုတ် `pass by pointer`) အကြား ကွာခြားချက်ကို နားလည်ရန် အရေးကြီးသည်။

**By value (တန်ဖိုးဖြင့် ပေးပို့ခြင်း):** Function တစ်ခုသို့ `struct` ကို တန်ဖိုးဖြင့် ပေးပို့သောအခါ၊ ထို `struct` ၏ အချက်အလက်များအားလုံးကို Function ထဲသို့ ကော်ပီ (copy) အဖြစ် ပေးပို့သည်။ Function အတွင်းမှ ပြုလုပ်သော ပြောင်းလဲမှုများသည် မူရင်း `struct` ၏ တန်ဖိုးကို မထိခိုက်ပါ။ ၎င်းသည် အသေးစား Data များအတွက် လုံလောက်သော်လည်း၊ ကြီးမားသော `struct` များအတွက် Memory ကို အလဟဿ အသုံးပြုစေနိုင်သည်။

**By pointer (ညွှန်ပြချက်ဖြင့် ပေးပို့ခြင်း):** Function တစ်ခုသို့ `struct` ၏ Memory Address (Pointer) ကို ပေးပို့သောအခါ၊ Function သည် မူရင်း `struct` ကို တိုက်ရိုက်မထိဘဲ ထို Memory Address မှတဆင့် မူရင်း `struct` သို့ ဝင်ရောက်၍ တန်ဖိုးများကို ပြောင်းလဲနိုင်သည်။ ဤနည်းလမ်းသည် ကြီးမားသော Data Structures များကို ကိုင်တွယ်ရာတွင် Memory ကို ထိရောက်စွာ အသုံးပြုနိုင်ပြီး ပိုမိုထိရောက်မှုရှိသည်။

Function များအား `struct` တစ်ခုကို လက်ခံပြီး ပြန်ပေးနိုင်ပုံကိုလည်း လေ့လာရမည်။ Function တစ်ခုသည် `struct` ကို ပြန်ပေးရန်အတွက် `return` statement ဖြင့် `struct` ကို တိုက်ရိုက်ပြန်ပေးနိုင်သည်မဟုတ်ဘဲ၊ `struct` ၏ Pointer ကို ပြန်ပေးရမည်။

ဥပမာအားဖြင့်၊ ကျောင်းသား၏ အမှတ်များကို ပြန်ပေးသည့် function တစ်ခုကို ဖန်တီးကြည့်ရအောင်။

```c
// Function definition: Function တစ်ခုက struct pointer ကို လက်ခံပြီး အမှတ်များကို ပြန်ပေးမည်
void displayMarks(struct Student *s) {
    printf("Name: %s, Age: %d, Marks: %.2f\n", s->name, s->age, s->marks);
}

int main() {
    struct Student student1;
    // Initialization... (အပေါ်ကအတိုင်း)

    // Function ကို call လုပ်ရာတွင် struct pointer ကို ပေးပို့ခြင်း (Call by reference)
    displayMarks(&student1); 

    return 0;
}
```

ဤဥပမာတွင် `displayMarks` function သည် `student1` ၏ လိပ်စာကို လက်ခံရရှိပြီး၊ `->` operator ကို အသုံးပြု၍ ထိုလိပ်စာမှတဆင့် အချက်အလက်များကို ဖတ်ယူပြသသည်။ ဤနည်းလမ်းသည် ပိုမိုထိရောက်ပြီး Memory ကို ချွေတာသည်။

### Nested Structs (အထပ်ထပ် Struct များ)

`nested struct` (အထပ်ထပ် Struct များ) ဆိုသည်မှာ `struct` တစ်ခု၏ အစိတ်အပိုင်းအဖြစ် နောက်ထပ် `struct` တစ်ခုကို ထည့်သွင်းခြင်း ဖြစ်သည်။ ဤနည်းလမ်းသည် ရှုပ်ထွေးသော Data များကို ပိုမိုကျိုးကြောင်းဆီလျော်စွာ စီစဉ်ပေးသည်။ ဥပမာအားဖြင့်၊ ကျောင်းသား၏ အချက်အလက်များတွင် အမည်၊ အသက်နှင့် ဆရာ၏ အချက်အလက်များကို ပေါင်းစပ်ထားသည့် ပုံစံမျိုး ဖြစ်သည်။

```c
struct Address {
    char street[50];
    char city[30];
    int zip_code;
};

struct Student {
    char name[50];
    int age;
    float marks;
    struct Address home_address; // Nested Struct
};
```

ဤပုံစံဖြင့် `struct Student` တစ်ခုသည် ၎င်း၏ကိုယ်ပိုင် အချက်အလက်များအပြင် `home_address` ဟု အမည်ပေးထားသော နောက်ထပ် `struct Address` တစ်ခုကို အစိတ်အပိုင်းအဖြစ် ထည့်သွင်းထားသည်။ ဤနည်းလမ်းဖြင့် Data များအား အကြောင်းအရာအလိုက် ခွဲခြားသတ်မှတ်နိုင်ပြီး ပရိုဂရမ်ကို ပိုမိုရှင်းလင်းစေသည်။

### Array of Structs (Struct များ၏ Array များ)

`array of structs` (Struct များ၏ Array များ) ဆိုသည်မှာ တူညီသော `struct` အမျိုးအစားများစွာကို Array အဖြစ် စီမံခန့်ခွဲခြင်း ဖြစ်သည်။ ဤနည်းလမ်းသည် ကျောင်းသားများ၊ ပစ္စည်းများ စသည့် အရာဝတ္ထုများစွာ၏ စာရင်းများကို စီမံခန့်ခွဲရန် အလွန်အသုံးဝင်သည်။

```c
#define NUM_STUDENTS 3
struct Student students[NUM_STUDENTS]; // Student struct ၏ အရာဝတ္ထု ၃ ခုအတွက် Array ဖန်တီးခြင်း

// Array ထဲရှိ အချက်အလက်များကို ထည့်သွင်းခြင်း
strcpy(students[0].name, "Aung Myint");
students[0].age = 20;
students[0].marks = 85.5;

strcpy(students[1].name, "Ma Hla");
students[1].age = 21;
students[1].marks = 92.0;

// Array ထဲရှိ အချက်အလက်များကို ပြသခြင်း
printf("Student 1 Name: %s, Age: %d\n", students[0].name, students[0].age);
printf("Student 2 Name: %s, Age: %d\n", students[1].name, students[1].age);
```

ဤနည်းလမ်းဖြင့် ကျွန်ုပ်တို့သည် ကျောင်းသား သုံးဦး၏ အချက်အလက်များကို Array တစ်ခုတည်းတွင် စနစ်တကျ သိမ်းဆည်းထားနိုင်ပြီး၊ `for` loop များဖြင့် ထိုအချက်အလက်များကို အလွယ်တကူ စီမံခန့်ခွဲနိုင်သည်။ ဤသည်မှာ Data Structure များ၏ အခြေခံကျသော နည်းလမ်းတစ်ခု ဖြစ်သည်။

## ၄။ Data Type ကို ပြောင်းလဲခြင်းနှင့် အမည်ပေးခြင်း (Aliasing and Constants)

### typedef (အမည်ပေးခြင်း)

`typedef` (အမည်ပေးခြင်း) သည် C Programming တွင် အလွန်အသုံးဝင်သော Keyword တစ်ခု ဖြစ်သည်။ ၎င်းသည် မည်သည့် Data Type ကိုမဆို အခြားအမည်တစ်ခု (alias) ပေးနိုင်ခြင်းဖြင့် Code ကို ပိုမိုဖတ်ရှုရလွယ်ကူစေသည်။ `typedef` ကို အသုံးပြုခြင်းဖြင့် ကျွန်ုပ်တို့သည် ရှည်လျားသော `struct` သို့မဟုတ် ရှုပ်ထွေးသော `typedef` များကို အစားထိုး၍ ပိုမိုတိုတောင်းသော အမည်များဖြင့် ရေးသားနိုင်ကြသည်။

ဥပမာအားဖြင့်၊ `struct Student` ကို `StudentRecord` ဟု အမည်ပေးခြင်းဖြင့် ရေးသားနိုင်သည်။

```c
typedef struct Student {
    char name[50];
    int age;
    float marks;
} StudentRecord; // Student struct ကို StudentRecord အမည်ဖြင့် သတ်မှတ်ခြင်း

int main() {
    // StudentRecord ကို အသုံးပြုခြင်း
    StudentRecord student1;
    strcpy(student1.name, "Aung Myint");
    student1.age = 20;
    student1.marks = 85.5;
    
    printf("Student Name: %s, Age: %d\n", student1.name, student1.age);
    return 0;
}
```

`typedef` ကို အသုံးပြုခြင်းဖြင့် Code ကို ပိုမိုရှင်းလင်းပြီး လူသားတို့ နားလည်လွယ်စေသည်။ ၎င်းသည် Data Type များအတွက် အမည်ပေးခြင်း (aliasing) သဘောတရားကို အခြေခံသည်။

### enum (Enumeration)

`enum` (Enumeration) သည် ဆက်စပ်နေသော မတူညီသည့် Constant (တန်ဖိုးများ) များကို အမည်ပေးခြင်း ဖြစ်သည်။ ၎င်းသည် ပရိုဂရမ်၏ အခြေအနေများ (states) သို့မဟုတ် ရွေးချယ်စရာများ (choices) ကို သတ်မှတ်ရန် အလွန်အသုံးဝင်သည်။ `enum` သည် တန်ဖိုးများကို နံပါတ်များဖြင့် ကိုယ်စားပြုသော်လည်း၊ ထိုနံပါတ်များကို ပိုမိုနားလည်လွယ်သော အမည်များဖြင့် အစားထိုးနိုင်စေသည်။

ဥပမာအားဖြင့်၊ ကျား/မ အခြေအနေအတွက် `0` နှင့် `1` ကို အစားထိုးမည့်အစား `Male` နှင့် `Female` ဟူသော အမည်များကို အသုံးပြုနိုင်သည်။

```c
enum Gender { MALE, FEMALE }; // Gender အတွက် enumeration ဖန်တီးခြင်း

int main() {
    enum Gender studentGender = MALE; // studentGender ကို MALE တန်ဖိုးဖြင့် သတ်မှတ်ခြင်း

    if (studentGender == MALE) {
        printf("Gender is Male\n");
    } else if (studentGender == FEMALE) {
        printf("Gender is Female\n");
    }
    
    return 0;
}
```

`enum` သည် Code ကို ပိုမိုရှင်းလင်းစေပြီး၊ မည်သည့် တန်ဖိုးများကို လက်ခံနိုင်သည်ကိုလည်း သတ်မှတ်ပေးသည်။ ဤနည်းလမ်းသည် ပရိုဂရမ်၏ Logic ကို ပိုမိုကောင်းမွန်စေသည်။

### Bit Field (Bit အခြေခံ)

`bit field` (Bit အခြေခံ) သည် Memory ကို `bit` (ဘစ်) အဆင့်တွင် ခွဲဝေခြင်း၏ အခြေခံသဘောတရား ဖြစ်သည်။ `bit` ဆိုသည်မှာ `0` သို့မဟုတ် `1` ကို ကိုယ်စားပြုသော အသေးဆုံး Data Unit ဖြစ်သည်။ `bit field` သည် ထို `bit` အစုအဝေးကို အသုံးပြု၍ Memory နေရာအနည်းငယ်ကို ထိရောက်စွာ အသုံးချနိုင်စေသည်။ ဥပမာအားဖြင့်၊ ကျား/မ အခြေအနေတစ်ခုကို သိမ်းဆည်းရန်အတွက် `char` တစ်လုံး (8 bit) အစား၊ `bit` ၃ လုံး (သို့မဟုတ် `unsigned int` တစ်ခု) ကို အသုံးပြု၍ ထိုအချက်အလက်ကို ကိုယ်စားပြုနိုင်သည်။ ဤနည်းလမ်းသည် အလွန်နည်းသော Memory နေရာကို ထိရောက်စွာ အသုံးချနိုင်စေသည်။

`bit field` ကို အသုံးပြုရာတွင် `struct` ထဲရှိ မည်သည့် Member ကို မည်သည့် `bit` အပိုင်းအတွက် သတ်မှတ်မည်ကို သေချာစွာ သိရှိရန် လိုအပ်သည်။ ၎င်းသည် Memory Management နှင့် Hardware Level တွင် အလွန်အရေးကြီးသော အခြေခံသဘောတရား ဖြစ်သည်။

## ၅။ Struct နှင့် Union ၏ ကွာခြားချက် (Struct vs. Union)

`struct` နှင့် `union` တို့သည် C Programming တွင် Data ကို စုစည်းရာ၌ အလွန်အရေးကြီးသော ကွာခြားချက်များ ရှိသည်။ ၎င်းတို့နှစ်ခုစလုံးသည် Data ကို စုစည်းရန် အသုံးပြုသော်လည်း Memory ကို စီမံခန့်ခွဲပုံနှင့် Data ကို ထည့်သွင်းပုံတွင် မတူညီကြပေ။

### Union ဆိုသည်မှာ အဘယ်နည်း

`union` (ပေါင်းစပ်မှု) ဆိုသည်မှာ `struct` နှင့် `union` တို့၏ အဓိက ကွာခြားချက်ကို ရှင်းပြသည်။ `union` သည် မတူညီသော Data အမျိုးအစားများထဲမှ တစ်ခုတည်းအတွက်သာ Memory ကို မျှဝေစေသည်။ ဆိုလိုသည်မှာ `union` တစ်ခုတွင် ထည့်သွင်းထားသော Data အားလုံးသည် တစ်ပြိုင်နက်တည်း Memory နေရာကို မျှဝေသည်။ မည်သည့် Data ကို အသုံးပြုမည်ကို ပရိုဂရမ်မှ ဆုံးဖြတ်ရမည် ဖြစ်သည်။

### Memory Allocation ၏ ကွာခြားချက်

**Struct ၏ Memory Allocation:** `struct` သည် ထည့်သွင်းထားသော Data အမျိုးအစားများ (ဥပမာ - `int`, `float`, `char`) အတွက် လိုအပ်သော Memory အားလုံးကို ထိန်းသိမ်းထားသည်။ ဥပမာအားဖြင့်၊ `struct Student` သည် `name` အတွက် လုံလောက်သော Memory၊ `age` အတွက် လုံလောက်သော Memory နှင့် `marks` အတွက် လုံလောက်သော Memory တို့အတွက် လိုအပ်သော စုစုပေါင်း Memory ပမာဏကို ထိန်းသိမ်းထားသည်။

**Union ၏ Memory Allocation:** `union` သည် ထည့်သွင်းထားသော Data များထဲမှ တစ်ခုတည်းအတွက်သာ Memory ကို မျှဝေသည်။ ဥပမာအားဖြင့်၊ `union Data` တစ်ခုတွင် `int` နှင့် `float` နှစ်ခုကို ထည့်သွင်းထားပါက၊ ထို `union` သည် `int` နှင့် `float` နှစ်ခုလုံးအတွက် လိုအပ်သော Memory ပမာဏကိုသာ ထိန်းသိမ်းသည်။ အကယ်၍ ကျွန်ုပ်တို့သည် `int` ကိုသာ အသုံးပြုမည်ဆိုလျှင် ထို Memory ကို `int` အတွက်သာ အသုံးပြုမည် ဖြစ်သည်။

### လက်တွေ့ ဥပမာဖြင့် နှိုင်းယှဉ်ခြင်း

ကျောင်းသားမှတ်တမ်းတွင် အသက် (`int`) နှင့် အမှတ် (`float`) နှစ်ခုကို သိမ်းဆည်းရာတွင် `struct` ကို အသုံးပြုခြင်းနှင့် `union` ကို အသုံးပြုခြင်း၏ ကွာခြားချက်ကို ဥပမာပေးကြည့်ရအောင်။

**Struct ဖြင့် သိမ်းဆည်းခြင်း:**

```c
struct Student {
    int age;
    float marks;
};
// struct Student သည် age နှင့် marks အတွက် လိုအပ်သော Memory အားလုံးကို ထိန်းသိမ်းသည်။
```

**Union ဖြင့် သိမ်းဆည်းခြင်း:**

```c
union Data {
    int age;
    float marks;
};
// union Data သည် age နှင့် marks နှစ်ခုအတွက် လိုအပ်သော Memory အားလုံးကို မျှဝေသည်။
```

Struct သည် Data အမျိုးအစားအလိုက် သီးခြား Memory နေရာများအတွက် သင့်လျော်ပြီး၊ Union သည် မတူညီသော Data အမျိုးအစားများထဲမှ တစ်ခုတည်းအတွက်သာ Memory ကို မျှဝေလိုသောအခါ အသုံးပြုသည်။ ဤကွာခြားချက်ကို နားလည်ခြင်းဖြင့် Memory ကို ထိရောက်စွာ စီမံခန့်ခွဲနိုင်မည် ဖြစ်သည်။

## ၆။ လက်တွေ့အသုံးချမှုနှင့် နိဂုံး (Practical Application and Conclusion)

### လက်တွေ့စီမံခန့်ခွဲရေး ပရိုဂရမ် (Practical Implementation)

အထက်ပါ သင်ယူခဲ့သော `struct` များ၏ အခြေခံများကို အသုံးပြု၍ ကျောင်းသား မှတ်တမ်းများ (Student Records) ကို စီမံခန့်ခွဲသော ပရိုဂရမ်ကို ရေးသားကြည့်ရအောင်။ ဤပရိုဂရမ်သည် `struct` များ၊ `array` များ၊ နှင့် `function` များတို့ကို ပေါင်းစပ်အသုံးပြုမည် ဖြစ်သည်။

```c
#include <stdio.h>
#include <string.h>

// 1. Struct Definition (အဆောက်အအုံကို သတ်မှတ်ခြင်း)
struct Student {
    char name[50];
    int age;
    float marks;
};

// 2. Function Definition (လုပ်ဆောင်ချက်များ)
void displayStudentInfo(struct Student s) { // By value (copy)
    printf("Name: %s, Age: %d, Marks: %.2f\n", s.name, s.age, s.marks);
}

void displayStudentInfoPtr(struct Student *s) { // By pointer (reference)
    printf("Name: %s, Age: %d, Marks: %.2f\n", s->name, s->age, s->marks);
}

int main() {
    // 3. Struct Declaration and Initialization (ကြေညာခြင်းနှင့် တန်ဖိုးထည့်သွင်းခြင်း)
    struct Student student1 = {"Aung Myint", 20, 85.5};
    struct Student student2 = {"Ma Hla", 21, 92.0};

    // 4. Array of Structs (Struct များ၏ Array များ)
    struct Student students[2] = {student1, student2};

    printf("--- Student Records ---\n");
    
    // 5. Accessing Members (အချက်အလက်များ ဝင်ရောက်ခြင်း)
    printf("Student 1 Details:\n");
    printf("Name: %s, Age: %d, Marks: %.2f\n", students[0].name, students[0].age, students[0].marks);
    
    printf("\nStudent 2 Details:\n");
    printf("Name: %s, Age: %d, Marks: %.2f\n", students[1].name, students[1].age, students[1].marks);

    printf("\n--- Display using Function ---\n");

    // Function ကို By value ဖြင့် ခေါ်ခြင်း
    displayStudentInfo(student1); 

    // Function ကို By pointer ဖြင့် ခေါ်ခြင်း
    displayStudentInfoPtr(&students[0]); // students[0] ၏ လိပ်စာကို ပေးပို့ခြင်း

    return 0;
}
```

ဤပရိုဂရမ်သည် `struct` များဖြင့် ကျောင်းသားအချက်အလက်များကို စီမံခန့်ခွဲပုံ၊ `array` ဖြင့် စုစည်းပုံနှင့် `function` များဖြင့် လုပ်ဆောင်ပုံတို့ကို ပြသထားသည်။ ဤသည်မှာ လက်တွေ့တွင် Data Structure များကို မည်သို့ တည်ဆောက်ရမည်ကို နားလည်စေသည်။

### Code Review နှင့် Debugging

`struct` နှင့် `pointer` များအသုံးပြုရာတွင် အမှားများဖြစ်တတ်သည်။ အထူးသဖြင့် `pointer` များတွင် `dangling pointer` (အညွှန်းပြချက် လွဲမှားခြင်း) သို့မဟုတ် `memory leak` (မှတ်ဉာဏ် ပေါက်ကြားခြင်း) များ ဖြစ်ပေါ်တတ်သည်။ `memory leak` ဆိုသည်မှာ ပရိုဂရမ်မှ အသုံးပြုပြီးနောက် ထို Memory ကို ပြန်လည်လွှတ်ပေးရန် မေ့လျော့ခြင်းကြောင့် ဖြစ်ပေါ်သည်။ ဤကဲ့သို့သော အမှားများကို ရှောင်ရှားရန်အတွက် `void*` (generic pointer) နှင့် `malloc`/`free` functions များကို သတိထား၍ အသုံးပြုရန် လိုအပ်သည်။

ဤကဲ့သို့သော ရှုပ်ထွေးသော အခြေအနေများကို Debug လုပ်ရန်အတွက် `GDB` (GNU Debugger) ကဲ့သို့သော Tool များကို အသုံးပြုခြင်းသည် အလွန်အရေးကြီးသည်။ `GDB` သည် ပရိုဂရမ်၏ Runtime အခြေအနေကို စစ်ဆေးနိုင်ပြီး၊ `memory` နှင့် `pointer` များ၏ မှန်ကန်မှုကို စစ်ဆေးရာတွင် အထောက်အကူပြုသည်။ `compile-time` (ကုဒ်ရေးသားချိန်) တွင် `compiler` မှ အမှားများကို ဖော်ထုတ်နိုင်သော်လည်း၊ `runtime` (ပရိုဂရမ်အလုပ်လုပ်ချိန်) တွင် ဖြစ်ပေါ်လာသော Memory related အမှားများကို Debug လုပ်ရန် `GDB` ကဲ့သို့သော Tool များသည် မရှိမဖြစ် လိုအပ်ပါသည်။

### Chapter Summary

ဤအခန်းတွင် ကျွန်ုပ်တို့သည် C Programming တွင် ကိုယ်ပိုင် Data Type များကို ဖန်တီးခြင်း၏ အခြေခံများကို လေ့လာခဲ့ကြသည်။

`struct` (အဆောက်အအုံ) သည် မတူညီသော Data အမျိုးအစားများကို တစ်စုတည်း ထုပ်ပိုးခြင်းဖြင့် Data ကို စနစ်တကျ စီမံခန့်ခွဲရန် အထောက်အကူပြုသည်။ `struct` ကို `dot operator` (`.`) ဖြင့် ဝင်ရောက်၍ အချက်အလက်များကို ရယူနိုင်ပြီး၊ `pointer` နှင့် `arrow operator` (`->`) ဖြင့် ထို Data များကို စီမံခန့်ခွဲနိုင်သည်။ ထို့အပြင် `nested struct` များဖြင့် ရှုပ်ထွေးသော Data များကို ပိုမိုရှင်းလင်းစွာ ဖော်ပြနိုင်သည်။

`union` သည် `struct` နှင့် ကွာခြားပြီး Memory Sharing (အမှတ်နေရာ မျှဝေခြင်း) ကို အခြေခံသည်။ `struct` သည် Data အားလုံးအတွက် Memory ကို ထိန်းသိမ်းသော်လည်း၊ `union` သည် ထည့်သွင်းထားသော Data များထဲမှ တစ်ခုတည်းအတွက်သာ Memory ကို မျှဝေသည်။ `typedef` သည် Data Type များအတွက် အမည်ပေးခြင်း (alias) လုပ်ရာတွင် အထောက်အကူပြုသည်။ `enum` သည် ဆက်စပ်နေသော Constant များအတွက် အမည်ပေးခြင်းဖြင့် Code ကို ပိုမိုဖတ်ရှုရလွယ်ကူစေသည်။ နောက်ဆုံးအနေဖြင့် `bit field` သည် Memory ကို `bit` အဆင့်တွင် ခွဲဝေ၍ ထိရောက်စွာ အသုံးချနိုင်သော နည်းလမ်းကို မိတ်ဆက်ပေးသည်။

ဤအယူအဆများသည် C Programming ၏ Memory Management နှင့် Data Structure များအတွက် အခြေခံအုတ်မြစ်များ ဖြစ်သည်။ `struct` နှင့် `union` များကို ကျွမ်းကျင်စွာ အသုံးပြုတတ်ခြင်းသည် ပိုမိုကြီးမားသောနှင့် ရှုပ်ထွေးသော ပရိုဂရမ်များကို ရေးသားရာတွင် အလွန်အရေးကြီးသော စွမ်းရည် ဖြစ်သည်။
