# Chapter 18: Bitwise Operation နှင့် Low-Level Programming

## ၁။ အခန်း၏ မိတ်ဆက်နှင့် အရေးပါပုံ (Introduction and Significance)

ကွန်ပျူတာ၏ အခြေခံအကျဆုံး အစိတ်အပိုင်းများဖြစ်သော Bit နှင့် Binary ကို နားလည်ခြင်းသည် Programming ၏ နောက်ကွယ်မှ လျှို့ဝှက်ဆန်းကျယ်သော အလုပ်လုပ်ပုံများကို ဖော်ထုတ်ပေးသည်။ ကျွန်ုပ်တို့နေ့စဉ်သုံးနေသည့် Software များသည် အလွန်ရှုပ်ထွေးသော အဆင့်များဖြင့် လုပ်ဆောင်နေသော်လည်း ထိုအရာအားလုံး၏ အခြေခံအုတ်မြစ်မှာ Binary (နှစ်မျိုးသော ဂဏန်းစနစ်) ဖြစ်သည်။ Bit တစ်ခုသည် အလွန်သေးငယ်သော အချက်အလက်တစ်ခုကို ကိုယ်စားပြုပြီး၊ 0 နှင့် 1 နှစ်ခုတည်းသော အခြေအနေများဖြင့်သာ အချက်အလက်များကို ကိုယ်စားပြုနိုင်သည်။ ဤအခြေခံအချက်ကို နားလည်ခြင်းသည် ကွန်ပျူတာ၏ Hardware အဆင့်တွင် ဖြစ်ပျက်နေသော လုပ်ဆောင်မှုများကို ပိုမိုနက်ရှိုင်းစွာ သိရှိနားလည်စေသည်။

C Programming တွင် Data ကို အဆင့်အထိ (Bit level) ကိုင်တွယ်နိုင်ခြင်းနှင့် Hardware နှင့် တိုက်ရိုက် ဆက်သွယ်နိုင်ခြင်းသည် အလွန်အရေးပါပါသည်။ အဆင့်မြင့် Programming များတွင် Data Structure များ၊ Algorithm များအပြင်၊ Memory ၏ အခြေခံဖွဲ့စည်းပုံနှင့် Hardware များ၏ လုပ်ဆောင်ပုံများကို နားလည်ရန် လိုအပ်သည်။ Bitwise Operation များသည် ဤအရာများ၏ အခြေခံကျသော ကိရိယာများဖြစ်ပြီး၊ Data များကို တိုက်ရိုက် မီဒီယာအဆင့်တွင် ပြုပြင်ထိန်းချုပ်နိုင်စွမ်းကို ပေးသည်။ ဤအခန်းတွင် ကျွန်ုပ်တို့သည် ယခင်အခန်းများတွင် လေ့လာခဲ့သော Memory နှင့် Data Type များ၏ အခြေခံအကြောင်းအရာများကို အခြေခံ၍ Data ၏ အသေးစိတ်ဖွဲ့စည်းပုံကို လေ့လာသွားမည် ဖြစ်သည်။ Bitwise Operation များသည် C Programming ၏ Low-Level Programming နှင့် System Programming များအတွက် မရှိမဖြစ်လိုအပ်သော အခြေခံကျသော ကျွမ်းကျင်မှုတစ်ခု ဖြစ်သည်။

## ၂။ Binary Representation နှင့် Memory အတွင်း Data သိမ်းဆည်းပုံ (Binary Representation and Data Storage in Memory)

Data တစ်ခုချင်းစီကို ကွန်ပျူတာအတွင်း၌ မည်သို့ သိမ်းဆည်းပုံကို နားလည်ရန်အတွက် Binary Representation ကို အရင်ဆုံး ပြန်လည်သုံးသပ်ရန် လိုအပ်သည်။ Data တစ်ခုစီကို Bit များအဖြစ် မည်သို့ ဖော်ပြပုံကို သိရှိခြင်းက Memory ၏ အခြေခံအကြောင်းအရာကို နားလည်စေသည်။

### Binary Representation

Data တစ်ခုချင်းစီကို Bit များအဖြစ် မည်သို့ ဖော်ပြပုံကို လေ့လာရမည်။ Bit ဆိုသည်မှာ အလွန်သေးငယ်သော အချက်အလက်တစ်ခုဖြစ်ပြီး 0 (Off/False) နှင့် 1 (On/True) ဟူသော တန်ဖိုးနှစ်ခုကိုသာ ကိုယ်စားပြုသည်။ စာလုံးတစ်လုံး၊ ဂဏန်းတစ်လုံး၊ သို့မဟုတ် အခြားသော Data အမျိုးအစားတစ်ခုလုံးကို ဖော်ပြရန်အတွက် Bit များစွာကို စုပေါင်းအသုံးပြုရသည်။ ဥပမာအားဖြင့်၊ တစ်ခုတည်းသော Bit သည် ၁ ဘိုက် (Byte) သို့မဟုတ် ရှစ်ဘိုက် (8-bit) အစား အလွန်နည်းသော အချက်အလက်ကိုသာ ကိုယ်စားပြုသည်။

### Integer Representation

`int`, `char` စသည့် Data Type များသည် Memory ထဲတွင် မည်သို့ သိမ်းဆည်းပုံကို လေ့လာရမည်။ C Programming တွင် `int` သည် အများအားဖြင့် 4 byte (32 bit) သို့မဟုတ် 8 byte (64 bit) ရှိသည်။ Data အမျိုးအစားတစ်ခုချင်းစီသည် Memory တွင် သတ်မှတ်ထားသော အရွယ်အစားအတိုင်း သိမ်းဆည်းသည်။ Data Type များသည် `signed` (အမှတ်အသားပါ) နှင့် `unsigned` (အမှတ်အသားမပါ) ဟူ၍ အမျိုးအစားနှစ်မျိုးရှိသည်။ `signed` သည် အပေါင်းနှင့် အနုတ် ဂဏန်းများကို ကိုယ်စားပြုနိုင်ပြီး၊ `unsigned` သည် အပေါင်း ဂဏန်းများကိုသာ ကိုယ်စားပြုနိုင်ပြီး၊ ပိုမိုကြီးမားသော တန်ဖိုးများကို ကိုင်တွယ်နိုင်စွမ်းရှိသည်။

### Two's Complement

Negative Number များကို မည်သို့ ကိုယ်စားပြုပုံနှင့် ၎င်း၏ အရေးပါပုံကို နားလည်ရမည်။ ကွန်ပျူတာများတွင် အနုတ်ဂဏန်းများကို ကိုယ်စားပြုရန်အတွက် Two's Complement (နှစ်ဆဖြည့်) နည်းလမ်းကို အသုံးပြုကြသည်။ ဤနည်းလမ်းသည် အနုတ်ဂဏန်းများကို Binary ပုံစံဖြင့် သိမ်းဆည်းရန်အတွက် အလွန်အရေးကြီးသည်။ ဤနည်းလမ်းသည် Memory တွင် Negative Number များ သိမ်းဆည်းပုံကို အသေးစိတ်ရှင်းပြသည်။ အနုတ်ဂဏန်းတစ်ခုကို ကိုယ်စားပြုရန်အတွက် အချက်အလက်များကို သတ်မှတ်ထားသော Bit အရေအတွက်အတိုင်း လှည့်ပတ်၍ ဖြည့်စွက်ရသည်။ ဤနည်းလမ်းကြောင့် Arithmetic Operations များ (ပေါင်းခြင်း၊ နုတ်ခြင်း) ကို Hardware များက အလွယ်တကူ ဆောင်ရွက်နိုင်စေသည်။

Memory ကို ဗီရိုအံဆွဲများဖြင့် နှိုင်းယှဉ်ပြီး Bit များဖြင့် Data ကို ဖော်ပြပုံကို ရှင်းပြခြင်းဖြင့် ဤအချက်ကို ပိုမိုရှင်းလင်းစွာ နားလည်နိုင်သည်။ ဥပမာအားဖြင့်၊ 8-bit Integer တစ်ခုသည် 8 ခုသော Bit များဖြင့်သာ အချက်အလက်တစ်ခုလုံးကို ကိုယ်စားပြုသည်။

```c
// Example of 8-bit representation
int x = 5;
// In binary: 0000 0101
```

## ၃။ Bitwise Operators နှင့် Data Manipulation (Bitwise Operators and Data Manipulation)

Bit Level တွင် Data များကို တိုက်ရိုက် ကိုင်တွယ်ရန်အတွက် Bitwise Operators များသည် အလွန်အရေးကြီးသည်။ ဤ Operator များသည် Data များ၏ တစ်ခုချင်းစီသော Bit များအပေါ် တိုက်ရိုက်အကျိုးသက်ရောက်မှုများ ပြုလုပ်နိုင်သည်။

### Bitwise Operators

Bitwise Operators များအဖြစ် `&` (AND), `|` (OR), `^` (XOR), `~` (NOT), `<<` (Left Shift), နှင့် `>>` (Right Shift) တို့ရှိသည်။ ဤ Operator များသည် Boolean (မှန်/မမှန်) တန်ဖိုးများကို အခြေခံ၍ လုပ်ဆောင်သည်။

*   **`&` (AND):** ဤ Operator သည် နှစ်ခုသော Bit များအားလုံးသည် 1 ဖြစ်မှသာ ရလဒ်ကို 1 ပြန်ပေးသည်။ ၎င်းကို Bit Masking များတွင် အဓိကအသုံးပြုသည်။
*   **`|` (OR):** ဤ Operator သည် နှစ်ခုသော Bit များထဲမှ တစ်ခုမဆို 1 ဖြစ်နေပါက ရလဒ်ကို 1 ပြန်ပေးသည်။
*   **`^` (XOR):** ဤ Operator သည် နှစ်ခုသော Bit များသည် မတူညီပါက 1 ဖြစ်ပြီး၊ တူညီပါက 0 ဖြစ်သည်။ ၎င်းသည် Data များ၏ လွှဲပြောင်းမှုများအတွက် အသုံးဝင်သည်။
*   **`~` (NOT):** ဤ Operator သည် Bit တစ်ခု၏ တန်ဖိုးကို မှောက်၍ ပြောင်းလဲပေးသည်။ 0 ကို 1 နှင့် 1 ကို 0 အဖြစ် ပြောင်းလဲသည်။
*   **`<<` (Left Shift):** ဤ Operator သည် Bit များကို ဘယ်ဘက်သို့ တစ်နေရာချင်းစီ ရွှေ့ပေးသည်။ ၎င်းသည် Power of 2 များဖြင့် မြှောက်ခြင်း (Multiplication) နှင့် အတူတူဖြစ်သည်။
*   **`>>` (Right Shift):** ဤ Operator သည် Bit များကို ညာဘက်သို့ တစ်နေရာချင်းစီ ရွှေ့ပေးသည်။ ၎င်းသည် Power of 2 များဖြင့် စားခြင်း (Division) နှင့် အတူတူဖြစ်သည်။

### Bit Masking

Specific Bit တစ်ခုကို Set, Clear, Toggle, Check လုပ်ရန်အတွက် Bit Mask များ ဖန်တီးခြင်းနှင့် ၎င်း၏ အသုံးချပုံကို လေ့လာရမည်။ Bit Mask ဆိုသည်မှာ သတ်မှတ်ထားသော Bit များ၏ ပုံစံကို ဖော်ပြရန်အတွက် အသုံးပြုသော အထူးတန်ဖိုးများ ဖြစ်သည်။ ဥပမာအားဖြင့်၊ တစ်ခုတည်းသော Bit ကို သတ်မှတ်ရန်အတွက် $1$ နှင့် Bitwise AND (`&`) ကို အသုံးပြုနိုင်သည်။ ဤနည်းလမ်းကို Permission System များအတွက် အသုံးချရာတွင် အလွန်အသုံးဝင်သည်။

### Bit Shifting

`<<` နှင့် `>>` Operator များဖြင့် Power of 2 များဖြင့် မြှောက်ခြင်း/စားခြင်း (Multiply/Divide by power of 2) ကို အသေးစိတ်ရှင်းပြရမည်။ ဘယ်ဘက်သို့ Shift လုပ်ခြင်း (`<<`) သည် $2^n$ ဖြင့် မြှောက်ခြင်းကို ဖော်ပြသည်။ ဥပမာအားဖြင့်၊ $x << 2$ သည် $x$ ကို $2^2 = 4$ ဖြင့် မြှောက်ခြင်းနှင့် တူညီသည်။ ညာဘက်သို့ Shift လုပ်ခြင်း (`>>`) သည် $x$ ကို $2^n$ ဖြင့် စားခြင်းကို ဖော်ပြသည်။ ဤနည်းလမ်းသည် အလွန်လျင်မြန်သော ဂဏန်းတွက်ချက်မှုများအတွက် အထောက်အကူပြုသည်။

```c
// Example of Bit Shifting
int x = 10; // Binary: 1010
int y = x << 1; // Left shift by 1 (1010 << 1 = 10100 = 20)
```

## ၄။ Advanced Bitwise Applications (အဆင့်မြင့် Bitwise အသုံးချမှုများ)

Bitwise Operation များသည် Data အဆင့်မှ Hardware အဆင့်သို့ ကူးပြောင်းရာတွင် အရေးပါလာသည်။ ဤအသုံးချမှုများကို လက်တွေ့ ပရိုဂရမ်ရေးသားရာတွင် မည်သို့ အသုံးချနိုင်ပုံကို လေ့လာရမည်။

### Flag/Permission System

Bitwise Operation များကို အသုံးပြု၍ System ၏ Flag များ (ဥပမာ: Read/Write/Execute permissions) ကို ဖန်တီးပြီး ထိန်းချုပ်ခြင်းကို လေ့လာရမည်။ ဤနည်းလမ်းဖြင့် Data တစ်ခုတည်းသော Variable ဖြင့် စနစ်၏ အခြေအနေများကို သိမ်းဆည်းနိုင်သည်။ ဥပမာအားဖြင့်၊ တစ်ခုချင်းစီသော Bit ကို Permission တစ်ခုအတွက် သတ်မှတ်နိုင်သည်။ Read permission အတွက် Bit တစ်ခု၊ Write permission အတွက် နောက် Bit တစ်ခု စသည်ဖြင့် သတ်မှတ်နိုင်သည်။ ဤအရာသည် File System များ သို့မဟုတ် Operating System များတွင် အသုံးပြုသော အခြေခံ နည်းလမ်းဖြစ်သည်။

### Endianness

Big-Endian နှင့် Little-Endian Byte Order များ၏ အဓိပ္ပာယ်နှင့် မည်သည့် System များတွင် အသုံးပြုပုံကို ရှင်းပြရမည်။ Endianness ဆိုသည်မှာ Memory တွင် Byte များ စီစဉ်ပုံနှင့် ပတ်သက်သည်။ Big-Endian တွင် အကြီးဆုံး (Most Significant Byte - MSB) သည် အလင်းဆုံး (Memory Address အနိမ့်ဆုံး) နေရာတွင် သိမ်းဆည်းသည်။ Little-Endian တွင် အသေးဆုံး (Least Significant Byte - LSB) သည် အလင်းဆုံး နေရာတွင် သိမ်းဆည်းသည်။ ဤအချက်သည် Network Communication များ သို့မဟုတ် Multi-processor System များတွင် Data များ ပို့လွှတ်ခြင်းနှင့် လက်ခံခြင်းတွင် အရေးပါသော ကိစ္စဖြစ်သည်။

## ၅။ Floating-Point Representation နှင့် Precision Issues (Floating-Point Representation and Precision Issues)

Bitwise Operation များသည် Integer များအတွက် အလွန်ကောင်းမွန်သော်လည်း ဒဿမ (Decimal) ဂဏန်းများကို ကိုယ်စားပြုရာတွင် ပြဿနာများ ရှိလာသည်။ ကွန်ပျူတာတွင် ဒဿမ ဂဏန်းများကို မည်သို့ သိမ်းဆည်းပုံနှင့် ၎င်း၏ အမှားအယွင်းများအကြောင်းကို လေ့လာရမည်။

### IEEE 754 Standard

Floating-point ဂဏန်းများကို စံနှုန်းသတ်မှတ်သည့် IEEE 754 Standard အကြောင်း ခြုံငုံသုံးသပ်ရမည်။ ဤ Standard သည် Real Number များကို Binary ပုံစံဖြင့် သိမ်းဆည်းရန်အတွက် Sign (အမှတ်အသား), Exponent (အထက်အဆင့်), နှင့် Mantissa (အသေးစိတ်ဒေတာ) ဟူသော အစိတ်အပိုင်းများဖြင့် ဖော်ပြသည်။ ဤဖွဲ့စည်းပုံသည် ကွန်ပျူတာများတွင် Floating-point ဂဏန်းများကို စံနှုန်းသတ်မှတ်သည်။

### Floating-Point Precision

ဘာကြောင့် `0.1 + 0.2 != 0.3` ဖြစ်ရသနည်း။ ဤမေးခွန်းသည် Binary Representation နှင့် Floating-point Error များ၏ အကြောင်းရင်းကို ရှင်းပြရမည်။ ဒဿမ ဂဏန်းများကို Binary ပုံစံဖြင့် သိမ်းဆည်းရာတွင် အပိုင်းအစများ (Fractional Parts) သည် သတ်မှတ်ထားသော Bit အရေအတွက်အတွင်း၌ အပြည့်အဝ ကိုယ်စားမပြုနိုင်သောကြောင့် အမှားအယွင်းများ ဖြစ်ပေါ်လာသည်။ ဤ Floating-point Precision ပြဿနာသည် Real-world Programming တွင် မည်သို့ အကျိုးသက်ရောက်ပုံကို ဆွေးနွေးရမည်။

## ၆။ အခန်း၏ အနှစ်ချုပ်နှင့် နိဂုံး (Chapter Summary and Conclusion)

Bitwise Operation များသည် Data အဆင့်မှ Hardware အဆင့်သို့ ကူးပြောင်းရာတွင် မည်မျှအရေးကြီးကြောင်း ပြန်လည်အကျဉ်းချုပ်ဖော်ပြရမည်။ ကျွန်ုပ်တို့သည် Binary Representation မှ စတင်၍ Memory တွင် Data များ သိမ်းဆည်းပုံ၊ Bitwise Logic များဖြင့် Data များကို တိုက်ရိုက် ကိုင်တွယ်ပုံ (Masking နှင့် Shifting)၊ Endianness ကဲ့သို့သော Hardware အခြေခံအကြောင်းအရာများနှင့် Floating-point ၏ အမှားအယွင်းများကို လေ့လာခဲ့ကြသည်။

Key Takeaways အဖြစ် Memory, Data Type, Bitwise Logic, Endianness, နှင့် Floating-point ၏ အခြေခံအကြောင်းအရာများကို အနှစ်ချုပ်ပေးရမည်။ ဤအသိပညာများသည် Low-Level Programming နှင့် Hardware အခြေခံများကို နားလည်ရန်အတွက် အလွန်အရေးကြီးပါသည်။ အခန်း ၁၈ မှ သင်ယူခဲ့သော အသိပညာများသည် Pointer နှင့် Memory Management အခန်းများအတွက် အခြေခံအုတ်မြစ်ဖြစ်ကြောင်း ဖော်ပြရမည်။ နောက်အခန်းများတွင် Pointer နှင့် Memory Management များသည် ဤ Bitwise အခြေခံအုတ်မြစ်ပေါ်တွင် တည်ဆောက်ထားသော အဆင့်မြင့် အကြောင်းအရာများ ဖြစ်ကြောင်း ဆက်စပ်တင်ပြကာ အခန်းကို နိဂုံးချုပ်ရမည် ဖြစ်သည်။
