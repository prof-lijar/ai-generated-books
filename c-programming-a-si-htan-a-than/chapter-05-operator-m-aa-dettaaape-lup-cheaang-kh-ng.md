# Chapter 5: Operator များ — ဒေတာအပေါ် လုပ်ဆောင်ခြင်း

## 1. Introduction: Operator များ၏ အခြေခံသဘောတရား (The Fundamentals of Operators)

C Programming တွင် သင်ရေးသားသော Code များသည် မည်သို့သော Data များကို စီမံခန့်ခွဲပြီး ပြောင်းလဲပေးသည်ကို နားလည်ရန်၊ သင်္ချာ၊ ယုക്തിနှင့် Bit Level တို့တွင် လုပ်ဆောင်မှုများ (Operations) ကို မည်သို့ အကောင်အထည်ဖော်ပုံကို လေ့လာမည်။ ပရိုဂရမ်တစ်ခုသည် အခြေခံအားဖြင့် Data များကို လက်ခံရရှိပြီး ထို Data များအပေါ် အခြေခံ၍ တွက်ချက်မှုများ၊ နှိုင်းယှဉ်မှုများနှင့် ထိန်းချုပ်မှုများ ပြုလုပ်ခြင်းဖြင့် အလုပ်လုပ်သည်။ ဤလုပ်ဆောင်မှုများကို အလိုအလျောက်နှင့် ထိရောက်စွာ လုပ်ဆောင်ပေးသည့် အရာများမှာ **Operator များ** ဖြစ်သည်။

Operator များဆိုသည်မှာ C Language တွင် သတ်မှတ်ထားသော သင်္ကေတများ (Symbols) ဖြစ်ပြီး၊ ၎င်းတို့သည် တစ်ခု သို့မဟုတ် တစ်ခုထက် ပိုသော Data များ (Operands) အပေါ်တွင် သတ်မှတ်ထားသော လုပ်ဆောင်မှု (Operation) တစ်ခုကို ပြုလုပ်ပေးသည်။ ဥပမာအားဖြင့်၊ သင်သည် `a + b` ဟု ရေးသောအခါ၊ `+` သည် Addition လုပ်ဆောင်မှုကို ကိုယ်စားပြုပြီး `a` နှင့် `b` ကို Operand များအဖြစ် သတ်မှတ်သည်။

Operator များသည် အဓိပ္ပာယ်နှင့် လုပ်ဆောင်ပုံတို့ဖြင့် အမျိုးအစားအလိုက် ခွဲခြားနိုင်ပါသည်။ ၎င်းတို့သည် ဂဏန်းများအပေါ် တွက်ချက်ခြင်း (Arithmetic)၊ အခြေအနေများ နှိုင်းယှဉ်ခြင်း (Relational)၊ ယုക്തിဆိုင်ရာ ဆုံးဖြတ်ချက်များ (Logical)၊ Bit Level တွင် တိုက်ရိုက် ကိုင်တွယ်ခြင်း (Bitwise)၊ တန်ဖိုးများ ပြောင်းလဲခြင်း (Assignment) နှင့် Data အမျိုးအစား ပြောင်းလဲခြင်း (Type Casting) စသည့် အလွှာပေါင်းစုံကို ပေါင်းစပ်ပေးသည်။

ဤ Operator များသည် C Language ၏ အခြေခံအုတ်မြစ်ဖြစ်ပြီး၊ Memory နှင့် CPU အဆင့်တွင် Data များ မည်သို့ ကိုယ်စားပြုသည်နှင့် မည်သို့ အလုပ်လုပ်သည်ကို နားလည်ရန် အရေးကြီးပုံကို သိရှိရမည်။ Operator များ၏ အစီအစဉ်နှင့် လုပ်ဆောင်ပုံကို နားလည်ခြင်းသည် ပိုမိုရှုပ်ထွေးသော ပရိုဂရမ်များကို ရေးသားရာတွင် မှားယွင်းမှုများကို ရှောင်ရှားနိုင်ရန်နှင့် စနစ်တကျ မှန်ကန်သော Logic ကို တည်ဆောက်နိုင်ရန် အထောက်အကူပြုသည်။

## 2. Arithmetic Operators: ဂဏန်းတွက်ချက်ခြင်း (Arithmetic Operations)

Arithmetic Operators များသည် ဂဏန်းများအပေါ် အခြေခံ၍ သင်္ချာဆိုင်ရာ လုပ်ဆောင်ချက်များကို လုပ်ဆောင်ပေးသည်။ ၎င်းတို့တွင် အခြေခံအားဖြင့် ပေါင်းခြင်း (`+`)၊ နုတ်ခြင်း (`-`)၊ မြှောက်ခြင်း (`*`)၊ စားခြင်း (`/`) နှင့် အကြွင်း (Modulo) (`%`) တို့ ပါဝင်သည်။ ဤလုပ်ဆောင်မှုများသည် ကွန်ပျူတာ၏ အခြေခံ ဂဏန်းတွက်ချက်မှုများကို တိုက်ရိုက် ကိုယ်စားပြုသည်။

### Integer vs. Floating-Point Division

Arithmetic Operations တွင် အထူးသဖြင့် စားခြင်း (`/`) နှင့် အကြွင်း (`%`) တို့ကို အသုံးပြုသောအခါ၊ ဂဏန်းအမျိုးအစား (Integer သို့မဟုတ် Floating-Point) အပေါ် မူတည်၍ ရလဒ်ကွာခြားမှုများ ရှိလာသည်။

**Integer Division (အစုံတွက်ချက်မှု)**
Integer Division ဆိုသည်မှာ နှစ်ခုသော အစုံဂဏန်းကို ပေါင်းစပ်လိုက်သောအခါ ရရှိသော အကျွင်း (Remainder) မရှိဘဲ အပြည့်အဝ အကျဉ်းကိုသာ ထုတ်ပေးခြင်း ဖြစ်သည်။ ဥပမာအားဖြင့်၊ `7 / 2` ကို ကွန်ပျူတာက 3 အဖြစ်သာ ပြန်ပေးသည်။ ဤသည်မှာ ဂဏန်းတွက်ချက်မှုများတွင် အရေးကြီးသော အချက်တစ်ချက်ဖြစ်ပြီး၊ အထူးသဖြင့် အင်ဂျင်နီယာဆိုင်ရာ သို့မဟုတ် စာရင်းအင်းဆိုင်ရာ တွက်ချက်မှုများတွင် အသုံးပြုသည်။ အကျိုးကျေးဇူးမှာ၊ ကွန်ပျူတာသည် အမှားအယွင်းမရှိဘဲ အမြန်ဆုံး တွက်ချက်မှုများ ပြုလုပ်နိုင်ခြင်းဖြစ်သည်။

**Floating-Point Division (အမြဲတမ်းတွက်ချက်မှု)**
သို့သော်၊ တိကျသော ရလဒ်များ လိုအပ်သောအခါ၊ ကျွန်ုပ်တို့သည် Floating-Point ဂဏန်းများဖြစ်သော `float` သို့မဟုတ် `double` ကို အသုံးပြုရမည်။ ဤအမျိုးအစားများဖြင့် စားခြင်းကို ပြုလုပ်ပါက၊ ရလဒ်တွင် ဒဿမ (Decimal) အပိုင်းများ ပါဝင်လာသည်။ ဥပမာအားဖြင့်၊ `7.0 / 2.0` ဆိုပါက ရလဒ်သည် `3.5` ဖြစ်လာသည်။ ဤနည်းလမ်းသည် ပိုမိုတိကျသော ရလဒ်များကို ရရှိစေပြီး၊ အထူးသဖြင့် အချိန်၊ အပူချိန်၊ ငွေကြေး စသည့် ဆက်စပ်မှုများရှိသော ကိန်းဂဏန်းများအတွက် မရှိမဖြစ်လိုအပ်သည်။

**Division နှင့် Modulo ၏ ကွာခြားချက်**
Division (`/`) နှင့် Modulo (`%`) တို့သည် အလွန်နီးစပ်သော်လည်း မတူညီသော အဓိပ္ပာယ်များ ရှိသည်။ Division သည် အကျဉ်းကို ပေးပြီး Remainder ကို မထည့်ဘဲ ပေးသည်။ Modulo (`%`) သည် Division ပြီးနောက် ကျန်ရှိသော အကြွင်း (Remainder) ကိုသာ ပြန်ပေးသည်။ ဥပမာအားဖြင့်၊ `7 % 2` သည် `1` ဖြစ်သည်။ ဤကွာခြားချက်သည် ပရိုဂရမ်များတွင် အလွန်အရေးကြီးသည်။ ဥပမာအားဖြင့်၊ အရာဝတ္ထုများကို အစုအဝေးများ (Groups) အဖြစ် ခွဲခြားရာတွင် Modulo ကို အသုံးပြုပြီး၊ အကျွင်းကို သိရှိရန် လိုအပ်သည်။

### Operator Precedence in Arithmetic

Arithmetic Operators များ အသုံးပြုရာတွင်၊ မည်သည့် Operator ကို အရင်ဆုံး လုပ်ဆောင်ရမည်ကို သိရှိရန် **Operator Precedence** စည်းမျဉ်းကို လိုက်နာရမည်။ C Language တွင် ဤစည်းမျဉ်းသည် PEMDAS သို့မဟုတ် BODMAS နှင့် ဆင်တူသည်။ ဆိုလိုသည်မှာ မြှောက်ခြင်းနှင့် စားခြင်းက ပေါင်းခြင်းနှင့် နုတ်ခြင်းထက် ဦးစားပေးသည်။

ဥပမာအားဖြင့်၊ `a + b * c` ဟု ရေးထားပါက၊ ကွန်ပျူတာသည် ပထမဆုံး `b * c` ကို တွက်ချက်ပြီးမှ ထိုရလဒ်ကို `a` နှင့် ပေါင်းမည်ဖြစ်သည်။

**Parentheses ( ) ၏ အခန်းကဏ္ဍ**
Operator Precedence ကို ပိုမိုတိကျစွာ ထိန်းချုပ်ရန်နှင့် ရည်ရွယ်ချက်ရှိသော တွက်ချက်မှုများကို လုပ်ဆောင်စေရန်အတွက် Parentheses `()` ကို အသုံးပြုရသည်။ Parentheses များသည် အခြား Operator များထက် ပိုမိုမြင့်မားသော ဦးစားပေးကို ပေးသည်။ ဥပမာအားဖြင့်၊ `(a + b) * c` ဟု ရေးပါက၊ `a + b` ကို အရင်ဆုံး ပေါင်းပြီးမှ ထိုရလဒ်ကို `c` နှင့် မြှောက်မည်ဖြစ်သည်။ ဤအချက်သည် ရှုပ်ထွေးသော သင်္ချာတွက်ချက်မှုများတွင် မှားယွင်းမှုများကို ရှောင်ရှားရန် အလွန်အရေးကြီးသည်။

### Practical Example

ဤ Operator များသည် စက်ရုံဆိုင်ရာ တွက်ချက်မှုများ သို့မဟုတ် စာရင်းအင်း (Statistics) တွက်ချက်မှုများတွင် အလွန်အသုံးဝင်သည်။ ဥပမာအားဖြင့်၊ ကုန်ပစ္စည်းတစ်ခု၏ စုစုပေါင်းဈေးနှုန်းကို တွက်ချက်ရာတွင် ပစ္စည်းအရေအတွက်နှင့် တစ်ခုချင်းစီ၏ ဈေးနှုန်းများကို မြှောက်ပြီး၊ ထို့နောက် စုစုပေါင်းကို ရှာဖွေရာတွင် ပေါင်းခြင်း၊ နုတ်ခြင်းနှင့် အကြွင်းများကို အသုံးပြုသည်။ ဤလုပ်ဆောင်မှုများသည် လက်တွေ့ဘဝမှ ဆုံးဖြတ်ချက်များ ချမှတ်ရာတွင် အခြေခံကျသော ကိန်းဂဏန်းဆိုင်ရာ အခြေခံအုတ်မြစ်များ ဖြစ်သည်။

**Transition:** ဂဏန်းများအပေါ် တွက်ချက်မှုများ (Arithmetic) ကို နားလည်ပြီးပြီဆိုလျှင်၊ ထိုဂဏန်းများ၏ ဆက်စပ်မှုများကို နှိုင်းယှဉ်ခြင်းနှင့် စုပေါင်းခြင်း (Relational and Logical) တို့ကို ဆက်လက်လေ့လာမည်။

## 3. Relational and Logical Operators: နှိုင်းယှဉ်ခြင်းနှင့် ယုക്തി (Comparison and Logic)

Arithmetic Operators များဖြင့် ဂဏန်းများကို တွက်ချက်နိုင်ပြီဖြစ်သောအခါ၊ ထိုဂဏန်းများအကြား ဆက်စပ်မှုများကို စစ်ဆေးခြင်းနှင့် အခြေအနေအပေါ် မူတည်၍ ဆုံးဖြတ်ချက်များ ချမှတ်ခြင်းတို့ကို လေ့လာရမည်။ ဤအပိုင်းတွင် Relational Operators နှင့် Logical Operators များသည် အဓိကကျသော အခန်းကဏ္ဍမှ ပါဝင်သည်။

### Relational Operators: နှိုင်းယှဉ်ခြင်း

Relational Operators များသည် နှစ်ခုသော Value များကို နှိုင်းယှဉ်ရန် အသုံးပြုသည်။ ၎င်းတို့၏ ရလဒ်သည် အမြဲတမ်း Boolean တန်ဖိုးဖြစ်သော **True** (မှန်သည်) သို့မဟုတ် **False** (မှားသည်) ကိုသာ ထုတ်ပေးသည်။

*   **အသုံးပြုပုံ:**
    *   **တူညီမှု:** `==` (နှစ်ခု တူညီပါသလား) နှင့် `!=` (နှစ်ခု မတူပါသလား)
    *   **ကြီးမားမှု/ငယ်မားမှု:** `>` (ကြီးသည်)၊ `<` (သေးသည်)၊ `>=` (ကြီးသည် သို့မဟုတ် တူသည်)၊ `<=` (သေးသည် သို့မဟုတ် တူသည်)

ဤ Operator များသည် ပရိုဂရမ်များတွင် အခြေအနေများပေါ် မူတည်၍ လမ်းကြောင်းပြောင်းလဲခြင်း (Conditional Flow) အတွက် မရှိမဖြစ်လိုအပ်သည်။ ဥပမာအားဖြင့်၊ အသုံးပြုသူ၏ အသက်အရွယ်သည် အရွယ်ရောက်ပြီလား၊ သို့မဟုတ် သတ်မှတ်ထားသော အသက်ထက် ပိုကြီးပါသလား စသည်တို့ကို စစ်ဆေးရာတွင် အသုံးပြုသည်။ ဤရလဒ်သည် နောက်ဆက်တွဲ `if-else` statement များ၏ အခြေခံဖြစ်သည်။

### Logical Operators: ယုക്തി (Logic)

Relational Operators များသည် တစ်ခုတည်းသော အခြေအနေကိုသာ စစ်ဆေးပေးသည်။ သို့သော်၊ ပိုမိုရှုပ်ထွေးသော အခြေအနေများစွာကို ပေါင်းစပ်တွေးခေါ်ရန်အတွက် **Logical Operators** များ လိုအပ်သည်။ ဤ Operator များသည် Boolean တန်ဖိုးများကို ပေါင်းစပ်ပေးသည်။

*   **AND (`&&`):** နှစ်ခုစလုံး မှန်မှန်ကန်ကန် ဖြစ်မှသာ True ဖြစ်သည်။
*   **OR (`||`):** နှစ်ခုထဲမှ တစ်ခုခု မှန်မှန်ကန်ကန် ဖြစ်ပါက True ဖြစ်သည်။
*   **NOT (`!`):** အခြေအနေတစ်ခု၏ တန်ဖိုးကို ပြောင်းပြန်လှန်ပေးသည်။

ဤ Operator များသည် Data Validation နှင့် Control Flow (Flow Control) တွင် အလွန်အရေးကြီးသည်။ ဥပမာအားဖြင့်၊ အသုံးပြုသူ၏ လစဉ်ဝင်ငွေသည် သတ်မှတ်ထားသော အနည်းဆုံးပမာဏထက် ပိုနေပြီး၊ ထို့အပြင် သူသည် အသက် ၁၈ နှစ်အထက် ဖြစ်ရမည်ဆိုပါက၊ `(income > minimum_income) && (age > 18)` ဟု ရေးခြင်းဖြင့် ထိုအခြေအနေ နှစ်ခုစလုံး မှန်မှန်ကန်ကန် ဖြစ်မှသာ အမှန်တကယ် True ဖြစ်သည်။

**Truth Tables ဖြင့် အသေးစိတ်လေ့လာခြင်း**
ဤ Logical Operators များ၏ လုပ်ဆောင်ပုံကို နားလည်စေရန် Truth Tables ဖြင့် လေ့လာခြင်းသည် အထောက်အကူပြုသည်။ ၎င်းသည် မည်သည့် Input များ ပေးလိုက်သောအခါ Output အဖြစ် မည်သည့် Boolean တန်ဖိုးကို ထုတ်ပေးသည်ကို ရှင်းလင်းစွာ ဖော်ပြပေးသည်။

### Application

Relational နှင့် Logical Operators များသည် Data Validation နှင့် Control Flow တွင် အဓိကအသုံးပြုသည်။ ပရိုဂရမ်များသည် အခြေအနေအလိုက် မတူညီသော လမ်းကြောင်းများအတိုင်း ဆက်လက်လုပ်ဆောင်ရန်၊ သို့မဟုတ် အခြေအနေတစ်ခုကို စစ်ဆေးပြီး သတ်မှတ်ထားသော လုပ်ဆောင်ချက်ကိုသာ လုပ်ဆောင်ရန်အတွက် ဤ Operator များသည် မရှိမဖြစ်လိုအပ်သည်။

**Transition:** Data များကို နှိုင်းယှဉ်ပြီး ယုക്തി (Logic) ကို ပေါင်းစပ်နိုင်ပြီဖြစ်သောအခါ၊ ကျွန်ုပ်တို့သည် C Code ၏ အနက်ရှိုင်းဆုံး အဆင့်ဖြစ်သော Memory နှင့် Hardware အဆင့်တွင် Data များ မည်သို့ ကိုယ်စားပြုသည်ကို လေ့လာရမည်။

## 4. Bitwise Operators: Bit Level တွင် ကိုင်တွယ်ခြင်း (Handling Data at the Bit Level)

Arithmetic နှင့် Logical Operators များသည် ဂဏန်းအဆင့် သို့မဟုတ် Boolean အဆင့်တွင် လုပ်ဆောင်သော်လည်း၊ C Language ၏ အနက်ရှိုင်းဆုံး အဆင့်ဖြစ်သော Memory နှင့် Hardware အဆင့်တွင် Data များ မည်သို့ ကိုယ်စားပြုသည်ကို နားလည်ရန်အတွက် **Bitwise Operators** များသည် အလွန်အရေးကြီးသည်။ Bitwise Operators များသည် Data များကို Binary (Bit) အဆင့်တွင် တိုက်ရိုက် ကိုင်တွယ်ပြီး Bit-level manipulation များ ပြုလုပ်သည်။

### Bitwise Operators များ

Bitwise Operators များသည် Data တစ်ခုချင်းစီ၏ အထပ်အလိုက် (Bit by Bit) လုပ်ဆောင်ချက်များကို လုပ်ဆောင်သည်။

*   **Bitwise AND (`&`):** ဤ Operator သည် နှစ်ခုသော Bit များအား နှိုင်းယှဉ်ပြီး၊ ထိုနှစ်ခုစလုံး **၁ (1)** ဖြစ်မှသာ ရလဒ်ကို **၁** အဖြစ် ပြန်ပေးသည်။ ၎င်းကို **Masking** နှင့် Bit Manipulation များတွင် အလွန်အသုံးဝင်သည်။ ဥပမာအားဖြင့်၊ အချက်အလက်တစ်ခု၏ အချို့သော Bit များကို ဖျက်ပစ်ခြင်း သို့မဟုတ် ထိန်းချုပ်ခြင်းများ ပြုလုပ်ရာတွင် အသုံးပြုသည်။
*   **Bitwise OR (`|`):** ဤ Operator သည် နှစ်ခုသော Bit များထဲမှ တစ်ခုခု **၁ (1)** ဖြစ်ပါက ရလဒ်ကို **၁** အဖြစ် ပြန်ပေးသည်။ ၎င်းသည် Bit များကို ပေါင်းစပ်ရန် အသုံးပြုသည်။
*   **Bitwise XOR (`^`):** XOR (Exclusive OR) သည် နှစ်ခုသော Bit များ မတူညီပါက **၁ (1)** ဖြစ်ပြီး၊ တူညီပါက **၀ (0)** ဖြစ်သည်။ ၎င်းသည် အလှည့်ကျ (Toggling) လုပ်ဆောင်ပုံများအတွက် အသုံးဝင်သည်။
*   **Bitwise NOT (`~`):** ဤ Operator သည် Bit တစ်ခု၏ တန်ဖိုးကို ပြောင်းပြန်လှန်ပေးသည်။ ဆိုလိုသည်မှာ Bit ၁ သည် ၀ ဖြစ်လျှင် ၁ ဖြစ်သွားမည်၊ ၁ သည် ၀ ဖြစ်လျှင် ၁ ဖြစ်သွားမည်။

### Bit Shifting Operators

Bitwise Operators များအပြင် **Bit Shifting Operators** များသည် Data များကို ဘယ်ဘက် သို့မဟုတ် ညာဘက်သို့ Shift လုပ်ခြင်းဖြင့် အလွန်အရေးကြီးသော လုပ်ဆောင်ချက်များကို လုပ်ဆောင်သည်။

*   **Left Shift (`<<`):** ဤ Operator သည် Bit တစ်ခုချင်းစီကို ဘယ်ဘက်သို့ Shift လုပ်သည်။ ဤလုပ်ဆောင်ချက်သည် အလွန်အရေးကြီးသော **Power of 2** ဖြင့် မြှောက်ခြင်းကို အစားထိုးသည်။ ဥပမာအားဖြင့်၊ `x << 1` သည် $x$ ကို 2 ဖြင့် မြှောက်ခြင်းနှင့် တူသည်။
*   **Right Shift (`>>`):** ဤ Operator သည် Bit တစ်ခုချင်းစီကို ညာဘက်သို့ Shift လုပ်သည်။ ဤလုပ်ဆောင်ချက်သည် အလွန်အရေးကြီးသော **Division by 2** ကို အစားထိုးသည်။ ဥပမာအားဖြင့်၊ `x >> 1` သည် $x$ ကို 2 ဖြင့် စားပြီး အကျွင်းကို မထည့်ဘဲ ပြန်ပေးခြင်းနှင့် တူသည်။

**Deep Dive: Shift Operators ၏ မူလအခြေခံ**
Shift Operators များသည် Data များ၏ မူလအခြေခံဖြစ်သော Binary representation နှင့် Memory Address များ၏ ဆက်စပ်မှုနှင့် တိုက်ရိုက်သက်ဆိုင်သည်။ ကွန်ပျူတာအတွင်း၌ Data များသည် Bit များအဖြစ်သာ ရှိနေသောကြောင့်၊ ဤ Shift လုပ်ဆောင်ချက်များသည် Data များကို အလွန်လျင်မြန်စွာ ပြောင်းလဲရန်နှင့် Memory Address များအကြား ဆက်စပ်မှုကို ဖော်ထုတ်ရန် အထောက်အကူပြုသည်။

### Practical Application

Bitwise Operators များသည် Flags Management (အခြေအနေများအား မှတ်သားခြင်း) နှင့် Low-level Data Encoding များတွင် အလွန်အသုံးဝင်သည်။ ဥပမာအားဖြင့်၊ အသုံးပြုသူ၏ အခွင့်အရေးများ (ဥပမာ- စာအုပ်ဖတ်ခွင့်၊ အင်တာနက်ဝင်ခွင့်) ကို ကိုယ်စားပြုသော အခြေအနေများကို Bit တစ်ခုချင်းစီတွင် ထည့်သွင်းသိမ်းဆည်းခြင်းဖြင့် စနစ်တကျ စီမံခန့်ခွဲနိုင်သည်။

**Transition:** Bitwise Operators သည် Data များ၏ အခြေခံအကျဆုံးအဆင့်ကို ဖော်ပြသည်။ ယခုအခါ၊ ဤလုပ်ဆောင်မှုများအပြင်၊ Operator များအသုံးပြုရာတွင် ဖြစ်ပေါ်လာနိုင်သော Data အမျိုးအစားပြောင်းလဲမှုများကို စီမံခန့်ခွဲရန် လိုအပ်သည်။

## 5. Assignment Operators နှင့် Increment/Decrement: တန်ဖိုးများ ပြောင်းလဲခြင်း (Modifying Values)

Data များကို တွက်ချက်ခြင်းနှင့် နှိုင်းယှဉ်ခြင်းတို့ကို လုပ်ဆောင်ပြီးနောက်၊ ထို Data များ၏ တန်ဖိုးများကို ပြောင်းလဲရန် လိုအပ်လာသည်။ ဤလုပ်ဆောင်မှုများကို လုပ်ဆောင်ရန်အတွက် **Assignment Operators** နှင့် **Increment/Decrement Operators** များ အသုံးပြုသည်။

### Assignment Operators

Assignment Operators များသည် ရလဒ်ကို သတ်မှတ်ထားသော Variable ထဲသို့ တန်ဖိုးများ ထည့်သွင်းရန် အသုံးပြုသည်။

*   **အခြေခံ Assignment (`=`):** `a = 10;` ဆိုသည်မှာ $a$ ၏ တန်ဖိုးကို 10 ဟု သတ်မှတ်သည်။
*   **Compound Assignment Operators:** ဤ Operator များသည် လက်ရှိတန်ဖိုးနှင့် လုပ်ဆောင်ချက်ကို ပေါင်းစပ်ပေးသည်။ ၎င်းတို့သည် Code ကို ပိုမိုတိုတောင်းပြီး ထိရောက်စေသည်။
    *   `+=`: $a = a + b$ (ဥပမာ: `a += 5` သည် $a = a + 5$ နှင့် တူသည်။)
    *   `-=` : $a = a - b$
    *   `*=` : $a = a * b$
    *   `/=` : $a = a / b$
    *   `%=` : $a = a \% b$

ဤ Compound Assignment Operators များသည် ပရိုဂရမ် ရေးသားရာတွင် အလွန်အသုံးဝင်ပြီး Code ကို ပိုမိုရှင်းလင်းပြီး ထိရောက်သော ပုံစံဖြင့် ရေးသားနိုင်စေသည်။

### Increment and Decrement Operators

Increment (`++`) နှင့် Decrement (`--`) Operators များသည် Variable များ၏ တန်ဖိုးကို တစ်တိုး သို့မဟုတ် တစ်လျှော့ လုပ်ဆောင်သည်။ ဤ Operator များတွင် **Pre-increment** နှင့် **Post-increment** ဟူ၍ နှစ်မျိုးရှိသည်။

*   **Pre-increment (`++a`):** Variable ၏ တန်ဖိုးကို **အမြဲတမ်း** တစ်တိုးပြီးမှ ထိုတန်ဖိုးကို အသုံးပြုသည်။
*   **Post-increment (`a++`):** Variable ၏ လက်ရှိတန်ဖိုးကို အရင်ဆုံး အသုံးပြုပြီးမှ တစ်တိုးသည်။

**ဘယ်အချိန်တွင် မည်သည့်ပုံစံကို အသုံးပြုရမည်နည်း**
ဤကွာခြားချက်သည် အထူးသဖြင့် Pointer များနှင့် Function များတွင် အရေးကြီးသည်။ Pointer များတွင်၊ Pointer တစ်ခုသည် Memory Address ကို ကိုယ်စားပြုသောကြောင့်၊ Pre-increment နှင့် Post-increment သည် Pointer များ၏ Address ကို မည်သို့ အကျိုးသက်ရောက်သည်ကို နားလည်ရန် လိုအပ်သည်။ Function များတွင်လည်း၊ တစ်စုံတစ်ခုကို တန်ဖိုးပြောင်းလဲရာတွင် မည်သည့်အချိန်၌ အသုံးပြုရမည်ကို မှန်ကန်စွာ ဆုံးဖြတ်ရန် လိုအပ်သည်။

### Operator Precedence Review

Operator များ အသုံးပြုရာတွင်၊ ၎င်းတို့၏ လုပ်ဆောင်မှု အစီအစဉ်ကို အမြဲတမ်း သတိထားရမည်။ အခြေခံအားဖြင့်၊ Operator များ၏ ဦးစားပေးအစီအစဉ်မှာ အောက်ပါအတိုင်းဖြစ်သည်။

၁။ Parentheses `()`
၂။ Unary Operators (ဥပမာ: `++`, `--`, `!`, `~`)
၃။ Arithmetic Operators (`+`, `-`, `*`, `/`, `%`) နှင့် Bitwise Operators (`&`, `|`, `^`, `~`, `<<`, `>>`)
၄။ Relational Operators (`<`, `>`, `<=`, `>=` နှင့် `==`, `!=`)
၅။ Logical Operators (`&&`, `||`, `!`)
၆။ Assignment Operators (`=`, `+=`, `-=`, စသည်)

ဤအစီအစဉ်ကို မှတ်သားထားခြင်းဖြင့်၊ ကျွန်ုပ်တို့သည် ပရိုဂရမ်၏ Logic ကို မှန်ကန်စွာ ထိန်းချုပ်နိုင်ပြီး၊ ရည်ရွယ်ထားသော အတိုင်းအတာအတိုင်း လုပ်ဆောင်မှုများ ဖြစ်ပေါ်လာစေမည်ဖြစ်သည်။

## 6. Type Casting: Data အမျိုးအစား ပြောင်းလဲခြင်း (Type Conversion)

Data အမျိုးအစားများသည် ကွန်ပျူတာ၏ Memory တွင် မည်သို့ သိမ်းဆည်းထားသည်ကို ဖော်ပြသည်။ တစ်ခါတစ်ရံတွင်၊ ကွဲပြားခြားနားသော Data အမျိုးအစားများအကြား အချက်အလက်များကို ပြောင်းလဲရန် လိုအပ်လာသည်။ ဤလုပ်ငန်းစဉ်ကို **Type Casting** ဟုခေါ်သည်။

### Implicit Type Casting (Automatic Conversion)

Implicit Type Casting ဆိုသည်မှာ Compiler မှ Data အမျိုးအစား နှစ်ခုကြား အချက်အလက်များကို **အလိုအလျောက်** ပြောင်းလဲပေးခြင်း ဖြစ်သည်။ ဥပမာအားဖြင့်၊ အင်ဂျင်နီယာများသည် `int` အမျိုးအစားမှ `float` သို့ ပြောင်းလဲရာတွင်၊ ကွန်ပျူတာသည် အလိုအလျောက် ပြုလုပ်ပေးသည်။ ဤ Implicit Conversion သည် အများအားဖြင့် အန္တရာယ်ကင်းသော လုပ်ဆောင်မှုဖြစ်ပြီး၊ Compiler မှ သတ်မှတ်ထားသော စည်းမျဉ်းများအရ အလုပ်လုပ်သည်။

### Explicit Type Casting (Casting)

Explicit Type Casting ဆိုသည်မှာ ပရိုဂရမ်ရေးသားသူက **ကိုယ်တိုင်** ထိုအချက်အလက်များကို ပြောင်းလဲရန် ဖော်ပြခြင်း ဖြစ်သည်။ ၎င်းကို Type Cast ဟုခေါ်သည်။ ဤလုပ်ဆောင်ချက်ကို ပြုလုပ်ရန်အတွက် `(type) expression` ပုံစံကို အသုံးပြုရသည်။ ဥပမာအားဖြင့်၊ `(float) 7.5` ဟု ရေးခြင်းသည် ဂဏန်း 7.5 ကို Floating-Point အမျိုးအစားအဖြစ် ပြောင်းလဲရန် တိုက်ရိုက်ညွှန်ကြားခြင်းဖြစ်သည်။

**Potential Loss of Data (Data Truncation) နှင့် Overflow များ**
Explicit Type Casting ပြုလုပ်ရာတွင် အလွန်သတိထားရမည်။ အထူးသဖြင့် Floating-Point မှ Integer သို့ သို့မဟုတ် အကြီးစား Integer မှ သေးငယ်သော Integer သို့ ပြောင်းလဲသည့်အခါ၊ အချက်အလက် အချို့ ဆုံးရှုံးသွားနိုင်ခြင်း (Data Truncation) သို့မဟုတ် Overflow များ ဖြစ်ပေါ်လာနိုင်ခြေရှိသည်။ ဥပမာအားဖြင့်၊ `float` မှ `int` သို့ ပြောင်းသောအခါ၊ ဒဿမအပိုင်းကို ပယ်ဖျက်ပြီး အကျွင်းကိုသာ ထိန်းသိမ်းမည်ဖြစ်သည်။ ထို့ကြောင့်၊ Type Casting ပြုလုပ်မည့် အခြေအနေများကို ကြိုတင်ဆင်ခြင်သုံးသပ်ရန် အရေးကြီးသည်။

### Pointer Context

Type Casting သည် Pointer များနှင့် အလွန်နီးစပ်စွာ ဆက်စပ်နေသည်။ Pointer များသည် Memory Address များကို ကိုယ်စားပြုသောကြောင့်၊ Pointer များမှတစ်ဆင့် Data များကို ဝင်ရောက်ကြည့်ရှုရာတွင်၊ ထို Data များ၏ အမျိုးအစားကို မှန်ကန်စွာ သိရှိရန် Type Casting လိုအပ်သည်။ Pointer များကို အသုံးပြု၍ Memory နေရာများကို စီမံခန့်ခွဲရာတွင် Type Casting သည် အလွန်အရေးပါသော ကိရိယာတစ်ခုဖြစ်သည်။

## 7. Chapter Summary and Conclusion

### Recap

ဤအခန်းတွင် C Programming ၏ အခြေခံအုတ်မြစ်ဖြစ်သော Operator များ၏ အဓိကအမျိုးအစားများနှင့် ၎င်းတို့၏ အခန်းကဏ္ဍများကို အသေးစိတ် လေ့လာခဲ့ကြသည်။ ကျွန်ုပ်တို့သည် အောက်ပါ အဓိကနယ်ပယ်များကို လေ့လာခဲ့သည်-

*   **Arithmetic Operators:** ဂဏန်းတွက်ချက်ခြင်းနှင့် အကြွင်းများ (Arithmetic, Modulo)။
*   **Relational and Logical Operators:** Data များအကြား နှိုင်းယှဉ်ခြင်းနှင့် ယုക്തിဆိုင်ရာ ဆုံးဖြတ်ချက်များ (Comparison, Logic)။
*   **Bitwise Operators:** Data များကို Binary အဆင့်တွင် တိုက်ရိုက် ကိုင်တွယ်ခြင်း (Bitwise, Shifting)။
*   **Assignment and Increment/Decrement Operators:** တန်ဖိုးများကို ပြောင်းလဲခြင်း (Assignment, Increment/Decrement)။
*   **Type Casting:** Data အမျိုးအစားများကြား အချက်အလက် ပြောင်းလဲခြင်း (Implicit/Explicit Conversion)။

### Key Takeaway

Operator များသည် C Program ၏ အခြေခံတွေးခေါ်မှု (Computational Thinking) နှင့် Machine Level ဆက်သွယ်မှုကို မည်သို့ အထောက်အကူပြုသည်ကို နားလည်ရမည်။ ၎င်းတို့သည် Data များအပေါ် မည်သို့ အလုပ်လုပ်ပုံကို တိကျစွာ ဖော်ပြပေးပြီး၊ ပရိုဂရမ်ရေးသားသူအား Data များအပေါ် အခြေခံ၍ ရှုပ်ထွေးသော လုပ်ဆောင်ချက်များကို ထိရောက်စွာ အကောင်အထည်ဖော်နိုင်ရန် ကိရိယာများ ပေးအပ်သည်။
