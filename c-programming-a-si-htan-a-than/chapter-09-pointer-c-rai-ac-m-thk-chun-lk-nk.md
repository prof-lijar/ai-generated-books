# Chapter 9: Pointer — C ရဲ့ အစွမ်းထက်ဆုံး လက်နက်

C Programming တွင် Memory (မှတ်ဉာဏ်) ကို နားလည်ခြင်းသည် ပရိုဂရမ်ရေးသားရာတွင် အလွန်အရေးကြီးပါသည်။ သင်သည် ပရိုဂရမ်တစ်ခုကို စနစ်တကျ ထိန်းချုပ်လိုပါက၊ သင်သည် ကွန်ပျူတာ၏ အခြေခံအဆောက်အအုံဖြစ်သော Memory ကို မည်သို့ အသုံးပြုပုံကို သိရှိထားရမည် ဖြစ်သည်။ ထိုအရာကို အထူးတလည် ထိန်းချုပ်နိုင်သော လက်နက်များထဲမှ တစ်ခုမှာ Pointer (ညွှန်ပြချက်) များ ဖြစ်သည်။ Pointer များသည် C ဘာသာစကား၏ အစွမ်းထက်ဆုံးနှင့် အစွမ်းအထက်ဆုံးသော အယူအဆများထဲမှ တစ်ခုဖြစ်ပြီး၊ ကွန်ပျူတာ၏ အလွှာ (Machine Level) တွင် တိုက်ရိုက် ထိန်းချုပ်နိုင်စွမ်းကို ပေးစွမ်းပါသည်။

## ၁။ မိတ်ဆက်နှင့် အခြေခံအယူအဆများ (Introduction and Basic Concepts)

### Memory Address ပြန်လည်သုံးသပ်ခြင်း (Reviewing Memory Addresses)

ပထမဆုံးအနေဖြင့် Variable တစ်ခုစီသည် ကွန်ပျူတာ၏ Memory အတွင်း၌ မည်သည့်နေရာတွင် တည်ရှိနေသည်ကို သိရှိရန် လိုအပ်ပါသည်။ C ဘာသာစကားတွင် ကျွန်ုပ်တို့ သတ်မှတ်ထားသော Variable တစ်ခုစီတိုင်းသည် Memory တွင် သတ်မှတ်ထားသော နေရာတစ်ခု၌ တည်ရှိနေပြီး ထိုနေရာကို Memory Address ဟု ခေါ်သည်။ ဤ Address များသည် တစ်ခုနှင့်တစ်ခု ဆက်စပ်နေပြီး၊ ဤဆက်စပ်မှုကို နားလည်မှသာ Pointer များ၏ အစွမ်းကို အပြည့်အဝ အသုံးချနိုင်မည် ဖြစ်သည်။

Variable တစ်ခု၏ Memory Address ကို ရယူရန်အတွက် `&` (Address-of) Operator ကို အသုံးပြုသည်။ ဥပမာအားဖြင့်၊ `int number = 10;` ဟု ကြေညာလိုက်သောအခါ `number` သည် Memory တွင် တစ်နေရာ၌ သိမ်းဆည်းထားသည်။ ထိုနေရာ၏ Address ကို သိရှိလိုပါက `&number` ကို အသုံးပြုရမည် ဖြစ်သည်။ ဤနေရာတွင် Address နှင့် Value များသည် တစ်ခုနှင့်တစ်ခု မည်သို့ ဆက်စပ်နေသည်ကို နားလည်ရန် အရေးကြီးပါသည်။

### Pointer ဆိုတာ ဘာလဲ (What is a Pointer?)

Pointer (ညွှန်ပြချက်) ဆိုသည်မှာ Memory Address များကို သိမ်းဆည်းထားသည့် Variable တစ်မျိုးပင် ဖြစ်သည်။ ပုံမှန် Variable များသည် တန်ဖိုး (Value) ကို သိမ်းဆည်းထားသော်လည်း Pointer များသည် အခြား Variable များ၏ Memory Address များကို သိမ်းဆည်းထားသည်။ ဥပမာအားဖြင့်၊ `int *ptr;` ဟု ကြေညာလိုက်သောအခါ `ptr` သည် `int` အမျိုးအစားရှိ Variable တစ်ခု၏ Address ကို လက်ခံနိုင်သည့် ညွှန်ပြချက် ဖြစ်သည်။ ဆိုလိုသည်မှာ `ptr` သည် တန်ဖိုးကို မသိဘဲ အခြားနေရာတစ်ခုကို ရည်ညွှန်းနေခြင်း ဖြစ်သည်။

Memory ကို ကျွန်ုပ်တို့၏ နေ့စဉ်ဘဝတွင် ဗီရိုအံဆွဲများဖြင့် နှိုင်းယှဉ်ကြည့်လျှင်၊ Memory သည် ကျယ်ပြန့်သော မြေပြင်တစ်ခုနှင့် တူသည်။ Variable များသည် ထိုမြေပြင်ပေါ်ရှိ သီးခြားနေရာများဖြစ်ပြီး၊ ထိုနေရာတိုင်းတွင် တန်ဖိုး (Value) တစ်ခုစီ ရှိသည်။ Pointer ဆိုသည်မှာ ထိုနေရာများ၏ လိပ်စာကတ်များ (Address Cards) ကဲ့သို့ ဖြစ်သည်။ ဤလိပ်စာကတ်များသည် တန်ဖိုးကို မညွှန်ပြဘဲ ထိုတန်ဖိုးရှိသော နေရာကိုသာ ညွှန်ပြပေးသည်။

### Address နှင့် Value များ၏ ဆက်စပ်မှု

Pointer များအသုံးပြုရာတွင် အရေးအကြီးဆုံးမှာ Address နှင့် Value များ မည်သို့ ဆက်စပ်နေသည်ကို နားလည်ရန် ဖြစ်သည်။

ပထမအချက်မှာ Address သည် Memory အတွင်းရှိ နေရာကို ဖော်ပြသည်။ ဒုတိယအချက်မှာ ထိုနေရာတွင် သိမ်းဆည်းထားသော တန်ဖိုး (Value) ဖြစ်သည်။ Pointer သည် ထို Address ကို သိမ်းဆည်းထားသည်။

Pointer မှတဆင့် တန်ဖိုးကို ဖတ်ခြင်းနှင့် ရေးခြင်းကို `*` (Dereference) Operator ဖြင့် လုပ်ဆောင်သည်။ `*` ကို Pointer ၏ ရှေ့တွင် ထားခြင်းသည် ထို Pointer က ညွှန်ပြနေသော Address ရှိ တန်ဖိုးကို ရည်ညွှန်းသည်။ ဥပမာအားဖြင့်၊ `*ptr` သည် `ptr` မှ သိမ်းဆည်းထားသော Address တွင် ရှိနေသည့် တန်ဖိုးကို ဖော်ပြသည်။ ထို့အပြင်၊ `*ptr = new_value;` ဟု ရေးလိုက်သောအခါ `ptr` မှ ညွှန်ပြနေသော Address တွင် `new_value` ကို သိမ်းဆည်းလိုက်ခြင်း ဖြစ်သည်။ ဤအချက်က Pointer များသည် Memory ကို တိုက်ရိုက် ထိန်းချုပ်နိုင်ပုံကို ပြသသည်။

## ၂။ Pointer ဖြင့် Data ကို ထိန်းချုပ်ခြင်း (Controlling Data with Pointers)

### Pointer နှင့် Array များ၏ ဆက်နွယ်ပုံ (Relationship between Pointer and Array)

Pointer များသည် Array များနှင့် အလွန်နီးစပ်စွာ ဆက်စပ်နေသည်။ C ဘာသာစကားတွင် Array များသည် Memory တွင် အဆက်မပြတ် (contiguous) ဖြစ်နေသော Memory နေရာများအဖြစ် သိမ်းဆည်းထားသည်။ ထို့ကြောင့် Array တစ်ခု၏ နာမည် (Array Name) ကို ထို Array ၏ ပထမဆုံး Element ၏ Address အဖြစ် သတ်မှတ်နိုင်သည်။ အခြားနည်းအားဖြင့် ဆိုရလျှင်၊ Array တစ်ခု၏ နာမည်သည် ထို Array ၏ ပထမဆုံး Element ၏ Memory Address ကို ကိုယ်စားပြုသည့် Pointer တစ်ခုနှင့် အတူတူပင် ဖြစ်သည်။

Array ဝင်ရိုး (Row-major order) ဖြင့် Memory Layout ကို ဥပမာပေးရမည်ဆိုလျှင်၊ `int arr[3] = {10, 20, 30};` ဟု ကြေညာလိုက်သောအခါ၊ `arr` သည် ပထမဆုံး Element (10) ၏ Address ကို ညွှန်ပြနေသည်။ ထို့နောက် ဒုတိယ Element (20) သည် ပထမ Element ၏ Address မှ နောက်ထပ် `sizeof(int)` (ဥပမာ- 4 byte) လောက် အကွာအဝေးတွင် ရှိနေမည် ဖြစ်သည်။ ဤအဆက်မပြတ်တည်ရှိမှုကြောင့် Pointer Arithmetic သည် Array များအတွက် အလွန်အသုံးဝင်လာသည်။

### Pointer Arithmetic (Pointer Arithmetic)

Pointer Arithmetic ဆိုသည်မှာ Pointer များပေါ်တွင် အခြေခံ သင်္ချာအော်ပရေတာများဖြစ်သော `+` နှင့် `-` ကို အသုံးပြုခြင်းကို ဆိုလိုသည်။ Pointer များသည် Address များကို ကိုယ်စားပြုသောကြောင့်၊ Pointer များပေါ်တွင် ဤအော်ပရေတာများကို အသုံးပြုသောအခါ ထို Address များပေါ်တွင် အခြေခံသော ရုပ်ပိုင်းဆိုင်ရာ လှုပ်ရှားမှုများ ဖြစ်ပေါ်လာသည်။

Pointer များပေါ်တွင် `+` Operator ကို အသုံးပြုသောအခါ၊ ထို Pointer သည် နောက်ထပ် Data အမျိုးအစားတစ်ခု၏ Size အတိုင်း အကွာအဝေး တိုးတက်သွားသည်။ ဥပမာအားဖြင့်၊ `ptr + 1` ဟု ရေးလိုက်သောအခါ၊ ၎င်းသည် `ptr` မှ နောက်ထပ် `sizeof(data_type)` (ဥပမာ- `sizeof(int)`) ပုံရှိ Memory နေရာကို ညွှန်ပြသည်။ ဤနည်းလမ်းကို အသုံးပြု၍ Array ထဲသို့ လှည့်ပတ်ခြင်း (Navigation) ကို အလွယ်တကူ လုပ်ဆောင်နိုင်သည်။ ဤ Pointer Arithmetic သည် ကွန်ပျူတာ၏ Memory အတွင်းရှိ လှုပ်ရှားမှုများကို တိုက်ရိုက် ထိန်းချုပ်နိုင်ခြင်း၏ အဓိကသော့ချက် ဖြစ်သည်။

### Pointer နှင့် Function များ (Pointers and Functions)

Function များတွင် Data များကို လက်ခံရယူပုံကို `Call by Value` (တန်ဖိုးဖြင့် ခေါ်ခြင်း) နှင့် `Call by Reference` (ကိုးကားချက်ဖြင့် ခေါ်ခြင်း) ဟူ၍ နှစ်မျိုးရှိသည်။ ပုံမှန်အားဖြင့် `Call by Value` တွင် Function တစ်ခုသည် Argument များ၏ ကော်ပီကိုသာ လက်ခံရရှိသည်။ သို့သော် Pointer များ အသုံးပြုခြင်းဖြင့် `Call by Reference` ပုံစံကို အတုယူနိုင်ပြီး၊ Function များအတွင်း၌ တန်ဖိုးများကို တိုက်ရိုက် ပြောင်းလဲနိုင်စွမ်း ရှိသည်။

ဥပမာအားဖြင့်၊ နှစ်ခုသော တန်ဖိုးကို လဲလှယ်ရန် `swap` function ကို ဖန်တီးရာတွင်၊ ပုံမှန်အားဖြင့် တန်ဖိုးများကို လက်ခံရုံဖြင့် ပြောင်းလဲ၍ မရပါ။ ထို့ကြောင့်၊ ထိုတန်ဖိုးများ၏ Memory Address များကို ပြန်ပေးရန်အတွက် Pointer များကို အသုံးပြုရသည်။ `swap` function တွင် Pointer များအား Argument များအဖြစ် လက်ခံရရှိပြီး၊ ထို Pointer များမှတဆင့် တိုက်ရိုက် Memory တွင် တည်ရှိသော တန်ဖိုးများကို ပြောင်းလဲပေးခြင်းဖြင့် `Call by Reference` အကျိုးကျေးဇူးကို ရရှိသည်။ ဤနည်းလမ်းသည် Pointer များ၏ အစွမ်းကို Function Programming တွင် အသုံးချနိုင်ပုံကို ပြသသည်။

## ၃။ ပိုမိုရှုပ်ထွေးသော Pointer အယူအဆများ (More Advanced Pointer Concepts)

### Pointer to Pointer (Double Pointer) (Pointer to Pointer)

Pointer များ၏ အစွမ်းကို ပိုမိုနက်ရှိုင်းစွာ အသုံးချရန်အတွက် Pointer to Pointer (Double Pointer) သဘောတရားကို လေ့လာရမည် ဖြစ်သည်။ Double Pointer ဆိုသည်မှာ Pointer တစ်ခုကို ထပ်မံ၍ Pointer အဖြစ် သိမ်းဆည်းထားသော Pointer ဖြစ်သည်။ ဤအရာကို `**` (Dereference) Operator ဖြင့် ဖော်ပြသည်။

`ptr` သည် Address တစ်ခုကို သိမ်းဆည်းထားသည်။ `*ptr` သည် ထို Address တွင် ရှိသော တန်ဖိုးကို ရည်ညွှန်းသည်။ ထို့ကြောင့် `**ptr` သည် `ptr` မှ သိမ်းဆည်းထားသော Address တွင် ရှိသော တန်ဖိုးကို ရည်ညွှန်းသည်။ ဤသည်မှာ Pointer များဖြင့် Pointer များကို ထိန်းချုပ်ခြင်း၏ သဘောတရား ဖြစ်သည်။ Double Pointer များသည် အထူးသဖြင့် Complex Data Structures များ၊ Matrix များ သို့မဟုတ် Function များမှ Return လုပ်သော Pointer များကို ပြောင်းလဲရန် လိုအပ်သည့်အခါ အလွန်အသုံးဝင်သည်။

### Pointer နှင့် NULL (NULL Pointers)

Pointer များ အသုံးပြုရာတွင် အလွန်အရေးကြီးသော အချက်တစ်ခုမှာ Pointer ကို မည်သည့် Address ကိုမျှ ညွှန်ပြခြင်းမရှိဘဲ ထားခြင်း ဖြစ်သည်။ ဤသို့ မညွှန်ပြဘဲ ထားသော Pointer များကို `NULL` Pointer ဟု ခေါ်သည်။ `NULL` Pointer ဆိုသည်မှာ မည်သည့် Memory Address ကိုမျှ မညွှန်ပြသော Pointer ဖြစ်သည်။

Pointer တစ်ခုကို Initialize မလုပ်ဘဲ အသုံးပြုပါက၊ ထို Pointer သည် မသတ်မှတ်ထားသော Memory Address တစ်ခုကို ညွှန်ပြနေနိုင်သည်။ ဤသို့ မသတ်မှတ်ထားသော Address များသို့ ရောက်ရှိသွားပါက၊ ပရိုဂရမ်သည် မှားယွင်းသော Memory ကို ဖတ်ရန် သို့မဟုတ် ရေးရန် ကြိုးစားရာမှ `Segmentation fault` (အပိုင်းအခြားမှားခြင်း) ကဲ့သို့သော ပြင်းထန်သော Error များ ဖြစ်ပေါ်လာနိုင်သည်။ ထို့အပြင်၊ Pointer ကို အသုံးပြုပြီး မည်သည့်နေရာမှ မည်သည့်နေရာသို့ ရောက်သည်ကို မသိသောကြောင့် `Dangling Pointer` (လွတ်လပ်သော Pointer) နှင့် `Wild Pointer` (မသတ်မှတ်ထားသော Pointer) များ ဖြစ်ပေါ်လာနိုင်သည်။ ဤအရာများသည် ပရိုဂရမ်၏ တည်ငြိမ်မှုကို ထိခိုက်စေသော အကြီးမားဆုံးသော ပြဿနာများထဲမှ တစ်ခု ဖြစ်သည်။ ထို့ကြောင့် Pointer များကို အသုံးပြုသည့်အခါ ၎င်းတို့ကို အမြဲတမ်း `NULL` ဖြင့် စတင်ပေးခြင်း (Initialization) သည် အလွန်အရေးကြီးပါသည်။

### Pointer အမျိုးအစားများ (Pointer Types)

Pointer များသည် အမျိုးမျိုးသော အခြေအနေများအတွက် အသုံးပြုနိုင်သော အမျိုးအစားများ ရှိသည်။

`void` Pointer ဆိုသည်မှာ မည်သည့် Data အမျိုးအစားကိုမျှ ညွှန်ပြနိုင်သော Pointer ဖြစ်သည်။ ဤ Pointer သည် Memory Address ကို သိမ်းဆည်းသော်လည်း၊ ထို Address မှ မည်သည့် Data ကို ဖတ်မည်ကို သတ်မှတ်မထားပေ။ `void` Pointer သည် Generic Pointer အဖြစ် အသုံးပြုနိုင်ပြီး၊ ၎င်းသည် မည်သည့် Data အမျိုးအစားနှင့်မဆို အခြေခံကျသော လိပ်စာကို ကိုယ်စားပြုနိုင်သည်။

`const` နှင့် Pointer တွဲဖက်အသုံးပြုခြင်းသည် Pointer များ၏ လုံခြုံရေးကို မြှင့်တင်ပေးသည်။ `const` Keyword ကို Pointer နှင့် တွဲဖက်အသုံးပြုခြင်းဖြင့် ထို Pointer မှတဆင့် တန်ဖိုးများကို ပြောင်းလဲ၍မရဘဲ ဖတ်ရုံသာ လုပ်ဆောင်နိုင်စေသည်။ ဥပမာအားဖြင့်၊ `const int *ptr;` ဟု ကြေညာလိုက်သောအခါ `ptr` သည် `const` ဖြစ်သောကြောင့် ထို Pointer မှတဆင့် တန်ဖိုးကို ရေးရန် ကြိုးစားပါက Compile-time တွင် Error ပေါ်လာမည် ဖြစ်သည်။ ဤနည်းလမ်းသည် Pointer များမှတစ်ဆင့် မတော်တဆ ပြောင်းလဲမှုများ ဖြစ်ပေါ်ခြင်းကို ကာကွယ်ပေးသည်။

## ၄။ အထူးပြု အသုံးပြုမှုများနှင့် စီမံခန့်ခွဲမှု (Special Applications and Management)

### Pointer နှင့် Memory Management (Memory Management)

Pointer များသည် Dynamic Memory Allocation (ပြောင်းလွယ်ပြင်လွယ်ရှိသော မှတ်ဉာဏ် ခွဲဝေခြင်း) ကို စီမံခန့်ခွဲရာတွင် အလွန်အရေးပါပါသည်။ C တွင် `malloc()`, `calloc()`, `realloc()`, နှင့် `free()` Functions များသည် ပရိုဂရမ် runtime အချိန်တွင် Heap Memory တွင် လိုအပ်သော Memory ကို တောင်းဆိုပြီး Pointer များမှတစ်ဆင့် ထို Memory များကို ရယူပေးသည်။

Heap Memory နှင့် Stack Memory တို့သည် Memory ၏ မတူညီသော အပိုင်းများ ဖြစ်သည်။ Stack Memory သည် Function များ၏ Local Variables နှင့် Return Address များ စသည်တို့ကို စီမံခန့်ခွဲသည်။ ၎င်းသည် ပုံသေ (Static) ပုံစံဖြင့် စီမံခန့်ခွဲသည်။ Heap Memory သည် `malloc()` စသည့် Function များဖြင့် တောင်းဆိုသော Dynamic Memory များအတွက် ဖြစ်ပြီး၊ ၎င်းသည် ပရိုဂရမ်၏ Runtime အချိန်တွင် လိုအပ်သလောက် Memory ကို တောင်းဆိုပြီး ပြန်လွှဲပေးရသည်။ Pointer များသည် ဤ Heap Memory ကို ထိန်းချုပ်ပြီး Memory Leak (မှတ်ဉာဏ် ပေါက်ကြားမှု) နှင့် Buffer Overflow (အလွန်အကျွံ ရေးထည့်မှု) ကဲ့သို့သော ပြဿနာများကို ရှောင်ရှားရန် လမ်းညွှန်ပေးသည်။ `free()` Function ကို အသုံးပြု၍ မလိုအပ်တော့သော Memory များကို ပြန်လွှဲပေးခြင်းဖြင့် Memory Leak ကို ကာကွယ်နိုင်သည်။

## ၅။ အခန်းနိဂုံးနှင့် အနှစ်ချုပ် (Conclusion and Summary)

### Pointer များ၏ အကျဉ်းချုပ် (Summary of Pointers)

Pointer များသည် C Programming ၏ အခြေခံကျသော အခွင့်အလမ်းများ (Power) ကို ပေးအပ်သည်။ ၎င်းတို့သည် Memory ကို အလွှာလိုက် (Layer by Layer) နားလည်စေပြီး၊ ပရိုဂရမ်မှ Memory ကို တိုက်ရိုက် ထိန်းချုပ်နိုင်စွမ်းကို ပေးသည်။ Address များ၊ တန်ဖိုးများနှင့် Data Structure များအကြားရှိ ဆက်စပ်မှုကို ပေါ်လွင်စေသည်။ Pointer များကို ကျွမ်းကျင်စွာ အသုံးပြုခြင်းသည် ကွန်ပျူတာ၏ Machine Level တွင် ဖြစ်ပျက်နေသော အခြေအနေများကို နားလည်ပြီး ထိန်းချုပ်နိုင်စွမ်းကို ပေးသည်။

### အနှစ်ချုပ် (Conclusion)

Pointer များသည် C ဘာသာစကား၏ အခြေခံကျသော အခွင့်အလမ်းများ ဖြစ်သည်။ ၎င်းတို့ကို ကျွမ်းကျင်စွာ အသုံးပြုနိုင်ခြင်းသည် ပရိုဂရမ်ရေးသားရာတွင် ပိုမိုထိရောက်မှုရှိစေသည်။ Pointer များမှတစ်ဆင့် Memory ၏ အသေးစိတ်အချက်အလက်များကို သိရှိနားလည်ခြင်းသည် ပရိုဂရမ်၏ စွမ်းဆောင်ရည်ကို မြှင့်တင်ပေးသည်။ Pointer များသည် Data Structure များ (ဥပမာ- Linked List, Tree) ကို တည်ဆောက်ပုံကို နားလည်ရန်အတွက် အခြေခံကျသော အုတ်မြစ် ဖြစ်သည်။

ဤအခန်းတွင် လေ့လာခဲ့သော Pointer များ၏ အယူအဆများသည် C Programming ၏ အရေးကြီးသော အစိတ်အပိုင်းတစ်ခု ဖြစ်သည်။ Pointer များကို မှန်ကန်စွာ အသုံးပြုတတ်ရန် လေ့ကျင့်မှုနှင့် လက်တွေ့အသုံးချမှုများ လိုအပ်ပါသည်။ နောက်အခန်းများတွင် Pointer များမှတစ်ဆင့် Data Structure များဖြစ်သော Linked List နှင့် Tree များအကြောင်းကို ဆက်လက်လေ့လာသွားမည် ဖြစ်သည်။
