# Chapter 17: Tree နှင့် Hash Table — Advanced Data Structure

## ၁။ အခန်း မိတ်ဆက်နှင့် အခြေခံသဘောတရားများ (Introduction and Foundational Concepts)

### ၁.၁ Data Structure များ၏ အရေးပါပုံ (Importance of Data Structures)

ကွန်ပျူတာ သိပ္ပံနှင့် Programming လောကတွင် Data Structure များသည် အခြေခံကျသော အခန်းကဏ္ဍမှစ၍ အလွန်အရေးကြီးသော အကြောင်းအရာတစ်ခု ဖြစ်ပါသည်။ Data Structure ဆိုသည်မှာ Data များကို စနစ်တကျ သိမ်းဆည်းခြင်း၊ စီစဉ်ခြင်းနှင့် ထိရောက်စွာ ကိုင်တွယ်ဖြေရှင်းနိုင်ရန်အတွက် သတ်မှတ်ထားသော နည်းလမ်းများ (methods) နှင့် အစီအစဉ်များ (organization) ကို ဆိုလိုပါသည်။ ကျွန်ုပ်တို့၏ ကွန်ပျူတာများတွင် သိမ်းဆည်းထားသော အချက်အလက်များ (Data) သည် ဂဏန်းများ၊ စာလုံးများ၊ ပုံရိပ်များအစရှိသည်ဖြင့် အမျိုးမျိုးရှိကြသည်။ ထိုအချက်အလက်များကို ရိုးရိုးရှင်းရှင်း သိမ်းဆည်းထားခြင်းထက်၊ ထိုအချက်အလက်များကို မည်သို့ စီစဉ်ထားသည်ကို မူတည်၍ ထိုအချက်အလက်များကို မည်မျှမြန်ဆန်စွာ ရှာဖွေနိုင်သည်၊ ထည့်သွင်းနိုင်သည်၊ ဖယ်ရှားနိုင်သည်တို့ကို ဆုံးဖြတ်ပေးပါသည်။ Data Structure များသည် Memory ထဲရှိ Data များကို အလွန်ထိရောက်စွာ သိမ်းဆည်းခြင်းနှင့် ထိရောက်စွာ ကိုင်တွယ်ခြင်း၏ အဓိက အခန်းကဏ္ဍမှ ပါဝင်သည်။ Data Structure များ မရှိပါက၊ ကျွန်ုပ်တို့သည် Data များကို ရိုးရိုးရှင်းရှင်း စာကြောင်းများအဖြစ်သာ သိမ်းဆည်းရမည်ဖြစ်ပြီး၊ ရှုပ်ထွေးသော ဆက်စပ်မှုများ (relationships) ကို ကိုင်တွယ်ရန် အလွန်ခက်ခဲသွားမည် ဖြစ်သည်။

### ၁.၂ Tree (သစ်ပင်) မိတ်ဆက်ခြင်း (Introduction to Tree)

Data Structure အမျိုးအစားများထဲမှ Tree (သစ်ပင်) သည် အထူးခြားဆုံးနှင့် အသုံးအများဆုံးသော အဖွဲ့အစည်းပုံစံများထဲမှ တစ်ခုဖြစ်သည်။ Tree ဆိုသည်မှာ အဆင့်ဆင့် အဖွဲ့အစည်း (hierarchical data structure) တစ်မျိုး ဖြစ်သည်။ ဤအဖွဲ့အစည်းတွင် Data တစ်ခုကို အမြစ် (root) အဖြစ် သတ်မှတ်ပြီး၊ ထိုအမြစ်မှ အောက်သို့ အကိုင်းအခက်များ (branches) အဖြစ် ဆက်တိုက် ဆင်းသွားပြီး အောက်ခြေတွင် အစင်းများ (leaves) အဖြစ် အဆုံးသတ်သွားသည့် ပုံစံကို ဆိုလိုသည်။ ဤဖွဲ့စည်းပုံသည် မိဘ-သား ဆက်နွှယ်မှု (parent-child relationship) ကို ဖော်ပြပေးသည်။ မိဘ (parent) သည် သား (child) တစ်ခုနှင့် သို့မဟုတ် ထို့ထက်ပိုသော သားများအား ထိန်းချုပ်ထားပြီး၊ အမြစ်မှ စတင်၍ အောက်သို့ ဆင်းသွားသော ဤဆက်နွှယ်မှုသည် Data များအကြား ရှုပ်ထွေးသော ဆက်စပ်မှုများကို စနစ်တကျ ဖော်ပြပေးနိုင်သည်။ ဥပမာအားဖြင့်၊ မိသားစုအဖွဲ့အစည်း (Family structure) သို့မဟုတ် ဖိုင်စနစ် (File system) များသည် Tree ပုံစံဖြင့် အလွန်ကောင်းမွန်စွာ ကိုယ်စားပြုနိုင်သည်။

### ၁.၃ Binary Tree (နှစ်ထပ်သစ်ပင်) (Binary Tree)

Tree များထဲမှ Binary Tree (နှစ်ထပ်သစ်ပင်) သည် အထူးသဖြင့် ကွန်ပျူတာ သိပ္ပံတွင် အလွန်အသုံးများသည်။ Binary Tree ဆိုသည်မှာ Node တစ်ခုချင်းစီတွင် Child နှစ်ခု (left နှင့် right) သာ ရှိသော သစ်ပင်အမျိုးအစား ဖြစ်သည်။ ဤအခြေခံဖွဲ့စည်းပုံသည် Data များအား နှစ်ပိုင်းခွဲ၍ စီမံခန့်ခွဲရာတွင် အလွန်အထောက်အကူပြုသည်။ ဘယ်ဘက် (left) နှင့် ညာဘက် (right) အကိုင်းများဖြင့် သစ်ပင်ကို ခွဲခြားထားသောကြောင့် Data များအား အလွယ်တကူ စီမံခန့်ခွဲနိုင်ပါသည်။ Binary Tree ၏ အခြေခံဖွဲ့စည်းပုံသည် ရှာဖွေမှုနှင့် အချက်အလက်များကို စီစဉ်ရာတွင် အလွန်ထိရောက်မှုရှိသည်။ ဤဖွဲ့စည်းပုံသည် Binary Search Tree (BST) ကဲ့သို့သော အထူး Data Structure များ၏ အခြေခံအုတ်မြစ်လည်း ဖြစ်သည်။

## ၂။ Binary Search Tree (BST) ၏ အခြေခံနှင့် လုပ်ဆောင်ချက်များ (Fundamentals and Operations of BST)

### ၂.၁ Binary Search Tree (BST) မိတ်ဆက်ခြင်း (Introduction to BST)

Binary Search Tree (BST) ဆိုသည်မှာ အစီအစဉ်တကျ စီစဉ်ထားသော Binary Tree တစ်မျိုး ဖြစ်သည်။ BST ၏ အဓိက အားသာချက်မှာ Data များအား စီစဉ်ထားခြင်းကြောင့် ရှာဖွေမှု (searching) နှင့် အခြားလုပ်ဆောင်မှုများ (insertion, deletion) ကို အလွန်လျင်မြန်စွာ လုပ်ဆောင်နိုင်ခြင်း ဖြစ်သည်။ BST ၏ အဓိက စည်းမျဉ်းမှာ အလွန်တိကျပြီး အရေးကြီးပါသည်။ အဆိုပါ စည်းမျဉ်းမှာ- ဘယ်ဘက်အကိုင်းရှိ Data များသည် လက်ရှိ Node ၏ Data ထက် သေးငယ်ရမည်၊ အလယ်က Node (root) မှ Data သည် အခြား Data များထက် အကြီးဆုံး ဖြစ်ရမည်၊ ညာဘက်အကိုင်းရှိ Data များသည် လက်ရှိ Node ၏ Data ထက် ကြီးမားရမည်ဟူသော စည်းမျဉ်းဖြစ်သည်။ ဤစည်းမျဉ်းကို **left < root < right** ဟု ရေးသားလေ့ရှိသည်။ ဤစည်းမျဉ်းကို လိုက်နာမှသာ BST သည် ထိရောက်သော ရှာဖွေမှု လုပ်ဆောင်ချက်များကို ပေးစွမ်းနိုင်မည် ဖြစ်သည်။

### ၂.၂ BST တွင် Data ထည့်သွင်းခြင်း (Insertion in BST)

BST ထဲသို့ Data တစ်ခု ထည့်သွင်းသည့် လုပ်ငန်းစဉ်ကို Insertion ဟုခေါ်သည်။ Data တစ်ခုကို ထည့်သွင်းရာတွင်၊ ထို Data သည် BST ၏ အမြစ် (root) မှ စတင်၍ သင့်လျော်သော နေရာ (placement) ကို ရှာဖွေရမည် ဖြစ်သည်။ ပထမဆုံးအနေဖြင့်၊ လက်ရှိ Node မှ စတင်၍ ဘယ်ဘက်သို့ သွားသည် (left)။ လက်ရှိ Node ၏ Data ထက် Data အသစ်က သေးငယ်ပါက ထိုဘယ်ဘက်သို့ ဆက်သွားသည်။ ဤလုပ်ငန်းစဉ်ကို ထပ်မံလုပ်ဆောင်ပြီးနောက်၊ ထို Node ၏ ညာဘက်သို့ ဆက်သွားသည် (right)။ ထို Node ၏ Data ထက် Data အသစ်က ကြီးမားပါက ထိုညာဘက်သို့ ဆက်သွားသည်။ ဤရှာဖွေမှုသည် အဆုံးသတ်သည်အထိ ဆက်လက်လုပ်ဆောင်ပြီး၊ နောက်ဆုံးတွင် Data ထည့်သွင်းရန် သင့်လျော်သော အကိုင်းနေရာ (empty spot) ကို ရှာတွေ့သည့်အခါ ထိုနေရာတွင် Data အသစ်ကို ထည့်သွင်းပေးရမည် ဖြစ်သည်။ ဤလုပ်ငန်းစဉ်သည် ထည့်သွင်းမှု (Insertion) နှင့် ရှာဖွေမှု (Search) ကို တစ်ပြိုင်နက်တည်း ဆောင်ရွက်နိုင်သောကြောင့် BST သည် အလွန်ထိရောက်သည်။

### ၂.၃ BST တွင် Data ရှာဖွေခြင်း (Searching in BST)

BST ထဲမှ သတ်မှတ်ထားသော Key တစ်ခုကို ရှာဖွေသည့် လုပ်ငန်းစဉ်ကို Searching ဟုခေါ်သည်။ BST တွင် Search လုပ်ဆောင်ပုံမှာ အလွန်ရိုးရှင်းသည်။ အမြစ် (root) မှ စတင်၍ လက်ရှိ Node ၏ Data နှင့် ရှာဖွေလိုသော Key ကို နှိုင်းယှဉ်ရမည်။
၁။ လက်ရှိ Node ၏ Data သည် ရှာဖွေလိုသော Key နှင့် တူညီပါက၊ ထိုနေရာတွင် Key ကို ရှာဖွေတွေ့ရှိကြောင်း ပြန်အကြောင်းကြားသည်။
၂။ လက်ရှိ Node ၏ Data သည် ရှာဖွေလိုသော Key ထက် သေးငယ်ပါက၊ ထို Node ၏ ဘယ်ဘက် (left) အကိုင်းသို့ ဆက်လက်သွားသည်။
၃။ လက်ရှိ Node ၏ Data သည် ရှာဖွေလိုသော Key ထက် ကြီးမားပါက၊ ထို Node ၏ ညာဘက် (right) အကိုင်းသို့ ဆက်လက်သွားသည်။
ဤလုပ်ငန်းစဉ်သည် အဆုံးသတ်မည့်အချိန်တွင် ရှာဖွေလိုသော Key ကို ရှာမတွေ့ပါက ထို Key သည် Tree ထဲတွင် မရှိကြောင်း ပြန်လည်အကြောင်းကြားမည် ဖြစ်သည်။ ဤရှာဖွေမှုသည် Tree ၏ အမြင့်ဆုံးအဆင့်အထိ ဆင်းသွားသောကြောင့် အလွန်မြန်ဆန်သည်။ BST တွင် Search လုပ်ဆောင်မှု၏ Time Complexity ကို Big O ဖြင့် ဖော်ပြရာတွင် အကောင်းဆုံး အခြေအနေ (best case) တွင် $O(\log n)$ (logarithmic time) ဖြစ်ပြီး၊ အဆိုးဆုံး အခြေအနေ (worst case) တွင် $O(n)$ (linear time) ဖြစ်နိုင်သည်။

### ၂.၄ BST တွင် Data ဖယ်ရှားခြင်း (Deletion in BST)

BST မှ Data တစ်ခုကို ဖယ်ရှားသည့် လုပ်ငန်းစဉ် (Deletion) သည် အနည်းငယ် ပိုမိုရှုပ်ထွေးသည်။ ဖယ်ရှားလိုသော Data ကို ရှာဖွေတွေ့ရှိပြီးနောက်၊ ထို Node ကို ဖယ်ရှားရမည်။ Node ကို ဖယ်ရှားသည့်အခါ အောက်ပါ အခြေအနေသုံးမျိုး ရှိနိုင်သည်။
၁။ **Node သည် Leaf Node ဖြစ်ပါက** (ဘယ်ဘက်နှင့် ညာဘက် အကိုင်းများ မရှိပါက)၊ ထို Node ကို ဖယ်ရှားပြီး ၎င်း၏ Parent Node မှ Pointer ကို ပြန်လည်ချိန်ညှိပေးရမည်။
၂။ **Node တွင် Child တစ်ခုတည်းသာ ရှိပါက** (ဥပမာ- Left သို့မဟုတ် Right သာရှိပါက)၊ ထို Node ကို ဖယ်ရှားပြီး ၎င်း၏ Parent Node မှ Pointer ကို ထို Node ၏ Child သို့ ပြောင်းလဲပေးရမည်။
၃။ **Node တွင် Child နှစ်ခုလုံး ရှိပါက** (Left နှင့် Right နှစ်ခုလုံး ရှိပါက)၊ ဤအခြေအနေတွင် ဖယ်ရှားလိုသော Node နေရာတွင် ထို Node ၏ Data နှင့် တူညီသော အသေးဆုံး Data (Inorder Successor သို့မဟုတ် Inorder Predecessor) ကို နေရာယူပေးရမည်။ ထို Successor Data ကို ဖယ်ရှားပြီး၊ ထို Successor ၏ Data ကို နေရာတွင် ထည့်သွင်းပေးရမည်။ ထို့နောက်၊ Successor Node ကို ဖယ်ရှားရန်အတွက် အထက်အဆင့်မှ Pointer များကို ပြန်လည်ချိန်ညှိပေးရမည်။ ဤလုပ်ငန်းစဉ်သည် Pointer များကို စနစ်တကျ ပြန်လည်ချိန်ညှိခြင်း (Pointer manipulation) ကို အဓိကထားသည်။

### ၂.၅ Tree Traversal နည်းလမ်းများ (Tree Traversal Methods)

Tree Traversal ဆိုသည်မှာ Tree တစ်ခုရှိ Node အားလုံးကို စနစ်တကျ၊ အစီအစဉ်တကျ လှည့်ပတ်ကြည့်ရှုခြင်းကို ဆိုလိုသည်။ ဤလုပ်ငန်းစဉ်ကို အလွယ်တကူ လုပ်ဆောင်နိုင်ရန်အတွက် Recursion (ပြန်လည်ခေါ်ဆိုမှု) နည်းလမ်းကို အသုံးပြုလေ့ရှိသည်။ Tree Traversal နည်းလမ်းများအဓိကမှာ အောက်ပါအတိုင်း ဖြစ်သည်။

၁။ **Inorder Traversal (အထဲမှ အပြင်သို့)** - ဤနည်းလမ်းသည် BST အတွက် အရေးကြီးသည်။ ၎င်းသည် Left Subtree မှ စတင်ပြီး Root ကို သွားကာ Right Subtree သို့ ဆက်သွားသည်။ ဤနည်းလမ်းသည် BST တွင် Data များအား တိကျသော အစီအစဉ် (sorted order) ဖြင့် ထုတ်ပေးသည်။
၂။ **Preorder Traversal (အစမှ အလယ်သို့)** - ဤနည်းလမ်းသည် Root ကို အရင်ကြည့်ပြီး၊ ထို့နောက် Left Subtree နှင့် Right Subtree များကို အလှည့်ကျ လိုက်ကြည့်ရှုသည်။
၃။ **Postorder Traversal (အပြင်မှ အစသို့)** - ဤနည်းလမ်းသည် Left Subtree နှင့် Right Subtree များကို အရင်ကြည့်ပြီး၊ နောက်ဆုံးတွင် Root ကို ကြည့်ရှုသည်။

ဤ Traversal နည်းလမ်းများကို အကောင်အထည်ဖော်ရာတွင် Recursion ကို အသုံးပြုခြင်းသည် အလွန်ရိုးရှင်းသည်။ Function တစ်ခုသည် မိမိကိုယ်ကို ပြန်လည်ခေါ်ဆိုခြင်းဖြင့် Tree တစ်ခုလုံးကို စီမံခန့်ခွဲနိုင်သည်။ ဥပမာအားဖြင့်၊ Inorder Traversal ကို Recursion ဖြင့် အကောင်အထည်ဖော်ပုံမှာ- လက်ရှိ Node ရှိ Left Child ကို Recursively ခေါ်ဆိုပြီး၊ ထို့နောက် လက်ရှိ Node ကို Process လုပ်ပြီး၊ နောက်ဆုံးတွင် Right Child ကို Recursively ခေါ်ဆိုခြင်းဖြင့် အကောင်အထည်ဖော်နိုင်သည်။ ဤနည်းလမ်းသည် Tree ၏ အစိတ်အပိုင်းများအား စနစ်တကျ စီမံခန့်ခွဲရန် အလွန်သင့်လျော်သည်။

## ၃။ Hash Table (ကုဒ်အစုအဝေး) ၏ အခြေခံသဘောတရားများ (Fundamentals of Hash Table)

### ၃.၁ Hash Table မိတ်ဆက်ခြင်း (Introduction to Hash Table)

Hash Table ဆိုသည်မှာ Key-Value Pair Data Structure တစ်မျိုး ဖြစ်သည်။ ဤ Data Structure ၏ အဓိက ရည်ရွယ်ချက်မှာ Key တစ်ခုကို အသုံးပြု၍ သက်ဆိုင်ရာ Value တစ်ခုကို အလွန်လျင်မြန်စွာ ရှာဖွေခြင်း (Lookup) ဖြစ်သည်။ Hash Table ၏ အကြီးမားဆုံး အားသာချက်မှာ အချိန် အတက်အကျ အလွန်နည်းခြင်း၊ ၎င်းကို $O(1)$ (Constant time) Lookup ဟု ဆိုသည်။ ၎င်းသည် Data များကို အင်မတန် မြန်ဆန်စွာ သိမ်းဆည်းရန်နှင့် ပြန်လည်ရယူရန် ရည်ရွယ်သည်။ ၎င်းကို အကောင်အထည်ဖော်ရာတွင်၊ Data များကို သိမ်းဆည်းရန်အတွက် Array တစ်ခုကို အသုံးပြုပြီး၊ Key ကို Hash Function တစ်ခုမှတစ်ဆင့် Array ၏ Index (နေရာ) သို့ ပြောင်းလဲပေးခြင်းဖြင့် Value ကို ထည့်သွင်းသည်။

### ၃.၂ Hash Function (ကုဒ်လုပ်ဆောင်ချက်) (Hash Function)

Hash Function ဆိုသည်မှာ Key တစ်ခုကို Array ၏ Index သို့ ပြောင်းလဲပေးသည့် သင်္ချာဆိုင်ရာ လုပ်ဆောင်ချက် (mathematical operation) တစ်ခု ဖြစ်သည်။ Hash Function ၏ အဓိက ရည်ရွယ်ချက်မှာ Key တစ်ခုကို ထိရောက်စွာ (evenly) Array ၏ Index များအကြားတွင် ခွဲခြားပေးရန် ဖြစ်သည်။ Effective Hash Function များသည် Key များ၏ မတူညီမှုများအား Array ၏ Index များအကြားတွင် တန်းတန်းစီ (distribution) ဖြစ်စေရန် အရေးကြီးသည်။ Hash Function များသည် Key ကို Index အဖြစ် ပြောင်းလဲပေးသည့်အခါ၊ ထို Index သည် Array ၏ အပိုင်းအစ (index) အတွင်း၌ ရှိရမည် ဖြစ်သည်။ Hash Function ၏ အားနည်းချက်မှာ၊ အချို့သော Hash Function များသည် Key များ၏ အနည်းငယ်သော ကွဲလွဲမှုများကြောင့် Index များ တူညီသော နေရာသို့ ရောက်ရှိစေခြင်း (Collision) ဖြစ်ပေါ်စေသည်။ ထို့ကြောင့် ထိရောက်သော Hash Function များသည် Collision ကို အနည်းဆုံးဖြစ်အောင် ထိန်းချုပ်ပေးနိုင်ရန် အရေးကြီးသည်။

### ၃.၃ Collision Handling (တိုက်မိခြင်းကို ကိုင်တွယ်ခြင်း)

Collision ဆိုသည်မှာ မတူညီသော Key များက Hash Function မှတစ်ဆင့် Array ၏ တစ်ခုတည်းသော Index သို့ ရောက်ရှိသွားခြင်းကို ဆိုလိုသည်။ ဤ Collision ကို ကိုင်တွယ်ဖြေရှင်းရန်အတွက် နည်းလမ်းများစွာ ရှိသည်။

၁။ **Chaining (Linked List ကို အသုံးပြုခြင်း)** - ဤနည်းလမ်းတွင်၊ Array ၏ Index တစ်ခုစီတွင် Data ကို တိုက်ရိုက်ထည့်သွင်းမည့်အစား၊ ထို Index တွင် Linked List တစ်ခုကို ထည့်သွင်းသည်။ Collision ဖြစ်ပေါ်သည့်အခါ၊ ထို Linked List ၏ နောက်ဆက်တွဲ (chain) တွင် ထို Data ကို ထည့်သွင်းသည်။ ဤနည်းလမ်းသည် Collision ကို အကောင်းဆုံး ကိုင်တွယ်နိုင်ပြီး၊ Data များအား စီစဉ်ထားရန် မလိုအပ်ဘဲ သိမ်းဆည်းနိုင်စေသည်။
၂။ **Open Addressing (အခြားနေရာများတွင် ထည့်သွင်းခြင်း)** - ဤနည်းလမ်းတွင်၊ Collision ဖြစ်ပေါ်သည့်အခါ၊ Hash Table ၏ အခြား ফাঁকা နေရာများကို ရှာဖွေ၍ ထိုနေရာတွင် Data ကို ထည့်သွင်းသည်။ ဥပမာအားဖြင့်၊ Linear Probing (Linear Probing) သည် Index မှ စတင်၍ ဆက်လက် ရှာဖွေခြင်း၊ Quadratic Probing သည် Index နှင့် ပတ်သက်သော ပုံစံဖြင့် ရှာဖွေခြင်းတို့ ပါဝင်သည်။ ဤနည်းလမ်းများသည် Memory ကို ထိရောက်စွာ အသုံးချနိုင်သော်လည်း၊ ရှာဖွေမှုနှင့် ဖယ်ရှားမှု လုပ်ငန်းစဉ်များတွင် ပိုမိုဂရုစိုက်ရန် လိုအပ်သည်။

## ၄။ Data Structure များ၏ နှိုင်းယှဉ်ချက်နှင့် ရွေးချယ်မှုများ (Comparison and Trade-offs)

### ၄.၁ Array, Linked List နှင့် Hash Table နှိုင်းယှဉ်ခြင်း (Comparison of Array, Linked List, and Hash Table)

Data Structure များ ရွေးချယ်ရာတွင်၊ ထို Data များအား မည်သို့ သိမ်းဆည်းမည်၊ မည်သို့ အသုံးပြုမည်ကို မူတည်၍ ရွေးချယ်ရမည် ဖြစ်သည်။ Array နှင့် Linked List တို့သည် Memory နှင့် Access ပုံစံတွင် သိသာသော ကွာခြားချက်များ ရှိသည်။ Array များသည် Memory တွင် အဆက်မပြတ် (contiguous) နေရာများတွင် သိမ်းဆည်းထားသောကြောင့် Data တစ်ခုကို Index ဖြင့် အလွန်လျင်မြန်စွာ (Direct access) ရယူနိုင်သော်လည်း၊ Data ကို ထည့်သွင်းခြင်း သို့မဟုတ် ဖယ်ရှားခြင်းများတွင် အခက်အခဲရှိနိုင်သည်။ Linked List များသည် Memory တွင် မည်သည့်နေရာမဆို ဖြန့်ကျက်စွာ သိမ်းဆည်းထားနိုင်ပြီး၊ Data ထည့်သွင်းခြင်းနှင့် ဖယ်ရှားခြင်းများတွင် ပိုမိုလွယ်ကူသည်။ သို့သော် Linked List တွင် Data တစ်ခုကို ရှာဖွေရန်အတွက် အစမှစ၍ တစ်ဆင့်ချင်း စစ်ဆေးရသောကြောင့် Access လုပ်ဆောင်မှုသည် Array ထက် နှေးကွေးသည်။

Hash Table ကို Array နှင့် Linked List တို့နှင့် နှိုင်းယှဉ်ကြည့်လျှင်၊ Hash Table သည် Key ကို အသုံးပြု၍ အလွန်လျင်မြန်သော Lookup ($O(1)$) ကို ပေးစွမ်းနိုင်သော်လည်း၊ Collision ဖြစ်ပေါ်လာပါက ထို Collision ကို ကိုင်တွယ်ရန်အတွက် အချိန်ပိုပေးရသည်။ Hash Table သည် Key-Value Mapping များအတွက် အကောင်းဆုံးဖြစ်ပြီး၊ အချက်အလက်များကို အစီအစဉ်တကျ မလိုအပ်ဘဲ မြန်ဆန်စွာ သိမ်းဆည်းရန် အထူးသင့်လျော်သည်။ ထို့ကြောင့်၊ ကျွန်ုပ်တို့သည် ရှာဖွေမှု (Searching) နှင့် ထည့်သွင်းမှု (Insertion) အပေါ် မူတည်၍ Time Complexity နှင့် Space Complexity (Trade-offs) ကို ထည့်သွင်းစဉ်းစားပြီး Data Structure ကို ရွေးချယ်သင့်သည်။

### ၄.၂ BST နှင့် Hash Table ၏ အသုံးချမှု (Applications of BST vs Hash Table)

BST နှင့် Hash Table တို့သည် မတူညီသော Data များအတွက် အကောင်းဆုံးဖြစ်သည်။ BST သည် Data များအား အစီအစဉ်တကျ (sorted order) ထားရှိရန် လိုအပ်သောအခါ အလွန်သင့်လျော်သည်။ ဥပမာအားဖြင့်၊ Dictionary Program များတွင် Key (စာလုံး) အလိုက် Value (အဓိပ္ပာယ်) များကို သိမ်းဆည်းရာတွင် BST သည် အလွန်ကောင်းမွန်သည်။ BST သည် Data များအား အစဉ်လိုက် စီစဉ်ထားသောကြောင့် Inorder Traversal ဖြင့် Data များကို အစဉ်တကျ ရယူနိုင်သည်။

Hash Table သည် Key-Value Mapping များအတွက် အကောင်းဆုံးဖြစ်သည်။ ဥပမာအားဖြင့်၊ Phone Book Program များတွင် နာမည် (Key) နှင့် ဖုန်းနံပါတ် (Value) ကို သိမ်းဆည်းရာတွင် Hash Table သည် အလွန်လျင်မြန်သည်။ Key ကို Hash Function ဖြင့် Index သို့ ပြောင်းလဲပြီး တိုက်ရိုက်ရှာဖွေနိုင်သောကြောင့် အချိန်ကုန်သက်သာသည်။ ထို့ကြောင့်၊ Data အစီအစဉ် (order) နှင့် အလွယ်တကူ ရှာဖွေမှု (fast access) တို့အပေါ် မူတည်၍ BST သို့မဟုတ် Hash Table ကို ရွေးချယ်အသုံးပြုရမည် ဖြစ်သည်။

## ၅။ လက်တွေ့ လေ့ကျင့်ခန်းများနှင့် နိဂုံး (Practical Exercises and Conclusion)

### ၅.၁ လက်တွေ့ BST Implement လုပ်ခြင်း (Practical BST Implementation)

C Language ကို အသုံးပြု၍ BST ကို လက်တွေ့ ရေးဆွဲခြင်းသည် Data Structure ၏ အတွင်းပိုင်း လုပ်ဆောင်ပုံကို နားလည်စေရန် အကောင်းဆုံး နည်းလမ်းဖြစ်သည်။ BST ကို Implement လုပ်ရာတွင် အခြေခံအားဖြင့် Node တစ်ခုကို သတ်မှတ်ရမည်။ ထို Node တစ်ခုတွင် Data တစ်ခုနှင့် Pointer နှစ်ခု (left နှင့် right) ပါဝင်ရမည်။ ထို့နောက် Insertion (ထည့်သွင်းခြင်း) နှင့် Search (ရှာဖွေခြင်း) အတွက် Function များ ရေးဆွဲရမည်။ Insertion လုပ်ရာတွင်၊ Data ကို ထည့်သွင်းရန် သင့်လျော်သော နေရာကို ရှာဖွေပြီး Pointer များကို ပြန်လည်ချိန်ညှိရန် လိုအပ်သည်။ Search လုပ်ရာတွင်လည်း၊ BST ၏ စည်းမျဉ်းအတိုင်း ဘယ်ဘက် သို့မဟုတ် ညာဘက်သို့ ဆက်လက် ရှာဖွေခြင်းဖြင့် အချိန်တိုအတွင်း Key ကို ရှာဖွေနိုင်မည် ဖြစ်သည်။ ထို့အပြင်၊ Inorder Traversal ကို Recursion ကို အသုံးပြု၍ အကောင်အထည်ဖော်ခြင်းဖြင့် Tree တစ်ခုလုံး၏ Data များကို အစီအစဉ်တကျ ထုတ်ပေးနိုင်မည် ဖြစ်သည်။ ဤလက်တွေ့ လေ့ကျင့်ခန်းမှတစ်ဆင့် Pointer များ၏ အလုပ်လုပ်ပုံနှင့် Recursion ၏ စွမ်းဆောင်ရည်ကို နားလည်လာမည် ဖြစ်သည်။

### ၅.၂ လက်တွေ့ Hash Table Implement လုပ်ခြင်း (Practical Hash Table Implementation)

Hash Table ကို လက်တွေ့ Implement လုပ်ရာတွင် Collision Handling ကို ထည့်သွင်းစဉ်းစားရန် အလွန်အရေးကြီးသည်။ Chaining နည်းလမ်းကို အသုံးပြု၍ Hash Table ကို ရေးဆွဲခြင်းသည် Collision ကို Linked List များဖြင့် ကိုင်တွယ်ဖြေရှင်းခြင်း ဖြစ်သည်။ Hash Table ကို Array တစ်ခုအဖြစ် စတင်သတ်မှတ်ပြီး၊ Hash Function ကို အသုံးပြု၍ Key မှ Index ကို ရရှိအောင် လုပ်ဆောင်ရမည်။ Collision ဖြစ်ပေါ်ပါက၊ ထို Index တွင် Linked List တစ်ခုကို ထည့်သွင်းပြီး၊ ထို List တွင် တိုက်မိသော Data များကို ဆက်တိုက် ထည့်သွင်းရမည်။ ဤနည်းလမ်းဖြင့် Phone Book သို့မဟုတ် Contact List များကဲ့သို့သော Key-Value Mapping များကို အလွန်မြန်ဆန်စွာ စီမံခန့်ခွဲနိုင်မည် ဖြစ်သည်။ Open Addressing နည်းလမ်းများ (ဥပမာ- Linear Probing) ကိုလည်း စမ်းသပ်လေ့ကျင့်ခြင်းဖြင့်၊ Collision ကို Array ၏ အခြားနေရာများတွင် ထည့်သွင်းခြင်းဖြင့် ကိုင်တွယ်ပုံကို နားလည်လာမည် ဖြစ်သည်။

### ၅.၃ အခန်း အကျဉ်းချုပ် (Chapter Summary)

ဤအခန်းတွင် ကျွန်ုပ်တို့သည် Data Structure များ၏ အထက်အဆင့်ဖြစ်သော Tree နှင့် Hash Table တို့ကို လေ့လာခဲ့ကြသည်။ Tree များသည် အဆင့်ဆင့် အဖွဲ့အစည်း (hierarchical structure) ကို ဖော်ပြရန်နှင့် အချက်အလက်များကို အစီအစဉ်တကျ စီစဉ်ရန်အတွက် အလွန်အသုံးဝင်သည်။ Binary Search Tree (BST) သည် ထို Tree များထဲမှ အထူးပုံစံတစ်ခုဖြစ်ပြီး၊ အစီအစဉ်တကျ ထိန်းသိမ်းထားသော Data များအတွက် အကောင်းဆုံးဖြစ်သည်။ BST တွင် Insertion, Search, Deletion နှင့် Traversal (Inorder, Preorder, Postorder) လုပ်ဆောင်ချက်များကို Recursion ကို အသုံးပြု၍ အကောင်အထည်ဖော်နိုင်ကြောင်း လေ့လာခဲ့သည်။

တစ်ဖက်တွင်မူ Hash Table သည် Key-Value Pair များကို အလွန်မြန်ဆန်စွာ ရှာဖွေရန် (O(1) Lookup) ရည်ရွယ်သော Data Structure ဖြစ်သည်။ Hash Function မှတစ်ဆင့် Key ကို Index သို့ ပြောင်းလဲပေးခြင်းနှင့် Collision Handling (Chaining သို့မဟုတ် Open Addressing) နည်းလမ်းများကို အသုံးပြုခြင်းဖြင့် ထိရောက်စွာ စီမံခန့်ခွဲနိုင်ကြောင်း သိရှိခဲ့သည်။

အချုပ်အားဖြင့်ဆိုရလျှင်၊ Data Structure များသည် ကွန်ပျူတာ၏ အခြေခံကျသော အုတ်မြစ်များ ဖြစ်သည်။ Tree များသည် ဆက်စပ်မှု (relationships) နှင့် အဆင့်အလိုက် အဖွဲ့အစည်းများကို ကိုင်တွယ်ရာတွင် အထူးကောင်းမွန်ပြီး၊ Hash Table များသည် အချက်အလက်များကို အလွန်လျင်မြန်စွာ ရှာဖွေရန်အတွက် အထူးသင့်လျော်သည်။ Memory Management နှင့် Data Structure များ၏ အခြေခံအကျိုးဆက်များကို နားလည်ခြင်းသည် ကွန်ပျူတာ Programming ၏ အောင်မြင်မှုအတွက် အလွန်အရေးကြီးသည်။ ဤအကြောင်းအရာများသည် အဆင့်မြင့် Data Structure များ၏ အခြေခံများကို နားလည်စေရန်အတွက် အခြေခံအုတ်မြစ်များ ဖြစ်ပါသည်။
