# Chapter 16: Sorting နှင့် Searching Algorithm — ဒေတာ ရှာဖွေခြင်းနှင့် စီခြင်း

## ၁။ နိဒါန်းနှင့် အခြေခံသဘောတရားများ (Introduction and Fundamentals)

ကွန်ပျူတာတွေဟာ အချက်အလက်တွေ အများကြီးကို ကိုင်တွယ်ပြီး လုပ်ဆောင်တဲ့အခါ၊ အဲဒီအချက်အလက်တွေထဲက မှန်ကန်တဲ့ အရာကို ဘယ်လို အချိန်တိုအတွင်း ရှာဖွေမလဲ၊ ဒါမှမဟုတ် အဲဒီအချက်အလက်တွေကို စနစ်တကျ စီစဉ်မလဲဆိုတဲ့ မေးခွန်းတွေဟာ အလွန်အရေးကြီးပါတယ်။ ဒီမေးခွန်းတွေကို အဖြေပေးဖို့အတွက် Algorithm တွေဆိုတဲ့ အစီအစဉ်တွေဟာ အဓိကကျပါတယ်။ ဒီအခန်းမှာတော့ ဒီမေးခွန်းတွေကို အဖြေပေးမယ့် Sorting နှင့် Searching Algorithm တွေအကြောင်းကို အသေးစိတ် လေ့လာသွားမှာ ဖြစ်ပါတယ်။

### Algorithm ဆိုတာ ဘာလဲ (What is an Algorithm)

Algorithm (အယ်ဂိုရီသမ်) ဆိုတာ ဘာလဲဆိုတော့ ပြဿနာတစ်ခုကို ဖြေရှင်းဖို့အတွက် အဆင့်ဆင့် လုပ်ဆောင်ရမယ့် တိကျရှင်းလင်းတဲ့ လုပ်ငန်းစဉ် (step-by-step procedure) ဖြစ်ပါတယ်။ ဒါဟာ လူသားတွေက နေ့စဉ်ဘဝမှာ အစားအစာ ချက်ပြုတ်တာ၊ လမ်းကြောင်းရှာတာ စတဲ့ အရာတွေကို လုပ်ဆောင်တဲ့ ပုံစံနဲ့ ဆင်တူပါတယ်။ ကွန်ပျူတာအတွက် Algorithm ဆိုတာကတော့ ထိုပြဿနာကို အမှားအမှန်မရှိဘဲ၊ အဆုံးသတ်ရမယ့် အဆင့်တိုင်းကို တိကျရှင်းလင်းစွာ ဖော်ပြထားတဲ့ လုပ်ငန်းစဉ်တစ်ခု ဖြစ်ပါတယ်။ ဒီ Algorithm တွေဟာ ကွန်ပျူတာတွေရဲ့ အခြေခံကျတဲ့ အင်ဂျင်ဖြစ်ပြီး၊ ဘယ်လို Input (ထည့်သွင်းမှု) ကို ရခဲ့သည်ဖြစ်စေ၊ ဘယ်လို Output (ထုတ်ပေးမှု) ကို ရမလဲဆိုတာကို ဆုံးဖြတ်ပေးပါတယ်။

ကွန်ပျူတာနဲ့ ပတ်သက်ပြီး Algorithm တွေရဲ့ အရေးပါပုံကတော့ အလွန်ကြီးမားပါတယ်။ ကျွန်ုပ်တို့ဟာ ကွန်ပျူတာတွေကို အချက်အလက်တွေ ထည့်သွင်းပြီး ပေးလိုက်တဲ့အခါ (Input)၊ အဲဒီအချက်အလက်တွေကနေ အသုံးဝင်တဲ့ အဖြေ (Output) ကို ထုတ်ပေးနိုင်ဖို့အတွက် Algorithm တွေ လိုအပ်ပါတယ်။ Sorting နဲ့ Searching Algorithm တွေဟာ အချက်အလက်တွေရဲ့ အကြီးမားဆုံးနဲ့ အမြန်ဆုံး အကျိုးကျေးဇူးကို ရရှိစေဖို့အတွက် မရှိမဖြစ် လိုအပ်တဲ့ နည်းလမ်းတွေ ဖြစ်ပါတယ်။

### Efficiency ကို တိုင်းတာခြင်း (Measuring Efficiency)

Algorithm တစ်ခုရဲ့ ထိရောက်မှု (efficiency) ကို ဘယ်လို တိုင်းတာမလဲဆိုတဲ့ မေးခွန်းဟာ အလွန်အရေးကြီးပါတယ်။ အချက်အလက် အမြောက်အမြားကို ကိုင်တွယ်တဲ့အခါမှာ Algorithm တစ်ခုက အချိန်ဘယ်လောက်ကြာမလဲ၊ မည်မျှသော Memory ကို အသုံးပြုမလဲဆိုတာကို သိဖို့ လိုအပ်ပါတယ်။ ဒီထိရောက်မှုကို တိုင်းတာဖို့အတွက် Time Complexity နဲ့ Space Complexity ဆိုတဲ့ အယူအဆတွေကို မိတ်ဆက်ပေးချင်ပါတယ်။

Time Complexity (အချိန် ရှုပ်ထွေးမှု) ဆိုတာ Algorithm တစ်ခုက ပေးထားတဲ့ Input ပေါ်မူတည်ပြီး လုပ်ဆောင်ရမယ့် အချိန်ကို တိုင်းတာတာ ဖြစ်ပါတယ်။ Space Complexity (နေရာ ရှုပ်ထွေးမှု) ဆိုတာ Algorithm တစ်ခုက လုပ်ဆောင်နေစဉ်အတွင်း အသုံးပြုမယ့် Memory ပမာဏကို တိုင်းတာတာ ဖြစ်ပါတယ်။

ဒီနှစ်ခုကို ပိုမိုတိကျစွာ ဖော်ပြနိုင်ဖို့အတွက် Big O notation (Big O notation) ကို အသုံးပြုပါတယ်။ Big O notation ဟာ Algorithm ရဲ့ အဆိုးဆုံး အခြေအနေ (Worst-case scenario) မှာ အချိန် သို့မဟုတ် Memory ပမာဏ ဘယ်လောက်အထိ တိုးလာမလဲဆိုတာကို ဖော်ပြပေးတဲ့ သင်္ကေတ (notation) တစ်မျိုး ဖြစ်ပါတယ်။

Big O notation ရဲ့ အဓိက အခြေအနေအချို့ကို ဥပမာများနှင့် နှိုင်းယှဉ်ပြီး ရှင်းပြပါမယ်။

*   **O(1) (Constant Time):** အချိန် မကြာမီ ဘယ်လောက်မဆို အချိန်ယူရမယ့် လုပ်ဆောင်မှုမျိုးကို ဆိုလိုပါတယ်။ ဥပမာ- တစ်ခုတည်းသော Variable တစ်ခုကို တန်ဖိုးပေးတာ၊ Array ရဲ့ ပထမဆုံး Element ကို ဝင်ကြည့်တာမျိုးဟာ အချိန်မရွေး အမြန်ဆုံး လုပ်ဆောင်နိုင်တာကို ဆိုလိုပါတယ်။ ဒါကို Memory ကို ဗီရိုအံဆွဲနဲ့ နှိုင်းယှဉ်ရင်၊ အချိန်မရွေး အတူတူပဲ အချိန်ပေးရတာမျိုး ဖြစ်ပါတယ်။
*   **O(n) (Linear Time):** Input ပမာဏ (n) နဲ့ တိုက်ရိုက်အချိုးကျ လုပ်ဆောင်ရမယ့် လုပ်ဆောင်မှုမျိုးကို ဆိုလိုပါတယ်။ ဥပမာ- Array တစ်ခုထဲက ဒေတာအားလုံးကို တစ်ခုချင်းစီ စစ်ဆေးတာ (Linear Search) ကို ဥပမာပေးရရင်၊ ဒေတာ ပမာဏ နှစ်ဆ တိုးလာရင် အချိန်လည်း နှစ်ဆ တိုးလာပါတယ်။
*   **O(n²) (Quadratic Time):** Input ပမာဏ နှစ်ခုကို အခြေခံပြီး လုပ်ဆောင်ရမယ့် လုပ်ဆောင်မှုမျိုးကို ဆိုလိုပါတယ်။ ဥပမာ- Bubble Sort လိုမျိုး Algorithm တွေဟာ အနီးအနားက ဒေတာနှစ်ခုကို မဖြစ်မနေ နှိုင်းယှဉ်ပြီး လဲလှယ်ရတဲ့အတွက် ဒေတာ ပမာဏ နှစ်ဆ တိုးလာရင် အချိန်က လေးဆ တိုးလာနိုင်ပါတယ်။
*   **O(log n) (Logarithmic Time):** Input ပမာဏ တိုးလာပေမယ့် အချိန်မှာ အလွန်နှေးကွေးစွာ တိုးလာမယ့် လုပ်ဆောင်မှုမျိုးကို ဆိုလိုပါတယ်။ ဥပမာ- Binary Search လိုမျိုး Algorithm တွေဟာ ဒေတာအစုအဝေးကို တစ်ဝက်စီ ခွဲပြီး ရှာဖွေတဲ့အတွက် အလွန်လျင်မြန်စွာ အဖြေရနိုင်ပါတယ်။

Big O notation ဟာ Algorithm တွေရဲ့ အကောင်းဆုံးနဲ့ အဆိုးဆုံး စွမ်းဆောင်ရည်ကို နှိုင်းယှဉ်ပြီး၊ အကြီးမားဆုံး အချက်အလက်ပမာဏတွေအတွက် ဘယ်လောက်ထိ ထိရောက်မှုရှိမလဲဆိုတာကို နားလည်ဖို့ အထောက်အကူပြုပါတယ်။

## ၂။ ဒေတာ ရှာဖွေခြင်း (Searching Algorithms)

ဒေတာ ရှာဖွေခြင်း (Searching) ဆိုတာကတော့ ရှိပြီးသား ဒေတာအစုအဝေးတစ်ခုထဲမှာ ကျွန်ုပ်တို့ လိုချင်တဲ့ အစိတ်အပိုင်းတစ်ခုကို ရှာဖွေခြင်းပဲ ဖြစ်ပါတယ်။ ဒီအပိုင်းမှာတော့ အလွယ်ဆုံးနဲ့ အထိရောက်ဆုံး ရှာဖွေရေး နည်းလမ်းနှစ်မျိုးကို လေ့လာသွားမှာ ဖြစ်ပါတယ်။

### Linear Search (မျဉ်းဖြောင့်ရှာဖွေခြင်း)

Linear Search ဆိုတာကတော့ ဒေတာအစုအဝေးတစ်ခုထဲက လိုချင်တဲ့ အစိတ်အပိုင်းတစ်ခုကို ရှာဖွေဖို့အတွက် အလွန်ရိုးရှင်းတဲ့ နည်းလမ်းပဲ ဖြစ်ပါတယ်။ ဒီနည်းလမ်းမှာ ဒေတာတစ်ခုချင်းစီကို စတင်ကနေ အဆုံးအထိ တစ်ခုချင်းစီ စစ်ဆေးပြီး ရှာဖွေခြင်းကို ဆိုလိုပါတယ်။

Linear Search ရဲ့ Time Complexity ဟာ O(n) ဖြစ်ပါတယ်။ ဆိုလိုတာက Input ပမာဏ (n) ပေါ်မူတည်ပြီး ရှာဖွေရမယ့် အချိန်ဟာ n နဲ့ တိုက်ရိုက်အချိုးကျ တိုးလာမှာ ဖြစ်ပါတယ်။

နေ့စဉ်ဘဝဥပမာနဲ့ နှိုင်းယှဉ်ကြည့်ရအောင်၊ စာအုပ်တစ်အုပ်ထဲမှာ သင့်နာမည်ကို ရှာဖွေမယ်လို့ ဆိုပါစို့။ သင်က စာအုပ်ရဲ့ စာမျက်နှာ ၁ မှ စတင်ပြီး နာမည်ကို စစ်ဆေးမယ်ဆိုရင်၊ အဲဒီနာမည်ကို စာမျက်နှာ ၁ မှာ မတွေ့ရင် စာမျက်နှာ ၂ မှာ စစ်ဆေးပြီး၊ မတွေ့ရင် စာမျက်နှာ ၃ မှာ စစ်ဆေးသွားရမှာ ဖြစ်ပါတယ်။ အကယ်၍ နာမည်က စာမျက်နှာ ၁၀၀ မှာ ရှိတယ်ဆိုရင်၊ သင်ဟာ အများဆုံး ၁၀၀ ကြိမ် စစ်ဆေးရမှာ ဖြစ်ပါတယ်။ ဒါကြောင့် ဒေတာ ပမာဏ (n) ပိုများလေလေ၊ ရှာဖွေရမယ့် အချိန်လည်း ပိုကြာလေလေ ဖြစ်ပါတယ်။ ဒါကို အလွန်အကျယ်ဆိုရရင်၊ စာအုပ်တစ်အုပ်ထဲက စာမျက်နှာ တစ်မျက်နှာချင်း စစ်ဆေးတာနဲ့ တူပါတယ်။

### Binary Search (နှစ်ထပ်ရှာဖွေခြင်း)

Linear Search ဟာ ရိုးရှင်းပေမယ့် ဒေတာအများကြီးအတွက် အချိန်ကုန်နိုင်တာကြောင့် ပိုမိုထိရောက်တဲ့ နည်းလမ်းတွေ ရှိပါတယ်။ အဲဒါကတော့ Binary Search (နှစ်ထပ်ရှာဖွေခြင်း) ဖြစ်ပါတယ်။ Binary Search ကို အသုံးပြုနိုင်ဖို့အတွက် ဒေတာအစုအဝေးဟာ **စီစဉ်ထားရပါမယ် (sorted)**။

Binary Search ရဲ့ အလုပ်လုပ်ပုံကတော့ အောက်ပါအတိုင်း အဆင့်ဆင့် ဆောင်ရွက်ရပါတယ်-

ပထမဆုံးအနေနဲ့ ဒေတာအစုအဝေးကို စီစဉ်ထားရပါမယ်။ ဥပမာအားဖြင့်၊ ဖုန်းထဲက Contact List တစ်ခုကို အမည်အလိုက် စီစဉ်ပြီး (A-Z အစီအစဉ်အတိုင်း) ထားရပါမယ်။

ဒုတိယအဆင့်အနေနဲ့၊ ရှာဖွေလိုသော အချက်အလက် (Target) ကို ရှာဖွေမည့် အလယ်မှတ် (Middle element) ကို ရှာဖွေပါတယ်။ ဒေတာအစုအဝေးရဲ့ အစ (Start) နဲ့ အဆုံး (End) ကြားက အလယ်မှတ်ကို ယူပြီး အဲဒီနေရာမှာ ရှိတဲ့ ဒေတာနဲ့ ရှာဖွေလိုတဲ့ အချက်အလက်ကို နှိုင်းယှဉ်ကြည့်ပါတယ်။

တတိယအဆင့်အနေနဲ့၊ ရှာဖွေလိုသော အချက်အလက်ဟာ အလယ်မှတ်ထက်သေးရင် အလယ်မှတ်ရဲ့ ဘယ်ဘက်ခြမ်းကိုသာ ရှာဖွေပြီး၊ အလယ်မှတ်ထက်ကြီးရင် အလယ်မှတ်ရဲ့ ညာဘက်ခြမ်းကိုသာ ရှာဖွေသွားပါတယ်။ ဒီလုပ်ငန်းစဉ်ကို အချက်အလက်တွေ လုံးဝမကျန်မကျန် ရှာဖွေနိုင်တဲ့အထိ ဆက်လက်လုပ်ဆောင်ရပါတယ်။

Binary Search ရဲ့ Time Complexity ဟာ O(log n) ဖြစ်ပါတယ်။ ဆိုလိုတာက Input ပမာဏ (n) နှစ်ဆ တိုးလာရင် ရှာဖွေရမယ့် အချိန်ဟာ နှစ်ဆ တိုးလာမှာ မဟုတ်ဘဲ အလွန်လျင်မြန်စွာ တိုးတက်လာမှာ ဖြစ်ပါတယ်။

ဖုန်းထဲက Contact List ကို ဥပမာပေးရရင်၊ သင်ဟာ စာအုပ်တစ်အုပ်လို အစီအစဉ်တကျ စီပြီး ရှာဖွေတာနဲ့ တူပါတယ်။ စာမျက်နှာ တစ်မျက်နှာချင်း မကြည့်ဘဲ၊ အလယ်စာမျက်နှာကိုကြည့်ပြီး အဲဒီစာမျက်နှာက နာမည်က ရှာနေတဲ့ နာမည်ထက်ကြီးလား သေးလားကို ဆုံးဖြတ်ပြီး လိုအပ်တဲ့ စာမျက်နှာအပိုင်းကို ချက်ချင်း ရွေးချယ်သွားတာမျိုး ဖြစ်ပါတယ်။ ဒါကြောင့် ဒေတာအစုအဝေးကြီးများအတွက် Linear Search ထက် အလွန်ထိရောက်ပါတယ်။

## ၃။ ဒေတာ စီစဉ်ခြင်း (Sorting Algorithms)

ဒေတာ စီစဉ်ခြင်း (Sorting) ဆိုတာကတော့ ဒေတာအစုအဝေးတစ်ခုကို သတ်မှတ်ထားသော စည်းမျဉ်းတစ်ခုအပေါ် မူတည်ပြီး စီစဉ်ခြင်းပဲ ဖြစ်ပါတယ်။ ဥပမာအားဖြင့်၊ ဂဏန်းတွေ၊ စာလုံးတွေ၊ သို့မဟုတ် အခြားသော ဒေတာအမျိုးအစားတွေကို အငယ်ဆုံးမှ အကြီးဆုံးအထိ သို့မဟုတ် အကြီးဆုံးမှ အငယ်ဆုံးအထိ စီစဉ်တာမျိုး ဖြစ်ပါတယ်။ Sorting Algorithm တွေဟာ ဒီလုပ်ငန်းစဉ်ကို အကောင်းဆုံးနဲ့ အမြန်ဆုံး လုပ်ဆောင်နိုင်ဖို့အတွက် ပုံစံအမျိုးမျိုးနဲ့ တည်ရှိကြပါတယ်။

### Sorting ရဲ့ အခြေခံသဘောတရားများ

Sorting Algorithm များရဲ့ အခြေခံကွာခြားချက်တွေကတော့ ၎င်းတို့ရဲ့ လုပ်ဆောင်ပုံနဲ့ Time/Space Complexity မှာ ရှိပါတယ်။ အချို့ Algorithm တွေဟာ ရိုးရှင်းပေမယ့် အချိန်အများကြီးယူနိုင်ပြီး အချို့ကတော့ ရှုပ်ထွေးပေမယ့် အလွန်မြန်ဆန်ပါတယ်။

### Bubble Sort (ပူဖောင်းစီခြင်း)

Bubble Sort ဟာ Sorting Algorithm များထဲမှာ အလွယ်ကူဆုံးနဲ့ အခြေခံကျဆုံး Algorithm တစ်ခု ဖြစ်ပါတယ်။ Bubble Sort ရဲ့ လုပ်ဆောင်ပုံကတော့ အနီးအနားက ဒေတာနှစ်ခုကို တစ်ခုနဲ့တစ်ခု နှိုင်းယှဉ်ပြီး မှားယွင်းနေပါက အဲဒီနှစ်ခုကို လဲလှယ်ခြင်းဖြင့် စီစဉ်ခြင်းပဲ ဖြစ်ပါတယ်။ အဲဒီလို လဲလှယ်မှုတွေကို ဒေတာအစုအဝေးတစ်ခုလုံးအတွက် အကြိမ်ပေါင်းများစွာ ထပ်ခါထပ်ခါ လုပ်ဆောင်ပြီး အဆုံးမှာ မှန်ကန်တဲ့ စီစဉ်မှုကို ရရှိအောင် လုပ်ဆောင်တာ ဖြစ်ပါတယ်။

Bubble Sort ရဲ့ အလုပ်လုပ်ပုံကို Animation-style နဲ့ ရှင်းပြရမယ်ဆိုရင်၊ ဒေတာတွေဟာ ရေထဲမှာရှိတဲ့ အစေ့တွေလိုမျိုး မြင်ယောင်ကြည့်နိုင်ပါတယ်။ အနီးအနားက အစေ့တွေဟာ အမြဲတမ်း အချင်းချင်း နှိုင်းယှဉ်ပြီး မှားယွင်းနေရင် အလှည့်ကျ လဲလှယ်သွားပါတယ်။ အချိန်ကြာလာတာနဲ့အမျှ အကြီးဆုံး အစေ့တွေဟာ အစုအဝေးရဲ့ အဆုံးထိ တဖြည်းဖြည်းချင်း ပူဖောင်းပြီး နေရာတကျ ရောက်သွားပါတယ်။

Bubble Sort ရဲ့ Time Complexity ဟာ Worst case မှာ O(n²) ဖြစ်ပါတယ်။ ဆိုလိုတာက Input ပမာဏ (n) နှစ်ဆ တိုးလာရင် အချိန်က လေးဆ တိုးလာနိုင်ပါတယ်။ ဒေတာအစုအဝေးကြီးများအတွက်တော့ O(n²) ဟာ အချိန်ကုန်နိုင်ပါတယ်။

### Selection Sort (ရွေးချယ်ပြီး စီစဉ်ခြင်း)

Selection Sort ဟာ ဒေတာအစုအဝေးကို စီစဉ်ဖို့အတွက် အသေးဆုံး (သို့မဟုတ် အကြီးဆုံး) အချက်အလက်ကို ရွေးချယ်ပြီး မှန်ကန်တဲ့ နေရာမှာ ထည့်သွင်းခြင်းဖြင့် စီစဉ်တဲ့ နည်းလမ်း ဖြစ်ပါတယ်။

Selection Sort ရဲ့ အလုပ်လုပ်ပုံကို အဆင့်ဆင့် ရှင်းပြရမယ်ဆိုရင်-

ပထမဆုံးအနေနဲ့၊ စီစဉ်မယ့် ဒေတာအစုအဝေးထဲက အသေးဆုံး (သို့မဟုတ် အကြီးဆုံး) အချက်အလက်ကို ရှာဖွေပါတယ်။
ဒုတိယအနေနဲ့၊ အဲဒီ အသေးဆုံး (သို့မဟုတ် အကြီးဆုံး) အချက်အလက်ကို စီစဉ်မယ့် အစဆုံးနေရာ (Position) မှာ ထားရှိဖို့အတွက် အဲဒီအချက်အလက်ကို ရွေးချယ်ပြီး ထားပါတယ်။
တတိယအနေနဲ့၊ ကျန်ရှိနေတဲ့ ဒေတာတွေထဲကနေ နောက်ထပ် အသေးဆုံး (သို့မဟုတ် အကြီးဆုံး) အချက်အလက်ကို ရှာပြီး နောက်နေရာမှာ ထားရှိပါတယ်။
ဒီလုပ်ငန်းစဉ်ကို ဒေတာအစုအဝေးထဲက ဒေတာအားလုံးကို စီစဉ်ပြီးတဲ့အထိ ဆက်လက်လုပ်ဆောင်ရပါတယ်။

Selection Sort ဟာ Bubble Sort ထက် ပိုပြီး လဲလှယ်မှု (swaps) နည်းပါးပေမယ့် အချိန်ကို ထိရောက်စွာ အသုံးပြုပါတယ်။ Time Complexity ဟာ အမြဲတမ်း O(n²) ဖြစ်ပါတယ်။

### Insertion Sort (ထည့်သွင်းစီစဉ်ခြင်း)

Insertion Sort ဟာ ကတ်ထုပ်စီစဉ်တဲ့ ပုံစံနဲ့ အလွန်တူပါတယ်။ အချက်အလက်အသစ်တစ်ခုကို လက်ရှိစီစဉ်ထားတဲ့ နေရာထဲသို့ ထည့်သွင်းပြီး မှန်ကန်တဲ့ နေရာမှာ ထည့်သွင်းဖို့အတွက် အရင်က ဒေတာတွေနဲ့ နှိုင်းယှဉ်ပြီး ပြန်စီစဉ်ခြင်းဖြင့် စီစဉ်တဲ့ နည်းလမ်း ဖြစ်ပါတယ်။

Insertion Sort ရဲ့ အလုပ်လုပ်ပုံကို ဥပမာပေးပြီး ရှင်းပြရမယ်ဆိုရင်-

သင်က ကတ်ထုပ်အစုံကို စီစဉ်ထားတယ်လို့ ဆိုပါစို့။ သင်က အသစ်ထည့်မယ့် ကတ်ထုပ်အသစ်ကို လက်ရှိ စီစဉ်ထားတဲ့ အစီအစဉ်အတိုင်း အစဉ်လိုက် ထည့်သွင်းရမှာ ဖြစ်ပါတယ်။ အသစ်ထည့်မယ့် ကတ်ထုပ်ကို လက်ရှိ အစီအစဉ်ထဲကို ထည့်လိုက်တဲ့အခါ၊ အဲဒီအသစ်ထည့်လိုက်တဲ့ ကတ်ထုပ်ဟာ မှန်ကန်တဲ့ နေရာကို ရောက်အောင် အရင်က ကတ်ထုပ်တွေနဲ့ နှိုင်းယှဉ်ပြီး နေရာပြောင်းလဲပေးရမှာ ဖြစ်ပါတယ်။ ဥပမာ၊ အကြီးဆုံးကနေ အငယ်ဆုံးအထိ စီစဉ်ထားတဲ့အခါ၊ အသစ်ထည့်လိုက်တဲ့ ကတ်ထုပ်ဟာ အဲဒီ အစီအစဉ်ထဲမှာ မှန်ကန်တဲ့ နေရာကို ရှာပြီး အဲဒီနေရာကို တွန်းထည့်ရမှာ ဖြစ်ပါတယ်။

Insertion Sort ရဲ့ Time Complexity ဟာ အကောင်းဆုံး အခြေအနေမှာ O(n) ဖြစ်ပါတယ်။ ဒေတာအစုအဝေးက အနည်းငယ်စီ ပြောင်းလဲနေတဲ့အခါမှာ အလွန်မြန်ဆန်စွာ စီစဉ်နိုင်ပါတယ်။

### Merge Sort (ပေါင်းစပ်စီစဉ်ခြင်း)

Merge Sort ဟာ Sorting Algorithm များထဲမှာ Divide and Conquer (ခွဲပြီး သိမ်းပြီး ပေါင်းစပ်ခြင်း) ဆိုတဲ့ နည်းလမ်းကို အသုံးပြုပြီး စီစဉ်တဲ့ နည်းလမ်း ဖြစ်ပါတယ်။

Merge Sort ရဲ့ အလုပ်လုပ်ပုံကို အဆင့်ဆင့် ရှင်းပြရမယ်ဆိုရင်-

ပထမဆုံးအနေနဲ့၊ ဒေတာအစုအဝေးကို အပိုင်းငယ်လေးတွေအဖြစ် ခွဲခြားပေးပါတယ်။ ဒီခွဲခြားတဲ့ လုပ်ငန်းစဉ်ကို Recursion (ပြန်လည်ခေါ်ဆိုမှု) ကို အသုံးပြုပြီး အဆုံးမရှိ ခွဲခြားနိုင်ပါတယ်။ အပိုင်းငယ်လေးတွေဟာ တစ်ခုချင်းစီမှာ အသေးဆုံးဖြစ်တဲ့အထိ ခွဲခြားခံရမှာ ဖြစ်ပါတယ်။

ဒုတိယအဆင့်အနေနဲ့၊ အပိုင်းငယ်လေးတွေဖြစ်တဲ့ အလွန်သေးငယ်တဲ့ အစုအဝေးနှစ်ခုကို ယူပြီး၊ အဲဒီနှစ်ခုကို မှန်ကန်တဲ့ စီစဉ်မှုနဲ့ ပေါင်းစပ်ပေးပါတယ်။ ဒီပေါင်းစပ်တဲ့ လုပ်ငန်းစဉ်က အလွန်အရေးကြီးပါတယ်။ အပိုင်းငယ်လေးတွေကနေ စပြီး ပေါင်းစပ်ပြီး အကြီးမားတဲ့ စီစဉ်မှုကို ရရှိအောင် ဆက်လက်လုပ်ဆောင်ရမှာ ဖြစ်ပါတယ်။

Merge Sort ရဲ့ Time Complexity ဟာ အမြဲတမ်း O(n log n) ဖြစ်ပါတယ်။ ဒါဟာ ဒေတာအစုအဝေးကြီးများအတွက် အလွန်ထိရောက်တဲ့ စီစဉ်မှု နည်းလမ်းတစ်ခု ဖြစ်ပါတယ်။

### Quick Sort (လျင်မြန်သော ခွဲခြားစီစဉ်ခြင်း)

Quick Sort ဟာလည်း Divide and Conquer နည်းလမ်းကို အသုံးပြုပြီး စီစဉ်တဲ့ နည်းလမ်း ဖြစ်ပါတယ်။ Quick Sort ရဲ့ အဓိကအချက်ကတော့ Pivot (အဓိကအချက်) တစ်ခုကို ရွေးချယ်ပြီး ဒေတာတွေကို Pivot နှင့် နှိုင်းယှဉ်၍ Partition (ခွဲခြား) လုပ်ခြင်းဖြင့် စီစဉ်ခြင်းပဲ ဖြစ်ပါတယ်။

Quick Sort ရဲ့ အလုပ်လုပ်ပုံကို ရှင်းပြရမယ်ဆိုရင်-

ပထမဆုံးအနေနဲ့၊ ဒေတာအစုအဝေးထဲကနေ Pivot တစ်ခုကို ရွေးချယ်ပါတယ်။ Pivot ရဲ့ တည်နေရာပေါ်မူတည်ပြီး အခြားဒေတာတွေကို ကြီးတဲ့ဘက်နဲ့ သေးတဲ့ဘက်အဖြစ် ခွဲခြားပေးပါတယ်။
ဒုတိယအနေနဲ့၊ Pivot ကို အခြေခံပြီး ဒေတာတွေကို Partition လုပ်ပါတယ်။ ဆိုလိုတာက Pivot ထက်ကြီးတဲ့ ဒေတာတွေဟာ သူ့ဘေးမှာ စုစည်းပြီး၊ Pivot ထက်သေးတဲ့ ဒေတာတွေဟာ သူ့ဘေးမှာ စုစည်းသွားအောင် လုပ်ဆောင်ခြင်း ဖြစ်ပါတယ်။
တတိယအနေနဲ့၊ Partition လုပ်ပြီးတဲ့အခါမှာ Pivot ဟာ မှန်ကန်တဲ့ နေရာမှာ ရှိနေပြီဖြစ်တဲ့အတွက် အဲဒီနေရာကို စီစဉ်ပြီး၊ Pivot မရှေ့က ဒေတာတွေနဲ့ Pivot နောက်က ဒေတာတွေကို သီးခြားစီ ခွဲပြီး နောက်ထပ် Quick Sort ကို ပြန်လည် အသုံးပြုပြီး အဲဒီအပိုင်းတွေကို စီစဉ်သွားပါတယ်။

Quick Sort ရဲ့ Average case Time Complexity ဟာ O(n log n) ဖြစ်ပါတယ်။ ဒါဟာ ဒေတာအစုအဝေးကြီးများအတွက် အလွန်လျင်မြန်ပြီး ထိရောက်တဲ့ စီစဉ်မှု နည်းလမ်းတစ်ခု ဖြစ်ပါတယ်။

## ၄။ Algorithm များ၏ နှိုင်းယှဉ်ချက်နှင့် လက်တွေ့အသုံးချမှု (Comparison and Practical Application)

Sorting နဲ့ Searching Algorithm တွေဟာ ကွန်ပျူတာ သိပ္ပံရဲ့ အခြေခံကျတဲ့ အစိတ်အပိုင်းတွေ ဖြစ်ပါတယ်။ ဘယ် Algorithm ကို ဘယ်အချိန်မှာ သုံးသင့်တယ်ဆိုတာကို သိဖို့အတွက် ၎င်းတို့ရဲ့ အားသာချက်၊ အားနည်းချက်တွေကို နှိုင်းယှဉ်လေ့လာဖို့ လိုအပ်ပါတယ်။

### Sorting Algorithm များ၏ နှိုင်းယှဉ်ချက်

**Bubble Sort, Selection Sort, Insertion Sort** တို့ဟာ ဒေတာအစုအဝေး အရွယ်အစား နည်းသောအခါ (Small Data Sets) သို့မဟုတ် ရိုးရှင်းသော အခြေအနေများအတွက် သင့်လျော်ပါတယ်။

*   **Bubble Sort:** အလွန်ရိုးရှင်းသော်လည်း အချိန်ယူရလေ့ရှိသည်။ အချိန်ရှုပ်ထွေးမှု O(n²) ရှိသောကြောင့် အကြီးမားဆုံး ဒေတာအစုအဝေးတွေအတွက် သင့်လျော်지 မဟုတ်ပါ။
*   **Selection Sort:** လဲလှယ်မှု (swaps) နည်းပါးသောကြောင့် အကောင်းဆုံးမဟုတ်သော်လည်း အချိန်အနည်းငယ် ပိုယူရနိုင်ပါသည်။
*   **Insertion Sort:** ဒေတာအစုအဝေးက အနည်းငယ်စီ ပြောင်းလဲနေတဲ့အခါ (nearly sorted) မှာ အလွန်မြန်ဆန်ပါတယ်။

**Merge Sort နှင့် Quick Sort** တို့ဟာ ဒေတာအစုအဝေးကြီးများ (Large Data Sets) အတွက် အလွန်ထိရောက်ပါတယ်။

*   **Merge Sort:** အမြဲတမ်း O(n log n) အချိန်ယူရပြီး၊ အဆိုးဆုံး အခြေအနေ (Worst-case) မှာတောင် ထိရောက်မှုရှိပါတယ်။ ဒေတာကို ခွဲပြီး ပေါင်းစပ်တဲ့ နည်းလမ်းကြောင့် တည်ငြိမ်တဲ့ စွမ်းဆောင်ရည်ရှိပါတယ်။
*   **Quick Sort:** Average case မှာ O(n log n) အချိန်ယူရပြီး၊ လက်တွေ့မှာ အလွန်မြန်ဆန်ပါတယ်။ ဒါပေမယ့် အဆိုးဆုံး အခြေအနေမှာ O(n²) ဖြစ်နိုင်ခြေရှိပါတယ်။

### Algorithm ရွေးချယ်မှု (Algorithm Selection)

Sorting နှင့် Searching Algorithm ကို ရွေးချယ်တဲ့အခါ အဓိကအားဖြင့် ဒေတာရဲ့ ပမာဏ (Data size) နဲ့ ရှာဖွေမှု သို့မဟုတ် စီစဉ်မှုရဲ့ လိုအပ်ချက်ပေါ် မူတည်ရပါတယ်။

*   **ဒေတာပမာဏအပေါ် မူတည်၍:** ဒေတာအစုအဝေးသေးငယ်သောအခါ (ဥပမာ- အနည်းငယ်သော Contact List များ) Bubble Sort သို့မဟုတ် Insertion Sort က လွယ်ကူစွာ အသုံးပြုနိုင်ပါတယ်။ ဒေတာအစုအဝေးကြီးများအတွက် Merge Sort သို့မဟုတ် Quick Sort က ပိုမိုထိရောက်မှုရှိပါတယ်။
*   **ရှာဖွေမှုအပေါ် မူတည်၍:** ဒေတာကို အမြဲတမ်း စီစဉ်ထားရမယ်ဆိုရင် Binary Search ကို အသုံးပြုသင့်ပါတယ်။ Linear Search ကိုတော့ စီစဉ်ထားခြင်းမရှိဘဲ ရှာဖွေရတဲ့အခါသာ အသုံးပြုသင့်ပါတယ်။
*   **တည်ငြိမ်မှု (Stability) အပေါ် မူတည်၍:** Merge Sort ဟာ ဒေတာကို စီစဉ်တဲ့အခါ တည်ငြိမ်မှုရှိတဲ့ (stable) စီစဉ်မှုဖြစ်ပြီး၊ Quick Sort က ပိုမိုမြန်ဆန်တဲ့ အဖြေကို ပေးနိုင်ပါတယ်။

### လက်တွေ့ဥပမာများ (Practical Examples)

Sorting နှင့် Searching ကို လက်တွေ့ C code ဖြင့် အကောင်အထည်ဖော်ခြင်းဖြင့် ၎င်းတို့ရဲ့ အကျိုးကျေးဇူးတွေကို ပိုမိုနားလည်နိုင်ပါတယ်။ ဥပမာအားဖြင့်၊ Array တစ်ခုထဲက ဂဏန်းတွေကို Bubble Sort နဲ့ စီစဉ်ပြီး၊ ထို့နောက် Binary Search ကို အသုံးပြုပြီး လိုချင်တဲ့ ဂဏန်းကို ရှာဖွေတာမျိုးကို C code ဖြင့် အကောင်အထည်ဖော်ကြည့်ခြင်းဖြင့် Algorithm တွေရဲ့ လက်တွေ့အသုံးချပုံကို လက်တွေ့မြင်တွေ့နိုင်မှာ ဖြစ်ပါတယ်။

## ၅။ နိဂုံးချုပ် (Conclusion)

အချုပ်အားဖြင့် ဆိုရလျှင် Sorting နှင့် Searching Algorithm တွေဟာ ကွန်ပျူတာ သိပ္ပံရဲ့ အခြေခံကျတဲ့ အစိတ်အပိုင်းတွေ ဖြစ်ပါတယ်။ ဤ Algorithm များ၏ Time Complexity နှင့် Space Complexity များ၊ အထူးသဖြင့် Big O notation များကို နားလည်ခြင်းက ကျွန်ုပ်တို့အား ပိုမိုထိရောက်သော နှင့် မြန်ဆန်သော Software များကို ရေးသားနိုင်ရန် အထောက်အကူပြုပါသည်။

ကျွန်ုပ်တို့ လေ့လာခဲ့သော Linear Search (O(n)) နှင့် Binary Search (O(log n)) တို့သည် ဒေတာကို ရှာဖွေရာတွင် မည်မျှထိရောက်မှုရှိနိုင်သည်ကို ပြသခဲ့ပြီး၊ Sorting Algorithm များဖြစ်သော Bubble Sort (O(n²)) မှ Merge Sort (O(n log n)) နှင့် Quick Sort (Average O(n log n)) အထိ ကွဲပြားခြားနားမှုများကို သိရှိနားလည်စေခဲ့ပါတယ်။

Algorithm ရွေးချယ်မှုမှာ ဒေတာရဲ့ ပမာဏနှင့် ရှာဖွေမှု သို့မဟုတ် စီစဉ်မှုရဲ့ လိုအပ်ချက်ပေါ် မူတည်ပြီး ဆုံးဖြတ်ရမှာ ဖြစ်ပါတယ်။ ထို့ကြောင့်၊ ကွန်ပျူတာနဲ့ အချက်အလက်တွေနဲ့ အလုပ်လုပ်တဲ့အခါ၊ အမြဲတမ်း အချိန်နှင့် နေရာအပေါ် အာရုံစိုက်ပြီး အကောင်းဆုံး Algorithm ကို ရွေးချယ်အသုံးပြုတတ်ဖို့ လိုအပ်ပါသည်။

နောက်ဆက်တွဲအနေဖြင့်၊ နောက်အခန်းများတွင် Data Structure များ (Linked List, Tree) နှင့် ပိုမိုရှုပ်ထွေးသော Algorithm များအကြောင်း ဆက်လက်လေ့လာသွားမည်ဖြစ်ပြီး၊ ၎င်းတို့သည် ထိုအခြေခံ Algorithm များကို ပိုမိုအဆင့်မြင့်သော ပြဿနာများအတွက် အသုံးချနိုင်မည့် နည်းလမ်းများ ဖြစ်လာမည် ဖြစ်ပါသည်။
