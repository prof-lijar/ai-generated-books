# Chapter 10: Dynamic Memory Allocation — Memory ကို ကိုယ်တိုင် စီမံခြင်း

## ၁။ အခန်းအစနှင့် မိတ်ဆက် (Introduction and Hook)

ပရိုဂရမ်တစ်ခုကို ရေးသားတဲ့အခါ ကျွန်ုပ်တို့သည် အချက်အလက်များ (data)၊ အရာဝတ္ထုများ (object) နှင့် လုပ်ဆောင်မှုများ (functions) ကို စီစဉ်ဖွဲ့စည်းရပါတယ်။ ဒီအရာတွေဟာ ကွန်ပျူတာရဲ့ မှတ်ဉာဏ် (Memory) ထဲမှာ ဘယ်လိုနေရာတွေမှာ ရှိနေသလဲ၊ ဘယ်လို သိမ်းဆည်းထားသလဲဆိုတဲ့ မေးခွန်းကို မေးခွန်းထုတ်ရမှာ ဖြစ်ပါတယ်။ ကျွန်ုပ်တို့ရဲ့ မှတ်ဉာဏ်ဆိုတာ ဘယ်လိုအလုပ်လုပ်သလဲ၊ ပရိုဂရမ်က ဘယ်လို အချက်အလက်တွေကို ဝင်ရောက်အသုံးပြုနိုင်မလဲဆိုတာကို နားလည်ဖို့က အလွန်အရေးကြီးပါတယ်။

ဒီအခန်းမှာ ကျွန်ုပ်တို့ဟာ Static memory နဲ့ Dynamic memory ရဲ့ ကွာခြားချက်ကို နားလည်ပြီး C Language မှာ Memory ကို ကိုယ်တိုင် စီမံခန့်ခွဲပုံ၊ ဆိုလိုတာက Dynamic Memory Allocation ကို ဘယ်လိုအသုံးပြုမလဲဆိုတာကို အသေးစိတ် လေ့လာသွားမှာ ဖြစ်ပါတယ်။ Memory Management ဟာ C Programming ရဲ့ အခြေခံကျတဲ့ အယူအဆတွေထဲမှာ အရေးအကြီးဆုံး အစိတ်အပိုင်းတစ်ခု ဖြစ်ပါတယ်။ ဘာကြောင့်လဲဆိုတော့ Memory ကို မှန်ကန်စွာ စီမံခန့်ခွဲနိုင်ခြင်းက ပရိုဂရမ်ရဲ့ စွမ်းဆောင်ရည်၊ တည်ငြိမ်မှုနဲ့ လုံခြုံမှုကို အဆုံးအထိ ဆုံးဖြတ်ပေးနိုင်လို့ပါပဲ။ ထို့အပြင် Memory Leak လိုမျိုး အမှားအယွင်းတွေကို ဘယ်လိုရှောင်ရှားမလဲ၊ မှတ်ဉာဏ်ကို မလိုအပ်ဘဲ ပိတ်ဆို့တာမျိုး မဖြစ်အောင် ဘယ်လို စီမံမလဲဆိုတဲ့ အကြောင်းတွေကိုလည်း ဒီအခန်းမှာ မိတ်ဆက်ပေးသွားမှာ ဖြစ်ပါတယ်။

## ၂။ Memory အမျိုးအစားများ၏ နားလည်မှု (Understanding Memory Types)

C Programming မှာ Memory ကို အခြေခံအားဖြင့် နှစ်မျိုးကြီးအဖြစ် ခွဲခြားနိုင်ပါတယ်- Stack Memory နဲ့ Heap Memory ဖြစ်ပါတယ်။ ဒီနှစ်ခုရဲ့ အလုပ်လုပ်ပုံနဲ့ အသုံးပြုပုံကို နားလည်ဖို့က Dynamic Memory Allocation ကို နားလည်ဖို့ အခြေခံကျပါတယ်။

### Stack Memory နှင့် Heap Memory နှိုင်းယှဉ်ချက်

Stack Memory နဲ့ Heap Memory တို့ဟာ ကွန်ပျူတာရဲ့ RAM (Random Access Memory) ထဲမှာ အချက်အလက်တွေကို သိမ်းဆည်းတဲ့ မတူညီတဲ့ နည်းလမ်းတွေ ဖြစ်ပါတယ်။

**Stack Memory (စတက် မက်မိုရီ):**
Stack Memory ဟာ အလွန်အထူးပုံစံနဲ့ စီမံခန့်ခွဲတဲ့ Memory အမျိုးအစား ဖြစ်ပါတယ်။ ဒါဟာ **LIFO (Last-In, First-Out)** ဆိုတဲ့ မူကို အခြေခံပါတယ်။ အချက်အလက်တွေဟာ စီမံခန့်ခွဲရတာ အလွန်လွယ်ကူပြီး မြန်ဆန်ပါတယ်။

Stack Memory မှာ အဓိက သိမ်းဆည်းတာတွေကတော့-
၁။ Function Calls (Function ခေါ်ဆိုမှုများ) များအတွက် လိုအပ်တဲ့ အချက်အလက်များ။
၂။ Local Variables (Function အတွင်းမှာ ကြေငြာထားတဲ့ ตัวแปรများ)။
၃။ Function Return Addresses များ။

Stack Memory မှာ အချက်အလက်တွေဟာ Function တစ်ခု စတင်တဲ့အခါ အပေါ်ကနေ အောက်ကို တန်းစီပြီး သိမ်းဆည်းပါတယ်။ Function တစ်ခု ပြီးဆုံးသွားတဲ့အခါ အဲဒီအပိုင်းက Memory ဟာ အလိုအလျောက် ပြန်လွှတ်ပေးခံရပါတယ်။ ဒါကြောင့် Stack Memory မှာ သိမ်းဆည်းတဲ့ အချက်အလက်တွေဟာ **Static** ဖြစ်ပြီး၊ ပရိုဂရမ် Run ဖြစ်နေစဉ်အတွင်းမှာ အချိန်တိုအတွင်းသာ အသုံးပြုပြီး ပျောက်ကွယ်သွားနိုင်ပါတယ်။

**Heap Memory (ဟီးပ် မက်မိုရီ):**
Heap Memory ဟာ Stack Memory နဲ့ လုံးဝကွဲပြားတဲ့ အလုပ်လုပ်ပုံရှိပါတယ်။ Heap Memory မှာ အချက်အလက်တွေဟာ **Dynamic** ဖြစ်ပါတယ်။ ဘာကြောင့် Dynamic ဖြစ်တာလဲဆိုတော့ ပရိုဂရမ် Runner (Compiler/Linker) က အဲဒီ Memory ကို လိုအပ်တဲ့အချိန်မှာပဲ ခွဲဝေပေးနိုင်လို့ ဖြစ်ပါတယ်။

Heap Memory မှာ အဓိက သိမ်းဆည်းတာကတော့-
၁။ Dynamic Data (ပြောင်းလဲနိုင်သော အချက်အလက်များ)။
၂။ Objects များ (Structs များ)။
၃။ Pointer များ (အခြား Memory လိပ်စာများကို သိမ်းဆည်းရန်)။

Heap Memory မှာ Memory ကို ခွဲဝေတာက ပိုပြီး ရှုပ်ထွေးပါတယ်။ ပရိုဂရမ်ရေးတဲ့အချိန်မှာ ဘယ်လောက် Memory လိုမလဲဆိုတာကို Run Time မှာ ဆုံးဖြတ်ရပါတယ်။ ဒါပေမယ့် ဒီနည်းလမ်းက ပရိုဂရမ်ကို ပိုမိုပြောင်းလွယ်ပြင်လွယ်ရှိစေပါတယ်။ Heap Memory မှာ Memory ကို လွှတ်ပေးဖို့အတွက် ပရိုဂရမ်က ကိုယ်တိုင် တာဝန်ယူရပါတယ်။

### အသုံးပြုရမည့် အခြေအနေများ (When to use)

Stack နဲ့ Heap ကို ဘယ်အချိန်မှာ ဘယ်လိုသုံးသင့်သလဲဆိုတာကို နေ့စဉ်ဘဝဥပမာတွေနဲ့ နှိုင်းယှဉ်ကြည့်ရအောင်။

**Stack ကို အသုံးပြုသင့်သော အခြေအနေများ:**
Stack ကို Local Variables တွေ၊ Function Call တွေအတွက် အသုံးပြုသင့်ပါတယ်။ ဥပမာ- သင်ဟာ စာမေးပွဲဖြေနေတဲ့အခါ စာအုပ်တစ်အုပ်ကို စားပွဲပေါ်မှာ ခဏတင်ထားတာနဲ့ တူပါတယ်။ သင်စာအုပ်ကို အသုံးပြုနေတဲ့အချိန်မှာပဲ လိုအပ်တဲ့ စာမျက်နှာတွေကို လွယ်လွယ်ကူကူ ဝင်ကြည့်နိုင်ပြီး အသုံးပြုပြီးတာနဲ့ စားပွဲပေါ်ကနေ ဖယ်ရှားလိုက်နိုင်ပါတယ်။ ဒါဟာ လိုအပ်တဲ့ အချက်အလက်တွေကို အချိန်တိုအတွင်းမှာ စီမံဖို့ အသင့်တော်ဆုံး ဖြစ်ပါတယ်။

**Heap ကို အသုံးပြုသင့်သော အခြေအနေများ:**
Heap ကို Dynamic Memory Allocation အတွက် အသုံးပြုသင့်ပါတယ်။ ဥပမာ- သင်ဟာ ပိုက်ဆံအိတ်ထဲမှာ အရာဝတ္ထုအမျိုးမျိုးကို ထည့်ထားတာနဲ့ တူပါတယ်။ သင် ဘယ်လောက် ပိုက်ဆံအိတ်ကို လိုချင်လဲဆိုတာကို အချိန်မရွေး ပြောင်းလဲနိုင်ပြီး လိုအပ်သလောက် ထည့်လို့ရပါတယ်။ ပရိုဂရမ်ထဲမှာလည်း အချက်အလက်တွေရဲ့ အရွယ်အစားကို Run Time မှာ ဆုံးဖြတ်ရတဲ့အခါ (ဥပမာ- User က ဘယ်လောက် Input ပေးမလဲဆိုတာကို မသိတဲ့အခါ) Heap ကို အသုံးပြုရပါတယ်။

### Memory Allocation ၏ အခြေခံသဘောတရား

Process တစ်ခုရဲ့ Memory ပုံစံ (Memory Layout) ကို အတိုချုပ် ဖော်ပြရရင်၊ ပုံမှန်အားဖြင့် Process တစ်ခုဟာ Stack နဲ့ Heap တို့ကို အသုံးပြုပါတယ်။

**Stack:** Memory ရဲ့ အပေါ်ပိုင်း (သို့မဟုတ် အနီးဆုံးနေရာ) မှာ သိမ်းဆည်းပြီး လွယ်ကူစွာ ဝင်ရောက်အသုံးပြုနိုင်ပါတယ်။ အချက်အလက်တွေဟာ Address များနဲ့ ဆက်စပ်နေပြီး၊ အလွန်တိကျတဲ့ စီမံခန့်ခွဲမှုကို ခံရပါတယ်။

**Heap:** Stack ရဲ့ အောက်ဘက်မှာ သို့မဟုတ် အခြားနေရာမှာ လိုအပ်သလောက် ကျယ်ပြန့်စွာ ခွဲဝေထားတဲ့ Memory ပိုင်းဖြစ်ပြီး၊ Dynamic Data တွေနဲ့ Object တွေကို သိမ်းဆည်းဖို့ အသုံးပြုပါတယ်။ Heap မှာရှိတဲ့ Memory တွေကို သိမ်းဆည်းဖို့အတွက် Pointer တွေက အဓိကကျပါတယ်။

Stack နဲ့ Heap တို့ဟာ Memory ရဲ့ လမ်းကြောင်းများ (Address) နဲ့ ဆက်စပ်နေပါတယ်။ Stack ရဲ့ Address တွေဟာ Function Call များနဲ့ ဆက်စပ်ပြီး စနစ်တကျရှိနေပြီး၊ Heap ရဲ့ Address တွေဟာ `malloc()` စတဲ့ Function တွေကနေ ရလာတဲ့ လိပ်စာတွေ ဖြစ်ပါတယ်။

## ၃။ Dynamic Memory Allocation Functions များ (The Core Functions)

C Language မှာ Heap Memory ကို ကိုယ်တိုင် စီမံခန့်ခွဲဖို့အတွက် စံသတ်မှတ်ထားတဲ့ Function ကြီးလေးခု ရှိပါတယ်။ ဒီ Function တွေဟာ Heap ပေါ်မှာ Memory ကို ခွဲဝေပေးတာ၊ ပြောင်းလဲပေးတာနဲ့ လွှတ်ပေးတာတွေကို လုပ်ဆောင်ပေးပါတယ်။

### `malloc()` (Memory Allocation)

`malloc()` (Memory Allocation) ဆိုတာ Heap Memory ပေါ်မှာ လိုအပ်တဲ့ အရွယ်အစားရှိတဲ့ Memory ကို တောင်းဆိုပြီး ခွဲဝေပေးတဲ့ Function ဖြစ်ပါတယ်။

**လုပ်ဆောင်ပုံ:**
`malloc()` ဟာ Runtime မှာ သတ်မှတ်ပေးလိုက်တဲ့ အရွယ်အစား (size) အတိုင်း byte တွေကို Heap ပေါ်ကနေ စုစည်းပြီး လိုအပ်တဲ့ Memory ကို ခွဲဝေပေးပါတယ်။ ဒီ Function ဟာ Heap ပေါ်မှာ အချက်အလက်တွေကို သိမ်းဆည်းပြီး အဲဒီ Memory ရဲ့ အစမှတ် (starting address) ကို `pointer` (ညွှန်ပြချက်) အနေနဲ့ ပြန်ပေးပါတယ်။

**Syntax နှင့် Parameter များ:**
`malloc()` ရဲ့ Syntax ဟာ အောက်ပါအတိုင်း ဖြစ်ပါတယ်-
`void *malloc(size_t *size, void *ptr);`

*   `size_t *size`: လိုချင်တဲ့ Memory ရဲ့ အရွယ်အစား (byte အရေအတွက်) ကို သိမ်းဆည်းမယ့် Pointer ဖြစ်ပါတယ်။
*   `void *ptr`: ခွဲဝေပေးမယ့် Memory ရဲ့ စတင်လိပ်စာကို သိမ်းဆည်းမယ့် Pointer ဖြစ်ပါတယ်။

ဒီ Function ကို အသုံးပြုတဲ့အခါမှာ ကျွန်ုပ်တို့ဟာ လိုချင်တဲ့ Memory ရဲ့ အရွယ်အစားကို `sizeof` operator ကို အသုံးပြုပြီး တွက်ချက်ရပါတယ်။

**ဥပမာဖြင့် လက်တွေ့ပြသခြင်း:**
ဥပမာအားဖြင့်၊ `int` တစ်ခုအတွက် Memory လိုချင်ရင်၊ ကျွန်ုပ်တို့ဟာ `sizeof(int)` ကို အသုံးပြုပြီး လိုအပ်တဲ့ byte အရေအတွက်ကို သိရမှာ ဖြစ်ပါတယ်။

```c
#include <stdio.h>
#include <stdlib.h> // malloc, free အတွက် လိုအပ်သည်

int main() {
    int *ptr; // Pointer ကို ကြေငြာသည်

    // 10 integer အတွက် Memory ခွဲဝေရန် တောင်းဆိုခြင်း
    // sizeof(int) * 10 ဆိုသည်မှာ 10 * sizeof(int) byte လိုချင်သည်ဟု ဆိုလိုသည်။
    ptr = (int *)malloc(10 * sizeof(int)); 

    // Memory ခွဲဝေမှု မှန်ကန်မှု စစ်ဆေးခြင်း
    if (ptr == NULL) {
        printf("Memory allocation failed!\n");
        return 1; // မှားယွင်းပါက ထွက်ခွာခြင်း
    }

    // Memory ထဲသို့ တန်ဖိုးထည့်သွင်းခြင်း
    for (int i = 0; i < 10; i++) {
        ptr[i] = i * 2;
        printf("Element %d: %d\n", i, ptr[i]);
    }

    // Memory ကို လွှတ်ပေးခြင်း
    free(ptr);
    ptr = NULL; // Pointer ကို NULL သို့ ပြောင်းခြင်း (Dangling Pointer မဖြစ်အောင်)

    return 0;
}
```
ဒီဥပမာမှာ `malloc` ဟာ Heap ပေါ်မှာ 10 * sizeof(int) byte ကို ခွဲဝေပေးပြီး အဲဒီ Memory ရဲ့ အစလိပ်စာကို `ptr` မှာ သိမ်းဆည်းပေးလိုက်ပါတယ်။ `ptr` ဟာ အဲဒီ Memory ကို ရည်ညွှန်းနေပါတယ်။

### `calloc()` (Contiguous Allocation)

`calloc()` (Contiguous Allocation) ဆိုတာ `malloc()` နဲ့ ဆင်တူပေမယ့် အဓိက ကွာခြားချက်ကတော့ အချက်အလက်တွေကို စတင်ခွဲဝေပေးတဲ့အခါ **အလိုအလျောက် 0 နဲ့ Initialize လုပ်ပေးခြင်း** ဖြစ်ပါတယ်။

**လုပ်ဆောင်ပုံ:**
`calloc()` ဟာ လိုချင်တဲ့ အရေအတွက် (number of elements) နဲ့ တစ်ခုချင်းစီရဲ့ အရွယ်အစား (size of each element) ကို လက်ခံပါတယ်။ ၎င်းသည် ထို Memory ကို ခွဲဝေပေးပြီး၊ ထို Memory အားလုံးကို ပထမဆုံး 0 (zero) နဲ့ Initialize လုပ်ပေးပါတယ်။

**`malloc()` နှင့် `calloc()` တို့၏ ကွာခြားချက်:**
*   `malloc()`: Memory ကို ခွဲဝေပေးရုံသာ လုပ်ဆောင်ပြီး၊ အဲဒီ Memory တွေရဲ့ အစမှတ်လိပ်စာကို ပြန်ပေးရုံသာ လုပ်ဆောင်သည်။ Memory အထဲမှာ ရှိပြီးသား တန်ဖိုးတွေကို ဘယ်လိုမှ မသတ်မှတ်ပေးပါ။
*   `calloc()`: Memory ကို ခွဲဝေပေးပြီး၊ အဲဒီ Memory အားလုံးကို 0 နဲ့ Initialize လုပ်ပေးပါတယ်။ ဒါကြောင့် ပထမဆုံး အချက်အလက်တွေဟာ 0 ဖြစ်နေမှာပါ။

**ဥပမာဖြင့် ရှင်းပြခြင်း:**
`malloc` နဲ့ `calloc` ကို နှိုင်းယှဉ်ကြည့်ရအောင်။

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr_malloc;
    int *ptr_calloc;

    // malloc ကို အသုံးပြုခြင်း
    ptr_malloc = (int *)malloc(5 * sizeof(int));

    // calloc ကို အသုံးပြုခြင်း (5 integer အတွက် Memory ခွဲဝေပြီး 0 နဲ့ Initialize)
    ptr_calloc = (int *)calloc(5, sizeof(int));

    if (ptr_malloc != NULL && ptr_calloc != NULL) {
        printf("malloc allocated memory successfully.\n");
        printf("calloc allocated memory successfully.\n");
    }

    // Memory ကို လွှတ်ပေးခြင်း
    free(ptr_malloc);
    free(ptr_calloc);

    return 0;
}
```
`calloc` ကို အသုံးပြုလိုက်တဲ့အခါ၊ `ptr_calloc` မှာ သိမ်းဆည်းထားတဲ့ 5 ခုစလုံးဟာ 0 ဖြစ်နေမှာ ဖြစ်ပါတယ်။ ဒါဟာ အထူးသဖြင့် Array တွေလိုမျိုး အချက်အလက်တွေ စတင်ချိန်မှာ မှားယွင်းတဲ့ တန်ဖိုးတွေ (garbage values) ထည့်မိတာမျိုးကို ရှောင်ရှားဖို့ အလွန်အသုံးဝင်ပါတယ်။

### `realloc()` (Reallocation)

`realloc()` (Reallocation) ဆိုတာ Heap ပေါ်မှာ **အရင်က ခွဲဝေပြီးသား Memory ကို အခြားအရွယ်အစားအသစ်တစ်ခုသို့ ပြောင်းလဲပေးတဲ့** Function ဖြစ်ပါတယ်။

**လုပ်ဆောင်ပုံ:**
`realloc()` ဟာ အရင်က `malloc()` သို့မဟုတ် `calloc()` နဲ့ ခွဲဝေထားတဲ့ Memory Block ကို ပြန်လည်စီစဉ်ပေးပါတယ်။ ကျွန်ုပ်တို့သည် အရင်က ခွဲဝေထားတဲ့ Memory ရဲ့ လိပ်စာကို ထိန်းသိမ်းထားပြီး၊ လိုအပ်တဲ့ အရွယ်အစားကို ပြောင်းလဲပေးပါတယ်။

**Memory ပြောင်းလဲမှုများတွင် သတိပြုရမည့်အချက်များ:**
`realloc()` ကို အသုံးပြုတဲ့အခါ အဓိက သတိထားရမယ့်အချက်ကတော့ အသစ်ခွဲဝေပေးလိုက်တဲ့ Memory ဟာ အရင်က Memory နဲ့ **တူညီတဲ့ လိပ်စာ (address)** ဖြစ်နေမှာ မဟုတ်ဘဲ **အသစ်** ဖြစ်နိုင်ပါတယ်။

၁။ **Pointer ကို သိမ်းဆည်းခြင်း:** `realloc()` ရဲ့ Return Value ကို အမြဲတမ်း သတိထားရပါမယ်။ အဲဒါက အသစ်ခွဲဝေပေးလိုက်တဲ့ Memory ရဲ့ စတင်လိပ်စာကို ပြန်ပေးပါတယ်။ ဒီလိပ်စာကို Pointer မှာ သတ်မှတ်ထားရပါမယ်။
၂။ **Size ပြောင်းလဲမှု:** အကယ်၍ အသစ်ခွဲဝေပေးမယ့် အရွယ်အစားက အရင်ကထက် ပိုကြီးလာရင်၊ အဲဒီ Memory ရဲ့ နောက်ဆက်တွဲအပိုင်းတွေမှာ လုံလောက်တဲ့ နေရာ ရှိမရှိ စစ်ဆေးရပါမယ်။ လုံလောက်မှုမရှိရင် `realloc()` ဟာ `NULL` ကို ပြန်ပေးပါလိမ့်မယ်။

**ဥပမာ:**
```c
// ... (malloc ဖြင့် အစပိုင်းတွင် ခွဲဝေပြီးသား memory ရှိနေသည်ဟု ယူဆပါ)

// Array ကို အရွယ်အစား 10 မှ 20 သို့ ပြောင်းလဲခြင်း
int *ptr_realloc = (int *)realloc(ptr_malloc, 20 * sizeof(int));

if (ptr_realloc == NULL) {
    printf("Reallocation failed!\n");
    // Memory ပြန်လွှတ်ရန် လိုအပ်သည်
} else {
    printf("Memory successfully reallocated to new size.\n");
    // အခု ptr_realloc ကို အသုံးပြုနိုင်သည်
}
```

### `free()` (Memory Deallocation)

`free()` (Memory Deallocation) ဟာ Dynamic Memory Allocation ရဲ့ အရေးအကြီးဆုံး အစိတ်အပိုင်း ဖြစ်ပါတယ်။

**လုပ်ဆောင်ပုံ:**
`free()` ရဲ့ အဓိကလုပ်ဆောင်ချက်က Heap ပေါ်မှာ ခွဲဝေထားပြီးသား Memory ကို ပရိုဂရမ်ကနေ **ပြန်လွှတ်ပေးခြင်း** ဖြစ်ပါတယ်။ Memory ကို လွှတ်ပေးလိုက်တဲ့အခါ အဲဒီ Memory ဟာ နောက်ထပ် Memory Allocation များအတွက် အသုံးပြုနိုင်တော့ပါတယ်။

**`free()` မရှိမဖြစ် လိုအပ်ရခြင်း အကြောင်းရင်း:**
`free()` မရှိဘဲ Memory ကို လွှတ်ပေးဖို့ ကြိုးစားရင် ပရိုဂရမ်ဟာ အဲဒီ Memory ကို ထိန်းချုပ်နိုင်စွမ်း မရှိတော့ဘဲ၊ အဲဒီ Memory ကို ပိုင်ဆိုင်နေတယ်လို့ မှတ်ယူနေပါတယ်။ ဒါကြောင့် ပရိုဂရမ် အဆုံးသတ်သွားတဲ့အခါ အဲဒီ Memory တွေဟာ System ထဲမှာ ပိတ်ဆို့နေပြီး အခြား ပရိုဂရမ်တွေအတွက် အသုံးပြုလို့ မရတော့ဘဲ **Memory Leak** အဖြစ် ပြောင်းလဲသွားနိုင်ပါတယ်။

**`free()` ကို မခေါ်မိခြင်း၏ အကျိုးဆက်များ (Memory Leak):**
Memory Leak ဆိုသည်မှာ ပရိုဂရမ်က Heap ပေါ်မှာ Memory ကို ခွဲဝေယူလိုက်ပေမယ့်၊ နောက်ထပ် အသုံးပြုရန် မလိုအပ်တော့တဲ့အချိန်မှာ `free()` ကို မခေါ်ဘဲ ပရိုဂရမ် ရပ်သွားတာမျိုး ဖြစ်ပါတယ်။ အဲဒီ Memory တွေဟာ ပရိုဂရမ် အဆုံးသတ်ပြီးနောက်မှာလည်း System ထဲမှာ ကျန်နေပြီး အခြား Process တွေအတွက် အသုံးပြုလို့ မရတော့ဘဲ ပျောက်ဆုံးသွားပါတယ်။ ဒါကို Memory Leak လို့ ခေါ်ပါတယ်။ ဒါဟာ ပရိုဂရမ်ရဲ့ တည်ငြိမ်မှုကို ထိခိုက်စေပြီး၊ အချိန်ကြာလာတာနဲ့အမျှ System ရဲ့ Memory ကို အလွန်အကျွံ အသုံးပြုစေနိုင်ပါတယ်။

**ဥပမာ:**
အပေါ်က `malloc` နဲ့ `free` ကို ပြန်ကြည့်ရအောင်။

```c
// ... (malloc ဖြင့် Memory ခွဲဝေပြီးသား)

// Memory ကို လွှတ်ပေးခြင်း
free(ptr);
ptr = NULL; // Pointer ကို NULL သို့ ပြောင်းခြင်း
```
`free(ptr)` ကို ခေါ်လိုက်တဲ့အခါ၊ `ptr` ကို ရည်ညွှန်းနေတဲ့ Heap Memory Block ဟာ System ရဲ့ Memory Pool ထဲကို ပြန်လွှတ်ပေးလိုက်ပါတယ်။ ဒါကြောင့် ပရိုဂရမ်က အဲဒီ Memory ကို နောက်ထပ် မသုံးတော့ဘဲ အခြားနေရာတွေမှာ အသုံးပြုနိုင်ပါတယ်။

## ၄။ Memory Leak နှင့် အမှားများ (Memory Leaks and Errors)

Dynamic Memory Allocation ကို ကိုယ်တိုင် စီမံခန့်ခွဲတဲ့အခါ Memory Leak နဲ့ Pointer နဲ့ ပတ်သက်တဲ့ အမှားတွေဟာ အလွန်အန္တရာယ်များပါတယ်။

### Memory Leak ဆိုသည်မှာ အဘယ်နည်း

Memory Leak ဆိုသည်မှာ ပရိုဂရမ်က Heap ပေါ်မှာ Memory ကို ခွဲဝေယူလိုက်ပေမယ့်၊ နောက်ထပ် အသုံးပြုရန် မလိုအပ်တော့တဲ့အချိန်မှာ `free()` ကို မခေါ်ဘဲ ထွက်ခွာသွားခြင်းကြောင့် အဲဒီ Memory တွေဟာ System ထဲမှာ ပိတ်ဆို့နေပြီး အခြား Process တွေအတွက် အသုံးပြုလို့ မရတော့ဘဲ ကျန်နေတာကို ဆိုလိုပါတယ်။

Memory Leak များသည် ဘယ်လိုဖြစ်ပေါ်လာပုံနှင့် ဘယ်လိုသက်ရောက်မှုရှိပုံ-
Memory Leak များဟာ ပရိုဂရမ်တွေမှာ အလွန်အဖြစ်များပါတယ်။ ပရိုဂရမ်တစ်ခုဟာ အချိန်ကြာကြာ Run နေတဲ့အခါ၊ အဲဒီပရိုဂရမ်က Heap မှာ Memory ကို ခွဲဝေယူပြီး၊ ထို Memory ကို လွှတ်ပေးဖို့ မေ့သွားတဲ့အခါ၊ အဲဒီ Memory တွေဟာ ပိတ်ဆို့နေပြီး ပရိုဂရမ်ရဲ့ အသုံးပြုနိုင်စွမ်းကို လျော့ကျစေပါတယ်။ အကြီးမားဆုံး အကျိုးဆက်ကတော့ ပရိုဂရမ်ရဲ့ Run Time မှာ Memory ကို အလွန်အကျွံ အသုံးပြုစေတာ (Resource Exhaustion) ဖြစ်ပါတယ်။

### Dangling Pointer နှင့် Invalid Pointer

`free()` ကို မှားယွင်းစွာ အသုံးပြုတဲ့အခါ Pointer တွေမှာ ပြဿနာတွေ ဖြစ်ပေါ်လာနိုင်ပါတယ်။

**Dangling Pointer (လွှတ်တင်လိုက်သော လိပ်စာ):**
`free()` လုပ်ပြီးသော Memory ကို ဆက်လက်အသုံးပြုခြင်းဟာ အလွန်အန္တရာယ်များပါတယ်။ `free()` လုပ်လိုက်တဲ့ Memory Block ကို ပရိုဂရမ်ကနေ လွှတ်ပေးလိုက်တဲ့အခါ၊ အဲဒီ Memory ရဲ့ လိပ်စာ (address) ဟာ ပျက်ပြယ်သွားပါတယ်။ ဒီလိပ်စာကို ဆက်လက်အသုံးပြုဖို့ ကြိုးစားတဲ့အခါ၊ အဲဒီလိပ်စာဟာ တကယ်တမ်း အခြား Memory ကို ရည်ညွှန်းနေနိုင်တာကြောင့် **Dangling Pointer (လွှတ်တင်လိုက်သော လိပ်စာ)** ဖြစ်သွားပါတယ်။ ဒီလိပ်စာကို အသုံးပြုတာဟာ မည်သည့် အချက်အလက်ကိုမှ မသိဘဲ မရေရာတဲ့ အမှားတွေ (Undefined Behavior) ကို ဖြစ်ပေါ်စေနိုင်ပါတယ်။

**Invalid Pointer (မှားယွင်းသော လိပ်စာ):**
Dangling Pointer ထက် ပိုဆိုးတဲ့ အခြေအနေကတော့ `Invalid Pointer` ဖြစ်တာပါ။ `Invalid Pointer` ဆိုတာဟာ တကယ်တမ်း ရှိမနေတဲ့ Memory လိပ်စာကို ရည်ညွှန်းတာမျိုး ဖြစ်ပါတယ်။ ဒါဟာ ပရိုဂရမ်ရဲ့ Memory Layout ကို လွဲမှားစေပြီး၊ နောက်ဆုံးမှာ **Segmentation Fault** လိုမျိုး ပြင်းထန်တဲ့ Error တွေကို ဖြစ်ပေါ်စေနိုင်ပါတယ်။

### Memory Error ရှာဖွေခြင်း (Debugging)

Memory Error များဟာ Debugging လုပ်ရတာ အလွန်ခက်ခဲပါတယ်။ ဘာကြောင့်လဲဆိုတော့ ဒီအမှားတွေဟာ ပရိုဂရမ်ရဲ့ Run Time မှာ အချိန်အကြာကြီး မပေါ်လာဘဲ၊ အချိန်အကြာကြီးကြာပြီးမှ ပေါ်လာတတ်တာမျိုး ရှိပါတယ်။

Memory Error များ၏ အန္တရာယ်မှာ ပရိုဂရမ် Crash ဖြစ်ခြင်း၊ မမှန်ကန်တဲ့ အချက်အလက်တွေ ထွက်ပေါ်လာခြင်း၊ ဒါမှမဟုတ် System ကို ပျက်စီးစေခြင်း စတာတွေ ဖြစ်နိုင်ပါတယ်။

**`Valgrind` ကို မိတ်ဆက်ခြင်း:**
ဒီလို Memory Error တွေကို ရှာဖွေဖို့အတွက် အလွန်အသုံးဝင်တဲ့ Tool တစ်ခုကတော့ `Valgrind` ဖြစ်ပါတယ်။ `Valgrind` ဟာ ပရိုဂရမ် Run ဖြစ်နေစဉ်အတွင်းမှာ Memory Access တွေကို စောင့်ကြည့်ပြီး၊ Memory Leak တွေ၊ Invalid Pointer တွေ၊ Buffer Overflows တွေနဲ့ အခြား Memory Error တွေကို အသေးစိတ် ဖော်ထုတ်ပေးနိုင်တဲ့ Debugging Tool တစ်ခု ဖြစ်ပါတယ်။ C Programming မှာ Memory ကို စီမံခန့်ခွဲတဲ့အခါ `Valgrind` ကို အသုံးပြုတာဟာ Memory Error တွေကို ရှာဖွေရာမှာ အလွန်အထောက်အကူပြုပါတယ်။

## ၅။ လက်တွေ့အသုံးချမှုနှင့် Dynamic Array များ (Practical Application: Dynamic Arrays)

Static `array` တွေဟာ Compile-time မှာ အရွယ်အစားကို သိပြီး အသုံးပြုရလွယ်ကူပေမယ့်၊ ပရိုဂရမ် ရေးတဲ့အချိန်မှာ အချက်အလက်တွေရဲ့ အရေအတွက် မသိတဲ့အခါ Dynamic `array` တွေကို အသုံးပြုဖို့ လိုအပ်လာပါတယ်။

### Dynamic Array ၏ လိုအပ်ချက်

Static `array` တွေဟာ `int arr[10];` လိုမျိုး ကြေငြာလိုက်တဲ့အခါ Memory ဟာ Compile-time မှာ သတ်မှတ်ပြီး အဲဒီအရွယ်အစားအတိုင်းပဲ ရှိနေပါတယ်။ ဒါပေမယ့် ကျွန်ုပ်တို့ရဲ့ ပရိုဂရမ်က User Input ကို အခြေခံပြီး Array ရဲ့ အရွယ်အစားကို Run Time မှာ ဆုံးဖြတ်ရတဲ့အခါ Static `array` တွေက အသုံးမဝင်တော့ပါဘူး။ ဒါကြောင့် Dynamic `array` တွေဟာ လိုအပ်တဲ့အချိန်မှာ Memory ကို တောင်းဆိုပြီး ခွဲဝေနိုင်တဲ့အတွက် အလွန်အရေးကြီးလာပါတယ်။

Dynamic `array` နှင့် Static `array` တို့၏ ခြားနားချက်-
*   **Static Array:** Memory ဟာ Compile-time မှာ သတ်မှတ်ပြီး၊ အရွယ်အစား ပြောင်းလို့မရပါ။ Memory Allocation ဟာ Static ဖြစ်ပါတယ်။
*   **Dynamic Array:** Memory ကို Run Time မှာ `malloc` သို့မဟုတ် `realloc` ကို အသုံးပြုပြီး ခွဲဝေယူပါတယ်။ အရွယ်အစားကို လိုအပ်သလောက် ပြောင်းလဲနိုင်ပါတယ်။

### Dynamic Array ဖန်တီးခြင်း (Implementation)

Dynamic `array` တစ်ခုကို ဖန်တီးဖို့အတွက် ကျွန်ုပ်တို့ဟာ `malloc()` ကို အသုံးပြုရပါမယ်။

**လုပ်ဆောင်ပုံ:**
User Input ကို အခြေခံပြီး Dynamic `array` တစ်ခုကို ဖန်တီးဖို့အတွက်၊ ပထမဆုံး `malloc()` ကို အသုံးပြုပြီး လိုအပ်တဲ့ Memory ပမာဏကို တောင်းဆိုရပါတယ်။

**လက်တွေ့ Code Example:**
အောက်ပါ Code ဥပမာက User Input ကို အခြေခံပြီး Dynamic Array တစ်ခုကို ဖန်တီးပုံကို ပြသထားပါတယ်။

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int n;
    printf("Enter the number of elements you want to store: ");
    scanf("%d", &n);

    if (n <= 0) {
        printf("Invalid size. Size must be positive.\n");
        return 1;
    }

    // n အရေအတွက်အတွက် Memory ခွဲဝေခြင်း
    int *arr = (int *)malloc(n * sizeof(int));

    if (arr == NULL) {
        printf("Memory allocation failed!\n");
        return 1;
    }

    // Array ထဲသို့ တန်ဖိုးထည့်သွင်းခြင်း
    printf("Enter %d integers:\n", n);
    for (int i = 0; i < n; i++) {
        printf("Element %d: ", i + 1);
        scanf("%d", &arr[i]); // arr[i] တွင် တန်ဖိုးထည့်သွင်းခြင်း
    }

    // Array ထဲမှ တန်ဖိုးများ ထုတ်ပြခြင်း
    printf("\nYou entered the following elements:\n");
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");

    // Memory ကို လွှတ်ပေးခြင်း
    free(arr);
    arr = NULL; // Pointer ကို NULL သို့ ပြောင်းခြင်း

    return 0;
}
```
ဒီ Code မှာ၊ User က ထည့်ပေးတဲ့ အရေအတွက် (`n`) ကို အခြေခံပြီး `n * sizeof(int)` byte ကို `malloc` နဲ့ တောင်းဆိုပါတယ်။ `malloc` ဟာ အဲဒီ Memory ကို ခွဲဝေပေးပြီး `arr` pointer မှာ လိပ်စာကို သိမ်းဆည်းပေးလိုက်ပါတယ်။ ထို့နောက် User ထည့်ပေးတဲ့ တန်ဖိုးတွေကို Array ထဲမှာ သိမ်းဆည်းပြီး ပြန်လည်ထုတ်ပေးပါတယ်။ အရေးကြီးဆုံးကတော့ ပရိုဂရမ်အဆုံးမှာ `free(arr)` ကို ခေါ်ပြီး Memory ကို ပြန်လွှတ်ပေးဖို့ မမေ့ရပါဘူး။

### `realloc()` ကို အသုံးပြု၍ Array Resize လုပ်ခြင်း

Dynamic Array တစ်ခုရဲ့ အရွယ်အစားကို လိုအပ်သလို ပြောင်းလဲဖို့အတွက် `realloc()` Function ကို အသုံးပြုပါတယ်။

**လုပ်ဆောင်ပုံ:**
`realloc()` ဟာ အရင်က `malloc()` သို့မဟုတ် `calloc()` နဲ့ ခွဲဝေထားတဲ့ Memory Block ရဲ့ အရွယ်အစားကို ပြောင်းလဲပေးပါတယ်။

**Size ပြောင်းလဲစဉ် Memory ကို ဘယ်လို စီမံရမလဲ:**
`realloc()` ကို အသုံးပြုတဲ့အခါ၊ အရင်က ပြောခဲ့သလို၊ အသစ်ခွဲဝေပေးမယ့် Memory ဟာ လုံလောက်မှု ရှိမရှိ စစ်ဆေးရပါတယ်။ အကယ်၍ လိုအပ်တဲ့ အရွယ်အစားထက် ပိုမိုကြီးတဲ့ Memory ကို တောင်းဆိုရင်၊ အဲဒီ Memory ဟာ အရင်က Memory ရဲ့ အဆုံးမှာ ရှိနေမယ့် နေရာမှာ ရှိနေနိုင်ပါတယ်။ ဒါပေမယ့် အကယ်၍ လိုအပ်တဲ့ အရွယ်အစားက ပိုကြီးလာရင်၊ `realloc()` ဟာ အဲဒီ Memory Block ကို လွှတ်ပေးပြီး အခြားနေရာမှာ ပိုကြီးတဲ့ Block အသစ်ကို ခွဲဝေပေးနိုင်ပါတယ်။

**လက်တွေ့ Code Example:**
အပေါ်က Dynamic Array ဥပမာကို `realloc()` ကို အသုံးပြုပြီး အရွယ်အစားကို ပြောင်းလဲပုံကို ပြသပါမယ်။

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int n_initial;
    printf("Enter the initial number of elements: ");
    scanf("%d", &n_initial);

    if (n_initial <= 0) {
        printf("Invalid size.\n");
        return 1;
    }

    // ပထမအကြိမ် Memory ခွဲဝေခြင်း
    int *arr = (int *)malloc(n_initial * sizeof(int));

    if (arr == NULL) {
        printf("Initial memory allocation failed!\n");
        return 1;
    }

    // Initial Data ထည့်သွင်းခြင်း
    for (int i = 0; i < n_initial; i++) {
        printf("Enter element %d: ", i + 1);
        scanf("%d", &arr[i]);
    }

    // Array အရွယ်အစားကို ပြောင်းလဲခြင်း (ဥပမာ - 2 เท่า ပြောင်းခြင်း)
    int n_new = n_initial * 2;
    int *arr_new = (int *)realloc(arr, n_new * sizeof(int));

    if (arr_new == NULL) {
        printf("Reallocation failed!\n");
        // အရင် Memory ကို free လုပ်ရန် လိုအပ်သည်
        free(arr);
        return 1;
    }

    // အသစ်ခွဲဝေထားသော Memory ကို အသုံးပြုခြင်း
    arr = arr_new; // Pointer ကို အသစ်ခွဲဝေထားသော လိပ်စာသို့ ပြောင်းလဲခြင်း
    printf("Successfully reallocated memory to size %d.\n", n_new);

    // အသစ်ထည့်သွင်းရန် လိုအပ်သော အချက်အလက်များ ထပ်ထည့်ခြင်း
    printf("Enter %d new elements:\n", n_new - n_initial);
    for (int i = n_initial; i < n_new; i++) {
        printf("Element %d: ", i + 1);
        scanf("%d", &arr[i]);
    }

    // Array ထဲမှ တန်ဖိုးများ ထုတ်ပြခြင်း
    printf("\nYou entered the updated elements:\n");
    for (int i = 0; i < n_new; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");

    // Memory ကို လွှတ်ပေးခြင်း
    free(arr);

    return 0;
}
```
ဒီ Code မှာ၊ ပထမအကြိမ် `malloc` နဲ့ Memory ကို ခွဲဝေပြီး၊ ထို့နောက် `realloc` ကို အသုံးပြုပြီး Array ရဲ့ အရွယ်အစားကို နှစ်ဆ တိုးလိုက်ပါတယ်။ `realloc` ဟာ Memory ကို ပြန်လည်စီစဉ်ပေးပြီး၊ `arr` pointer ကို အသစ်ခွဲဝေထားတဲ့ Memory ကို ရည်ညွှန်းစေပါတယ်။ နောက်ဆုံးမှာ `free(arr)` ကို ခေါ်ပြီး Memory ကို လွှတ်ပေးလိုက်ပါတယ်။

## ၆။ နိဂုံးချုပ်နှင့် အခန်းအနှစ်ချုပ် (Conclusion and Summary)

Dynamic Memory Allocation ဟာ C Programming မှာ Memory ကို ကိုယ်တိုင် ထိန်းချုပ်ပြီး စီမံခန့်ခွဲနိုင်ဖို့အတွက် အလွန်အရေးကြီးတဲ့ နည်းလမ်းတစ်ခု ဖြစ်ပါတယ်။ Stack နဲ့ Heap တို့ရဲ့ ကွာခြားချက်ကို နားလည်ခြင်းက Memory Management ရဲ့ အခြေခံအုတ်မြစ်ကို ပေးပါတယ်။ Stack ဟာ စီမံခန့်ခွဲရလွယ်ကူပြီး လုံခြုံတဲ့ Memory ဖြစ်ပြီး၊ Heap ဟာ ပိုမိုပြောင်းလွယ်ပြင်လွယ်ရှိတဲ့ Dynamic Memory Allocation အတွက် အသုံးပြုရပါတယ်။

`malloc()`, `calloc()`, `realloc()`, နှင့် `free()` ဟူသော Function ကြီးလေးခုဟာ Heap Memory ကို စီမံခန့်ခွဲဖို့အတွက် အဓိကသော့ချက်တွေ ဖြစ်ပါတယ်။ `malloc()` ဟာ Memory ကို တောင်းဆိုပြီး ခွဲဝေပေးတာ၊ `calloc()` ဟာ 0 နဲ့ Initialize လုပ်ပြီး ခွဲဝေပေးတာ၊ `realloc()` ဟာ ရှိပြီးသား Memory ကို ပြောင်းလဲပေးတာ၊ နှင့် `free()` ဟာ မလိုအပ်တော့တဲ့ Memory ကို ပြန်လွှတ်ပေးတာတွေကို လုပ်ဆောင်ပေးပါတယ်။ ဒီ Function တွေကို မှန်ကန်စွာ အသုံးပြုခြင်းက ပရိုဂရမ်ရဲ့ Memory ကို လုံခြုံစေပြီး၊ Memory Leak လိုမျိုး အမှားအယွင်းတွေကို ရှောင်ရှားနိုင်စေပါတယ်။

Dynamic Array တွေဟာ Compile-time မှာ အရွယ်အစား မသိတဲ့အခါမှာ အလွန်အသုံးဝင်ပြီး၊ `malloc` နဲ့ `realloc` ကို အသုံးပြုခြင်းဖြင့် ပရိုဂရမ်က Run Time မှာ လိုအပ်တဲ့ အရွယ်အစားအတိုင်း Memory ကို တောင်းဆိုနိုင်ပါတယ်။ အထူးသဖြင့် `realloc()` ကို မှန်ကန်စွာ အသုံးပြုခြင်းဖြင့် Array တွေရဲ့ အရွယ်အစားကို အလွယ်တကူ ပြောင်းလဲနိုင်တာကြောင့် ပရိုဂရမ်တွေဟာ ပိုမိုပြောင်းလွယ်ပြင်လွယ်ရှိလာပါတယ်။

အချုပ်အားဖြင့်ဆိုရရင်၊ Dynamic Memory Allocation ဟာ C Programming မှာ ပရိုဂရမ်တွေရဲ့ ပြောင်းလွယ်ပြင်လွယ်ရှိမှုကို ပေးစွမ်းပါတယ်။ Memory ကို ကိုယ်တိုင် စီမံခန့်ခွဲတတ်ဖို့ဆိုတာ C Programming ရဲ့ အခြေခံကျတဲ့ ကျွမ်းကျင်မှုတစ်ခု ဖြစ်ပါတယ်။ ဒီအခန်းမှာ သင်ယူခဲ့တဲ့ အယူအဆတွေကို နားလည်ပြီး၊ နောက်အခန်းတွေမှာ Data Structure များ (ဥပမာ- Linked List) မှာ Memory ကို ပိုမိုထိရောက်စွာ စီမံခန့်ခွဲပုံကို ဆက်လက်လေ့လာသွားမှာ ဖြစ်ပါတယ်။ Memory ကို မှန်ကန်စွာ စီမံခန့်ခွဲတတ်ဖို့ဆိုတာ C Programmer တစ်ယောက်အတွက် မရှိမဖြစ် လိုအပ်တဲ့ ကျွမ်းကျင်မှုတစ်ခု ဖြစ်ကြောင်း မှတ်သားထားသင့်ပါတယ်။
