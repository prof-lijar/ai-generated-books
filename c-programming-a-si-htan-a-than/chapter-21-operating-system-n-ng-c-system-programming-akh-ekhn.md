# Chapter 21: Operating System နှင့် C — System Programming အခြေခံ

## ၁။ နိဒါန်းနှင့် စတင်မိတ်ဆက်ခြင်း (Introduction and Hook)

ကွန်ပျူတာစနစ်တစ်ခု၏ အခြေခံအုတ်မြစ်ကို နားလည်ရန်အတွက် Operating System (OS) နှင့် C Programming ကို ပေါင်းစပ်လေ့လာခြင်းသည် အလွန်အရေးကြီးပါသည်။ OS ဆိုသည်မှာ Hardware (အထူးပစ္စည်းများ) နှင့် Software (ပရိုဂရမ်များ) တို့ကြားတွင် အလယ်အလတ် အခန်းကဏ္ဍမှ ဆက်သွယ်ပေးသည့် အဓိကအဖွဲ့အစည်းတစ်ခု ဖြစ်သည်။ ဥပမာအားဖြင့်၊ ကားမောင်းသူ (User) သည် ကားကို မောင်းနှင်သော်လည်း အင်ဂျင်နီယာ (Hardware) ၏ ရှုပ်ထွေးသော စနစ်များကို တိုက်ရိုက်ထိန်းချုပ်ရန် မလိုအပ်ဘဲ၊ ကားမောင်းသူနှင့် အင်ဂျင်နီယာတို့ကြားတွင် ဆက်သွယ်ရေးလမ်းကြောင်းတစ်ခု (Interface) ရှိနေခြင်းကဲ့သို့ပင် OS သည် Hardware နှင့် Application များအကြား ဆက်သွယ်ရေးလမ်းကြောင်းကို တည်ဆောက်ပေးပါသည်။ ဤဆက်သွယ်ရေးလမ်းကြောင်းကို ထိန်းချုပ်ပေးသည့် အဓိက အစိတ်အပိုင်းများမှာ Kernel နှင့် Shell များ ဖြစ်ကြသည်။

ဤနေရာတွင် C Programming ကို Operating System Development တွင် အသုံးပြုရခြင်း၏ အကြောင်းရင်းကို ဆွေးနွေးရမည် ဖြစ်သည်။ C သည် Hardware ကို တိုက်ရိုက်ထိန်းချုပ်နိုင်စွမ်းရှိသောကြောင့်၊ အခြားသော ပရိုဂရမ်ဘာသာများထက် ပိုမိုကောင်းမွန်သော Performance နှင့် စွမ်းဆောင်ရည်ကို ရရှိစေသည်။ ကွန်ပျူတာ၏ အနိမ့်ဆုံးအဆင့်ဖြစ်သော Hardware များကို တိုက်ရိုက်ထိန်းချုပ်ပြီး Memory ကို စီမံခန့်ခွဲကာ၊ OS ၏ အစိတ်အပိုင်းများကို တိုက်ရိုက်တည်ဆောက်နိုင်ခြင်းကြောင့် System Programming များအတွက် အကောင်းဆုံးသော ကိရိယာတစ်ခု ဖြစ်လာသည်။ System Programming ဆိုသည်မှာ OS ၏ အခြေခံအဆောက်အအုံများအပေါ်တွင် တည်ဆောက်ခြင်းနှင့် ထိန်းချုပ်ခြင်းကို ဆိုလိုသည်။ ထို့ကြောင့် C ကို အသုံးပြုခြင်းဖြင့် ကျွန်ုပ်တို့သည် ကွန်ပျူတာစနစ်၏ အခြေခံအလုပ်လုပ်ပုံကို နားလည်ပြီး၊ ထိုစနစ်ကို အမြင့်ဆုံးစွမ်းဆောင်ရည်ဖြင့် ထိန်းချုပ်နိုင်သော System Programming များ ဖန်တီးနိုင်ကြသည်။

System Programming ၏ အခြေခံသဘောတရားများကို လေ့လာရာတွင်၊ ကျွန်ုပ်တို့သည် ကွန်ပျူတာစနစ်အဆင့် (System level) တွင် အလုပ်လုပ်ပုံကို မည်သို့ ပုံဖော်ရမည်ကို ရှာဖွေရမည် ဖြစ်သည်။ C သည် Memory ကို အလွန်နီးစပ်စွာ ထိန်းချုပ်နိုင်သောကြောင့်၊ OS ၏ အတွင်းပိုင်း အလုပ်လုပ်ပုံများကို နားလည်ပြီး၊ ထိုအခြေခံအပေါ်တွင် ပိုမိုရှုပ်ထွေးသော စနစ်များကို တည်ဆောက်ပုံကို မိတ်ဆက်ပေးမည် ဖြစ်သည်။

## ၂။ Operating System ၏ အခြေခံဖွဲ့စည်းပုံ (Fundamentals of Operating System Structure)

Operating System သည် ရှုပ်ထွေးသော စနစ်တစ်ခုဖြစ်သော်လည်း၊ ၎င်း၏ အခြေခံဖွဲ့စည်းပုံကို နားလည်ခြင်းဖြင့် ပိုမိုလွယ်ကူစွာ ကိုင်တွယ်နိုင်မည် ဖြစ်သည်။ OS ၏ အဓိက အစိတ်အပိုင်းများမှာ Kernel နှင့် Shell တို့ဖြင့် အဓိက ခွဲခြားနိုင်သည်။

### Kernel နှင့် Shell ၏ အခန်းကဏ္ဍ

Kernel (အဓိကအနှစ်သာရ) ဆိုသည်မှာ Operating System ၏ အဓိကအနှစ်သာရဖြစ်သည်။ ၎င်းသည် Hardware နှင့် Software များကြားတွင် အဓိက ဆက်သွယ်ရေးလမ်းကြောင်းအဖြစ် အလုပ်လုပ်သည်။ Kernel သည် ကွန်ပျူတာ၏ အရေးကြီးဆုံး လုပ်ငန်းစဉ်များဖြစ်သော Memory Management၊ Process Management နှင့် Device Management စသည့် အရာများကို ထိန်းချုပ်ပြီး Hardware နှင့် တိုက်ရိုက် ဆက်သွယ်ဆောင်ရွက်သည်။ ၎င်းသည် OS ၏ အလယ်ဗဟိုဖြစ်ပြီး၊ အခြားသော Application များသည် Kernel မှတစ်ဆင့်သာ Hardware နှင့် အပြန်အလှန် ဆက်သွယ်နိုင်ကြသည်။

Shell (ခွံ) ဆိုသည်မှာ User နှင့် Kernel တို့ကြားရှိ အပြန်အလှန် ဆက်သွယ်မှု (Interface) ဖြစ်သည်။ Shell သည် User များက Terminal သို့မဟုတ် Command Line မှတစ်ဆင့် စာသားအမိန့်များ (Commands) ကို ထည့်သွင်း၍ Kernel အား ထိုအမိန့်များကို အကောင်အထည်ဖော်စေရန် အသုံးပြုသည့် ပတ်ဝန်းကျင်ဖြစ်သည်။ Shell သည် User များ၏ လိုအပ်ချက်များကို Kernel ထံသို့ ပို့ဆောင်ပေးပြီး Kernel မှ လုပ်ဆောင်ပြီးသော အဖြေများကို User များထံ ပြန်လည်တင်ပြပေးခြင်းဖြင့် အပြန်အလှန် ဆက်သွယ်မှုကို ဖြစ်ပေါ်စေသည်။ ထို့ကြောင့် Kernel သည် စနစ်၏ အင်ဂျင်နီယာဖြစ်ပြီး Shell သည် အသုံးပြုသူအတွက် အသုံးပြုရလွယ်ကူသော အထက်ပတ်ဝန်းကျင် ဖြစ်သည်။

### System Call (စနစ်ခေါ်ဆိုမှု) ဆိုတာ ဘာလဲ?

System Call (စနစ်ခေါ်ဆိုမှု) ဆိုသည်မှာ Application (C Program) တစ်ခုက Operating System (Kernel) မှ ပေးထားသော Service များ (ဥပမာ- File ကိုဖွင့်ခြင်း၊ ဖိုင်ထဲသို့ ရေးခြင်း၊ Process အသစ်ဖန်တီးခြင်း စသည်) ကို ခေါ်ယူရန် အသုံးပြုသည့် နည်းလမ်းဖြစ်သည်။ Application များသည် Hardware ကို တိုက်ရိုက်ထိန်းချုပ်မရသောကြောင့်၊ OS ၏ စီမံခန့်ခွဲမှုများကို အသုံးပြုရန်အတွက် System Call များကို အသုံးပြုရသည်။ System Call များသည် Hardware နှင့် တိုက်ရိုက်ဆက်စပ်နေပြီး၊ Kernel ကို အကြောင်းကြား၍ လုပ်ဆောင်စေသည်။ ဥပမာအားဖြင့်၊ ဖိုင်တစ်ခုကို ဖွင့်ရန်အတွက် C Program တစ်ခုသည် `open()` ဟူသော System Call ကို ခေါ်ဆိုရပြီး၊ OS သည် ထိုအမိန့်ကို လက်ခံ၍ File System ကို စစ်ဆေးပြီး လိုအပ်သော အချက်အလက်များကို ပြန်ပေးသည်။ System Call များသည် Hardware နှင့် တိုက်ရိုက် ဆက်စပ်နေသောကြောင့်၊ ၎င်းတို့၏ မှန်ကန်မှုသည် စနစ်တစ်ခုလုံး၏ စွမ်းဆောင်ရည်ကို အကြီးအကျယ် ထိခိုက်နိုင်သည်။

Transition Note: OS ၏ အခြေခံဖွဲ့စည်းပုံနှင့် System Call များ၏ အခြေခံသဘောတရားများကို နားလည်ပြီးပြီဆိုတော့၊ ထို OS မှ ပေးအပ်သော Service များ (System Calls) ကို C Program များဖြင့် မည်သို့ အသုံးပြု၍ စနစ်ကို ထိန်းချုပ်နိုင်မလဲဆိုသည့် အပိုင်းကို ဆက်လက်လေ့လာရမည် ဖြစ်သည်။

## ၃။ Process နှင့် Memory Management (Process and Memory Management)

OS ၏ အခြေခံဖွဲ့စည်းပုံကို သိရှိပြီးနောက်၊ အောက်ပါ အခန်းတွင် Process များ မည်သို့ ရှိနေပုံနှင့် Memory ကို မည်သို့ စီမံခန့်ခွဲပုံကို လေ့လာသွားမည် ဖြစ်သည်။

### Process ဆိုတာ ဘာလဲ?

Process (လုပ်ငန်းစဉ်) ဆိုသည်မှာ Running Program တစ်ခု၏ အခြေအနေကို ဆိုလိုသည်။ ၎င်းသည် ပရိုဂရမ်တစ်ခုが実行されている (running) အခြေအနေကို ဖော်ပြသည်။ တစ်ချိန်တည်းတွင် လုပ်ဆောင်နေသော Program များသည် Process များအဖြစ် ရှိနေကြသည်။ Process တစ်ခုချင်းစီသည် မိမိ၏ ကိုယ်ပိုင် Memory Space နှင့် Resource များ (ဥပမာ- CPU အချိန်၊ File Handle များ) ကို ပိုင်ဆိုင်သည်။

Process တစ်ခုချင်းစီ၏ Memory Layout ကို နားလည်ရန် အရေးကြီးသည်။ Process တစ်ခု၏ Memory သည် အပိုင်း (Segments) များအဖြစ် ခွဲခြားထားသည်။ အဓိကအားဖြင့် Text Segment (ကုဒ်အပိုင်း)၊ Data Segment (အချက်အလက်အပိုင်း)၊ Heap Segment နှင့် Stack Segment များဖြင့် ဖွဲ့စည်းထားသည်။ ဤအပိုင်းအလိုက် ခွဲခြားထားခြင်းသည် Memory ကို စနစ်တကျ စီမံခန့်ခွဲရန်အတွက် အခြေခံဖြစ်သည်။

### Memory Segmentation ၏ အသေးစိတ်

Process များ၏ Memory Layout ကို နားလည်ရန် Memory Segmentation ၏ အသေးစိတ်ကို လေ့လာရမည်။

Text Segment (ကုဒ်အပိုင်း) သည် ပရိုဂရမ်၏ Executable Code များအား သိမ်းဆည်းထားသည်။ ဤအပိုင်းသည် အများအားဖြင့် Read-only ဖြစ်ပြီး၊ ပရိုဂရမ်၏ Instruction များ ပါဝင်သည်။ Data Segment (အချက်အလက်အပိုင်း) သည် ပရိုဂရမ်၏ Variable များ၊ Constants များ နှင့်その他の Data များအား သိမ်းဆည်းထားသည်။ ဤနှစ်ပိုင်းသည် ပရိုဂရမ်၏ လုပ်ဆောင်ချက်နှင့် သိမ်းဆည်းထားသော အချက်အလက်များကို ခွဲခြားပေးသည်။

Heap Segment နှင့် Stack Segment များသည် Process တစ်ခု၏ Dynamic Memory Allocation အတွက် အသုံးပြုသည်။ Stack Segment သည် Function Calls များ၊ Local Variables များ နှင့် Return Addresses များအား သိမ်းဆည်းထားပြီး၊ Function တစ်ခုမှ အခြား Function တစ်ခုသို့ ပြန်သွားသည့်အခါ အချက်အလက်များကို စနစ်တကျ ထိန်းသိမ်းထားသည်။ Heap Segment သည် ပရိုဂရမ်က Runtime အတွင်းတွင် လိုအပ်သလောက် Memory ကို တောင်းဆို၍ ရရှိသော Dynamic Data များအတွက် အသုံးပြုသည်။ Memory ကို ဗီရိုအံဆွဲ (Byte) နှင့် နှိုင်းယှဉ်လျှင်၊ Stack နှင့် Heap တို့သည် Memory ၏ အစိတ်အပိုင်းများကို အသုံးပြုပုံနှင့် ထိန်းချုပ်ပုံကို ဖော်ပြသည်။

Process Management ၏ အခြေခံများတွင် Process များ မည်သို့ အစီအစဉ်တကျ ရှိနေပုံကိုလည်း လေ့လာရမည်။ Process Scheduling ဆိုသည်မှာ CPU သည် မည်သည့် Process ကို မည်မျှကြာအောင် အသုံးပြုရမည်ကို ဆုံးဖြတ်သည့် စနစ်ဖြစ်သည်။ ဤအစီအစဉ်ကို ထိန်းချုပ်ခြင်းဖြင့် Process များ အချင်းချင်း မတိုက်မိအောင်နှင့် စနစ်၏ စွမ်းဆောင်ရည်ကို အမြင့်ဆုံးဖြစ်အောင် ထိန်းသိမ်းနိုင်မည် ဖြစ်သည်။

Transition Note: Process တစ်ခုချင်းစီ၏ Memory နှင့် Process Scheduling အခြေခံများကို သိရှိပြီးပြီဆိုတော့၊ C Program တစ်ခုက တစ်ခုတည်းသော Process ထက် ပိုမိုရှုပ်ထွေးသော Process များ (Child Process) ကို ဖန်တီးပြီး အချင်းချင်း ဆက်သွယ်နိုင်မလားဆိုသည့် အပိုင်းကို ဆက်လက်လေ့လာရမည် ဖြစ်သည်။

## ၄။ File I/O နှင့် File Descriptor (File I/O and File Descriptor)

Process နှင့် Memory Management ကို နားလည်ပြီးနောက်၊ ကွန်ပျူတာစနစ်တွင် အချက်အလက်များကို မည်သို့ သိမ်းဆည်းပုံနှင့် ထိန်းချုပ်ပုံဖြစ်သော File I/O နှင့် File Descriptor အကြောင်းကို လေ့လာရမည် ဖြစ်သည်။

### File I/O ၏ အခြေခံသဘောတရား

File I/O (ဖိုင် အဝင်အထွက်) ဆိုသည်မှာ File နှင့် Data များအား Disk သို့မဟုတ် Storage ထဲတွင် မည်သို့ သိမ်းဆည်းပုံနှင့် ထုတ်ယူပုံကို ဆိုလိုသည်။ ဒေတာများကို Disk ပေါ်တွင် သိမ်းဆည်းရာတွင်၊ Physical Memory (RAM) မှ Disk သို့ Data များအား ရွှေ့ပြောင်းခြင်းနှင့် ထိန်းသိမ်းခြင်းတို့ လိုအပ်သည်။ File I/O သည် ဤလုပ်ငန်းစဉ်များကို OS မှတစ်ဆင့် စီမံခန့်ခွဲပေးသည်။

### File Descriptor ဆိုတာ ဘာလဲ?

File Descriptor (ဖိုင် ဖော်ပြချက်) ဆိုသည်မှာ Operating System က File တစ်ခုကို မှတ်သားထားပုံကို ဆိုလိုသည်။ File တစ်ခုကို OS သည် ၎င်း၏ Internal Structure နှင့် တည်နေရာကို သိမ်းဆည်းထားသည်။ C Programming တွင် File I/O လုပ်ဆောင်ရာ၌၊ File တစ်ခုကို ကိုယ်စားပြုသည့် ဂဏန်းတန်ဖိုး (Integer value) တစ်ခုကို File Descriptor ဟု ခေါ်သည်။ ဤ File Descriptor များသည် OS ၏ File Table တွင် မှတ်သားထားသော အချက်အလက်များကို ရည်ညွှန်းပေးသည်။

Standard Streams များသည် File Descriptor များ၏ အထူးအမှုတွဲများ ဖြစ်သည်။ ၎င်းတို့မှာ Standard Input (stdin) သည် ပုံမှန်အားဖြင့် Keyboard မှ Data များ လက်ခံရရှိရန်၊ Standard Output (stdout) သည် ပုံမှန်အားဖြင့် Screen သို့ Data များ ထုတ်ပေးရန်၊ နှင့် Standard Error (stderr) သည် Error Messages များ ထုတ်ပေးရန် ဖြစ်သည်။ ဤ Stream များသည် C Program များအတွက် အချက်အလက်များကို ထည့်သွင်းခြင်းနှင့် ထုတ်ပေးခြင်းကို အလွယ်တကူ လုပ်ဆောင်နိုင်စေသည်။ ဥပမာအားဖြင့်၊ `stdin` (Descriptor 0) မှ Data များ လက်ခံခြင်း၊ `stdout` (Descriptor 1) မှ Output များ ထုတ်ပေးခြင်းနှင့် `stderr` (Descriptor 2) မှ Error များ ထုတ်ပေးခြင်းတို့ ဖြစ်သည်။

### Low-Level File I/O Functions

C Programming တွင် File I/O ကို လုပ်ဆောင်ရန်အတွက် Low-Level Functions များကို အသုံးပြုရသည်။ Standard I/O ကို C ဖြင့် အသုံးပြုပုံမှာ `stdin`, `stdout`, နှင့် `stderr` တို့ကို တိုက်ရိုက်အသုံးပြုခြင်းဖြင့် ဖြစ်သည်။

File Handle များအား ကိုင်တွယ်ရန်အတွက် `open()`, `read()`, `write()`, နှင့် `close()` ကဲ့သို့သော System Call များ အပြင် C Library မှ ပံ့ပိုးပေးသော Function များဖြစ်သည့် `fopen()`, `fread()`, `fwrite()`, နှင့် `fclose()` တို့ကို အသုံးပြုသည်။ `fopen()` သည် File တစ်ခုကို ဖွင့်ပြီး File Pointer တစ်ခုကို ပြန်ပေးသည်။ ထို့နောက် `fread()` သို့မဟုတ် `fwrite()` ကို အသုံးပြု၍ Data များကို ဖိုင်ထဲသို့ ရေးသွင်းခြင်း သို့မဟုတ် ဖိုင်ထဲမှ ဖတ်ယူခြင်းများ ပြုလုပ်သည်။ `fclose()` သည် File ကို ပိတ်ပြီး OS ထံသို့ အကြောင်းကြားခြင်း ဖြစ်သည်။ ဤ Function များသည် OS ၏ File System နှင့် ဆက်သွယ်ပြီး Data များကို Disk ပေါ်သို့ ရေးသားခြင်းနှင့် ဖတ်ယူခြင်းများကို အထောက်အကူပြုသည်။

Transition Note: File I/O နှင့် File Descriptor များအကြောင်းကို နားလည်ပြီးပြီဆိုတော့၊ C Program တစ်ခုက တစ်ခုတည်းသော Process ထက် ပိုမိုရှုပ်ထွေးသော Process များ (Child Process) ကို ဖန်တီးပြီး အချင်းချင်း ဆက်သွယ်နိုင်မလားဆိုတာကို လေ့လာရမည် ဖြစ်သည်။

## ၅။ Process Creation နှင့် Inter-process Communication (IPC)

Process များ ဖန်တီးပြီး အချင်းချင်း ဆက်သွယ်နိုင်မည့် နည်းလမ်းများကို လေ့လာရမည် ဖြစ်သည်။ ဤအပိုင်းတွင် Process များကြား အချက်အလက်များ (Data) ကို မည်သို့ ပို့ဆောင်နိုင်ပုံကို အဓိကထားမည် ဖြစ်သည်။

### Process Creation ၏ အခြေခံ (fork() Function)

Process Creation (လုပ်ငန်းစဉ် ဖန်တီးခြင်း) သည် OS ၏ အရေးကြီးသော လုပ်ဆောင်ချက်များထဲမှ တစ်ခုဖြစ်သည်။ Process တစ်ခုမှ Child Process တစ်ခုကို ဖန်တီးရန်အတွက် အသုံးအများဆုံး နည်းလမ်းမှာ `fork()` Function ဖြစ်သည်။ `fork()` Function ကို C Program တွင် အသုံးပြုသောအခါ၊ ပုံမှန်အားဖြင့် Parent Process နှင့် Child Process နှစ်ခုလုံးကို ဖန်တီးပေးသည်။ ဤလုပ်ငန်းစဉ်တွင် Parent နှင့် Child Process များသည် မိမိတို့၏ Memory Space နှင့် Resource များကို မည်သို့ ပူးတွဲရှိပုံကို နားလည်ရန် လိုအပ်သည်။ Child Process သည် Parent Process ၏ Memory များကို ကော်ပီယူ (Copy) ရမည် သို့မဟုတ် Share လုပ်ရမည် ဖြစ်သည်။ ဤအချက်သည် Process များကြား အချက်အလက်များ အပြန်အလှန် ဆက်သွယ်နိုင်ရန် အခြေခံအုတ်မြစ် ဖြစ်သည်။

### Inter-process Communication (IPC) အခြေခံ (pipe() Function)

Process များကြား အချက်အလက်များ (Data) ကို မည်သို့ ပို့ဆောင်နိုင်ပုံကို Inter-process Communication (IPC) ဟု ခေါ်သည်။ IPC သည် Process များအချင်းချင်း ဆက်သွယ်၍ အလုပ်လုပ်နိုင်ရန် လိုအပ်သော နည်းလမ်းများဖြစ်သည်။ Pipe (ပိုက်) နည်းလမ်းသည် Process များကြား Data Flow ကို ထိန်းချုပ်ရန်အတွက် အလွန်အသုံးဝင်သော နည်းလမ်းတစ်ခု ဖြစ်သည်။ Pipe သည် Data များအား တစ်လမ်းသွား (One-way) သို့မဟုတ် နှစ်လမ်းသွား (Two-way) အဖြစ် ပို့ဆောင်ရန်အတွက် Buffer တစ်ခုအဖြစ် အသုံးပြုသည်။ Process နှစ်ခုသည် Pipe တစ်ခုကို အသုံးပြု၍ Data များကို တစ်ခုနှင့်တစ်ခု ပို့နိုင်ပြီး၊ ထို Data များသည် Pipe တွင် စောင့်ဆိုင်းနေနိုင်သည်။ ဤနည်းလမ်းဖြင့် Process များသည် အချိန်နှင့်တပြေးညီ အချက်အလက်များကို လဲလှယ်နိုင်ကြသည်။

Transition Note: Process များ ဖန်တီးပြီး အချင်းချင်း ဆက်သွယ်နိုင်ပြီဆိုတော့၊ အရေးပေါ်အခြေအနေများတွင် Process များကြား စနစ်တကျ ဆက်သွယ်မှုနှင့် ထိန်းချုပ်မှု (Signal Handling) အကြောင်းကို လေ့လာရမည် ဖြစ်သည်။

## ၆။ Signal Handling နှင့် နိဂုံး (Signal Handling and Conclusion)

Process များကြား ဆက်သွယ်နိုင်ပြီဆိုတော့၊ အရေးပေါ်အခြေအနေများတွင် Process များကြား စနစ်တကျ ဆက်သွယ်မှုနှင့် ထိန်းချုပ်မှု (Signal Handling) အကြောင်းကို လေ့လာရမည် ဖြစ်သည်။

### Signal Handling (အချက်ပေးမှု စီမံခန့်ခွဲမှု)

Signal (အချက်ပေးမှု) ဆိုသည်မှာ OS မှ Process တစ်ခုသို့ အပြင်မှ အကြောင်းကြားရန် သို့မဟုတ် ထိန်းချုပ်ရန် ပေးသော အချက်အလက်များ ဖြစ်သည်။ ဥပမာအားဖြင့်၊ SIGINT (Interrupt) သည် ပရိုဂရမ်ကို ရပ်တန့်ရန်၊ SIGTERM (Terminate) သည် ပရိုဂရမ်ကို ပြီးဆုံးစေရန် အသုံးပြုသည်။ Signal များသည် Process များအား အပြင်မှ ထိန်းချုပ်ရန်နှင့် အကြောင်းကြားရန် အလွန်အရေးကြီးသည်။ Process များသည် အခြား Process များ၏ အခြေအနေကို သိရှိပြီး၊ အရေးပေါ်အခြေအနေများတွင် ထိန်းချုပ်နိုင်ရန် Signal များမှတစ်ဆင့် အကြောင်းကြားခြင်းနှင့် တုံ့ပြန်ခြင်းများ ပြုလုပ်သည်။ Signal များဖြင့် Process များကို ရပ်တန့်ခြင်း (Termination) သို့မဟုတ် ရပ်ဆိုင်းခြင်း (Suspension) များ ပြုလုပ်နိုင်သည်။

### C ကို OS Development တွင် အသုံးပြုရခြင်း၏ အကျိုးကျေးဇူးများ

C Programming ကို Operating System Development တွင် အသုံးပြုရခြင်း၏ အဓိက အကျိုးကျေးဇူးများမှာ အောက်ပါအတိုင်း ဖြစ်သည်။

ပထမအချက်မှာ Hardware Access (Hardware ကို တိုက်ရိုက်ထိန်းချုပ်နိုင်ခြင်း) ဖြစ်သည်။ C သည် Hardware နှင့် အနီးစပ်ဆုံး အလုပ်လုပ်နိုင်သောကြောင့်၊ OS ၏ Kernel အဆင့်တွင် Hardware များနှင့် တိုက်ရိုက် ဆက်သွယ်ပြီး ထိန်းချုပ်နိုင်ခြင်းသည် မရှိမဖြစ်လိုအပ်သည်။

ဒုတိယအချက်မှာ Performance (စွမ်းဆောင်ရည်) ဖြစ်သည်။ C သည် Machine Code နှင့် အနီးစပ်ဆုံး အလုပ်လုပ်သောကြောင့်၊ အခြားဘာသာစကားများနှင့် နှိုင်းယှဉ်လျှင် အလွန်မြန်ဆန်ပြီး စွမ်းဆောင်ရည်မြင့်မားသော ပရိုဂရမ်များကို ရေးသားနိုင်ခြင်းဖြစ်သည်။

တတိယအချက်မှာ System Programming အတွက် C ၏ အခြေခံကျကျရှိသော သဘောတရားများ ဖြစ်သည်။ Memory Management၊ Process Scheduling၊ File I/O နှင့် IPC စသည့် OS ၏ အခြေခံအယူအဆများကို C ဖြင့် အခြေခံကျကျ တည်ဆောက်နိုင်ခြင်းသည် System Programming များအတွက် အလွန်အကျိုးရှိသည်။

### Chapter Summary

ဤအခန်းတွင် Operating System ၏ အခြေခံဖွဲ့စည်းပုံဖြစ်သော Kernel နှင့် Shell၊ System Call များ၊ Process နှင့် Memory Layout (Text, Data, Heap, Stack)၊ File I/O နှင့် File Descriptor များ၊ Process Creation (fork()) နှင့် Inter-process Communication (pipe())၊ Signal Handling များအကြောင်းကို လေ့လာခဲ့ပြီး ဖြစ်သည်။ ဤအကြောင်းအရာများသည် Modern Operating System များ၏ အခြေခံကျသော အစိတ်အပိုင်းများ ဖြစ်ကြသည်။ C Programming ကို အသုံးပြု၍ ဤအခြေခံများကို နားလည်ပြီး၊ Hardware ကို ထိန်းချုပ်နိုင်သည့် System Programming များ တည်ဆောက်နိုင်ရန် အခြေခံအုတ်မြစ်ကို ချမှတ်နိုင်ခဲ့ပြီ ဖြစ်သည်။
