# Chapter 2: CPU, Memory နှင့် ကွန်ပျူတာ ဗိသုကာ — အတွင်းပိုင်းကို ဖွင့်ကြည့်ခြင်း

## 1. အဖွင့်အစနှင့် မိတ်ဆက် (Opening Hook & Introduction)

ကွန်ပျူတာတွေက ဘာကြောင့် အလုပ်လုပ်ရတာလဲ။ ကျွန်ုပ်တို့ နေ့စဉ်သုံးနေတဲ့ စမတ်ဖုန်း၊ ကွန်ပျူတာတွေဟာ ကျွန်ုပ်တို့ရဲ့ စကား၊ ပုံရိပ်၊ ဗီဒီယိုတွေကို ပြသပေးနိုင်ပြီး၊ ဆက်သွယ်ရေးလုပ်ငန်းတွေ၊ ဂိမ်းတွေ၊ ပရိုဂရမ်တွေ အားလုံးကို ထိန်းချုပ်ပေးနိုင်ပါတယ်။ ဒီလို ရှုပ်ထွေးတဲ့ လုပ်ဆောင်ချက်တွေဟာ မြင်ရတာထက် ပိုပြီး နက်ရှိုင်းတဲ့ အခြေခံအုတ်မြစ်တွေပေါ်မှာ တည်ဆောက်ထားတာ ဖြစ်ပါတယ်။ ကျွန်ုပ်တို့ဟာ C Programming လိုမျိုး ဘာသာစကားတွေကို သင်ယူပြီး ပရိုဂရမ်တွေ ရေးသားတဲ့အခါ၊ ကျွန်ုပ်တို့ဟာ ဘာသာစကားတွေရဲ့ အပေါ်ယံကိုသာ မကြည့်ဘဲ၊ အဲဒီဘာသာစကားတွေ နောက်ကွယ်မှာ ဘယ်လို အလုပ်လုပ်နေလဲဆိုတဲ့ Hardware နဲ့ Software ဆက်စပ်မှုတွေကို နားလည်ဖို့ လိုအပ်ပါတယ်။

C Programming ကို သင်ယူခြင်းရဲ့ နောက်ကွယ်က အခြေခံအုတ်မြစ်တွေဟာ ကွန်ပျူတာရဲ့ အခြေခံ Hardware (သို့မဟုတ်) ဗိသုကာ (Architecture) နဲ့ တိုက်ရိုက်ဆက်စပ်နေပါတယ်။ CPU (Central Processing Unit)၊ Memory (မှတ်ဉာဏ်) နဲ့ ကွန်ပျူတာရဲ့ အကြီးစား စီစဉ်ဖွဲ့စည်းပုံတွေဟာ ကျွန်ုပ်တို့ ရေးသားတဲ့ Code တစ်ကြောင်းချင်းစီက ဘယ်လို အကောင်အထည်ဖော်ခံရသလဲဆိုတာကို ဆုံးဖြတ်ပေးပါတယ်။ ဒီအခန်းရဲ့ ရည်ရွယ်ချက်ကတော့ ဒီအခြေခံ Hardware နှင့် Software တို့ကြားက ဆက်စပ်မှုကို လက်တွေ့ကျကျ နားလည်အောင် ဖော်ထုတ်ပေးဖို့ ဖြစ်ပါတယ်။ ကျွန်ုပ်တို့ဟာ ပရိုဂရမ်မဆရာတွေအဖြစ် အလုပ်လုပ်နေတဲ့အတွက်၊ ပရိုဂရမ်တစ်ခုက CPU ကို ဘယ်လို အမိန့်ပေးပြီး၊ အဲဒီအမိန့်တွေဟာ Memory ထဲမှာ ဘယ်လို သိမ်းဆည်းပြီး၊ နောက်ဆုံးမှာ ဘယ်လို အကျိုးရလဒ်တွေ ထွက်ပေါ်လာသလဲဆိုတဲ့ အတွင်းပိုင်း အကြောင်းအရာတွေကို လေ့လာသွားမှာ ဖြစ်ပါတယ်။

## 2. CPU: ကွန်ပျူတာရဲ့ ဦးနှောက် (The Central Processing Unit)

CPU ဆိုတာ ကွန်ပျူတာရဲ့ အဓိက အစိတ်အပိုင်းဖြစ်ပြီး "Central Processing Unit" လို့ အမည်ရသည်။ ၎င်းကို ကွန်ပျူတာရဲ့ ဦးနှောက်အဖြစ် သတ်မှတ်နိုင်ပါတယ်။ CPU ရဲ့ အဓိကလုပ်ဆောင်ချက်ကတော့ အချက်အလက်တွေကို လက်ခံခြင်း၊ တွက်ချက်ခြင်း၊ နှင့် အမိန့်ပေးခြင်း (Control) တို့ကို အဓိက လုပ်ဆောင်ခြင်းပဲ ဖြစ်ပါတယ်။ ကွန်ပျူတာထဲမှာ ဖြစ်ပေါ်နေတဲ့ အရာအားလုံးဟာ CPU ရဲ့ အမိန့်ပေးမှုအပေါ် အခြေခံပါတယ်။

### CPU ၏ အခန်းကဏ္ဍများ

CPU ဟာ အဓိကအားဖြင့် သုံးမျိုးသော အခန်းကဏ္ဍများကို လုပ်ဆောင်ပေးပါတယ်။ ၎င်းတို့ကို အောက်ပါအတိုင်း ခွဲခြားနိုင်ပါတယ်-

၁။ **Control (ထိန်းချုပ်မှု):** CPU ဟာ ပရိုဂရမ်တစ်ခုလုံးကို စီမံခန့်ခွဲပေးတဲ့ အခန်းကဏ္ဍကို ထမ်းဆောင်ပါတယ်။ Instruction တစ်ခုပြီးတစ်ခု ဘယ်အချိန်မှာ ဘယ်နေရာကို သွားမလဲဆိုတာကို ထိန်းချုပ်ပေးပါတယ်။
၂။ **Calculation (တွက်ချက်မှု):** သင်္ချာဆိုင်ရာ (Addition, Subtraction) နှင့် ယုတ္တိဗေဒဆိုင်ရာ (Logical operations, AND, OR, NOT) လိုမျိုး ဂဏန်းတွေနဲ့ တန်ဖိုးတွေကို တွက်ချက်ပေးတဲ့ လုပ်ငန်းတွေကို လုပ်ဆောင်ပါတယ်။
၃။ **Execution (အကောင်အထည်ဖော်မှု):** Control ၏ အမိန့်များအပေါ် အခြေခံပြီး၊ လက်တွေ့ကျကျ လုပ်ဆောင်မှုတွေကို အကောင်အထည်ဖော်ပေးခြင်း ဖြစ်ပါတယ်။

### Fetch-Decode-Execute Cycle: အမိန့်ပေးမှု စက်ဝိုင်း

CPU ဟာ Instruction တစ်ခုကို လက်ခံရရှိပြီး လုပ်ဆောင်တဲ့အခါ အလွန်တိကျတဲ့ အဆင့်ဆင့် လုပ်ဆောင်မှု စက်ဝိုင်းတစ်ခုကို အသုံးပြုပါတယ်။ ဒီလုပ်ဆောင်မှု စက်ဝိုင်းကို **Fetch-Decode-Execute Cycle** လို့ ခေါ်ပါတယ်။

**၁။ Fetch (ရှာဖွေခြင်း):**
ပထမဆုံး အဆင့်မှာ CPU ဟာ Memory ထဲကနေ လုပ်ဆောင်ရမယ့် Instruction (အမိန့်) တစ်ခုကို ရှာဖွေပြီး သူ့ရဲ့ လက်ရှိနေရာကနေ ခေါ်ယူပါတယ်။ ဒီအမိန့်ကို "Instruction Register" (IR) လို့ခေါ်တဲ့ အထူးနေရာတစ်ခုထဲကို သယ်ယူရရှိပါတယ်။ ဒီအမိန့်ရဲ့ လိပ်စာကို သိမ်းဆည်းထားတဲ့နေရာကတော့ **Program Counter (PC)** ဖြစ်ပါတယ်။

**၂။ Decode (အဓိပ္ပာယ်ဖွင့်ဆိုခြင်း):**
Fetch လုပ်ပြီးတဲ့ Instruction ဟာ ဘာကို ဆိုလိုတာလဲ၊ ဘယ်လိုလုပ်ဆောင်ရမလဲဆိုတဲ့ အဓိပ္ပာယ်ကို CPU ရဲ့ Control Unit (CU) က ဖတ်ရှုပြီး အဓိပ္ပာယ်ဖွင့်ဆိုပါတယ်။ ဒီအဆင့်မှာ Instruction ထဲက Command တွေကို CPU ရဲ့ အတွင်းပိုင်း ပရိုဂရမ်များနဲ့ တိုက်ဆိုင်စစ်ဆေးပါတယ်။

**၃။ Execute (အကောင်အထည်ဖော်ခြင်း):**
Decode လုပ်ပြီးတဲ့အခါမှာ CPU ဟာ အဲဒီ Instruction ကို လက်တွေ့ကျကျ အကောင်အထည်ဖော်ဖို့အတွက် လိုအပ်တဲ့ လုပ်ဆောင်ချက်ကို စတင်ပါတယ်။ ဥပမာအားဖြင့်၊ တွက်ချက်မှု (ALU ကို အသုံးပြုခြင်း)၊ မှတ်ဉာဏ်ထဲသို့ ရေးထည့်ခြင်း (Memory ထံသို့) သို့မဟုတ် အခြား Register များထဲသို့ အချက်အလက်များကို ရွှေ့ပြောင်းခြင်း စတဲ့ လုပ်ဆောင်မှုတွေ ဖြစ်ပါတယ်။

ဒီ Fetch-Decode-Execute စက်ဝိုင်းဟာ ကွန်ပျူတာရဲ့ အခြေခံလုပ်ဆောင်မှုပုံစံဖြစ်ပြီး၊ ပရိုဂရမ်တစ်ခုလုံးဟာ ဒီ Cycle ကို ထပ်ခါထပ်ခါ အသုံးပြုပြီး အကောင်အထည်ဖော်ခံရတာ ဖြစ်ပါတယ်။

### CPU ၏ အတွင်းပိုင်း အစိတ်အပိုင်းများ

CPU ဟာ အဆင့်မြင့်တဲ့ စက်ဖြစ်တဲ့အတွက် သူ့ရဲ့ အတွင်းပိုင်းမှာ အထူးပြုလုပ်ဆောင်မှုများအတွက် သီးခြားအစိတ်အပိုင်းများ ပါဝင်ပါတယ်။ အဓိက အစိတ်အပိုင်းသုံးခုကို ဖော်ပြရအောင်။

**Registers (မှတ်စုများ):**
Registers ဆိုတာ CPU ရဲ့ အလွန်မြန်ဆန်တဲ့၊ ခေတ္တအချက်အလက်တွေကို လက်ရှိ သိမ်းဆည်းထားမယ့် အလွန်သေးငယ်တဲ့ မှတ်ဉာဏ်နေရာတွေ ဖြစ်ပါတယ်။ ၎င်းတို့သည် CPU အတွင်းမှာ အချက်အလက်များကို အလွန်လျင်မြန်စွာ ရယူသုံးစွဲနိုင်ရန်အတွက် အရေးကြီးပါတယ်။ ဥပမာအားဖြင့်၊ လက်ရှိ လုပ်ဆောင်နေသော တန်ဖိုးများ၊ Address များ၊ သို့မဟုတ် လုပ်ဆောင်မှုများ၏ အခြေအနေများကို ခေတ္တ သိမ်းဆည်းထားသည်။ Register များသည် Memory ထက် အဆပေါင်းများစွာ ပိုမိုမြန်ဆန်စွာ အသုံးပြုနိုင်သောကြောင့် CPU ၏ အမြန်နှုန်းကို မြှင့်တင်ပေးသည်။

**ALU (Arithmetic Logic Unit):**
ALU ဆိုတာ CPU ရဲ့ တွက်ချက်မှုနှင့် ယုတ္တိဗေဒဆိုင်ရာ လုပ်ဆောင်မှုများကို တာဝန်ယူတဲ့ အပိုင်း ဖြစ်ပါတယ်။ ၎င်းသည် အခြေခံအားဖြင့် ဂဏန်းဆိုင်ရာ သင်္ချာတွက်ချက်မှုများ (ပေါင်း၊ နုတ်၊ မြှောက်၊ စား) နှင့် ယုတ္တိဗေဒဆိုင်ရာ နှိုင်းယှဉ်မှုများ (ထပ်တူ၊ မညီ၊ ပိတ်၊ ဖွင့် စသည်) ကို လုပ်ဆောင်ပေးပါတယ်။ ALU သည် Instruction မှ ရရှိသော အချက်အလက်များကို လက်ခံပြီး လိုအပ်သော တွက်ချက်မှုများကို အလွန်လျင်မြန်စွာ ပြုလုပ်ပေးနိုင်စွမ်းရှိသည်။

**Program Counter (PC):**
Program Counter (PC) ဆိုတာ CPU ရဲ့ အလွန်အရေးကြီးတဲ့ Register တစ်ခု ဖြစ်ပါတယ်။ ၎င်း၏ အဓိကတာဝန်ကတော့ နောက်ထပ် Fetch လုပ်ရမယ့် Instruction ရဲ့ လိပ်စာ (Memory Address) ကို သိမ်းဆည်းထားဖို့ ဖြစ်ပါတယ်။ Fetch-Decode-Execute Cycle တစ်ခုပြီးဆုံးသွားတဲ့အခါ PC ဟာ နောက် Instruction ရဲ့ လိပ်စာကို အလိုအလျောက် ပြောင်းလဲပေးပါတယ်။ ဒါကြောင့် CPU ဟာ ဘယ်ကနေ နောက် Instruction ကို ဆက်လက် Fetch လုပ်ရမလဲဆိုတာကို မှတ်သားထားနိုင်ပါတယ်။

CPU သည် ဤအစိတ်အပိုင်းများ (Registers, ALU, PC) ပေါင်းစပ်ပြီး Fetch-Decode-Execute Cycle ကို အကြိမ်ပေါင်းများစွာ ထပ်ခါထပ်ခါ လုပ်ဆောင်ခြင်းဖြင့် ကွန်ပျူတာရဲ့ အမိန့်များကို လက်တွေ့ကျကျ အကောင်အထည်ဖော်ပေးနိုင်ပါတယ်။

CPU က အလုပ်လုပ်ဖို့အတွက် အချက်အလက်တွေ လိုအပ်ပါတယ်။ အဲဒီအချက်အလက်တွေက ဘယ်မှာ သိမ်းထားသလဲဆိုတာကို ဆက်လက်လေ့လာရအောင်။

## 3. Memory System: အချက်အလက်များ သိမ်းဆည်းရာနေရာများ (RAM, ROM, နှင့် Address)

CPU ဟာ အမိန့်ပေးတဲ့ ဦးနှောက်ဖြစ်ပြီး၊ အဲဒီအမိန့်တွေ အလုပ်လုပ်ဖို့အတွက် အချက်အလက်တွေ လိုအပ်ပါတယ်။ ဒီအချက်အလက်တွေကို သိမ်းဆည်းထားတဲ့ နေရာတွေကတော့ Memory System ဖြစ်ပါတယ်။ Memory System ဟာ အချက်အလက်များကို စနစ်တကျ သိမ်းဆည်းပြီး CPU နှင့် အခြား Hardware များအကြား ဆက်သွယ်ပေးတဲ့ အလယ်အလတ် ဆောင်ရွက်ပေးတဲ့ အခန်းကဏ္ဍ ဖြစ်ပါတယ်။

### RAM နှင့် ROM

Memory System ကို အဓိကအားဖြင့် အမျိုးအစားနှစ်မျိုးအဖြစ် ခွဲခြားနိုင်ပါတယ်- RAM နှင့် ROM တို့ ဖြစ်ပါတယ်။

**RAM (Random Access Memory):**
RAM ဆိုတာ ကွန်ပျူတာရဲ့ **ခေတ္တမှတ်ဉာဏ် (Volatile Memory)** ဖြစ်ပါတယ်။ RAM ဟာ CPU က လက်ရှိ အလုပ်လုပ်နေတဲ့ ပရိုဂရမ်တွေနဲ့ အချက်အလက်တွေကို ခေတ္တမျှ သိမ်းဆည်းထားတဲ့ နေရာ ဖြစ်ပါတယ်။ RAM ထဲမှာ သိမ်းဆည်းထားတဲ့ အချက်အလက်တွေဟာ CPU က လိုအပ်တဲ့အခါ အလွန်လျင်မြန်စွာ ဝင်ရောက်ဖတ်ရှုနိုင်ပြီး ပြောင်းလဲနိုင်ပါတယ်။ အရေးကြီးတဲ့အချက်က RAM ဟာ **Volatile** ဖြစ်တယ်ဆိုတာပါ၊ ဆိုလိုတာက ပါဝါပိတ်လိုက်တာနဲ့ RAM ထဲက အချက်အလက်တွေ အကုန်လုံး ပျောက်ဆုံးသွားပါတယ်။ ဒါကြောင့် RAM ကို အမြဲတမ်း အသုံးပြုနေတဲ့ အချက်အလက်တွေနဲ့ လက်ရှိလုပ်ဆောင်နေတဲ့ ပရိုဂရမ်တွေအတွက် အဓိက အသုံးပြုကြပါတယ်။

**ROM (Read-Only Memory):**
ROM ဆိုတာ **Read-Only Memory** ဖြစ်ပါတယ်။ ROM ဟာ ကွန်ပျူတာ စတင်ချိန်မှစ၍ ထည့်သွင်းထားတဲ့ အခြေခံအချက်အလက်တွေနဲ့ Booting လုပ်ဖို့အတွက် လိုအပ်တဲ့ မူလပရိုဂရမ်တွေ (Firmware) ကို သိမ်းဆည်းထားတဲ့ နေရာ ဖြစ်ပါတယ်။ ROM ထဲက အချက်အလက်တွေကို အလွယ်တကူ ဖတ်ရှုနိုင်ပေမယ့်၊ ပုံမှန်အားဖြင့် အဲဒီအချက်အလက်တွေကို ပြန်လည်ရေးသားတာ (Write) မပြုလုပ်နိုင်ပါဘူး။ ROM သည် ကွန်ပျူတာ စတင်ချိန်မှာ အမြဲတမ်း ထိန်းသိမ်းထားရမယ့် အခြေခံ စနစ်များအတွက် အရေးကြီးပါတယ်။

### Memory Address ဆိုတာ ဘာလဲ?

Memory ထဲမှာ ရှိတဲ့ အချက်အလက် တစ်ခုချင်းစီကို CPU က ဘယ်နေရာမှာ သိမ်းထားသလဲဆိုတာကို သိဖို့အတွက် **Memory Address** ဆိုတဲ့ သဘောတရားကို နားလည်ဖို့ လိုအပ်ပါတယ်။ Memory တစ်ခုလုံးဟာ Byte (၈ bit) များဖြင့် စီစဉ်ထားပြီး၊ Byte တစ်ခုချင်းစီတိုင်းမှာ ထူးခြားတဲ့ လိပ်စာ (Address) တစ်ခုစီ ရှိပါတယ်။

ဥပမာအားဖြင့်၊ Memory တစ်ခုလုံးကို စဉ်ဆက်မပြတ် Byte များဖြင့် စီစဉ်ထားတယ်ဆိုပါစို့။
*   Address 0 မှ 7 အထိ Byte 0 ဖြစ်သည်။
*   Address 8 မှ 15 အထိ Byte 1 ဖြစ်သည်။
*   Address 16 မှ 23 အထိ Byte 2 ဖြစ်သည်။

ဒါကို ပိုမိုလက်တွေ့ကျကျ ဥပမာပေးကြည့်ရအောင်။ ကျွန်ုပ်တို့သည် Array တစ်ခုကို Memory ထဲတွင် သိမ်းဆည်းထားသည်ဟု ဆိုကြပါစို့။ C Programming တွင် Array များသည် Memory ထဲတွင် အစဉ်လိုက် (Contiguous) အနေဖြင့် သိမ်းဆည်းထားလေ့ရှိသည်။

အကယ်၍ ကျွန်ုပ်တို့သည် `int arr[3] = {10, 20, 30};` ဟူသော Array ကို Memory ထဲတွင် သိမ်းဆည်းထားသည်ဆိုပါစို့။
*   `arr[0]` (တန်ဖိုး ၁၀) သည် Memory Address $X$ တွင် ရှိသည်။
*   `arr[1]` (တန်ဖိုး ၂၀) သည် Memory Address $X+4$ တွင် ရှိသည်။ (အကယ်၍ `int` သည် 4 Byte ဖြစ်သည်ဟု ယူဆပါက)
*   `arr[2]` (တန်ဖိုး ၃၀) သည် Memory Address $X+8$ တွင် ရှိသည်။

ဤကဲ့သို့ဖြင့်၊ Memory Address သည် အချက်အလက်တစ်ခုချင်းစီ၏ တိကျသော နေရာကို ဖော်ပြပေးပါသည်။ CPU သည် Instruction တစ်ခုကို Fetch လုပ်သောအခါ၊ ၎င်းသည် Memory Address ကို အသုံးပြု၍ ထိုလိပ်စာမှ အချက်အလက်ကို ရယူမည် ဖြစ်သည်။ Memory Addressing သည် ပရိုဂရမ်နှင့် Data များ မည်သို့ ဆက်စပ်နေပုံကို အခြေခံကျကျ ဖော်ပြပေးပါသည်။

### Memory Hierarchy Overview

Memory System သည် အဆင့်အလိုက် စီစဉ်ထားသော အဆင့်ဆင့် စနစ်တစ်ခု ဖြစ်ပါတယ်။ အချက်အလက်များကို သိမ်းဆည်းရာတွင် မြန်နှုန်း၊ စွမ်းဆောင်ရည်နှင့် ကုန်ကျစရိတ်တို့ကို ထည့်သွင်းစဉ်းစားရပါတယ်။ Memory Hierarchy တွင် အောက်ပါအဆင့်များ ပါဝင်သည်-

၁။ **Registers (CPU အတွင်း):** အမြန်ဆုံးနှင့် အနီးဆုံးမှတ်ဉာဏ် (အလွန်သေးငယ်ပြီး CPU အတွင်း၌ တိုက်ရိုက်တည်ရှိသည်။)
၂။ **Cache Memory (L1, L2, L3):** CPU နှင့် RAM ကြားတွင် တည်ရှိပြီး CPU အတွက် အလွန်အရေးကြီးသော အချက်အလက်များကို ခေတ္တသိမ်းဆည်းထားသည်။ ၎င်းတို့သည် RAM ထက် ပိုမိုမြန်ဆန်သည်။
၃။ **RAM (Main Memory):** လက်ရှိအသုံးပြုနေသော ပရိုဂရမ်များနှင့် အချက်အလက်များကို သိမ်းဆည်းထားသော အဓိက မှတ်ဉာဏ်။
၄။ **Secondary Storage (HDD/SSD):** အချက်အလက်များကို အချိန်ကြာရှည်စွာ သိမ်းဆည်းထားသော နေရာ (ဥပမာ- Hard Disk Drive သို့မဟုတ် Solid State Drive)။ ၎င်းသည် RAM ထက် နှေးကွေးသော်လည်း အချက်အလက်များကို အမြဲတမ်း သိမ်းဆည်းထားနိုင်သော နေရာ ဖြစ်သည်။

ဤအဆင့်ဆင့် စီစဉ်ဖွဲ့စည်းပုံကြောင့် ကွန်ပျူတာများသည် အချက်အလက်များကို လိုအပ်သည့် အချိန်နှင့် နေရာတွင် အထိရောက်ဆုံး ရရှိနိုင်စေပါသည်။

Memory ထဲမှာ အချက်အလက်တွေ ဘယ်လို စီစဉ်ထားလဲဆိုတာကို နားလည်သွားပြီဆိုရင်၊ ဒီအချက်အလက်တွေကို ဘယ်လို ပေါင်းစပ်ပြီး သိမ်းဆည်းလဲဆိုတဲ့ အကြီးစား သဘောတရားကို ဆက်လက်ကြည့်ရအောင်။

## 4. Von Neumann Architecture နှင့် Data/Program Synchronization

CPU နှင့် Memory တို့၏ အခြေခံဖွဲ့စည်းပုံကို နားလည်ပြီးပြီဆိုတော့၊ အခုအခန်းမှာ ကွန်ပျူတာရဲ့ အကြီးစား ဗိသုကာ (Conceptual Framework) အကြောင်းကို လေ့လာရပါမယ်။ ဤဗိသုကာသည် ခေတ်မီ ကွန်ပျူတာများ၏ အခြေခံအုတ်မြစ်ဖြစ်ပြီး၊ ပရိုဂရမ်နှင့် အချက်အလက်များကို မည်သို့ စီမံခန့်ခွဲပုံကို ဖော်ပြပေးပါသည်။

### Von Neumann Architecture

ကွန်ပျူတာများ၏ အများစုသည် **Von Neumann Architecture** ကို အခြေခံ၍ တည်ဆောက်ထားကြသည်။ ဤဗိသုကာ၏ အဓိက သဘောတရားမှာ **Program နှင့် Data နှစ်မျိုးလုံးကို Memory တစ်ခုတည်းတွင် အတူတကွ သိမ်းဆည်းထားခြင်း** ဖြစ်သည်။ ဤအယူအဆသည် ကွန်ပျူတာများအတွက် အလွန်အရေးကြီးသော ပြောင်းလဲမှုတစ်ခု ဖြစ်ခဲ့သည်။

ဤဗိသုကာ၏ အဓိပ္ပာယ်ကို နားလည်ရန်အတွက် Program နှင့် Data တို့ကို Memory ထဲတွင် မည်သို့ သိမ်းဆည်းထားပုံကို ထည့်သွင်းစဉ်းစားရပါမည်။

### Program Storage vs. Data Storage

Von Neumann Architecture တွင် Memory တစ်ခုတည်းကို အသုံးပြုသောကြောင့်၊ ထို Memory အတွင်းရှိ အချက်အလက်များသည် Instruction များ (ပရိုဂရမ်) နှင့် Data များ (အချက်အလက်များ) ဟူ၍ နှစ်မျိုး ကွဲပြားပါသည်။

**Program Storage (ပရိုဂရမ် သိမ်းဆည်းရာ):**
ဤနေရာတွင် သိမ်းဆည်းထားသော အချက်အလက်များသည် CPU ကို ဘယ်လိုလုပ်ဆောင်ရမလဲဆိုသည့် အမိန့်များ၊ Instruction များ၊ နှင့် Control Logic များ ဖြစ်သည်။ ဤအချက်အလက်များသည် ပရိုဂရမ်၏ အစီအစဉ်ကို ဖော်ပြသည်။

**Data Storage (အချက်အလက် သိမ်းဆည်းရာ):**
ဤနေရာတွင် သိမ်းဆည်းထားသော အချက်အလက်များသည် ပရိုဂရမ်များက လုပ်ဆောင်မည့် အကြောင်းအရာများ (ဥပမာ- ဂဏန်းများ၊ စာသားများ၊ ပုံရိပ်များ) ဖြစ်သည်။

ဤနှစ်မျိုးလုံးသည် Memory Address Space တွင် အတူတကွ တည်ရှိနေသည်။ Program သည် Data နှင့် မည်သို့ ဆက်စပ်နေသနည်းဆိုသည်မှာ၊ ပရိုဂရမ်တစ်ခုသည် အခြား Data များကို အသုံးပြုရန်အတွက် ထို Data များ၏ Memory Address များကို သိရှိရန် လိုအပ်ပါသည်။

### Instruction vs. Operand

ပရိုဂရမ်ကို နားလည်ရန်အတွက် Instruction နှင့် Operand တို့၏ ကွာခြားချက်ကို သိရှိရန် အရေးကြီးပါသည်။

**Instruction (အမိန့်):**
Instruction ဆိုသည်မှာ CPU ကို အတိအကျ လုပ်ဆောင်ခိုင်းသည့် အမိန့်တစ်ခု ဖြစ်သည်။ ဥပမာအားဖြင့်၊ "Add (ပေါင်း) လုပ်ပါ" သို့မဟုတ် "Move (ရွှေ့) လုပ်ပါ" စသည်ဖြင့် ဖြစ်သည်။ Instruction များသည် CPU ၏ Control Unit မှ ထိန်းချုပ်ပြီး Fetch-Decode-Execute Cycle တွင် အဓိက အခန်းကဏ္ဍမှ ပါဝင်သည်။

**Operand (အချက်အလက်):**
Operand ဆိုသည်မှာ Instruction များက လုပ်ဆောင်ရမည့် အချက်အလက်များ ဖြစ်သည်။ Instruction တစ်ခုသည် အမြဲတမ်း Operand တစ်ခု သို့မဟုတ် တစ်ခုထက်မက လိုအပ်ပါသည်။ ဥပမာအားဖြင့်၊ "Add (ပေါင်း) လုပ်ပါ" ဆိုသော Instruction တွင် "ဘယ်နှစ်ခုကို ပေါင်းမလဲ" ဆိုသော အချက်အလက် (Operand) သည် ပေါင်းမည့် ဂဏန်းများ ဖြစ်သည်။

Instruction နှင့် Operand တို့သည် ပူးပေါင်း၍ လုပ်ဆောင်သောအခါမှသာ ပရိုဂရမ်သည် မှန်ကန်သော အကျိုးရလဒ်ကို ရရှိနိုင်ပါသည်။

### Memory Addressing Concept

Von Neumann Architecture တွင် Program နှင့် Data များသည် အတူတကွ Memory ထဲတွင် ရှိနေသောကြောင့် Memory Addressing Concept သည် အလွန်အရေးကြီးလာပါသည်။ CPU သည် Instruction တစ်ခုကို Fetch လုပ်သောအခါ၊ ၎င်းသည် Memory Address ကို အသုံးပြု၍ ထိုလိပ်စာမှ Instruction ကို ရယူသည်။ ထို့နောက် ထို Instruction ထဲတွင် ပါရှိသော Operand များ၏ Address များကို ပြန်လည်ရှာဖွေပြီး ထိုလိပ်စာများမှ Data များကို ရယူကာ ALU တွင် တွက်ချက်မှုများ ပြုလုပ်စေသည်။

ဤနည်းအားဖြင့်၊ ပရိုဂရမ်သည် Data များအား မည်သို့ အသုံးပြုရမည်ကို Memory Address များဖြင့် ဆက်စပ်ထားခြင်း ဖြစ်သည်။ ဤအခြေခံဗိသုကာကို နားလည်ပြီဆိုတော့၊ C Language ၏ ပထမဆုံး အခြေခံအဆင့်ဖြစ်တဲ့ Data နှင့် Pointer များကို နားလည်ဖို့အတွက် အခြေခံအဆောက်အအုံကို ပိုမိုနက်ရှိုင်းစွာ လေ့လာရပါမယ်။

## 5. အနှစ်ချုပ်နှင့် နောက်ဆက်တွဲလမ်းကြောင်း (Conclusion and Next Steps)

ယနေ့ သင်ခန်းစာတွင် ကျွန်ုပ်တို့သည် ကွန်ပျူတာ၏ အတွင်းပိုင်းအခြေခံများဖြစ်သော CPU၊ Memory နှင့် ကွန်ပျူတာ ဗိသုကာ (Architecture) တို့ကို လေ့လာခဲ့ကြပါသည်။

### Chapter Summary

ကျွန်ုပ်တို့သည် CPU ကို ကွန်ပျူတာ၏ ဦးနှောက်အဖြစ်၊ ၎င်း၏ Fetch-Decode-Execute Cycle နှင့် Registers, ALU, PC ကဲ့သို့သော အတွင်းပိုင်း အစိတ်အပိုင်းများကို လေ့လာခဲ့ပါသည်။ ထို့အပြင် Memory System ၏ အခန်းကဏ္ဍကို RAM (ခေတ္တမှတ်ဉာဏ်) နှင့် ROM (အခြေခံပရိုဂရမ်) ကဲ့သို့သော အမျိုးအစားများ၊ Memory Address ၏ အဓိပ္ပာယ်နှင့် Memory Hierarchy (Cache, RAM, Storage) တို့ကို နားလည်ခဲ့ပါသည်။ နောက်ဆုံးအနေဖြင့်၊ Program နှင့် Data များကို Memory တစ်ခုတည်းတွင် သိမ်းဆည်းထားသည့် Von Neumann Architecture ၏ အကြီးစား သဘောတရားနှင့် Instruction နှင့် Operand တို့၏ ဆက်စပ်မှုကို ဆွေးနွေးခဲ့ပါသည်။

### Key Takeaways

Hardware အခြေခံများကို နားလည်ခြင်းသည် Software ကို နားလည်ရာတွင် မည်သို့ အထောက်အကူပြုသနည်းကို ကျွန်ုပ်တို့ တွေ့ရှိခဲ့ရပါသည်။ ပရိုဂရမ်တစ်ခုသည် ဘယ်လို အမိန့်ပေးပြီး၊ ထိုအမိန့်များက Hardware တွင် မည်သို့ အကောင်အထည်ဖော်ခံရသည်ကို နားလည်မှသာ၊ ကျွန်ုပ်တို့ရေးသားသော C Code များသည် ကွန်ပျူတာ၏ အတွင်းပိုင်းတွင် မည်သို့ အလုပ်လုပ်နေသည်ကို ပိုမိုနက်ရှိုင်းစွာ နားလည်နိုင်မည် ဖြစ်ပါသည်။ CPU နှင့် Memory တို့၏ ဆက်စပ်မှုကို နားလည်ခြင်းက Data Structures နှင့် Pointer များကဲ့သို့သော အဆင့်မြင့်သော Programming Concept များအတွက် အလွန်အရေးပါသော အခြေခံအုတ်မြစ် ဖြစ်ပါသည်။

### Look Ahead

ဤအခန်းသည် ကျွန်ုပ်တို့အတွက် အလွန်အရေးကြီးသော အခြေခံအုတ်မြစ်တစ်ခု ဖြစ်သည်။ နောက်အခန်းများတွင် ကျွန်ုပ်တို့သည် ဤ Hardware အခြေခံများကို အသုံးပြု၍ လက်တွေ့ C Programming ကို စတင်လေ့လာသွားမည် ဖြစ်သည်။ ကျွန်ုပ်တို့သည် Data Types၊ Variables များ၊ နှင့် Pointer များ၏ အခြေခံသဘောတရားများကို လေ့လာမည်ဖြစ်ပြီး၊ ဤအခြေခံများမှစ၍ Memory ထဲရှိ အချက်အလက်များကို ကိုင်တွယ်ပုံနှင့် Memory Address များကို ကိုယ်တိုင်စီမံခန့်ခွဲပုံတို့ကို နားလည်အောင် လေ့ကျင့်သွားကြပါမည်။ Hardware နှင့် Software တို့၏ အပြန်အလှန် ဆက်စပ်မှုကို နားလည်ထားခြင်းက ကျွန်ုပ်တို့၏ Programming ကျွမ်းကျင်မှုကို အရှိန်အဟုန်ဖြင့် မြှင့်တင်ပေးလိမ့်မည်ဟု ယုံကြည်ပါသည်။
