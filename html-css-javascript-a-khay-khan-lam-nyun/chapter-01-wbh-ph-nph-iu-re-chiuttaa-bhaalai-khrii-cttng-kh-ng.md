# Chapter 1: ဝဘ်ဖွံ့ဖြိုးရေးဆိုတာ ဘာလဲ — ခရီးစတင်ခြင်း

## 🌟 Opening Hook & Introduction Concept

သင်နေ့စဉ်သုံးနေတဲ့ Facebook, YouTube, Google တို့ဟာ ဘယ်လိုအရာတွေနဲ့ တည်ဆောက်ထားလဲ သိဖူးပါသလား? ကျွန်ုပ်တို့သည် အင်တာနက်ပေါ်တွင် အချိန်တိုင်းလိုလို အချက်အလက်တွေ၊ ပုံရိပ်တွေနဲ့ အပြန်အလှန်ဆက်သွယ်မှုတွေ ပေးနေတဲ့ ကမ္ဘာကြီးထဲမှာ နေထိုင်နေကြပါတယ်။ ဒီနေ့ကစပြီး အဲ့ဒီအရာတွေရဲ့ နောက်ကွယ်က လျှို့ဝှက်ချက်တွေကို ဖော်ထုတ်ပြီး၊ ကိုယ်တိုင် ဒီကမ္ဘာကြီးကို တည်ဆောက်နိုင်မယ့် ခရီးလမ်းကို စတင်ကြမယ်။ Web Development ဆိုတာ ဘာလဲ၊ ဘယ်လိုစတင်မလဲဆိုတဲ့ အခြေခံအချက်တွေကို ဒီအခန်းမှာ အတူတကွ လေ့လာသွားကြရအောင်။

Web Development ဆိုတာ ရိုးရှင်းလွယ်ကူရှင်းလင်းစွာ ဆိုရမယ်ဆိုရင်၊ အင်တာနက်ပေါ်မှာ ရှိတဲ့ ဝဘ်ဆိုက်တွေ၊ အက်ပ်လီကေးရှင်းတွေ၊ ဝဘ်ဝန်ဆောင်မှုတွေဟာ ဘယ်လိုအလုပ်လုပ်လဲ၊ ဘယ်လိုတည်ဆောက်လဲဆိုတဲ့ နည်းပညာပိုင်းဆိုင်ရာ အတတ်ပညာပဲ ဖြစ်ပါတယ်။ ဒါဟာ ရိုးရိုးစာရိုက်တာ သို့မဟုတ် ပုံဆွဲတာမျိုး မဟုတ်ဘဲ၊ ကွန်ပျူတာတွေနဲ့ ဆက်သွယ်ပြီး အသုံးပြုသူတွေအတွက် အသုံးဝင်တဲ့ အရာတွေကို ဖန်တီးပေးတဲ့ နည်းပညာနယ်ပယ်တစ်ခု ဖြစ်ပါတယ်။ ဘာကြောင့် ဒီအရာကို လေ့လာသင့်သလဲဆိုတော့၊ ဒီနေ့ခေတ်မှာ အချက်အလက်တွေဟာ အင်တာနက်ပေါ်မှာ အဓိကတည်ရှိနေပြီး၊ ကိုယ်တိုင် ဖန်တီးနိုင်တဲ့ စွမ်းရည်က အလွန်တန်ဖိုးရှိလာပြီ ဖြစ်ပါတယ်။ Web Development ကို လေ့လာခြင်းဟာ သင့်ကို ဒီနည်းပညာလောကရဲ့ အခြေခံကို နားလည်စေပြီး၊ ကိုယ်ပိုင် အရာတွေကို ဖန်တီးနိုင်မယ့် စိတ်အားထက်သန်မှုကို ပေးအပ်မှာ ဖြစ်ပါတယ်။

## 1. ဝဘ်ဖွံ့ဖြိုးရေးဆိုတာ ဘာလဲ? — အခြေခံသဘောတရားများ (What is Web Development? — Basic Concepts)

ဝဘ်ဆိုက်တွေ ဘယ်လိုအလုပ်လုပ်သလဲဆိုတဲ့ အခြေခံအကြောင်းအရာတွေကို နားလည်မှသာ နောက်ဆက်တွဲ သင်ခန်းစာတွေမှာ ဘာကြောင့် HTML, CSS, JavaScript လို အရာတွေကို လိုအပ်တာလဲဆိုတာ ပိုပြီး နားလည်လာပါလိမ့်မယ်။

### Web ဆိုတာ ဘာလဲ? (အင်တာနက်နဲ့ ဝဘ်ဆိုက်ရဲ့ အခြေခံသဘောတရား)

Web ဆိုတာ အင်တာနက် (Internet) ပေါ်မှာ တည်ရှိတဲ့ အချက်အလက်တွေနဲ့ အရင်းအမြစ်တွေကို ဖလှယ်နိုင်တဲ့ စနစ်တစ်ခု ဖြစ်ပါတယ်။ ကျွန်ုပ်တို့ အများစု သိကျွမ်းနေတဲ့ ဝဘ်ဆိုက်ဆိုတာက ဒီအင်တာနက်ပေါ်မှာ ရှိတဲ့ အချက်အလက်တွေကို စနစ်တကျ စီစဉ်ပြီး၊ Web Browser (Chrome, Firefox စသည်) ဆိုတဲ့ ဆော့ဖ်ဝဲလ်တွေကနေတစ်ဆင့် ကျွန်ုပ်တို့ မြင်တွေ့နိုင်အောင် ပြသပေးတဲ့ အရာတွေပဲ ဖြစ်ပါတယ်။

Web Browser ရဲ့ အခန်းကဏ္ဍက အလွန်အရေးကြီးပါတယ်။ Browser ဟာ ကျွန်ုပ်တို့ရဲ့ Computer မှာရှိတဲ့ Code တွေကို ဖတ်ပြီး၊ အဲ့ဒီ Code တွေကို စနစ်တကျ ပြောင်းလဲပြီး၊ ကျွန်ုပ်တို့ မြင်တွေ့နိုင်တဲ့ ပုံရိပ်၊ စာသားနဲ့ အပြန်အလှန်တုံ့ပြန်မှုတွေအဖြစ် ပြောင်းလဲပေးတဲ့ အဓိက ကိရိယာပဲ ဖြစ်ပါတယ်။ URL (Website Address) ဆိုတာကတော့ ဒီWebဆိုက်တစ်ခုရဲ့ ထူးခြားတဲ့ လိပ်စာပဲ ဖြစ်ပါတယ်။ အင်တာနက်ပေါ်မှာရှိတဲ့ အချက်အလက်တွေဟာ ဒီURL ကို အသုံးပြုပြီး Server တစ်ခုကနေ Client (Browser) ဆီကို ပေးပို့တာ ဖြစ်ပါတယ်။

### Web Site တွေ ဘယ်လိုအလုပ်လုပ်သလဲ? (Client-Server Model အခြေခံ)

ဝဘ်ဆိုက်တွေဟာ Client-Server Model ဆိုတဲ့ အခြေခံသဘောတရားပေါ်မှာ အခြေခံပြီး အလုပ်လုပ်ကြပါတယ်။

**Client (Browser):** ဒါဟာ အသုံးပြုသူရဲ့ ဘက်ကို ဆိုလိုပါတယ်။ ကျွန်ုပ်တို့ရဲ့ ကွန်ပျူတာပေါ်မှာရှိတဲ့ Web Browser (ဥပမာ- Chrome) ဟာ Client ဖြစ်ပါတယ်။ Client က Server ကို တစ်ခုခုကို တောင်းဆိုတဲ့အခါ (Request) အချက်အလက်တွေ ရရှိဖို့အတွက် ဆက်သွယ်မှု ပြုလုပ်ပါတယ်။

**Server (Data Storage):** ဒါကတော့ နောက်ကွယ်မှာ ရှိနေတဲ့ အစိတ်အပိုင်း ဖြစ်ပါတယ်။ Server ဆိုတာက အချက်အလက်တွေ၊ ဖိုင်တွေနဲ့ ဝဘ်ဆိုက်ရဲ့ Logic တွေကို သိမ်းဆည်းထားတဲ့ ကွန်ပျူတာစနစ် (သို့မဟုတ် ဆာဗာ) ဖြစ်ပါတယ်။

**Request နှင့် Response (တောင်းဆိုမှုနှင့် တုံ့ပြန်မှု):** ဒီနှစ်ခုဟာ Client နဲ့ Server ကြားက ဆက်သွယ်မှုရဲ့ အဓိကပုံစံပဲ ဖြစ်ပါတယ်။ Client က Server ကို "ဒီစာမျက်နှာကို ပေးပါ" လို့ Request ပို့လိုက်တဲ့အခါ၊ Server ဟာ အဲ့ဒီတောင်းဆိုမှုကို လက်ခံပြီး လိုအပ်တဲ့ အချက်အလက် (HTML ဖိုင် စတာတွေ) ကို ပြန်လည် ပေးပို့လိုက်တာကို Response လို့ ခေါ်ပါတယ်။ ဒီ Request/Response စနစ်ကြောင့် ကျွန်ုပ်တို့ဟာ အင်တာနက်ပေါ်မှာ အချက်အလက်တွေ ဖလှယ်နိုင်ပြီး ဝဘ်ဆိုက်တွေကို ကြည့်ရှုနိုင်ခြင်း ဖြစ်ပေါ်လာတာ ဖြစ်ပါတယ်။

## 2. Frontend နှင့် Backend ၏ ကွာခြားချက် (Frontend vs. Backend: The Difference)

ဝဘ်ဆိုက်တစ်ခုကို တည်ဆောက်တဲ့အခါ အစိတ်အပိုင်း နှစ်ပိုင်း ရှိပါတယ်။ ဒီနှစ်ပိုင်းကို နားလည်မှသာ HTML, CSS နဲ့ JavaScript တို့ရဲ့ အခန်းကဏ္ဍတွေကို ပိုပြီး သေချာစွာ ခွဲခြားနိုင်မှာ ဖြစ်ပါတယ်။

### Frontend (ဖောက်ထွင်းမြင်ရသော အပိုင်း)

Frontend ဆိုတာ အသုံးပြုသူ မြင်တွေ့ရတဲ့ အရာတွေနဲ့ တိုက်ရိုက်ထိတွေ့ဆက်ဆံတဲ့ အပိုင်း ဖြစ်ပါတယ်။ ဒါဟာ ဝဘ်ဆိုက်ရဲ့ အပြင်အဆင်၊ အလှအပ၊ Layout (ပုံစံ) နဲ့ အပြန်အလှန်တုံ့ပြန်မှု (Interactivity) တွေကို ပုံဖော်ပေးတာ ဖြစ်ပါတယ်။ အသုံးပြုသူတွေက Browser မှာ မြင်နေရတဲ့ အရာအားလုံးဟာ Frontend အပိုင်းပဲ ဖြစ်ပါတယ်။ Frontend ကို တည်ဆောက်ဖို့အတွက် အဓိက အသုံးပြုရမယ့် နည်းပညာတွေကတော့ HTML (Structure), CSS (Style), နဲ့ JavaScript (Behavior) တို့ ဖြစ်ပါတယ်။

### Backend (နောက်ကွယ် အပိုင်း)

Backend ဆိုတာ အသုံးပြုသူ မမြင်ရတဲ့ နောက်ကွယ်မှာ ဖြစ်ပျက်နေတဲ့ လုပ်ဆောင်ချက်တွေ၊ Logic တွေနဲ့ အချက်အလက်တွေကို စီမံခန့်ခွဲတဲ့ အပိုင်း ဖြစ်ပါတယ်။ ဥပမာအားဖြင့်၊ အသုံးပြုသူရဲ့ Login အချက်အလက်တွေ သိမ်းဆည်းတာ၊ ဒေတာဘေ့စ်ထဲကနေ အချက်အလက်တွေ ထုတ်ယူတာ၊ လုံခြုံရေး စစ်ဆေးတာ စတဲ့ လုပ်ဆောင်ချက်တွေအားလုံးဟာ Backend ရဲ့ အခန်းကဏ္ဍ ဖြစ်ပါတယ်။ Backend ကို အဓိကထားပြီး တည်ဆောက်ရတဲ့အတွက် Server-side programming ဘာသာစကားတွေ (ဥပမာ- Python, PHP, Node.js စသည်) တွေကို အသုံးပြုလေ့ရှိပါတယ်။

### နှိုင်းယှဉ်ချက်: အိမ်ဆောက်ခြင်းနှင့် နှိုင်းယှဉ်ချက်

ဒီနှစ်ပိုင်းကို အိမ်ဆောက်ခြင်းနဲ့ နှိုင်းယှဉ်ကြည့်ရင် ပိုပြီး နားလည်လွယ်ပါလိမ့်မယ်။

**Frontend (အပြင်အဆင်):** ဒါဟာ အိမ်ရဲ့ အပြင်ဘက် အဆောက်အအုံ၊ ပြတင်းပေါက်တွေ၊ အလှဆင်မှုတွေ၊ အတွင်းခန်းတွေရဲ့ ဒီဇိုင်းပုံစံတွေနဲ့ အလှအပကို ဆိုလိုပါတယ်။ ဒါတွေက အသုံးပြုသူတွေ မြင်တွေ့ရတဲ့ အရာတွေ ဖြစ်ပါတယ်။

**Backend (အတွင်းပိုင်းအခြေခံ):** ဒါဟာ အိမ်ရဲ့ အုတ်မြစ်၊ အဆောက်အအုံရဲ့ အခြေခံဖွဲ့စည်းပုံ၊ လျှပ်စစ်မီးတွေ၊ ရေပိုက်တွေ စတဲ့ အဆောက်အအုံရဲ့ အတွင်းပိုင်းမှာ ဖြစ်ပျက်နေတဲ့ စနစ်တွေနဲ့ အခြေခံအဆောက်အအုံတွေကို ဆိုလိုပါတယ်။ Frontend ဟာ Backend ရဲ့ အပေါ်မှာ တည်ဆောက်ပြီး Backend ကနေ လိုအပ်တဲ့ အချက်အလက်တွေကို Frontend ကို ပေးပို့ပေးတာ ဖြစ်ပါတယ်။

## 3. Web Development ၏ အစိတ်အပိုင်းများ — ပူးပေါင်းဆောင်ရွက်မှု (The Components — Collaboration)

Frontend နဲ့ Backend က ဘာတွေလဲဆိုတာ သိပြီဆိုတော့၊ ဒီအရာတွေကို ဘယ်လို ပေါင်းစပ်ပြီး အသက်ဝင်တဲ့ ဝဘ်ဆိုက်တစ်ခု ဖြစ်လာအောင် လုပ်ဆောင်ရမလဲဆိုတဲ့ အဆင့်ကို ဆက်ပြီး လေ့လာကြရအောင်။ ဒီအရာတွေဟာ တစ်ခုနဲ့တစ်ခု အပြန်အလှန် မှီခိုပြီး အလုပ်လုပ်ကြပါတယ်။

### HTML (Structure): ဝဘ်စာမျက်နှာရဲ့ အ뼈 (Skeleton) ကို ဖန်တီးခြင်း

Web Development ရဲ့ အခြေခံအကျဆုံး အစိတ်အပိုင်းကတော့ HTML (HyperText Markup Language) ဖြစ်ပါတယ်။ HTML ဟာ ဝဘ်စာမျက်နှာရဲ့ အ뼈 (Skeleton) ကို ဖန်တီးပေးတာ ဖြစ်ပါတယ်။ HTML ကို အသုံးပြုပြီး စာသားတွေ၊ ပုံတွေ၊ လင့်ခ်တွေ၊ စာရင်းတွေ စတဲ့ အကြောင်းအရာတွေကို စနစ်တကျ စီစဉ်ပေးပါတယ်။ HTML ဟာ ဝဘ်စာမျက်နှာရဲ့ အကြောင်းအရာကို ဘယ်လို စီစဉ်မလဲဆိုတဲ့ အခြေခံပုံစံကို သတ်မှတ်ပေးတာ ဖြစ်ပါတယ်။

### CSS (Style): အ뼈ကို လှပအောင် ပုံဖော်ခြင်း (အလှဆင်ခြင်း)

HTML က အ뼈ကို ဖန်တီးပေးလိုက်ပြီဆိုရင်၊ အဲ့ဒီအ뼈ကို လူတွေ မြင်တွေ့ရအောင် လှပအောင် ပုံဖော်ဖို့အတွက် CSS (Cascading Style Sheets) ကို အသုံးပြုပါတယ်။ CSS ဟာ HTML မှာရှိတဲ့ အကြောင်းအရာတွေကို အရောင်၊ ဖောင့်ပုံစံ၊ Layout (နေရာချထားမှု)၊ အနားသတ်တွေနဲ့ လှပအောင် ပုံဖော်ပေးတာ ဖြစ်ပါတယ်။ CSS မရှိရင် ဝဘ်စာမျက်နှာဟာ ရိုးရှင်းပြီး စာသားတွေပဲ အစီအစဉ်အတိုင်း ရေးထားတဲ့ စာရွက်တစ်ရွက်လို ဖြစ်နေပါလိမ့်မယ်။

### JavaScript (Behavior): ဝဘ်ဆိုက်ကို အသက်ဝင်အောင် လုပ်ဆောင်ခြင်း (အပြန်အလှန်တုံ့ပြန်မှုများ)

HTML က အ뼈၊ CSS က အလှအပကို ဖန်တီးပေးလိုက်ပြီဆိုတော့၊ JavaScript (JS) ဟာ ဝဘ်ဆိုက်ကို အသက်ဝင်အောင် လုပ်ဆောင်ပေးတဲ့ အခန်းကဏ္ဍကို တာဝန်ယူပါတယ်။ JavaScript ဟာ ဝဘ်ဆိုက်ကို အပြန်အလှန်တုံ့ပြန်မှုတွေ (Interactivity) ပေးနိုင်အောင် လုပ်ဆောင်ပေးပါတယ်။ ဥပမာအားဖြင့်၊ ခလုတ်တစ်ခုကို နှိပ်လိုက်တဲ့အခါ ပုံပြောင်းတာ၊ မီနူးတွေ ပေါ်လာတာ၊ ဖောင်တွေမှာ မှန်ကန်မှုရှိမရှိ စစ်ဆေးတာ စတဲ့ လုပ်ဆောင်ချက်တွေဟာ JavaScript ရဲ့ အခန်းကဏ္ဍ ဖြစ်ပါတယ်။

### သုံးခု ပူးပေါင်းပုံ: အိမ်ဆောက်ခြင်းနှင့် နှိုင်းယှဉ်ချက်

ဒီသုံးခုကို ပေါင်းစပ်လိုက်တဲ့အခါ အလွန်အစွမ်းထက်တဲ့ ပုံစံတစ်ခု ဖြစ်ပေါ်လာပါတယ်။

*   **HTML (အုတ်):** ဝဘ်ဆိုက်ရဲ့ အခြေခံအဆောက်အအုံကို တည်ဆောက်ပေးပါတယ်။
*   **CSS (ဆေး):** အဲ့ဒီအုတ်တွေကို လှပပြီး စနစ်တကျ ပုံဖော်ပေးပါတယ်။
*   **JavaScript (လျှပ်စစ်မီး):** အဲ့ဒီအဆောက်အအုံကို အသက်ဝင်စေပြီး အပြန်အလှန်တုံ့ပြန်မှုတွေ ပေးနိုင်အောင် လုပ်ဆောင်ပေးပါတယ်။

ဒီသုံးခု ပူးပေါင်းဆောင်ရွက်မှသာ ကျွန်ုပ်တို့ဟာ အသုံးပြုသူတွေနဲ့ အပြန်အလှန် ဆက်သွယ်နိုင်တဲ့၊ အချက်အလက်တွေ ပြသနိုင်တဲ့၊ လုပ်ဆောင်နိုင်တဲ့ အသက်ဝင်တဲ့ ဝဘ်ဆိုက်တစ်ခုကို တည်ဆောက်နိုင်မှာ ဖြစ်ပါတယ်။

## 4. စတင်ရန် လိုအပ်သော ကိရိယာများ (Tools for Getting Started)

အခြေခံသဘောတရားတွေနဲ့ ပူးပေါင်းဆောင်ရွက်မှုတွေ ဘယ်လိုအလုပ်လုပ်သလဲဆိုတာ နားလည်ပြီဆိုတော့၊ လက်တွေ့မှာ ကုဒ်တွေ ရေးသားဖို့အတွက် ဘယ်လို Software တွေ လိုအပ်သလဲ၊ ဘယ်လို Environment ကို ပြင်ဆင်ရမလဲဆိုတာကို ဆွေးနွေးကြရအောင်။

### Text Editor ရွေးချယ်ခြင်း: VS Code (Visual Studio Code)

Web Development အတွက် Code ရေးသားတဲ့အခါ အရေးကြီးဆုံးကတော့ ကောင်းမွန်တဲ့ Text Editor တစ်ခုကို ရွေးချယ်ဖို့ ဖြစ်ပါတယ်။ အများစုကတော့ VS Code (Visual Studio Code) ကို အသုံးပြုကြပါတယ်။ VS Code ဟာ အခမဲ့ဖြစ်ပြီး၊ HTML, CSS, JavaScript လို ဘာသာစကားတွေအတွက် အကောင်းဆုံး ပံ့ပိုးပေးတဲ့ Feature တွေ၊ Code ကို အလွယ်တကူ ရေးသားနိုင်ဖို့ အထောက်အကူပြုတဲ့ အတိုကောက်များ (Syntax Highlighting) နဲ့ Debugging လုပ်နိုင်တဲ့ စွမ်းဆောင်ရည်တွေ ပေးစွမ်းနိုင်ပါတယ်။ ဒါကြောင့် Web Developer တစ်ယောက်အနေနဲ့ VS Code ကို အသုံးပြုဖို့ အထူးအကြံပြုလိုပါတယ်။

### Web Browser ပြင်ဆင်ခြင်း: Developer Tools (Inspect Element)

Code တွေ ရေးသားပြီးတာနဲ့ အဲ့ဒီ Code တွေ မှန်ကန်စွာ အလုပ်လုပ်ရဲ့လား၊ ဘယ်နေရာမှာ အမှားရှိလဲဆိုတာကို စစ်ဆေးဖို့အတွက် Web Browser ရဲ့ Developer Tools (သို့မဟုတ် Inspect Element) ကို အခြေခံအသုံးပြုပုံကို လေ့လာဖို့ လိုအပ်ပါတယ်။ ဒီ Tools တွေကနေတစ်ဆင့် ကျွန်ုပ်တို့ဟာ Browser ထဲမှာ တိုက်ရိုက် Code တွေကို စစ်ဆေးနိုင်ပြီး၊ Website ရဲ့ အစိတ်အပိုင်းတစ်ခုချင်းစီရဲ့ Style နဲ့ Structure ကို ဘယ်လို ပြောင်းလဲလဲဆိုတာကို Real-time မှာ မြင်တွေ့နိုင်ပါတယ်။ ဒါဟာ Debugging လုပ်ရာမှာ အလွန်အရေးကြီးတဲ့ စွမ်းဆောင်ရည် ဖြစ်ပါတယ်။

### အခြေခံ Setup: လိုအပ်သော Software များ တင်သွင်းခြင်း

စတင်လေ့လာဖို့အတွက် အခြေခံအားဖြင့် ကွန်ပျူတာမှာ Text Editor နဲ့ Web Browser တစ်ခုသာ လိုအပ်ပါတယ်။ ဒါပေမယ့် နောက်ပိုင်းမှာ ပိုမိုအဆင့်မြင့်တဲ့ လုပ်ဆောင်မှုတွေအတွက် Version Control (Git) လိုမျိုး ကိရိယာတွေ၊ Code ကို အလွယ်တကူ စီမံခန့်ခွဲနိုင်မယ့် Environment တွေ (ဥပမာ- Node.js) ကိုပါ တင်သွင်းဖို့ လိုအပ်လာပါလိမ့်မယ်။ ဒါပေမယ့် ဒီခရီးရဲ့ အစမှာတော့ အပေါ်က နှစ်ခုကို အခြေခံအနေနဲ့ အသုံးပြုနိုင်ပြီ ဖြစ်ပါတယ်။

## 5. ပထမဆုံး လက်တွေ့ အတွေ့အကြုံ (First Practical Experience)

အခြေခံသဘောတရားတွေနဲ့ ကိရိယာတွေ ပြင်ဆင်ပြီးပြီဆိုတော့၊ အခုမှ လက်တွေ့မှာ ပထမဆုံး HTML ဖိုင်တစ်ခုကို ဖန်တီးပြီး Browser မှာ မြင်ရအောင် လုပ်ကြည့်ကြမယ်။ လက်တွေ့အတွေ့အကြုံကတော့ အကောင်းဆုံး သင်ယူမှု နည်းလမ်းပဲ ဖြစ်ပါတယ်။

### HTML ဖိုင် ဖန်တီးပုံ: `.html` extension ရဲ့ အရေးပါပုံ

Web Development ရဲ့ အစမှာ ကျွန်ုပ်တို့ရဲ့ ပထမဆုံး ဖိုင်ကတော့ HTML ဖိုင် ဖြစ်ပါတယ်။ ဒီဖိုင်တွေကို ဖန်တီးတဲ့အခါ ဖိုင်အမည်ကို `.html` extension နဲ့ အဆုံးသတ်ပေးဖို့ အလွန်အရေးကြီးပါတယ်။ ဒီ extension ဟာ ကွန်ပျူတာနဲ့ Browser တွေဟာ အဲ့ဒီဖိုင်ကို HTML စာမျက်နှာအဖြစ် မှတ်ယူပြီး မှန်ကန်စွာ ဖတ်ရှုနိုင်ဖို့ အထောက်အကူပြုပါတယ်။

### Code ရေးသားခြင်း: အခြေခံ HTML Tags များ

ယခုအခါ ကျွန်ုပ်တို့ဟာ VS Code ကို ဖွင့်ပြီး အခြေခံ HTML ဖိုင်တစ်ခုကို ဖန်တီးကြပါမယ်။ အဲ့ဒီဖိုင်ထဲမှာ ဝဘ်စာမျက်နှာရဲ့ အခြေခံဖွဲ့စည်းပုံကို ဖော်ပြပေးတဲ့ အဓိက Tags တွေကို ရေးသားကြည့်ရအောင်။

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Webpage</title>
</head>
<body>
    <h1>Hello, Web Development World!</h1>
    <p>This is my very first web page.</p>
</body>
</html>
```

အပေါ်က Code မှာ ပါဝင်တဲ့ အခြေခံ Tags တွေကို နားလည်အောင် ကြည့်ရအောင်။

*   `<!DOCTYPE html>`: ဒါက HTML5 ကို အသုံးပြုနေကြောင်း Browser ကို အသိပေးတဲ့ အမိန့်စာ ဖြစ်ပါတယ်။
*   `<html lang="en">`: ဒါဟာ စာမျက်နှာတစ်ခုလုံးရဲ့ အဓိက အစမှတ် ဖြစ်ပြီး၊ ဘာသာစကားကို သတ်မှတ်ပေးပါတယ်။
*   `<head>`: ဒီအပိုင်းက စာမျက်နှာရဲ့ အချက်အလက်တွေ (Metadata) ကို ထည့်သွင်းပေးတဲ့ နေရာ ဖြစ်ပါတယ်။ ဥပမာ- စာမျက်နှာရဲ့ Title ကို သတ်မှတ်တာမျိုး ဖြစ်ပါတယ်။
*   `<body>`: ဒီအပိုင်းကတော့ အသုံးပြုသူတွေ မြင်တွေ့ရမယ့် အကြောင်းအရာတွေအားလုံးကို ထည့်သွင်းပေးတဲ့ နေရာ ဖြစ်ပါတယ်။ ကျွန်ုပ်တို့ရဲ့ စာသားတွေ၊ ပုံတွေ၊ လင့်ခ်တွေအားလုံးဟာ ဒီ `<body>` အတွင်းမှာ ရှိရပါမယ်။

### Browser တွင် ဖွင့်ကြည့်ခြင်း: ရလဒ်ကို စစ်ဆေးခြင်း

Code ရေးသားပြီးသွားပြီဆိုရင်၊ အဲ့ဒီ `.html` ဖိုင်ကို သိမ်းဆည်းပြီး Browser နဲ့ ဖွင့်ကြည့်ရမှာ ဖြစ်ပါတယ်။ ဖိုင်ကို Right-click လုပ်ပြီး "Open with" ကို ရွေးပြီး သင့်ရဲ့ Web Browser (Chrome, Firefox စသည်) နဲ့ ဖွင့်လိုက်ပါ။

အဲ့ဒီအခါမှာ သင်ရေးသားခဲ့တဲ့ `<h1>` ခေါင်းစဉ်နဲ့ စာသားတွေဟာ Browser မှာ မြင်တွေ့ရတဲ့ ပုံစံအတိုင်း ပေါ်လာပါလိမ့်မယ်။ ဒါဟာ သင်ရေးသားခဲ့တဲ့ Code က ဘယ်လို အလုပ်လုပ်တယ်ဆိုတာကို လက်တွေ့မှာ မြင်တွေ့ရတဲ့ ပထမဆုံး အတွေ့အကြုံပဲ ဖြစ်ပါတယ်။ ဒီလိုမျိုး လက်တွေ့လုပ်ဆောင်မှုတွေက သင်ယူမှုကို အရှိန်အဟုန်မြှင့်ပေးပြီး သင်ဟာ ကိုယ်တိုင် ဖန်တီးနိုင်တဲ့ စွမ်းရည်ကို တိုးတက်စေမှာ ဖြစ်ပါတယ်။

## 📝 Chapter Summary & Next Steps

ဒီအခန်းမှာ Web Development ဆိုတာ ဘာလဲဆိုတာကို အခြေခံသဘောတရားတွေ၊ Web ဆိုတာ ဘယ်လိုအလုပ်လုပ်သလဲဆိုတဲ့ အခြေခံအကြောင်းအရာတွေကို နားလည်ခဲ့ပြီးပြီ ဖြစ်ပါတယ်။ Frontend နဲ့ Backend ရဲ့ ကွာခြားချက်ကို သိရှိခဲ့ပြီး၊ HTML, CSS, JavaScript တို့ဟာ ဝဘ်ဆိုက်တစ်ခုကို အသက်ဝင်စေဖို့ ဘယ်လို ပူးပေါင်းဆောင်ရွက်လဲဆိုတာကို အိမ်ဆောက်ခြင်းနဲ့ နှိုင်းယှဉ်ပြီး ရှင်းလင်းစွာ ဖော်ပြခဲ့ပါတယ်။ ထို့အပြင် Code ရေးသားဖို့အတွက် လိုအပ်တဲ့ ကိရိယာတွေ (VS Code, Browser Developer Tools) နဲ့ ပထမဆုံး HTML ဖိုင်တစ်ခုကို ဖန်တီးပြီး Browser မှာ မြင်ရအောင် လုပ်ကြည့်ခြင်းဖြင့် လက်တွေ့အတွေ့အကြုံကို ရရှိခဲ့ပါတယ်။

နောက်အခန်းမှာတော့ ဒီအခြေခံအုတ်မြစ်တွေပေါ်မှာ အခြေခံကျကျ ဆက်လက်တည်ဆောက်သွားမှာ ဖြစ်ပါတယ်။ နောက်အခန်းမှာ HTML ကို အသေးစိတ် လေ့လာပြီး ဝဘ်ဆိုက်ရဲ့ အခြေခံ အဆောက်အအုံကို တည်ဆောက်မယ့် အဆင့်ကို စတင်ပါမယ်။
