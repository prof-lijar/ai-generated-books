# Chapter 12: JavaScript Operator များနှင့် Control Flow

## 🚀 1. Introduction: JavaScript ၏ မောင်းနှင်အား (The Engine of JavaScript)

မည်သို့ဖြင့် JavaScript သည် ရိုးရှင်းသော စာကြောင်းများမှ အမှန်တကယ် ဆုံးဖြတ်ချက်များ ချမှတ်နိုင်ပြီး၊ အလုပ်များကို အလိုအလျောက် လုပ်ဆောင်နိုင်သော စွမ်းအားကို ရရှိလာသနည်း။ ပရိုဂရမ်ရေးသားခြင်း၏ အခြေခံသည် အချက်အလက်များကို စီမံခန့်ခွဲခြင်းနှင့် ထိုအချက်အလက်များအပေါ် အခြေခံ၍ ဆုံးဖြတ်ချက်များ ချမှတ်ခြင်းပင် ဖြစ်သည်။ JavaScript သည် ထိုအရာများကို လုပ်ဆောင်နိုင်သော စွမ်းအားကို ပေးစွမ်းသည်။ ရိုးရှင်းသော စာကြောင်းများ၊ သင်္ချာတွက်ချက်မှုများ၊ နှင့် အခြေအနေအလိုက် လုပ်ဆောင်မှုများဖြင့် ကျွန်ုပ်တို့သည် စာမျက်နှာများ (Webpages) ကို အသက်ဝင်စေပြီး၊ အသုံးပြုသူများ၏ အပြန်အလှန်တုံ့ပြန်မှုအပေါ် မူတည်၍ ပြောင်းလဲနေသော အတွေ့အကြုံများကို ဖန်တီးနိုင်သည်။

ဤအခန်းတွင် သင်သည် JavaScript ၏ အခြေခံတွက်ချက်မှုများ (Operators) နှင့် ပရိုဂရမ်၏ လမ်းကြောင်း (Control Flow) ကို မည်သို့အသုံးပြုရမည်ကို လေ့လာရမည် ဖြစ်သည်။ Operator များသည် သင်၏ ဒေတာများကို ပြုပြင်ပြောင်းလဲရန်၊ နှိုင်းယှဉ်ရန်နှင့် ပေါင်းစပ်ရန် ကူညီပေးသည်။ Control Flow များသည် ပရိုဂရမ်၏ အစီအစဉ်ကို ထိန်းချုပ်ပြီး၊ အခြေအနေများပေါ်မူတည်၍ မည်သည့်ကုဒ်များကို မည်သည့်အချိန်တွင် အက निष्ဆောင် (Execute) လုပ်ဆောင်ရမည်ကို ဆုံးဖြတ်ပေးသည်။ ဤကျွမ်းကျင်မှုနှစ်ခုကို ပေါင်းစပ်လိုက်သောအခါ သင်သည် စွမ်းဆောင်ရည်မြင့်မားသော၊ တုံ့ပြန်မှုရှိသော (Responsive) နှင့် ရှုပ်ထွေးသော လုပ်ဆောင်ချက်များကို ဖန်တီးနိုင်လာမည် ဖြစ်သည်။

ယခင်အခန်းများတွင် သင်သည် ဒေတာ (Data) ကို သိရှိခဲ့ပြီ ဖြစ်သည်။ ယခုအခါ ထိုဒေတာများကို မည်သို့ **တွက်ချက်**၊ **နှိုင်းယှဉ်** နှင့် **ဆုံးဖြတ်** ချနိုင်ပုံကို လေ့လာကြပါမည်။ ဤအခန်းသည် သင်၏ JavaScript ကျွမ်းကျင်မှုကို အခြေခံအုတ်မြစ်များပေါ်တွင် တည်ဆောက်ပေးမည့် အရေးကြီးဆုံး အခန်းတစ်ခန်း ဖြစ်ပါသည်။

## ⚙️ 2. Arithmetic နှင့် Assignment Operators: ဂဏန်းများနှင့် တန်ဖိုးများအား ကိုင်တွယ်ခြင်း (Handling Numbers and Values)

ပရိုဂရမ်ရေးသားရာတွင် အခြေခံအချက်အလက်များကို ကိုင်တွယ်နိုင်ရန်အတွက် ဂဏန်းများနှင့် တန်ဖိုးများကို ကိုင်တွယ်တတ်ရန် လိုအပ်သည်။ ဤလုပ်ငန်းစဉ်အတွက် ကျွန်ုပ်တို့သည် သင်္ချာတွက်ချက်မှုများနှင့် တန်ဖိုးများအား သတ်မှတ်ခြင်းဆိုင်ရာ အခြေခံများကို လေ့လာရမည် ဖြစ်သည်။

### Arithmetic Operators: သင်္ချာတွက်ချက်မှုများ

JavaScript တွင် အခြေခံ သင်္ချာတွက်ချက်မှုများကို ပြုလုပ်ရန်အတွက် အထူးသင်္ကေတများ (Operators) ကို အသုံးပြုသည်။ ဤသင်္ကေတများသည် ပုံမှန်အားဖြင့် သင်ကျက်မှတ်ထားသော သင်္ချာအခြေခံများနှင့် တူညီသော်လည်း၊ JavaScript တွင် ၎င်းတို့ကို အသုံးပြုပုံသည် အလွန်တိကျသည်။

*   **Addition (`+`):** နှစ်ခုသော ဂဏန်းကို ပေါင်းခြင်း။ ဥပမာ- `5 + 3` သည် `8` ဖြစ်သည်။
*   **Subtraction (`-`):** တစ်ခုမှ နောက်တစ်ခုကို နုတ်ခြင်း။ ဥပမာ- `10 - 4` သည် `6` ဖြစ်သည်။
*   **Multiplication (`*`):** ဂဏန်းနှစ်ခုကို မြှောက်ခြင်း။ ဥပမာ- `6 * 7` သည် `42` ဖြစ်သည်။
*   **Division (`/`):** တစ်ခုကို နောက်တစ်ခုဖြင့် စားခြင်း။ ဥပမာ- `20 / 5` သည် `4` ဖြစ်သည်။
*   **Modulus (`%`):** အစားထိုး (Remainder) ကို ရှာဖွေခြင်း။ ဥပမာ- `10 % 3` သည် `1` ဖြစ်သည်။ (၁၀ ကို ၃ ဖြင့် စားသောအခါ အကြွင်းသည် ၁ ဖြစ်သည်။)

ဤသင်္ချာတွက်ချက်မှုများသည် ဂဏန်းများအပေါ် အခြေခံသော အခြေခံလုပ်ဆောင်မှုများ ဖြစ်သည်။ ထို့အပြင်၊ အမှားအယွင်းများနှင့် အထူးမှတ်သားစရာများ ရှိသည်။ ဥပမာအားဖြင့်၊ အပြည့်အဝ စားခြင်း (Integer Division) နှင့် Floating-point ဂဏန်းများ (Decimal Numbers) ကို ခွဲခြားသိရှိရန် အရေးကြီးသည်။

### Assignment Operators: တန်ဖိုးများအား သတ်မှတ်ခြင်း

ဂဏန်းများကို တွက်ချက်ရုံသာမက၊ ထိုတွက်ချက်မှုများ၏ ရလဒ်များကို သတ်မှတ်ထားသော ভેરအမည် (Variable) များတွင် သိမ်းဆည်းထားရန် လိုအပ်သည်။ ဤလုပ်ငန်းစဉ်အတွက် Assignment Operators ကို အသုံးပြုသည်။

*   **Single Assignment (`=`):** တန်ဖိုးတစ်ခုကို ভેરအမည်ထဲသို့ တိုက်ရိုက်သတ်မှတ်ခြင်း။ ဥပမာ- `let score = 100;`
*   **Compound Assignment Operators:** ဤအရာများသည် သင်္ချာတွက်ချက်မှုများနှင့် တန်ဖိုးသတ်မှတ်ခြင်းကို တစ်ပြိုင်နက်တည်း လုပ်ဆောင်ပေးသည်။ ၎င်းတို့သည် ကုဒ်ကို ပိုမိုတိုတောင်းစေပြီး ထိရောက်စေသည်။
    *   `+=`: ပေါင်းပြီး သတ်မှတ်ခြင်း။ ဥပမာ- `score += 5;` သည် `score = score + 5` နှင့် တူညီသည်။
    *   `-=` : နုတ်ပြီး သတ်မှတ်ခြင်း။ ဥပမာ- `score -= 2;` သည် `score = score - 2` နှင့် တူညီသည်။
    *   `*=` : မြှောက်ပြီး သတ်မှတ်ခြင်း။ ဥပမာ- `score *= 3;` သည် `score = score * 3` နှင့် တူညီသည်။
    *   `/=` : စားပြီး သတ်မှတ်ခြင်း။ ဥပမာ- `score /= 2;` သည် `score = score / 2` နှင့် တူညီသည်။

ဤ Assignment Operators များသည် ပရိုဂရမ်၏ ထိရောက်မှုကို သိသိသာသာ မြှင့်တင်ပေးသည်။ ၎င်းတို့သည် ထပ်ခါထပ်ခါ ရေးရမည့် ရှည်လျားသော ဖော်မြူလာများကို ရှောင်ရှားပြီး၊ တန်ဖိုးများကို စီမံခန့်ခွဲရာတွင် ပိုမိုရှင်းလင်းစေသည်။

### Practical Example: ကွန်ပျူတာတွင် အခြေခံ တွက်ချက်မှုများ

ဥပမာအားဖြင့်၊ သင်သည် စာမေးပွဲရမှတ်များကို တွက်ချက်လိုသည်ဟု ဆိုကြပါစို့။

```javascript
let marks1 = 85;
let marks2 = 92;

// Arithmetic Operators ကို အသုံးပြု၍ ပေါင်းခြင်း
let totalMarks = marks1 + marks2;
console.log("စုစုပေါင်းမှတ်:", totalMarks); // Output: 177

// Assignment Operators ကို အသုံးပြု၍ ရမှတ်များကို နောက်ထပ် ထပ်ပေါင်းခြင်း
let totalMarks2 = 0;
totalMarks2 += marks1; // totalMarks2 သည် 85 ဖြစ်သွားသည်
totalMarks2 += marks2; // totalMarks2 သည် 177 ဖြစ်သွားသည်
console.log("နောက်ထပ်စုစုပေါင်းမှတ်:", totalMarks2); // Output: 177
```

ဤဥပမာမှ သိသာသည်မှာ၊ Arithmetic Operators များသည် ဂဏန်းများကို တွက်ချက်ရန်၊ Assignment Operators များသည် ထိုတွက်ချက်မှုရလဒ်များကို သတ်မှတ်ထားသောနေရာများတွင် သိမ်းဆည်းရန် အသုံးပြုကြောင်း ဖြစ်သည်။

ယခုအခါ ကျွန်ုပ်တို့သည် ဂဏန်းများကို တွက်ချက်ရုံသာမက၊ ထိုဂဏန်းများကို တစ်ခုနှင့်တစ်ခု **နှိုင်းယှဉ်** လုပ်နိုင်ပုံကို သိရှိရမည် ဖြစ်သည်။

## ⚖️ 3. Comparison Operators: တန်ဖိုးများကို နှိုင်းယှဉ်ခြင်း (Comparing Values)

ဂဏန်းများကို တွက်ချက်ခြင်းပြီးနောက်၊ ထိုဂဏန်းများသည် ကျွန်ုပ်တို့၏ လိုအပ်ချက်များနှင့် ကိုက်ညီမှုရှိမရှိ စစ်ဆေးရန် လိုအပ်သည်။ JavaScript တွင် တန်ဖိုးနှစ်ခုကို မည်သို့ နှိုင်းယှဉ်ပြီး မှန်/မှား (True/False) အဖြေကို ထုတ်ပေးနိုင်ပုံကို Comparison Operators များက ကူညီပေးသည်။

### Relational Operators: တန်းတူမှုနှင့် မတူညီမှု စစ်ဆေးခြင်း

ဤ Operator များသည် တန်ဖိုးနှစ်ခုအကြား ဆက်ဆံရေးကို ဖော်ပြသည်။ ၎င်းတို့၏ အဖြေသည် အမြဲတမ်း Boolean တန်ဖိုးဖြစ်သော `true` (မှန်) သို့မဟုတ် `false` (မှား) ဖြစ်သည်။

*   **Equality Operators:**
    *   `==` (Loose Equality): တန်ဖိုးများကို နှိုင်းယှဉ်ရာတွင်၊ ၎င်းတို့၏ အမျိုးအစား (Type) ကို မထည့်ဘဲ တန်ဖိုးများကိုသာ နှိုင်းယှဉ်သည်။ ဥပမာ- `5 == "5"` သည် `true` ဖြစ်သည်။ (Type Coercion ရှိသောကြောင့် သတိထားရမည်။)
    *   `===` (Strict Equality): တန်ဖိုးနှင့် အမျိုးအစား နှစ်ခုလုံး တူညီမှသာ `true` ဖြစ်သည်။ ဥပမာ- `5 === "5"` သည် `false` ဖြစ်သည်။ (ဤအရာကို အမြဲတမ်း အသုံးပြုသင့်သည်။)
    *   `!=` (Not Equal To): တန်ဖိုးနှစ်ခု မတူညီကြောင်း စစ်ဆေးသည်။
    *   `!==` (Not Equal To): တန်ဖိုးနှင့် အမျိုးအစား နှစ်ခုလုံး မတူညီကြောင်း စစ်ဆေးသည်။

*   **Relational Operators:**
    *   `>` (Greater Than): ပထမအရာက ဒုတိယအရာထက် ပိုကြီးခြင်း။
    *   `<` (Less Than): ပထမအရာက ဒုတိယအရာထက် ပိုသေးခြင်း။
    *   `>=` (Greater Than or Equal To): ပထမအရာက ဒုတိယအရာနှင့် တန်းတူ သို့မဟုတ် ပိုကြီးခြင်း။
    *   `<=` (Less Than or Equal To): ပထမအရာက ဒုတိယအရာနှင့် တန်းတူ သို့မဟုတ် ပိုသေးခြင်း။

### Loose vs. Strict Equality: အရေးပါသော ကွာခြားချက်

`==` နှင့် `===` တို့၏ ကွာခြားချက်သည် JavaScript တွင် အလွန်အရေးကြီးသည်။

`==` (Loose Equality) သည် Type Coercion (အမျိုးအစား ပြောင်းလဲခြင်း) ကို အသုံးပြုသည်။ ဥပမာ- စာသား (String) နှင့် ဂဏန်း (Number) တို့ကို နှိုင်းယှဉ်ရာတွင်၊ JavaScript သည် ၎င်းတို့ကို တွက်ချက်နိုင်ရန်အတွက် တစ်ခုကို အခြားတစ်ခု၏ အမျိုးအစားသို့ ပြောင်းလဲရန် ကြိုးစားသည်။ ဤအပြုအမူကြောင့် မထင်မှတ်ဘဲ မှားယွင်းသော ရလဒ်များ (Unexpected results) ထွက်ပေါ်လာနိုင်သည်။

`===` (Strict Equality) သည် Type ကိုပါ စစ်ဆေးသည်။ ဤနည်းလမ်းကို အသုံးပြုခြင်းဖြင့် သင်သည် ပရိုဂရမ်၏ အပြုအမူကို ပိုမိုထင်ရှားစေပြီး၊ အမှားအယွင်းများကို ရှောင်ရှားနိုင်မည် ဖြစ်သည်။ ထို့ကြောင့်၊ သင်သည် ဘယ်အချိန်မဆို တန်ဖိုးများကို နှိုင်းယှဉ်သည့်အခါ `===` ကို အသုံးပြုရန် အထူးအကြံပြုလိုပါသည်။

ယခုအခါ ကျွန်ုပ်တို့သည် တစ်ခုချင်းစီကို စစ်ဆေးရုံသာမက၊ အခြေအနေတစ်ခုပေါ်မူတည်၍ ကွဲပြားသော လုပ်ဆောင်မှုများကို လုပ်ဆောင်နိုင်ရမည် ဖြစ်သည်။

## 🧠 4. Logical Operators: ဆုံးဖြတ်ချက်များ ပေါင်းစပ်ခြင်း (Combining Decisions)

ပရိုဂရမ်များသည် ပိုမိုရှုပ်ထွေးသော ဆုံးဖြတ်ချက်များ ချမှတ်နိုင်ရန်အတွက်၊ တစ်ခုတည်းသော အခြေအနေကို မဟုတ်ဘဲ၊ အခြေအနေများစွာကို ပေါင်းစပ်ရန် လိုအပ်သည်။ ဤနေရာတွင် Logical Operators များက အလွန်အရေးပါသော အခန်းကဏ္ဍမှ ပါဝင်လာသည်။

### Logical Operators: AND, OR, NOT

Logical Operators များသည် Boolean တန်ဖိုးများ (True/False) ကို ပေါင်းစပ်ရန် အသုံးပြုသည်။

*   **Logical AND (`&&`):** ဤ Operator သည် နှစ်ခုလုံး မှန်မှန်ကန်ကန် ဖြစ်မှသာ အမှန် (True) အဖြေကို ပေးသည်။ အခြားတစ်ဖက်မှ မှားပါက အမှန် (False) ဖြစ်သွားသည်။ ဥပမာ- `(age >= 18 && hasLicense)` ဆိုသည်မှာ အသက် ၁၈ နှစ်အထက်ရှိပြီး လိုင်စင်ရှိမှသာ မှန်သည်။
*   **Logical OR (`||`):** ဤ Operator သည် နှစ်ခုထဲမှ တစ်ခုခု မှန်ပါက အမှန် (True) အဖြေကို ပေးသည်။ တစ်ခုလုံး မှားမှသာ အမှန် (False) ဖြစ်သည်။ ဥပမာ- `(isRaining || isSnowing)` ဆိုသည်မှာ မိုးရွာနေသည်ဖြစ်စေ၊ နှင်းမုန်တိုင်းဖြစ်နေသည်ဖြစ်စေ မှန်သည်။
*   **Logical NOT (`!`):** ဤ Operator သည် အခြေအနေကို ပြောင်းပြန်လှန်သည်။ `true` ကို `false` သို့၊ `false` ကို `true` သို့ ပြောင်းလဲပေးသည်။ ဥပမာ- `!isLoggedIn` ဆိုသည်မှာ `isLoggedIn` သည် `false` ဖြစ်နေပါက `true` ဖြစ်သွားသည်။

### Operator Precedence: အစီအစဉ်အတိုင်း လုပ်ဆောင်ပုံ

Logical Operators များအပြင်၊ JavaScript တွင် Operator များ အစီအစဉ်အတိုင်း လုပ်ဆောင်ပုံ (Operator Precedence) ကို နားလည်ရန် အလွန်အရေးကြီးသည်။ ၎င်းသည် ပရိုဂရမ်က မည်သည့်အရာကို ဦးစားပေးမည်ကို ဆုံးဖြတ်ပေးသည်။ ဥပမာ၊ သင်္ချာတွက်ချက်မှုများသည် သတ်မှတ်ထားသော စည်းမျဉ်းများအရ အရင်လုပ်ဆောင်ပြီး၊ ထို့နောက် Logical Operators များဖြင့် ပေါင်းစပ်သည်။

### Transition: အခြေအနေစစ်ဆေးမှုများ (if/else)

အခြေအနေများစွာကို ပေါင်းစပ်နိုင်ပြီဖြစ်သောကြောင့်၊ ထိုအခြေအနေများပေါ်မူတည်၍ ပရိုဂရမ်၏ လမ်းကြောင်းကို ပြောင်းလဲနိုင်ရန် လိုအပ်သည်။ ထိုအခြေအနေများကို စစ်ဆေးပြီး၊ ထိုအခြေအနေများပေါ်မူတည်၍ ကွဲပြားသော လုပ်ဆောင်မှုများကို လုပ်ဆောင်နိုင်ရမည် ဖြစ်သည်။

## 🚦 5. Conditional Statements: အခြေအနေအလိုက် လုပ်ဆောင်ခြင်း (Executing Based on Conditions)

Conditional Statements များသည် ပရိုဂရမ်၏ အဓိက အစိတ်အပိုင်းတစ်ခုဖြစ်သည်။ ၎င်းတို့သည် ပရိုဂရမ်ကို "စဉ်းစား" ပြီး၊ အခြေအနေပေါ်မူတည်၍ မည်သည့်ကုဒ်များကို အက निष्ဆောင် (Execute) လုပ်ရမည်ကို ဆုံးဖြတ်ပေးသည်။

### If Statement နှင့် If-Else Statement

*   **If Statement:** အခြေအနေတစ်ခု မှန်ပါက (True ဖြစ်ပါက) ထိုကုဒ်များကိုသာ အက निष्ဆောင် လုပ်ဆောင်သည်။
    ```javascript
    let age = 20;
    if (age >= 18) {
        console.log("သင်သည် မဲပေးခွင့်ရှိသည်။");
    }
    ```
*   **If-Else Statement:** အခြေအနေ မှန်ပါက A ကို၊ မမှန်ပါက B ကို လုပ်ဆောင်ရန် ရွေးချယ်စရာ (Alternative) ပေးသည်။
    ```javascript
    let temperature = 35;
    if (temperature > 30) {
        console.log("နေ့သည် ပူနေသည်။");
    } else {
        console.log("နေ့သည် အေးနေသည်။");
    }
    ```

### If-Else If-Else Ladder: အဆင့်ဆင့် ဆုံးဖြတ်ချက်

ပိုမိုရှုပ်ထွေးသော အခြေအနေများအတွက်၊ တစ်ခုပြီးတစ်ခု စစ်ဆေးရန် `else if` ကို အသုံးပြုသည်။ ၎င်းသည် အဆင့်ဆင့် ဆုံးဖြတ်ချက် ချမှတ်ရာတွင် အလွန်အသုံးဝင်သည်။

```javascript
let score = 75;
if (score >= 90) {
    console.log("အဆင့် A");
} else if (score >= 70) {
    console.log("အဆင့် B");
} else if (score >= 50) {
    console.log("အဆင့် C");
} else {
    console.log("ကျရှုံး");
}
```

### Switch Statement: အလားတူ အခြေအနေများအတွက်

အကယ်၍ သင်သည် တစ်ခုတည်းသော တန်ဖိုးကို စစ်ဆေးရန် လိုအပ်ပါက `switch` statement သည် `if-else if` များထက် ပိုမိုရှင်းလင်းပြီး ထိရောက်သည်။ ၎င်းသည် တစ်ခုတည်းသော အခြေအနေအတွက် အလွန်ကောင်းမွန်သည်။

```javascript
let day = 3;
switch (day) {
    case 1:
        console.log("တနင်္ဂနွေ");
        break;
    case 2:
        console.log("တနင်္လာ");
        break;
    case 3:
        console.log("အင်္ဂါ");
        break;
    default:
        console.log("မှားယွင်းသော နေ့ရက်");
}
```

ယခုအခါတွင်၊ အခြေအနေအလိုက် ဆုံးဖြတ်ချက်ချမှတ်နိုင်ပြီဆိုလျှင်၊ ထိုလုပ်ဆောင်မှုများကို တစ်ကြိမ်တည်း မဟုတ်ဘဲ၊ ထပ်ခါထပ်ခါ အလိုအလျောက် လုပ်ဆောင်စေရန် လိုအပ်သည်။

## 🔁 6. Looping Constructs: ထပ်ခါထပ်ခါ လုပ်ဆောင်ခြင်း (Repeating Actions)

အခြေအနေများကို စစ်ဆေးပြီး၊ ထိုအခြေအနေများပေါ်မူတည်၍ ထပ်ခါထပ်ခါ လုပ်ဆောင်နိုင်ပြီ ဖြစ်သည်။ ထိုလုပ်ဆောင်မှုများကို ထိရောက်စွာ အလိုအလျောက် လုပ်ဆောင်နိုင်ရန်အတွက် Loop များအသုံးပြုသည်။ Loop များသည် ပရိုဂရမ်၏ အချိန်ကုန်သက်သာမှုကို အလွန်တိုးတက်စေသည်။

### For Loop: သတ်မှတ်ထားသော အကြိမ်အရေအတွက်အတွက်

`for` loop သည် သတ်မှတ်ထားသော အကြိမ်အရေအတွက်အတွက် ထပ်ခါထပ်ခါ လုပ်ဆောင်ရန် အသုံးအများဆုံးနှင့် အလွယ်ကူဆုံး ပုံစံဖြစ်သည်။ ၎င်းသည် စတင်မည့်အမှတ်၊ အဆုံးသတ်မည့်အမှတ်နှင့် အတက်အကျ (Increment/Decrement) ကို သတ်မှတ်ပေးရမည်။

```javascript
// i သည် 0 မှ 4 အထိ တန်ဖိုးများဖြင့် ထပ်ခါထပ်ခါ လုပ်ဆောင်မည်
for (let i = 0; i < 5; i++) {
    console.log("Iteration အမှတ်: " + i);
}
```

### While Loop: အခြေအနေတစ်ခု မှန်နေသမျှ အချိန်အထိ

`while` loop သည် အခြေအနေတစ်ခု မှန်နေသမျှ အချိန်အထိ ထပ်ခါထပ်ခါ လုပ်ဆောင်ရန် အသုံးပြုသည်။ အခြေအနေ မှန်နေသမျှသာ Loop သည် ဆက်လက်အလုပ်လုပ်သည်။

```javascript
let count = 0;
while (count < 3) {
    console.log("While Loop မှ အလုပ်လုပ်နေသည်: " + count);
    count++; // အရေးကြီးသည်- အခြေအနေကို ပြောင်းလဲပေးရမည်။
}
```

### Loop Control: ထိန်းချုပ်ခြင်း

Loop များ၏ အလုပ်လုပ်ပုံကို ထိန်းချုပ်ရန်အတွက် အရေးပါသော Control Keywords များ ရှိသည်။

*   **`break`:** ဤ Keyword ကို အသုံးပြုသောအခါ လက်ရှိ Loop ကို ချက်ချင်း ရပ်တန့်လိုက်သည်။
*   **`continue`:** ဤ Keyword ကို အသုံးပြုသောအခါ လက်ရှိအကြိမ်ကို ကျော်ပြီး နောက်တစ်ကြိမ်သို့ ဆက်သွားသည်။

ဤ Control Keywords များသည် Loop များ၏ စီမံခန့်ခွဲမှုကို ပိုမိုတိကျစေသည်။

### Practical Application: ဂဏန်းများ၏ စာရင်းများကို စစ်ဆေးခြင်း

ဤ Loop များအား အသုံးပြု၍ ဂဏန်းများ၏ စာရင်းများကို စစ်ဆေးခြင်းနှင့် ဖော်ပြခြင်းသည် ပရိုဂရမ်၏ စွမ်းဆောင်ရည်ကို မြှင့်တင်ပေးသည်။

```javascript
// 1 မှ 10 အထိ ဂဏန်းများကို စစ်ဆေးခြင်း
for (let num = 1; num <= 10; num++) {
    console.log(num + " သည် စာရင်းတွင် ရှိသည်။");
}

// While Loop ဖြင့် နောက်ထပ် လေ့ကျင့်ခြင်း
let guess = 0;
while (guess < 5) {
    guess++;
    console.log("ခန့်မှန်းမှု: " + guess);
}
```

## 🏁 7. Chapter Summary & Practical Exercise (အနှစ်ချုပ်နှင့် လက်တွေ့လေ့ကျင့်ခန်း)

### Summary Review

ဤအခန်းတွင် ကျွန်ုပ်တို့သည် JavaScript ၏ အခြေခံတွက်ချက်မှုများမှသည် ပရိုဂရမ်၏ လမ်းကြောင်းထိန်းချုပ်မှုများအထိ အရေးကြီးသော အခြေခံများကို လေ့လာခဲ့ကြသည်။

*   **Operators များ:** သင်္ချာတွက်ချက်မှု (`+`, `-`, `*`, `/`, `%`) နှင့် Assignment Operators များဖြင့် ဒေတာများကို ကိုင်တွယ်ခြင်းကို သင်ယူခဲ့သည်။
*   **Comparison Operators များ:** `==`, `===`, `!=`, `!==`, `>`, `<` တို့ဖြင့် ဒေတာနှစ်ခုကို နှိုင်းယှဉ်ပြီး မှန်/မှား အဖြေများ ရရှိပုံနှင့် Strict Equality ၏ အရေးပါပုံကို နားလည်ခဲ့သည်။
*   **Logical Operators များ:** `&&` (AND), `||` (OR), `!` (NOT) တို့ဖြင့် ရှုပ်ထွေးသော အခြေအနေများကို ပေါင်းစပ်၍ ဆုံးဖြတ်ချက်များ ချမှတ်ပုံကို လေ့လာခဲ့သည်။
*   **Conditional Statements များ:** `if`, `else if`, `else` နှင့် `switch` statement များကို အသုံးပြု၍ ပရိုဂရမ်၏ လမ်းကြောင်းကို အခြေအနေအပေါ် မူတည်၍ ပြောင်းလဲနိုင်ပုံကို သင်ယူခဲ့သည်။
*   **Looping Constructs များ:** `for` loop နှင့် `while` loop များကို အသုံးပြု၍ လုပ်ဆောင်မှုများကို ထပ်ခါထပ်ခါ အလိုအလျောက် လုပ်ဆောင်နိုင်ပုံနှင့် `break`, `continue` များဖြင့် ထိန်းချုပ်ပုံကို လေ့လာခဲ့သည်။

### Final Practical Challenge: "ရိုးရှင်းသော ဂဏန်းခန့်မှန်းခြင်း ဂိမ်း ဖန်တီးခြင်း"

ယခု သင်လေ့လာခဲ့သော အကြောင်းအရာများအားလုံးကို ပေါင်းစပ်၍ လက်တွေ့ကျသော ပရောဂျက်တစ်ခုကို ဖန်တီးကြည့်ကြပါမည်။ ကျွန်ုပ်တို့၏ ရည်မှန်းချက်မှာ အသုံးပြုသူထံမှ ဂဏန်းတစ်ခုကို လက်ခံပြီး၊ JavaScript ၏ **If/Else** နှင့် **While Loop** ကို အသုံးပြု၍ မှန်ကန်သော ဂဏန်းကို ခန့်မှန်းခိုင်းသည့် ဂိမ်းငယ်တစ်ခုကို အဆင့်ဆင့် ဖန်တီးရန် ဖြစ်သည်။

**ရည်ရွယ်ချက်:** အသုံးပြုသူသည် သတ်မှတ်ထားသော အပိုင်းအစအတွင်းမှ လျှို့ဝှက်ဂဏန်းကို ခန့်မှန်းရမည်။

**အဆင့် ၁: ဂဏန်းကို သတ်မှတ်ခြင်းနှင့် စတင်ခြင်း**
ကျွန်ုပ်တို့သည် လျှို့ဝှက်ဂဏန်းတစ်ခုကို သတ်မှတ်ပြီး၊ အသုံးပြုသူထံမှ ခန့်မှန်းမှုကို လက်ခံရန် လိုအပ်သည်။

**အဆင့် ၂: While Loop ဖြင့် စစ်ဆေးခြင်း**
အသုံးပြုသူ၏ ခန့်မှန်းမှု မှန်ကန်သည်အထိ ထပ်ခါထပ်ခါ စစ်ဆေးရန် `while` loop ကို အသုံးပြုမည်။ ခန့်မှန်းမှု မှားနေသမျှ အချိန်အထိ Loop ကို ဆက်လက်အလုပ်လုပ်စေမည်။

**အဆင့် ၃: If/Else ဖြင့် ရလဒ်ပေးခြင်း**
Loop က အဆုံးသတ်သောအခါ၊ ခန့်မှန်းမှု မှန်ပါက "ဂိမ်းဆော့ကစားနိုင်ပြီ" ဟု ပြသမည်။ မှားပါက "နောက်ထပ်ကြိုးစားပါ" ဟု ပြသမည်။

ဤလက်တွေ့လေ့ကျင့်ခန်းသည် သင်သည် Operator များ၊ Comparison များ၊ Logical များ၊ Conditional များ၊ နှင့် Loop များအား ပေါင်းစပ်၍ အမှန်တကယ် အလုပ်လုပ်သော ပရိုဂရမ်များကို ဖန်တီးနိုင်ကြောင်း သက်သေပြပေးမည် ဖြစ်သည်။

### Conclusion

JavaScript ၏ Operator များနှင့် Control Flow များသည် ပရိုဂရမ်ရေးသားခြင်း၏ အင်မတန် အရေးကြီးသော အခြေခံများ ဖြစ်သည်။ ဤအခြေခံများကို နားလည်ခြင်းနှင့် ကျွမ်းကျင်စွာ အသုံးပြုနိုင်ခြင်းသည် သင်သည် ရိုးရှင်းသော စာကြောင်းများမှ အမှန်တကယ် ဆုံးဖြတ်ချက်များ ချမှတ်နိုင်ပြီး၊ အလုပ်များကို အလိုအလျောက် လုပ်ဆောင်နိုင်သော စွမ်းအားကို ရရှိလာစေသည်။ သင်သည် ဤကျွမ်းကျင်မှုများကို ပေါင်းစပ်လိုက်သောအခါ၊ သင်သည် အချက်အလက်များကို စီမံခန့်ခွဲနိုင်ပြီး၊ အသုံးပြုသူများအတွက် တုံ့ပြန်မှုရှိသော၊ ရှုပ်ထွေးသော Web Applications များကို အသက်ဝင်စေမည့် စွမ်းဆောင်ရည်ကို ပိုင်ဆိုင်လာမည် ဖြစ်သည်။ ဤအခန်းမှ သင်ရရှိသော အခြေခံများသည် သင့်အား ပိုမိုရှုပ်ထွေးသော JavaScript ပရောဂျက်များကို တည်ဆောက်ရန်အတွက် ခိုင်မာသော အုတ်မြစ်များ ဖြစ်စေမည် ဖြစ်သည်။
