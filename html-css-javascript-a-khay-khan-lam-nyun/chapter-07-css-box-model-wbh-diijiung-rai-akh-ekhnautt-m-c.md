# Chapter 7: CSS Box Model — ဝဘ်ဒီဇိုင်းရဲ့ အခြေခံအုတ်မြစ်

ဝဘ်ဆိုဒ်တစ်ခုကို "လှပစွာ" ဒီဇိုင်းဆွဲခြင်းဆိုတာ ဘာကို ဆိုလိုတာလဲ။ HTML နဲ့ CSS ကို လေ့လာနေတဲ့အခါ ကျွန်ုပ်တို့သည် အရာဝတ္ထုများ (Elements) ကို စာမျက်နှာပေါ်တွင် စီစဉ်နေခြင်း ဖြစ်သည်။ သို့သော် အဆိုပါ အရာဝတ္ထုများဟာ ဘယ်လိုနေရာယူနေသလဲ၊ အချင်းချင်း ဘယ်လိုဆက်ဆံနေသလဲဆိုတဲ့ အခြေခံ မေးခွန်းကို ဖြေရှင်းပေးမယ့် အရေးကြီးဆုံး အယူအဆတစ်ခု ရှိနေပါသည်။ ထိုအယူအဆမှာ CSS Box Model ဖြစ်သည်။ Box Model ဆိုသည်မှာ HTML Element တစ်ခုစီတိုင်းကို စတုရန်းပုံသဏ္ဍာန်ဖြင့် ပတ်လည်ဖုံးအုပ်ထားသော အခြေခံ မူဘောင်တစ်ခုကို ဆိုလိုသည်။ ဤအခန်းတွင် ကျွန်ုပ်တို့သည် Box Model ဆိုတာ ဘာလဲ၊ ဘာကြောင့် အရေးကြီးလဲ၊ ဘယ်လို အစိတ်အပိုင်းတွေနဲ့ ပတ်သက်ပြီး သိဖို့ လိုအပ်လဲဆိုတာကို အသေးစိတ် လေ့လာသွားကြမည် ဖြစ်သည်။

## CSS Box Model ဆိုတာ ဘာလဲ? (The Foundation: Understanding the Box)

CSS Box Model ဆိုသည်မှာ HTML Element တစ်ခုစီကို စတုရန်းပုံသဏ္ဍာန်ဖြင့် ဖော်ပြပြီး၊ ထိုစတုရန်းပုံသဏ္ဍာန်အတွင်းတွင် အကြောင်းအရာ (Content)၊ အတွင်း အနား (Padding)၊ အစွန်း (Border) နှင့် အပြင် အနား (Margin) ဟူသော အစိတ်အပိုင်းလေးခုကို ထည့်သွင်းစဉ်းစားခြင်းပင် ဖြစ်သည်။ ဤအယူအဆကို နားလည်ခြင်းဖြင့် ကျွန်ုပ်တို့သည် CSS ဖြင့် အရာဝတ္ထုများကို တိကျစွာ ထိန်းချုပ်နိုင်လာမည် ဖြစ်သည်။

### အဓိက အစိတ်အပိုင်းများ (The Four Components)

Box Model ကို အဓိကအားဖြင့် အစိတ်အပိုင်းလေးခုအဖြစ် ခွဲခြားနိုင်သည်။ ဤအစိတ်အပိုင်းများသည် အရာဝတ္ထုတစ်ခု၏ အပြင်ပိုင်းနှင့် အတွင်းပိုင်းကို ပုံဖော်ပေးသည်။

ပထမအချက်မှာ **Content (အကြောင်းအရာ)** ဖြစ်သည်။ ၎င်းသည် Element အတွင်း၌ တကယ်ပါဝင်သော စာသား၊ ပုံများ သို့မဟုတ် အခြားသော အကြောင်းအရာများ ဖြစ်သည်။ ၎င်းသည် Box ၏ အဓိက အစိတ်အပိုင်းဖြစ်သည်။

ဒုတိယအချက်မှာ **Padding (အတွင်း အနား)** ဖြစ်သည်။ ၎င်းသည် Content နှင့် Border ကြားရှိ အလွတ်နေရာကို ဆိုလိုသည်။ Padding သည် အကြောင်းအရာနှင့် အစွန်းကြားရှိ အတွင်းပိုင်း အကွာအဝေးကို ထိန်းချုပ်ပေးပြီး အကြောင်းအရာကို အစွန်းမှ ခွဲခြားပေးသည်။

တတိယအချက်မှာ **Border (အစွန်း)** ဖြစ်သည်။ Border သည် Box ၏ အပြင်ဘက်တွင်ရှိသော အစွန်းမျဉ်းကြောင်းဖြစ်သည်။ ၎င်းသည် Box ကို အခြား အရာဝတ္ထုများနှင့် ခွဲခြားပေးသည့် အနားသတ်မှတ်ချက်ဖြစ်သည်။

နောက်ဆုံးအချက်မှာ **Margin (အပြင် အနား)** ဖြစ်သည်။ Margin သည် Border နှင့် အခြား Element များကြားရှိ အလွတ်နေရာကို ဆိုလိုသည်။ Margin သည် Box ကို အခြား Element များနှင့် ခွဲခြားရန် အသုံးပြုပြီး အပြင်ဘက်ရှိ အကွာအဝေးကို ထိန်းချုပ်သည်။

ဤအချက်လေးချက်ကို ပုံတစ်ပုံဖြင့် မြင်ယောင်ကြည့်လျှင်၊ အကြောင်းအရာ (Content) သည် အလယ်တွင် ရှိပြီး၊ ထိုအကြောင်းအရာကို Padding ဖြင့် ဝန်းရံထားကာ၊ ထို့နောက် Border ဖြင့် အစွန်းပေးပြီး၊ နောက်ဆုံးတွင် Margin ဖြင့် အခြား အရာဝတ္ထုများနှင့် အကွာအဝေး ထားရှိခြင်းကို မြင်တွေ့ရမည် ဖြစ်သည်။ ဤပုံစံသည် CSS ဖြင့် HTML Element များအား မည်သို့ ပေါင်းစပ်နေပုံကို အလွယ်တကူ နားလည်စေသည်။

## အရွယ်အစားနှင့် စီမံခန့်ခွဲမှု (Sizing and Control)

Box Model ကို နားလည်ပြီဆိုတော့၊ ယခုအခါ ကျွန်ုပ်တို့သည် ဤ Box များ၏ အရွယ်အစား (Width နှင့် Height) ကို မည်သို့ သတ်မှတ်မည်၊ ထိန်းချုပ်မည်ကို ဆက်လက်လေ့လာကြရမည် ဖြစ်သည်။

### Width နှင့် Height သတ်မှတ်ခြင်း

CSS တွင် Box တစ်ခု၏ အကျယ် (Width) နှင့် အမြင့် (Height) ကို သတ်မှတ်ရာတွင် Pixel (px)၊ Percentage (%) နှင့် Viewport အခြေခံဖြင့် သတ်မှတ်နိုင်သည်။ Pixel များသည် တိကျသော အတိုင်းအတာကို ပေးနိုင်သော်လည်း၊ Responsive Design အတွက် Percentage နှင့် Viewport အခြေခံကို အသုံးပြုခြင်းက ပိုမို အကျိုးရှိသည်။

ဥပမာအားဖြင့်၊ `width: 300px;` သည် Box ၏ အကျယ်ကို သတ်မှတ်သည်။ ထို့အပြင်၊ `height: 200px;` သည် Box ၏ အမြင့်ကို သတ်မှတ်သည်။ ထို့နောက် Padding နှင့် Border များ ထည့်သွင်းလိုက်သောအခါ၊ Box ၏ စုစုပေါင်း အရွယ်အစားသည် ဤတန်ဖိုးများနှင့် အပေါ်က ပေါင်းစပ်မှုများဖြင့် ဖြစ်ပေါ်လာမည် ဖြစ်သည်။

### `box-sizing` Property ၏ အရေးပါပုံ

Box Model ၏ အလုပ်လုပ်ပုံတွင် အရေးကြီးဆုံးနှင့် အဖြစ်အများဆုံး အမှားအယွင်းဖြစ်စေသော အချက်မှာ `box-sizing` property ဖြစ်သည်။ ဤ property သည် Box တစ်ခု၏ စုစုပေါင်း အရွယ်အစားကို မည်သို့ တွက်ချက်မည်ကို ဆုံးဖြတ်ပေးသည်။

#### `content-box` (Default)

CSS တွင် ပုံသေအားဖြင့် သတ်မှတ်ထားသော `box-sizing` တန်ဖိုးမှာ `content-box` ဖြစ်သည်။ `content-box` ဖြင့် အလုပ်လုပ်သောအခါ၊ သင်သတ်မှတ်လိုက်သော `width` နှင့် `height` သည် Box ၏ အကြောင်းအရာ (Content) အတွက်သာ ဖြစ်သည်။ Padding နှင့် Border တို့ကို ထပ်ပေါင်း၍ Box ၏ စုစုပေါင်း အရွယ်အစားကို ရှည်ရှည်နှင့် ကျယ်ကျယ် ဖြစ်စေသည်။ ဥပမာအားဖြင့်၊ သင်သည် `width: 100px;` သတ်မှတ်လိုက်ပါက၊ အမှန်တကယ် Box ၏ အကျယ်သည် 100px ထက် ပိုသွားမည် ဖြစ်သည်။ ဤပုံစံသည် Layout များတွင် တွက်ချက်ရခက်ခဲစေပြီး အမှားအယွင်းများ ဖြစ်ပေါ်စေနိုင်သည်။

#### `border-box` (အရေးကြီးဆုံး)

Layout များတွင် အမှားအယွင်းမရှိဘဲ တိကျစွာ ထိန်းချုပ်လိုပါက `border-box` ကို အသုံးပြုရန် အလွန်အရေးကြီးသည်။ `border-box` ဖြင့် အလုပ်လုပ်သောအခါ၊ သင်သတ်မှတ်လိုက်သော `width` နှင့် `height` သည် Content၊ Padding နှင့် Border များအားလုံးကို အကျုံးဝင်သော စုစုပေါင်း အရွယ်အစား ဖြစ်သည်။ ဆိုလိုသည်မှာ၊ သင်သည် `width: 100px;` ဟု သတ်မှတ်လိုက်ပါက၊ ထို Box ၏ အကျယ်သည် Padding နှင့် Border များအားလုံးကို ထည့်သွင်းပြီး ၁၀၀px ဖြစ်သွားမည် ဖြစ်သည်။ ဤနည်းလမ်းသည် Layout များကို ပိုမိုလွယ်ကူစွာ စီမံခန့်ခွဲနိုင်စေပြီး ရှုပ်ထွေးသော Layout ပြဿနာများကို ဖြေရှင်းပေးနိုင်သောကြောင့် အလွန်အရေးကြီးပါသည်။

ဤ `box-sizing: border-box;` ကို စတင်အသုံးပြုခြင်းဖြင့် ကျွန်ုပ်တို့သည် CSS ဖြင့် Layout များ၏ အခြေခံအုတ်မြစ်ကို ခိုင်မာစွာ တည်ဆောက်နိုင်ပြီ ဖြစ်သည်။

## Margin Collapse နှင့် Display Properties (Layout Behavior)

Box Model ကို နားလည်ပြီးပြီဆိုတော့၊ ယခုအခါ Box များကြားရှိ အကွာအဝေး (Spacing) နှင့် Element များ၏ စီစဉ်ပုံ (Flow) ကို လေ့လာရမည် ဖြစ်သည်။ ဤအချက်နှစ်ချက်သည် ဝဘ်ဒီဇိုင်း၏ အလှအပကို ဆုံးဖြတ်ပေးသည်။

### Margin Collapse ဆိုတာ ဘာလဲ?

Margin Collapse ဆိုသည်မှာ Box များကြားရှိ အလွတ်နေရာများ မည်သို့ အကျိုးသက်ရောက်မှုရှိသည်ကို ဖော်ပြသည်။ အထူးသဖြင့် `margin` များကို အသုံးပြုသောအခါ၊ အနီးအနားရှိ Box များကြားတွင် မလိုအပ်သော အလွတ်နေရာများ ပေါင်းစပ်သွားပြီး ပျောက်ကွယ်သွားခြင်းကို Margin Collapse ဟုခေါ်သည်။ ဤဖြစ်စဉ်သည် အထူးသဖြင့် `margin` များနှင့် `border` တို့ကို ပေါင်းစပ်သောအခါ အထင်ကြီးစရာ မကောင်းသော Layout များ ဖြစ်ပေါ်လာစေသည်။ ဤအချက်ကို နားလည်ခြင်းဖြင့် ကျွန်ုပ်တို့သည် မလိုအပ်သော အလွတ်နေရာများကို ထိန်းချုပ်နိုင်မည် ဖြစ်သည်။

### Display Property များ၏ ကွာခြားချက်

Element များ မည်သို့ စီစဉ်နေသည်ကို ထိန်းချုပ်ရန်အတွက် CSS တွင် `display` property များသည် အလွန်အရေးကြီးသည်။ ဤ property များသည် Element များ မည်သို့ အကြောင်းအရာများကို စီစဉ်ပေးပုံကို အဓိပ္ပာယ်ဖွင့်ဆိုသည်။

#### `block`

`display: block;` သည် Element တစ်ခုကို အလျားလိုက် အပြည့် အကျယ်ရှိသော အရာအဖြစ် သတ်မှတ်သည်။ Block Element များသည် အမြဲတမ်း အပေါ်မှ အောက်သို့ တစ်ကြောင်းတည်းဖြင့် ဆက်တိုက် ရေးသားသွားသည်။ Block Element များသည် အကျယ်နှင့် အမြင့်ကို အလိုအလျောက် ပြောင်းလဲနိုင်ပြီး အခြား Element များနှင့် အလျားလိုက် စီစဉ်နေသည်။

#### `inline`

`display: inline;` သည် Element တစ်ခုကို စာသားအခြေခံအဖြစ် သတ်မှတ်သည်။ Inline Element များသည် အလျားလိုက် စာသားများကဲ့သို့ အခြား Element များနှင့် အတူတူ ရေးသားနေသည်။ Inline Element များသည် အကျယ်နှင့် အမြင့်ကို တိကျစွာ သတ်မှတ်ရန် ခက်ခဲပြီး၊ ၎င်းတို့၏ အနားအနီးရှိ အလွတ်နေရာများကိုသာ ထိန်းချုပ်နိုင်သည်။

#### `inline-block`

`display: inline-block;` သည် `block` နှင့် `inline` တို့၏ အကောင်းဆုံး ပေါင်းစပ်မှု ဖြစ်သည်။ Inline-block Element များသည် Inline အနေဖြင့် အခြား Element များနှင့် အတူတူ ရေးသားနိုင်သော်လည်း၊ Block Element များကဲ့သို့ အကျယ်နှင့် အမြင့်ကိုလည်း သတ်မှတ်နိုင်ပြီး အခြား Element များနှင့် အလျားလိုက် စီစဉ်နေနိုင်သည်။ ဤ property သည် Layout များတွင် အလွန်အသုံးဝင်ပြီး၊ အလွှာလိုက် စီစဉ်မှုကို ထိန်းချုပ်ရန်အတွက် အကောင်းဆုံး ရွေးချယ်မှုတစ်ခု ဖြစ်သည်။

ဤ `display` အမျိုးအစားများကို မှန်ကန်စွာ အသုံးပြုခြင်းဖြင့် ကျွန်ုပ်တို့သည် Element များ၏ Layout ပုံစံကို ထိန်းချုပ်နိုင်ပြီး၊ Margin Collapse ကဲ့သို့သော ပြဿနာများကိုလည်း ထိန်းချုပ်နိုင်လာသည်။

## လက်တွေ့ လေ့ကျင့်ခန်း (Practical Application: Building a Card Component)

အခုချိန်ထိ Box Model နှင့် Layout အခြေခံများကို လေ့လာပြီးပြီ ဖြစ်သည်။ ဤအယူအဆများကို လက်တွေ့တွင် အသုံးချပြီး လှပသော ဒီဇိုင်းတစ်ခုကို ဖန်တီးကြည့်ကြရအောင်။ ကျွန်ုပ်တို့၏ ပရောဂျက်မှာ အချက်အလက်များကို ပြသသည့် Card တစ်ခုကို CSS Box Model နှင့် Display Property များကို အသုံးပြု၍ ဒီဇိုင်းဆွဲခြင်း ဖြစ်သည်။

### Project: CSS Card Component ဖန်တီးခြင်း

ဤပရောဂျက်တွင် Box Model ၏ အစိတ်အပိုင်းလေးခုကို အသုံးပြု၍ Card တစ်ခုကို ဖန်တီးမည် ဖြစ်သည်။

**အဆင့် ၁: HTML Structure ပြင်ဆင်ခြင်း**

ပထမအဆင့်အနေဖြင့် Card အတွက် လိုအပ်သော HTML အခြေခံကို တည်ဆောက်ရမည်။

```html
<div class="card">
    <div class="card-content">
        <h2>Card Title</h2>
        <p>This is the content of the card. We are using the Box Model to control the spacing and boundaries.</p>
    </div>
</div>
```

**အဆင့် ၂: Box Model ကို အသုံးပြု၍ အနားပေးခြင်း**

အောက်ပါ CSS ကုဒ်များဖြင့် Box Model ၏ အစိတ်အပိုင်းများကို အသုံးပြု၍ အနားပေးခြင်းကို ပြုလုပ်မည်။

```css
.card {
    width: 300px;
    border: 1px solid #ccc; /* Border */
    padding: 20px;       /* Padding ( အတွင်း အနား) */
    margin: 20px;        /* Margin (အပြင် အနား) */
    background-color: #f9f9f9;
    box-sizing: border-box; /* အရေးကြီးဆုံး အချက် */
}

.card-content {
    /* Card အတွင်းရှိ အကြောင်းအရာအတွက် နောက်ထပ် Padding ထပ်ထည့်ခြင်း */
    padding: 15px;
    background-color: #ffffff;
    border: 1px solid #eee;
}
```

ဤအဆင့်တွင် ကျွန်ုပ်တို့သည် `border`၊ `padding` နှင့် `margin` တို့ကို အသုံးပြု၍ Card ၏ အပြင်ပိုင်းနှင့် အတွင်းပိုင်းကို ပုံဖော်လိုက်သည်။ အထူးသဖြင့် `box-sizing: border-box;` ကို အသုံးပြုခြင်းဖြင့်၊ Card ၏ စုစုပေါင်း အရွယ်အစားသည် လိုချင်သောအတိုင်း ထိန်းချုပ်နိုင်ပြီး Layout များတွင် အမှားအယွင်းများ လျော့နည်းသွားသည်။

**အဆင့် ၃: Display အမျိုးအစားကို အသုံးပြု၍ Layout ပုံစံကို ထိန်းချုပ်ခြင်း**

Card တစ်ခုကို စာမျက်နှာပေါ်တွင် အလယ်တည့်တည့်နှင့် သပ်သပ်ရပ်ရပ် ရှိစေရန် `display` property ကို အသုံးပြုမည်။

```css
.card {
    /* Box ကို အခြား Element များနှင့် အလွယ်တကူ စီစဉ်နိုင်ရန် */
    display: block; 
}
```

ဤဥပမာတွင် `display: block;` ကို အသုံးပြုခြင်းဖြင့် Card သည် အလျားလိုက် အပြည့် အကျယ်ရှိသော Block အဖြစ် ရှိနေသည်။ ၎င်းသည် အခြား Element များနှင့် အလျားလိုက် စီစဉ်နေသည်။

## အခန်းနိဂုံးနှင့် အနှစ်ချုပ် (Conclusion and Summary)

CSS Box Model သည် ဝဘ်ဒီဇိုင်း၏ အခြေခံအုတ်မြစ်ပင် ဖြစ်သည်။ Content၊ Padding၊ Border နှင့် Margin ဟူသော အစိတ်အပိုင်းလေးခုကို နားလည်ခြင်းသည် HTML Element များ မည်သို့ အတူတကွ ဆက်ဆံနေပုံကို သိရှိစေသည်။ ထို့အပြင် `box-sizing: border-box;` ကို အသုံးပြုခြင်းဖြင့် Layout များတွင် တိကျမှန်ကန်သော ထိန်းချုပ်မှုကို ရရှိနိုင်မည် ဖြစ်သည်။ Margin Collapse နှင့် Display Property များ (block, inline, inline-block) တို့ကို နားလည်ခြင်းဖြင့် Element များ၏ စီစဉ်ပုံကို ထိန်းချုပ်နိုင်မည် ဖြစ်သည်။

ကျွန်ုပ်တို့သည် ဤအခြေခံအုတ်မြစ်ကို ကျွမ်းကျင်စွာ သိရှိပြီးပြီ ဖြစ်သည်။ Box Model ကို ကျွမ်းကျင်မှသာ နောက်ထပ် ရှုပ်ထွေးသော Layout များဖြစ်သည့် CSS Flexbox နှင့် Grid တို့ကို အသုံးပြု၍ ပိုမိုရှုပ်ထွေးသော နှင့် လှပသော Interface များကို ဖန်တီးနိုင်မည် ဖြစ်သည်။ နောက်အခန်းများတွင် ကျွန်ုပ်တို့သည် CSS Flexbox နှင့် Grid တို့ဖြင့် မည်သို့ Layout များကို စနစ်တကျ ဖန်တီးပုံကို ဆက်လက်လေ့လာသွားကြမည် ဖြစ်သည်။
