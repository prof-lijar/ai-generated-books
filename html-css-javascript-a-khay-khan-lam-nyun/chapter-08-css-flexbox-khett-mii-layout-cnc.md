# Chapter 8: CSS Flexbox — ခေတ်မီ Layout စနစ်

## 💡 Introduction: Layout ရဲ့ အခက်အခဲနှင့် Flexbox ရဲ့ အရှိန်အဟုန် (Opening Hook)

HTML နှင့် CSS ကို စတင်လေ့လာသူတိုင်းသည် မျက်နှာစာကို လှပစွာ ဖန်တီးနိုင်သော်လည်း၊ ထိုအရာများကို စနစ်တကျ စီစဉ်ချထားခြင်း (Layout) တွင် အခက်အခဲများဖြင့် ရုန်းကန်ရလေ့ရှိသည်။ HTML တွင် အကြောင်းအရာများကို ဘယ်လိုချမလဲ၊ တစ်ခုနှင့်တစ်ခု မည်သို့ ဆက်စပ်နေမလဲဆိုသည့် မေးခွန်းများသည် Layout ကို ရေးဆွဲရာတွင် အဓိက စိန်ခေါ်မှုများ ဖြစ်လာသည်။ ရှေးခေတ်ကမှ ဆက်လက်အသုံးပြုခဲ့သော `float` နှင့် `inline-block` ကဲ့သို့သော နည်းလမ်းများသည် အချိန်ပေးပြီး ရှုပ်ထွေးသော အလွှာလိုက် စီစဉ်မှုများကို ဖန်တီးနိုင်ခဲ့သော်လည်း၊ ၎င်းတို့သည် ထိန်းချုပ်ရလွယ်ကူမှုမရှိဘဲ အလွန်အချိန်ကုန်၊ ကြိုးစားအားထုတ်မှုများ လိုအပ်ခဲ့သည်။ ဤနည်းလမ်းများသည် အထူးသဖြင့် လိုအပ်ချက်အလိုက် ပြောင်းလဲနေသော (Responsive) ဒီဇိုင်းများအတွက် အလွန်မသင့်လျော်သည်။

Flexbox မတိုင်မီက Layout များသည် အလွှာလိုက် (one-dimensional) စီစဉ်မှုများကို ရရှိရန်အတွက် ရှုပ်ထွေးသော ကန့်သတ်ချက်များနှင့် အချိန်ကုန်ခံရသည်။ ထိုအချိန်တွင်၊ CSS သည် အကြောင်းအရာများကို တစ်ခုနှင့်တစ်ခု တိကျစွာ ထိန်းချုပ်နိုင်သော စွမ်းဆောင်ရည်ကို ပေးမည့် နည်းလမ်းတစ်ခုကို မိတ်ဆက်ခဲ့သည်။ ထိုနည်းလမ်းမှာ CSS Flexbox ဖြစ်သည်။ Flexbox ဆိုသည်မှာ အကြောင်းအရာများကို တစ်ဖက်တည်း (အလျားလိုက် သို့မဟုတ် ဒေါင်လိုက်) စီစဉ်ရာတွင် အလွန်ထိရောက်ပြီး ရိုးရှင်းသော စနစ်တစ်ခုကို ပေးစွမ်းသည်။ ၎င်း၏ အဓိက အကျိုးကျေးဇူးမှာ ကုဒ်ကို ရေးသားရလွယ်ကူစေခြင်း၊ ပြောင်းလဲမှုများကို အလွယ်တကူ ထိန်းချုပ်နိုင်ခြင်းနှင့် အထူးသဖြင့် Responsive Design အတွက် အလွန်သင့်လျော်ခြင်းတို့ ဖြစ်သည်။

ဤအခန်းတွင် ကျွန်ုပ်တို့သည် CSS Flexbox ၏ အခြေခံသဘောတရားများမှသည် ၎င်း၏ အဓိက Property များအထိ လေ့လာသွားမည် ဖြစ်သည်။ ကျွန်ုပ်တို့၏ ရည်မှန်းချက်မှာ Flexbox ကို နားလည်ပြီး၊ ၎င်းကို အသုံးပြု၍ ရှုပ်ထွေးသော Layout များကို အလွယ်တကူ၊ ထိရောက်စွာ ဖန်တီးနိုင်ရန် ဖြစ်သည်။ ဤအခန်းပြီးဆုံးသောအခါ၊ သင်သည် မည်သို့ Layout များကို ပိုမိုလှပပြီး ပိုမိုထိရောက်စွာ ထိန်းချုပ်နိုင်ကြောင်း လက်တွေ့ကျကျ သိရှိသွားမည် ဖြစ်သည်။

## 1. Layout ရဲ့ ရှေ့ပြေးခေတ်နှင့် Flexbox မတိုင်မီ အခက်အခဲများ (The Pre-Flex Era)

CSS Layout ကို Flexbox မမိတ်ဆက်မီက ကျွန်ုပ်တို့သည် အဓိကအားဖြင့် `float` နှင့် `inline-block` ဟူသော နည်းလမ်းများကို အသုံးပြုခဲ့ရသည်။ ဤနည်းလမ်းများသည် အချိန်အနည်းငယ်အတွင်း Layout တစ်ခုကို ဖန်တီးနိုင်ခဲ့သော်လည်း၊ ၎င်းတို့တွင် မဖြေရှင်းနိုင်သော အားနည်းချက်များစွာ ရှိနေသည်။

### 1.1 Float နှင့် Inline-Block ၏ အားနည်းချက်များ

`float` ကို အသုံးပြုသောအခါ၊ အကြောင်းအရာများကို ဘေးတိုက်တွင် စီစဉ်နိုင်သော်လည်း၊ ၎င်းတို့သည် အခြားသော ကွန်တိန်နာများနှင့် အလွယ်တကူ ဆက်စပ်၍မရဘဲ ပြဿနာများ ဖြစ်ပေါ်စေသည်။ ဥပမာအားဖြင့်၊ `float` လုပ်ထားသော အကြောင်းအရာများသည် မိမိတို့၏ ပုံသဏ္ဌာန်အရ မဟုတ်ဘဲ၊ မိမိတို့၏ အပေါ်မှ အကြောင်းအရာများကို မထိန်းချုပ်နိုင်ဘဲ ပုံစံတကျ လွှမ်းမိုးနေတတ်သည်။ ထို့အပြင်၊ အကြောင်းအရာများကို ပြန်လည် အစုအဝေး (Container) အတွင်းသို့ ထည့်သွင်းရန် (Contain) အခက်အခဲများ ရှိသည်။

အခြားတစ်ဖက်တွင် `inline-block` ကို အသုံးပြုသောအခါ၊ အကြောင်းအရာများသည် စာသားများကဲ့သို့ အလျားလိုက် စီစဉ်နိုင်သော်လည်း၊ ၎င်းတို့သည် အလွှာလိုက် (vertical) စီစဉ်မှုများကို ထိန်းချုပ်ရန် အလွန်ရှုပ်ထွေးလာသည်။ အကြောင်းအရာတစ်ခုချင်းစီ၏ အနံ (height) နှင့် အန寬 (width) ကို ထိန်းချုပ်ရန်အတွက် အထပ်ထပ် CSS ကို ရေးသားရန် လိုအပ်လာပြီး၊ အလွန်ရှုပ်ထွေးသော အစီအစဉ်ချမှုများ ဖြစ်ပေါ်လာသည်။ ဤနည်းလမ်းများဖြင့် Layout ကို စီမံခန့်ခွဲရာတွင် လိုအပ်သော အချိန်နှင့် ကြိုးစားအားထုတ်မှုများစွာကို ပေးဆပ်ခဲ့ရသည်။

### 1.2 Layout ပြဿနာ၏ အနှစ်ချုပ်

ဤရှေ့ပြေးခေတ်၏ Layout ပြဿနာများကို အနှစ်ချုပ်ဆိုရလျှင်၊ အဓိကအားဖြင့် Layout များသည် အလိုအလျောက် ပြောင်းလဲမှု (Responsive Design) များနှင့် မကိုက်ညီဘဲ၊ ကျွန်ုပ်တို့သည် Layout တစ်ခုချင်းစီအတွက် အချိန်ကုန်ခံ၍ ကိုယ်တိုင် စီမံခန့်ခွဲရသည်။ ထို့အပြင်၊ ကုဒ်၏ ဖတ်ရှုရလွယ်ကူမှု (Readability) လည်း လျော့နည်းသွားပြီး၊ ရှုပ်ထွေးသော CSS ပုံစံများကြောင့် အမှားအယွင်းများ ဖြစ်ပေါ်နိုင်ခြေ မြင့်တက်လာသည်။

ဤအခက်အခဲများကို ကျော်လွှားရန်အတွက်၊ Layout ကို အလွယ်တကူ၊ ထိရောက်စွာ စီမံခန့်ခွဲနိုင်မည့် ပိုမိုကောင်းမွန်သော စနစ်တစ်ခု လိုအပ်လာသည်။ ထိုလိုအပ်ချက်ကို ဖြည့်ဆည်းပေးမည့် စနစ်မှာ CSS Flexbox ဖြစ်သည်။ Flexbox သည် အကြောင်းအရာများကို တစ်ဖက်တည်း စီစဉ်ရာတွင် အလွန်ထိရောက်သော စွမ်းဆောင်ရည်ကို ပေးစွမ်းနိုင်မည် ဖြစ်သည်။

## 2. Flexbox ၏ အခြေခံသဘောတရားများ (The Core Concepts)

Flexbox ကို နားလည်ရန်အတွက် အခြေခံသဘောတရားများကို မှန်ကန်စွာ သိရှိရန် လိုအပ်သည်။ Flexbox သည် အကြောင်းအရာများကို စီစဉ်ရာတွင် အဓိကအားဖြင့် နှစ်ဖက်စီ (One-dimensional layout) ကို အဓိကထားသည်။

### 2.1 Flexbox ဆိုသည်မှာ အဘယ်နည်း?

Flexbox ဆိုသည်မှာ CSS ၏ Layout စနစ်တစ်ခု ဖြစ်ပြီး၊ အကြောင်းအရာများကို တစ်ဖက်တည်း (အလျားလိုက် သို့မဟုတ် ဒေါင်လိုက်) စီစဉ်ရာတွင် အလွန်ကောင်းမွန်သော စွမ်းဆောင်ရည်ကို ပေးစွမ်းသည်။ Flexbox တွင် အဓိက အခန်းကဏ္ဍနှစ်ခုရှိသည်- **Flex Container** နှင့် **Flex Item** တို့ဖြစ်သည်။

**Flex Container** ဆိုသည်မှာ Layout ကို ထိန်းချုပ်မည့် အဓိက ကွန်တိန်နာ ဖြစ်သည်။ ၎င်းသည် `display: flex;` ဟု သတ်မှတ်ခြင်းဖြင့် မိမိ၏ အတွင်းမှ အကြောင်းအရာများ (Items) အပေါ်တွင် စီမံခန့်ခွဲမှုများ ပြုလုပ်နိုင်သော အဓိက အခန်းကဏ္ဍဖြစ်သည်။

**Flex Item** များသည် Flex Container အတွင်းရှိ အကြောင်းအရာများ ဖြစ်ကြသည်။ ၎င်းတို့သည် Container ၏ စည်းမျဉ်းများကို လိုက်နာပြီး အလိုအလျောက် ပြောင်းလဲနိုင်သော အစိတ်အပိုင်းများ ဖြစ်သည်။ Flexbox ၏ အားသာချက်မှာ Container သည် Item များ၏ အနေအထားနှင့် အရွယ်အစားကို အလိုအလျောက် ချိန်ညှိပေးနိုင်ခြင်းပင် ဖြစ်သည်။

### 2.2 Flexbox ၏ အဓိက အကျိုးကျေးဇူးများ

Flexbox ကို အသုံးပြုခြင်းဖြင့် ကျွန်ုပ်တို့သည် အောက်ပါ အကျိုးကျေးဇူးများကို ရရှိစေသည်။

ပထမအချက်မှာ **One-dimensional Layout** အတွက် အကောင်းဆုံးဖြစ်ခြင်း ဖြစ်သည်။ Flexbox သည် အလျားလိုက် (Row) သို့မဟုတ် ဒေါင်လိုက် (Column) စီစဉ်မှုများကို အလွန်လွယ်ကူစွာ ထိန်းချုပ်ပေးနိုင်ပြီး၊ ထိုအချက်များသည် အကြောင်းအရာများကို တစ်ဖက်တည်း စီစဉ်ရာတွင် အလွန်ထိရောက်သည်။

ဒုတိယအချက်မှာ **Dynamic Layouts** များ ဖန်တီးနိုင်ခြင်း ဖြစ်သည်။ Layout များသည် အကြောင်းအရာများ၏ အရွယ်အစား၊ အကွာအဝေးနှင့် စီစဉ်မှုကို အလိုအလျောက် ပြောင်းလဲနိုင်သောကြောင့်၊ အသုံးပြုသူ၏ စက်ပစ္စည်းအရွယ်အစား (Screen Size) အလိုက် Layout များ အလိုအလျောက် ပြောင်းလဲသွားသည် (Responsive) ပုံစံမျိုးကို အလွယ်တကူ ဖန်တီးနိုင်သည်။

တတိယအချက်မှာ **Responsive Design** အတွက် အထောက်အကူပြုခြင်း ဖြစ်သည်။ Flexbox သည် အကြောင်းအရာများကို တစ်ခုနှင့်တစ်ခု အညီအမျှ ဖြန့်ကျက်ခြင်းနှင့် အညီအမျှ စီစဉ်ခြင်းတို့ကို အလွယ်တကူ လုပ်ဆောင်ပေးနိုင်သောကြောင့်၊ မည်သည့် စက်ပစ္စည်းပေါ်တွင်မဆို လှပသော Layout များ တည်ရှိစေသည်။

## 3. Flexbox ၏ အဓိက Property များ (The Essential Properties)

Flexbox ၏ စွမ်းဆောင်ရည်ကို အပြည့်အဝ အသုံးချနိုင်ရန်အတွက် အဓိက Property များအကြောင်းကို နားလည်ရန် လိုအပ်သည်။ ဤ Property များသည် Flex Container နှင့် Flex Item များအပေါ် မည်သို့ သက်ရောက်သည်ကို သိရှိရန် အရေးကြီးသည်။

### 3.1 Direction နှင့် Alignment (အ దిశနှင့် စီစဉ်မှု)

Flexbox ၏ အခြေခံအကျဆုံး Property များမှာ အကြောင်းအရာများ မည်သို့ စီစဉ်မလဲနှင့် မည်သို့ ချိန်ညှိမလဲ ဆိုသည်ကို သတ်မှတ်ခြင်းတို့ ဖြစ်သည်။

`flex-direction`: ဤ Property သည် Flex Container အတွင်းရှိ အကြောင်းအရာများကို အလျားလိုက် (row) သို့မဟုတ် ဒေါင်လိုက် (column) ဖြင့် စီစဉ်မည့် အ దిశကို သတ်မှတ်သည်။ ၎င်းကို `row` (အလျားလိုက်) သို့မဟုတ် `column` (ဒေါင်လိုက်) အဖြစ် သတ်မှတ်နိုင်သည်။

`justify-content`: ဤ Property သည် Main Axis (အဓိက အဝင်အထွက်) ပေါ်တွင် အကြောင်းအရာများကို မည်သို့ ဖြန့်ကျက်မလဲ၊ အလယ်တည့်တည့်ထားမလဲ စသည်တို့ကို ထိန်းချုပ်သည်။ ဥပမာအားဖြင့်၊ အလယ်တွင်ထားရန် (`center`)၊ အစွန်းတွင်ထားရန် (`flex-start` သို့မဟုတ် `flex-end`)၊ သို့မဟုတ် အညီအမျှ ဖြန့်ကျက်ရန် (`space-between`, `space-around`) စသည်တို့ကို သတ်မှတ်နိုင်သည်။

`align-items`: ဤ Property သည် Cross Axis (ဒုတိယ အဝင်အထွက်) ပေါ်တွင် အကြောင်းအရာများကို မည်သို့ အညီအမျှ စီစဉ်မလဲ၊ အထူးသဖြင့် ဒေါင်လိုက် (vertical) စီစဉ်မှုကို ထိန်းချုပ်သည်။ ၎င်းကို `stretch` (အလိုအလျောက် ဆန့်တန်းခြင်း) သို့မဟုတ် `center` (အလယ်တွင်ထားခြင်း) စသည်ဖြင့် သတ်မှတ်နိုင်သည်။

### 3.2 Item Size နှင့် Growth (အကြောင်းအရာ အရွယ်အစားနှင့် ကြီးထွားမှု)

Item များ၏ အရွယ်အစားနှင့် ကြီးထွားမှုများကို ထိန်းချုပ်ရန်အတွက် အောက်ပါ Property များသည် အလွန်အရေးကြီးသည်။

`flex-grow`: ဤ Property သည် ကျန်ရှိသော နေရာလွတ်ကို Item တစ်ခုက မည်မျှ ယူမလဲကို သတ်မှတ်သည်။ ၎င်းကို ပုံသေနံ (value) အဖြစ် ဂဏန်း (ဥပမာ- `1`, `2`) ဖြင့် သတ်မှတ်ပေးနိုင်သည်။ ဥပမာအားဖြင့်၊ အခြား Item များထက် ပိုမိုကျယ်လာစေလိုပါက `flex-grow: 1` ဟု သတ်မှတ်နိုင်သည်။

`flex-shrink`: ဤ Property သည် Flex Container ၏ နေရာကျဉ်းလာသောအခါ Item တစ်ခုက မည်မျှ ကျုံ့မလဲကို ထိန်းချုပ်သည်။ ၎င်းသည် အလိုအလျောက် လုပ်ဆောင်ပေးပြီး Item များ မကျုံ့ဘဲ ကျုံ့ရန် ကြိုးစားသည်။

`flex-basis`: ဤ Property သည် Item တစ်ခု၏ အစအဦး အရွယ်အစားကို သတ်မှတ်သည်။ ၎င်းသည် Item များ၏ စတင်သော အကျယ် (width) သို့မဟုတ် အမြင့် (height) ကို သတ်မှတ်ရာတွင် အသုံးပြုသည်။

### 3.3 Space Management (နေရာလွတ် စီမံခန့်ခွဲမှု)

Item များကြားရှိ အကွာအဝေးများကို ထိန်းချုပ်ရန်အတွက် Flexbox သည် အလွန်အသုံးဝင်သည်။

`flex-wrap`: ဤ Property သည် အကြောင်းအရာများ တစ်ကြောင်းတည်းတွင် မကျရောက်ပါက ဘယ်သို့ ဆက်သွားမလဲကို သတ်မှတ်သည်။ ၎င်းကို `nowrap` (တစ်ကြောင်းတည်းထားရန်) သို့မဟုတ် `wrap` (အပေါ်သို့ အလွှာလိုက် ဆက်ရန်) အဖြစ် သတ်မှတ်နိုင်သည်။ ဤအချက်သည် Responsive Layout များအတွက် အလွန်အရေးကြီးသည်။

`gap`: နောက်ဆုံးပေါ်တွင် ထပ်မံဖြည့်ဆည်းရန်အတွက် `gap` Property ကို အသုံးပြုသည်။ ဤ Property သည် Item များကြားရှိ အကွာအဝေးကို အလွန်လွယ်ကူစွာ ထိန်းချုပ်ပေးသည်။ ၎င်းသည် `margin` ကို အသုံးပြု၍ အကွာအဝေးကို ထိန်းချုပ်ရမည့် ရှုပ်ထွေးမှုကို လျှော့ချပေးသည်။

## 4. လက်တွေ့ လေ့ကျင့်ခန်းများ (Practical Implementation)

အခြေခံသဘောတရားများနှင့် Property များကို နားလည်ပြီဆိုလျှင်၊ ယခုအခါ ထိုအင်ဂျင်များကို လက်တွေ့တွင် အသုံးချပြီး Layout များ ဖန်တီးကြည့်ကြမည် ဖြစ်သည်။

### 4.1 လက်တွေ့ လေ့ကျင့်ခန်း (Exercise 1): Navigation Bar ဖန်တီးခြင်း

ပထမဆုံးအနေဖြင့်၊ Flexbox ကို အသုံးပြု၍ Navigation Bar တစ်ခုကို အလွယ်တကူ စီစဉ်ကြည့်ရအောင်။ Navigation Bar များသည် အလျားလိုက် စီစဉ်ပြီး၊ လိုအပ်သော နေရာလွတ်များကို အညီအမျှ ဖြန့်ကျက်ရန် လိုအပ်သည်။

**ရည်မှန်းချက်:** Flexbox ကို အသုံးပြု၍ Navigation Bar တစ်ခုကို အလွယ်တကူ စီစဉ်ခြင်း။

**အသုံးပြုမည့် Property များ:** `display: flex;`၊ `justify-content` (space distribution)၊ `align-items` (vertical alignment)။

ဤလေ့ကျင့်ခန်းတွင်၊ Navigation Bar ကို Flex Container အဖြစ် သတ်မှတ်ပြီး၊ လင့်ခ်များကို အလျားလိုက် စီစဉ်ရန် `display: flex;` ကို အသုံးပြုသည်။ ထို့နောက်၊ လင့်ခ်များကို ဘယ်ဘက်တွင် စုစည်းရန်နှင့် ညာဘက်တွင် စုစည်းရန်အတွက် `justify-content: space-between;` ကို အသုံးပြုသည်။ ထို့အပြင်၊ လင့်ခ်များ၏ အမြင့်ကို အညီအမျှ ထိန်းရန်အတွက် `align-items: center;` ကို သတ်မှတ်လိုက်သည်။ ဤနည်းဖြင့်၊ ကျွန်ုပ်တို့သည် ရှုပ်ထွေးသော `float` သို့မဟုတ် `margin` များဖြင့် စီစဉ်ရမည့်အစား၊ အလွန်ရိုးရှင်းသော ကုဒ်ဖြင့် လှပသော Navigation Bar တစ်ခုကို ရရှိခဲ့သည်။ ဤလေ့ကျင့်ခန်းမှတစ်ဆင့်၊ Flexbox သည် အခြေခံ Layout များအတွက် မည်မျှ အကျိုးရှိကြောင်းကို လက်တွေ့ မြင်တွေ့ရမည် ဖြစ်သည်။

### 4.2 လက်တွေ့ လေ့ကျင့်ခန်း (Exercise 2): Card Layout ဖန်တီးခြင်း

နောက်တစ်ဆင့်အနေဖြင့်၊ Flexbox နှင့် `flex-wrap`, `gap` တို့ကို အသုံးပြု၍ ကတ် (Cards) များ အလွှာလိုက် (Row) နှင့် အလိုအလျောက် စီစဉ်ခြင်းကို လေ့ကျင့်ကြည့်ကြရအောင်။

**ရည်မှန်းချက်:** Flexbox နှင့် `flex-wrap`, `gap` ကို အသုံးပြု၍ ကတ် (Cards) များ အလွှာလိုက် (Row) နှင့် အလိုအလျောက် စီစဉ်ခြင်း။

ဤလေ့ကျင့်ခန်းတွင်၊ ကတ်များကို Flex Container အဖြစ် သတ်မှတ်ပြီး၊ ကတ်များ အလွှာလိုက် စီစဉ်ရန်အတွက် `flex-wrap: wrap;` ကို အသုံးပြုသည်။ ထို့နောက်၊ ကတ်များကြားရှိ အကွာအဝေးကို ထိန်းချုပ်ရန်အတွက် `gap` Property ကို အသုံးပြုလိုက်သည်။ ဤနည်းလမ်းဖြင့်၊ ကတ်များသည် နေရာကျဉ်းလာသည့်အခါတွင် အလိုအလျောက် အပေါ်သို့ အလွှာလိုက် ဆက်သွားပြီး၊ ကတ်များကြားတွင် လိုအပ်သော အကွာအဝေးကို အလိုအလျောက် ထိန်းချုပ်ပေးသည်။ ဤလေ့ကျင့်ခန်းမှတစ်ဆင့်၊ Flexbox သည် အကြောင်းအရာများ၏ အရွယ်အစားနှင့် အကွာအဝေးများကို စီမံခန့်ခွဲရာတွင် မည်မျှ ထိရောက်ကြောင်းကို သိရှိရမည် ဖြစ်သည်။

## 5. နိဂုံးနှင့် အနှစ်ချုပ် (Conclusion and Summary)

### 5.1 Flexbox ၏ အကျဉ်းချုပ် ပြန်လည်သုံးသပ်ခြင်း

ကျွန်ုပ်တို့ ယနေ့ သင်ယူခဲ့သော Flexbox ၏ အနှစ်ချုပ်ကို ပြန်လည်သုံးသပ်ကြည့်ရအောင်။ Layout ပြောင်းလဲမှု၏ ရှေ့ခေတ်မှ (Float မှ Flexbox သို့) ကျွန်ုပ်တို့သည် အလွှာလိုက် စီစဉ်မှုများကို ထိန်းချုပ်ရန်အတွက် အချိန်ကုန်ခံရသည်။ Flexbox သည် ဤရှုပ်ထွေးမှုကို အလွယ်တကူ၊ ထိရောက်စွာ စီမံခန့်ခွဲနိုင်သော စွမ်းဆောင်ရည်ကို ပေးစွမ်းသည်။ အခြေခံအားဖြင့် Flexbox သည် **Flex Container** နှင့် **Flex Item** တို့၏ ဆက်စပ်မှုကို အခြေခံသည်။

အဓိက Property များမှာ- အကြောင်းအရာများ၏ အနေအထားနှင့် စီစဉ်မှုအတွက် `flex-direction`, `justify-content`, `align-items` တို့ဖြစ်သည်။ ထို့အပြင် Item များ၏ အရွယ်အစားနှင့် ကြီးထွားမှုအတွက် `flex-grow`, `flex-shrink`, နှင့် `flex-basis` တို့ကို အသုံးပြုသည်။ ထို့အပြင် နေရာလွတ်များကို စီမံခန့်ခွဲရန်အတွက် `flex-wrap` နှင့် `gap` တို့သည် အလွန်အသုံးဝင်သည်။

### 5.2 ရှေ့ဆက်မည့် လမ်းကြောင်း

Flexbox သည် Web Layout များအတွက် မရှိမဖြစ် လိုအပ်သော စွမ်းဆောင်ရည်တစ်ခု ဖြစ်ကြောင်း ကျွန်ုပ်တို့ လက်တွေ့ လေ့ကျင့်ခန်းများမှတစ်ဆင့် မြင်တွေ့ခဲ့ရသည်။ ၎င်းသည် လှပသော၊ ထိရောက်သော နှင့် Responsive ဖြစ်သော Layout များကို ဖန်တီးနိုင်သည့် အင်ဂျင်တစ်ခု ဖြစ်သည်။

Flexbox ကို ကျွမ်းကျင်စွာ အသုံးပြုပြီးပါက၊ ကျွန်ုပ်တို့သည် CSS Layout ၏ နောက်ထပ် အဆင့်များသို့ ကူးပြောင်းနိုင်ရန် အသင့်ရှိနေပြီ ဖြစ်သည်။ နောက်အခန်းများတွင်၊ ကျွန်ုပ်တို့သည် Flexbox မှ Grid စနစ်သို့ ကူးပြောင်းခြင်းနှင့် CSS Layout ၏ နောက်ထပ် အဆင့်များအကြောင်းကို ဆက်လက်လေ့လာသွားမည် ဖြစ်သည်။

**Chapter Summary:** CSS Flexbox သည် ခေတ်မီ Web Layout များအတွက် မရှိမဖြစ် လိုအပ်သော စွမ်းဆောင်ရည်ဖြစ်ကြောင်း အလေးပေးဖော်ပြပါသည်။ ၎င်းသည် Layout များအား တစ်ဖက်တည်း စီစဉ်ရာတွင် အလွန်ထိရောက်ပြီး၊ Dynamic နှင့် Responsive ဖြစ်သော ဒီဇိုင်းများကို အလွယ်တကူ ဖန်တီးနိုင်စေသည်။
