# Chapter 17: CSS Animation နှင့် Transition — ဝဘ်ကို သက်ဝင်လှုပ်ရှားစေခြင်း

သင့်ရဲ့ Static (ပုံသေ) ဝဘ်စာမျက်နှာတွေကို ဘယ်လို ပိုမိုဆွဲဆောင်မှုရှိပြီး အသက်ဝင်လာအောင် ပြောင်းလဲနိုင်မလဲ? ဒီအခန်းမှာ CSS ရဲ့ အစွမ်းထက်တဲ့ Animation နဲ့ Transition နည်းပညာတွေကို အသုံးပြုပြီး သင့်ရဲ့ ဝဘ်ဒီဇိုင်းတွေကို မြင်တွေ့ရမယ့်အထိ လှုပ်ရှားနေစေမယ့် နည်းလမ်းတွေကို လက်တွေ့လေ့ကျင့်ပြီး သင်ယူကြပါမယ်။ ဒီနည်းပညာတွေဟာ သင့်ရဲ့ ဝဘ်စာမျက်နှာတွေကို ရိုးရှင်းတဲ့ စာသားတွေအပြင် အမြင်အာရုံနဲ့ စိတ်ခံစားမှုကိုပါ ပေးစွမ်းနိုင်တဲ့ အင်မတန် အရေးကြီးတဲ့ အစိတ်အပိုင်းတွေ ဖြစ်ပါတယ်။

ဒီအခန်းမှာ ကျွန်ုပ်တို့သည် CSS Transition နှင့် CSS Animation တို့၏ အခြေခံသဘောတရားများ၊ ၎င်းတို့၏ ကွာခြားချက်များ၊ နှင့် လက်တွေ့တွင် မည်သို့အသုံးပြုရမည်ကို အသေးစိတ်လေ့လာသွားကြပါမည်။ HTML နှင့် CSS အခြေခံဗဟုသုတများ ရှိရမည်ဖြစ်ပြီး၊ ဤအခန်းကို အကောင်းဆုံး နားလည်နိုင်ရန်အတွက် အခြေခံများမှ စတင်၍ လက်တွေ့လေ့ကျင့်ခန်းများမှတစ်ဆင့် အသုံးချပုံများကို လေ့လာသွားမည် ဖြစ်ပါသည်။

## CSS Transition: ချောမွေ့သော ပြောင်းလဲမှုများ (Smooth Changes with CSS Transitions)

CSS Transition ဆိုသည်မှာ CSS သတ်မှတ်ထားသော property များတစ်ခုမှ အခြား property တစ်ခုသို့ ပြောင်းလဲသည့်အခါ ထိုပြောင်းလဲမှုများအား တစ်ကြိမ်တည်း မပြောင်းဘဲ အချိန်ပေးပြီး ချောမွေ့စွာ ပြောင်းလဲသွားစေသည့် နည်းပညာတစ်ခု ဖြစ်ပါတယ်။ ၎င်း၏ အဓိက ရည်ရွယ်ချက်မှာ အသုံးပြုသူများအတွက် အမြင်အာရုံအရ ပိုမိုကောင်းမွန်ပြီး သက်တောင့်သက်သာရှိသော အတွေ့အကြုံကို ပေးစွမ်းရန် ဖြစ်သည်။

### Transition Property များ

Transition ကို အသုံးပြုရန်အတွက် အဓိကအားဖြင့် အောက်ပါ property များနှင့် တန်ဖိုးများကို သတ်မှတ်ပေးရပါမည်။

*   **`transition`**: ဤ property သည် Transition ကို စတင်ရန်အတွက် အဓိကကျသော အချက်ဖြစ်သည်။ ၎င်းသည် မည်သည့် property များအပေါ် Transition ကို အသက်ဝင်စေမည်ကို သတ်မှတ်ပေးသည်။
*   **`transition-property`**: မည်သည့် CSS property ကို ပြောင်းလဲမည်ကို သတ်မှတ်ပေးသည်။ ဥပမာအားဖြင့် `background-color`၊ `width`၊ သို့မဟုတ် `opacity` စသည်တို့ကို သတ်မှတ်နိုင်သည်။
*   **`transition-duration`**: ပြောင်းလဲမှု ဖြစ်ပေါ်မည့် အချိန်ကာလကို သတ်မှတ်ပေးသည်။ ၎င်းကို စက္ကန့် (seconds) သို့မဟုတ် မီလီစက္ကန့် (milliseconds) ဖြင့် ဖော်ပြရမည်။ ဥပမာအားဖြင့် `transition-duration: 0.5s;` ဟု ရေးခြင်းသည် ပြောင်းလဲမှုသည် ၀.၅ စက္ကန့်အတွင်း ပြီးဆုံးသွားမည်ဟု ဆိုလိုသည်။
*   **`transition-timing-function`**: ပြောင်းလဲမှု၏ အရှိန်အဟုန်ကို ထိန်းချုပ်သည်။ ၎င်းသည် ပြောင်းလဲမှု မည်သို့ စတင်မည်၊ မည်သို့ အရှိန်အဟုန်ဖြင့် ဖြစ်ပေါ်မည်ကို ထိန်းချုပ်ပေးသည်။ အသုံးများသော function များမှာ `ease` (အစနှင့်အဆုံး ဖြည်းဖြည်းချင်း)၊ `linear` (တသမတ်တည်း အရှိန်)၊ `ease-in` (အစက နှေးပြီး အဆုံးမှာ မြန်)၊ `ease-out` (အစမှာ မြန်ပြီး အဆုံးမှာ နှေး) တို့ ဖြစ်ကြသည်။
*   **`transition-delay`**: ပြောင်းလဲမှု စတင်မည့် အချိန်ကို ထိန်းချုပ်သည်။ ဤ property ကို အသုံးပြုခြင်းဖြင့် အခြားအရာများနှင့် ပေါင်းစပ်၍ လိုအပ်သည့် အချိန်အကွာအဝေးကို ထိန်းချုပ်နိုင်သည်။

### လက်တွေ့လေ့ကျင့်ခန်းများဖြင့် Transition ကို အသုံးချခြင်း

Transition ကို လက်တွေ့တွင် အသုံးပြုပုံကို နားလည်စေရန် အောက်ပါလေ့ကျင့်ခန်းများကို ဆောင်ရွက်ကြည့်ရအောင်။

**Hover Effect (Mouse ကိုတင်လိုက်သည့်အခါ ပြောင်းလဲခြင်း):**
အသုံးအများဆုံး Transition အသုံးပြုပုံမှာ Mouse ကို Element ပေါ်သို့ တင်လိုက်သည့်အခါ အကြောင်းအရာများ ချောမွေ့စွာ ပြောင်းလဲခြင်း (Hover Effect) ဖြစ်သည်။ ဥပမာအားဖြင့်၊ မောက်ကိုတင်လိုက်သည့်အခါ Background Color ကို ဖြည်းဖြည်းချင်း ပြောင်းလဲခြင်းဖြင့် ပိုမိုဆွဲဆောင်မှုရှိသော အတွေ့အကြုံကို ဖန်တီးနိုင်သည်။

**Smooth Width/Height Change:**
Box တစ်ခု၏ အရွယ်အစားကို Hover လုပ်လိုက်သည့်အခါ အရှိန်အဟုန်မရှိဘဲ ဖြည်းဖြည်းချင်း ချဲ့ထွင်ခြင်း (Smooth Scaling) ကိုလည်း Transition ဖြင့် အကောင်အထည်ဖော်နိုင်သည်။ ၎င်းသည် အသုံးပြုသူအား စာမျက်နှာပေါ်ရှိ အရာဝတ္ထုများနှင့် အပြန်အလှန် ဆက်သွယ်မှုရှိနေသည်ဟု ခံစားရစေသည်။

**Slide Animation (Basic):**
Transition ကို အသုံးပြု၍ Element တစ်ခုကို ဘေးတိုက် ရွှေ့ခြင်း (Slide Animation) ကိုလည်း လုပ်ဆောင်နိုင်သည်။ ၎င်းသည် Element တစ်ခုသည် နေရာတစ်ခုမှ နောက်နေရာတစ်ခုသို့ ချောမွေ့စွာ ရွေ့လျားသွားပုံကို ပြသပေးနိုင်သည်။

### Transition မှ Animation သို့ ကူးပြောင်းခြင်း

Transition နှင့် Animation တို့သည် အလွန်နီးစပ်သော နည်းပညာများဖြစ်သော်လည်း ၎င်းတို့၏ ရည်ရွယ်ချက်နှင့် အသုံးပြုပုံတွင် အဓိက ကွာခြားမှုများ ရှိပါသည်။ **Transition** သည် အခြေအနေတစ်ခုမှ အခြားအခြေအနေတစ်ခုသို့ **တစ်ကြိမ်တည်း** ပြောင်းလဲသည့်အခါ ထိုပြောင်းလဲမှုကို ချောမွေ့စေရန် အသုံးပြုသည်။ ၎င်းသည် အဖြစ်အပျက်တစ်ခု (Event) နှင့် တိုက်ရိုက်ဆက်စပ်နေသည်။

ယင်းနှင့် ဆန့်ကျင်ဘက်အားဖြင့် **Animation** သည် အချိန်ကြာရှည်စွာ **လည်ပတ်နေသော**၊ ပိုမိုရှုပ်ထွေးသော လှုပ်ရှားမှုများ (ဥပမာ- အချိန်နှင့်အမျှ ပုံစံပြောင်းလဲခြင်း သို့မဟုတ် အကြိမ်ပေါင်းများစွာ ထပ်ခါထပ်ခါ လုပ်ဆောင်ခြင်း) အတွက် အသုံးပြုသည်။ Animation များသည် `@keyframes` စနစ်ကို အခြေခံ၍ ဖန်တီးပြီး ပိုမိုရှုပ်ထွေးသော လှုပ်ရှားမှုများ ဖန်တီးနိုင်စွမ်းရှိသည်။

## CSS Animation: @keyframes ဖြင့် လှုပ်ရှားမှု ဖန်တီးခြင်း (Creating Motion with @keyframes)

CSS Animation သည် အချိန်နှင့်အမျှ အသွင်အပြင်များ ပြောင်းလဲနေသော၊ ပိုမိုရှုပ်ထွေးသော လှုပ်ရှားမှုများကို ဖန်တီးရာတွင် အသုံးဝင်သည်။ ၎င်းကို အဓိကအားဖြင့် `@keyframes` စနစ်ကို အခြေခံ၍ အသုံးပြုသည်။

### @keyframes စနစ်

`@keyframes` စနစ်သည် Animation များအတွက် အခြေခံ Framework ဖြစ်သည်။ ၎င်းသည် သတ်မှတ်ထားသော အချိန်ကာလအတွင်း Element တစ်ခု၏ အခြေအနေများကို အဆင့်ဆင့် (Keyframes) သတ်မှတ်ပေးသည်။

*   **Keyframes များ၏ အခန်းကဏ္ဍ:** `@keyframes` အတွင်း၌ 0% မှ 100% အထိ အဆင့်များကို သတ်မှတ်ပေးရသည်။ ဤအဆင့်များသည် Animation တစ်ခုလုံး၏ အစ၊ အလယ်၊ အဆုံး အခြေအနေများကို ကိုယ်စားပြုသည်။ ဥပမာအားဖြင့်၊ `0%` တွင် အစအခြေအနေနှင့် `100%` တွင် နောက်ဆုံးအခြေအနေကို သတ်မှတ်နိုင်သည်။

### Animation Property များ

`@keyframes` ကို အသုံးပြု၍ Animation များကို အသက်ဝင်စေရန်အတွက် အောက်ပါ Property များကို သတ်မှတ်ရမည်။

*   **`animation-name`**: မည်သည့် `@keyframes` ကို အသုံးပြုမည်ကို သတ်မှတ်ပေးသည်။ ၎င်းသည် မည်သည့် လှုပ်ရှားမှုကို အသုံးပြုမည်ကို ညွှန်ပြသည်။
*   **`animation-duration`**: Animation တစ်ခုလုံး မည်မျှကြာမည်ကို သတ်မှတ်ပေးသည်။ ၎င်းသည် Transition ၏ `transition-duration` နှင့် ဆင်တူသော်လည်း Animation အတွက် အသုံးပြုသည်။
*   **`animation-iteration-count`**: Animation ကို မည်မျှအကြိမ် ထပ်ခါထပ်ခါ လုပ်ဆောင်မည်ကို သတ်မှတ်ပေးသည်။ ဥပမာအားဖြင့်၊ `infinite` ဟု သတ်မှတ်ပါက Animation သည် အဆက်မပြတ် လည်ပတ်နေမည်ဖြစ်သည်။
*   **`animation-direction`**: လှုပ်ရှားမှု၏ ဦးတည်ရာကို ထိန်းချုပ်သည်။ `forwards` (အဆုံးအထိ ဆက်လက်လုပ်ဆောင်မည်)၊ `backwards` (အစအထိ ဆက်လက်လုပ်ဆောင်မည်)၊ သို့မဟုတ် `alternate` (အစနှင့်အဆုံးကို ပြောင်းပြန်လုပ်ဆောင်မည်) စသည်တို့ကို အသုံးပြုနိုင်သည်။

### Animation ကို အဆင့်မြှင့်တင်ခြင်း

Animation များကို ပိုမိုကောင်းမွန်စွာ ဖန်တီးနိုင်ရန်အတွက် Timing Functions များကို အသုံးပြုခြင်းသည် အလွန်အရေးကြီးသည်။

*   **Looping Animation:** အဆက်မပြတ် လည်ပတ်နေသော Animation များ ဖန်တီးရန် `animation-iteration-count: infinite;` ကို အသုံးပြုသည်။
*   **Timing Functions ၏ အသုံးပြုပုံ:** Timing Functions များသည် လှုပ်ရှားမှု၏ အရှိန်အဟုန်ကို ထိန်းချုပ်ပေးသည်။ `ease-in` သည် အစတွင် နှေးပြီး အဆုံးတွင် အရှိန်ဖြင့် ပြောင်းလဲသည်။ `ease-out` သည် အစတွင် အရှိန်ဖြင့် စတင်ပြီး အဆုံးတွင် နှေးသွားသည်။ `ease-in-out` သည် အစနှင့်အဆုံးတွင် ညီမျှသော အရှိန်ဖြင့် လှုပ်ရှားသည်။ `linear` သည် တသမတ်တည်း အရှိန်ဖြင့် လှုပ်ရှားသည်။ ဤ Function များဖြင့် လှုပ်ရှားမှုပုံစံကို ကိုယ်တိုင် ထိန်းချုပ်နိုင်ပြီး ပိုမိုစိတ်ဝင်စားဖွယ် ဖြစ်စေသည်။

### လက်တွေ့လေ့ကျင့်ခန်းများဖြင့် Animation ကို အသုံးချခြင်း

Animation များကို လက်တွေ့တွင် အသုံးပြုပုံကို နားလည်စေရန် အောက်ပါလေ့ကျင့်ခန်းများကို ဆောင်ရွက်ကြည့်ရအောင်။

**Color Cycling (အရောင်များ အစဉ်လိုက် ပြောင်းလဲခြင်း):**
`@keyframes` ကို အသုံးပြု၍ Background Color ကို အချိန်နှင့်အမျှ အစဉ်လိုက် ပြောင်းလဲခြင်းဖြင့် ဆွဲဆောင်မှုရှိသော အသွင်အပြင်ကို ဖန်တီးနိုင်သည်။ ဥပမာအားဖြင့်၊ အစတွင် အနီရောင်မှ စတင်၍ အဆုံးတွင် အပြာရောင်သို့ အစဉ်လိုက် ပြောင်းလဲစေနိုင်သည်။

**Pulse Effect (ရောင်ခြည်ထွက်ခြင်း):**
Element တစ်ခုကို အချိန်နှင့်အမျှ အရွယ်အစား (Scale) ကို တိုးတက်စေခြင်းဖြင့် "Pulse" သို့မဟုတ် "Breathe" အကျိုးသက်ရောက်မှုကို ဖန်တီးနိုင်သည်။ ၎င်းသည် Element တစ်ခုသည် အသက်ဝင်နေပုံကို ပေးစွမ်းနိုင်သည်။

## CSS Transform: ပုံသဏ္ဍာန် ပြောင်းလဲခြင်း (Manipulating Shape with CSS Transforms)

Transition နှင့် Animation များက အခြေအနေနှင့် အချိန်အလိုက် ပြောင်းလဲမှုကို ထိန်းချုပ်ပေးသော်လည်း၊ CSS Transform များသည် Element များ၏ **ပုံသဏ္ဍာန်နှင့် နေရာကို တိုက်ရိုက် ပြောင်းလဲပေးခြင်း** အတွက် အသုံးပြုသည်။ ၎င်းသည် Element ၏ မျက်နှာပြင်ပေါ်ရှိ အနေအထားကို လှုပ်ရှားစေသည်။

### Transform Property များ၏ အခန်းကဏ္ဍ

Transform Property များသည် CSS Element များ၏ အနေအထားနှင့် ပုံသဏ္ဍာန်ကို ပြောင်းလဲရန် အဓိကကျသည်။

*   **`translate(x, y)`**: ဤ function သည် Element ကို X နှင့် Y ဝင်ရိုးများပေါ်တွင် ရွှေ့ခြင်း (Movement) ကို ပြုလုပ်သည်။ `translate(100px, 50px)` ဟု ရေးလိုက်ပါက Element သည် ဘယ်ဘက်သို့ ၁၀၀ ပစ်နှင့် အပေါ်သို့ ၅၀ ပစ် ရွှေ့သွားမည်ဖြစ်သည်။
*   **`rotate(angle)`**: ဤ function သည် Element ကို သတ်မှတ်ထားသော ဒီဂရီ (degree) ပမာဏဖြင့် လှည့်ခြင်း (Rotation) ကို ပြုလုပ်သည်။ ဥပမာအားဖြင့်၊ `rotate(180deg)` ဟု ရေးလိုက်ပါက Element သည် ၁၈၀ ဒီဂရီ လှည့်သွားမည်ဖြစ်သည်။
*   **`scale(x, y)`**: ဤ function သည် Element ကို ချဲ့ထွင်ခြင်း (Enlarge) သို့မဟုတ် ကျုံ့ခြင်း (Shrink) ကို ပြုလုပ်သည်။ `scale(2, 1)` ဟု ရေးလိုက်ပါက Element သည် X အဝင်ရိုးတွင် နှစ်ဆနှင့် Y အဝင်ရိုးတွင် တစ်ဆယ်အထိ ချဲ့ထွင်သွားမည်ဖြစ်သည်။
*   **`skew(x-angle, y-angle)`**: ဤ function သည် Element ကို စောင်းခြင်း (Skewing) ကို ပြုလုပ်သည်။ ၎င်းသည် Element ကို အလျားလိုက် သို့မဟုတ် ဒေါင်လိုက် စောင်းစေသည်။

### Transform ကို Animation နှင့် ပေါင်းစပ်ခြင်း

Transform Property များကို `@keyframes` များနှင့် ပေါင်းစပ်၍ ပိုမိုရှုပ်ထွေးပြီး စိတ်ဝင်စားဖွယ်ကောင်းသော လှုပ်ရှားမှုများ ဖန်တီးနိုင်သည်။ ဥပမာအားဖြင့်၊ Element တစ်ခုကို လှည့်ရင်း (Rotate)၊ ချဲ့၍ (Scale)၊ ရွှေ့ခြင်း (Translate) တို့ကို တစ်ပြိုင်နက်တည်း Animation လုပ်ခြင်းဖြင့် 3D ပုံစံမျိုးသော လှုပ်ရှားမှုများကို ဖန်တီးနိုင်သည်။ ဤနည်းလမ်းသည် CSS ကို အသုံးပြု၍ ပုံသဏ္ဍာန်ကို အချိန်နှင့်အမျှ ပြောင်းလဲပေးနိုင်သည့် အစွမ်းကို ပေးသည်။

### လက်တွေ့လေ့ကျင့်ခန်းများဖြင့် Transform ကို အသုံးချခြင်း

**3D Effect ဖန်တီးခြင်း:**
အထက်ပါ Transform Functions များကို Animation နှင့် ပေါင်းစပ်ခြင်းဖြင့် Element တစ်ခုကို လှည့်၍ (Rotate)၊ ချဲ့၍ (Scale)၊ ရွှေ့ခြင်း (Translate) တို့ကို တစ်ပြိုင်နက်တည်း Animation လုပ်ခြင်းဖြင့် ပိုမိုရှုပ်ထွေးသော 3D အကျိုးသက်ရောက်မှုများကို ဖန်တီးနိုင်သည်။ ဤနည်းပညာသည် ဝဘ်ဒီဇိုင်းကို ပိုမိုနက်ရှိုင်းပြီး စိတ်ဝင်စားဖွယ် ဖြစ်စေသည်။

## အနှစ်ချုပ်နှင့် လက်တွေ့အသုံးချခြင်း (Conclusion and Practical Application)

ယခုအခါတွင် CSS Transition နှင့် Animation တို့၏ အခြေခံသဘောတရားများ၊ လက်တွေ့အသုံးချပုံများနှင့် အသုံးအဆောင် နည်းပညာများကို လေ့လာပြီး ဖြစ်ပါသည်။

### Transition နှင့် Animation နှိုင်းယှဉ်ချက်

Transition နှင့် Animation တို့ကို ပြန်လည် အကျဉ်းချုပ် နှိုင်းယှဉ်ကြည့်ရအောင်။

*   **Transition:** ၎င်းသည် **တစ်ကြိမ်တည်း ပြောင်းလဲမှုများ** အတွက် အသုံးပြုသည်။ ၎င်းသည် အဖြစ်အပျက်တစ်ခု (ဥပမာ- Hover) ဖြစ်ပေါ်သည့်အခါ အခြေအနေတစ်ခုမှ အခြားတစ်ခုသို့ **ချောမွေ့စွာ** ပြောင်းလဲစေသည်။ ၎င်းသည် အချိန်တိုအတွင်း ဖြစ်ပေါ်သည်။
*   **Animation:** ၎င်းသည် **အချိန်ကြာရှည်စွာ လည်ပတ်နေသော**၊ ပိုမိုရှုပ်ထွေးသော လှုပ်ရှားမှုများအတွက် အသုံးပြုသည်။ ၎င်းသည် `@keyframes` ကို အခြေခံ၍ အချိန်နှင့်အမျှ အဆင့်ဆင့် လှုပ်ရှားမှုများကို ဖန်တီးပေးသည်။

အနှစ်ချုပ်အားဖြင့်၊ Transition သည် **ပြောင်းလဲမှု၏ ချောမွေ့မှု**ကို အာရုံစိုက်ပြီး Animation သည် **အချိန်နှင့်အမျှ ဖြစ်ပေါ်နေသော လှုပ်ရှားမှု**ကို ဖန်တီးပေးသည်။ နှစ်ခုစလုံးသည် CSS ဖြင့် ဝဘ်ဒီဇိုင်းကို အသက်ဝင်စေရန် အဓိက အခန်းကဏ္ဍမှ ပါဝင်ကြသည်။

### အကောင်းဆုံး အလေ့အကျင့် (Best Practices)

Animation နှင့် Transition များကို အသုံးပြုရာတွင် Performance ကို ထိန်းသိမ်းရန် အလွန်အရေးကြီးသည်။

*   **Hardware Acceleration:** Transform (translate, rotate, scale) ကို အသုံးပြုသောအခါ သင့်ရဲ့ Browser သည် ၎င်းတို့ကို GPU (Graphics Processing Unit) မှတစ်ဆင့် ပိုမိုမြန်ဆန်စွာ လုပ်ဆောင်နိုင်သောကြောင့် Transition နှင့် Animation များအတွက် အကောင်းဆုံးဖြစ်သည်။
*   **Property များ ရွေးချယ်ခြင်း:** မလိုအပ်သော property များကို Animation တွင် အသုံးပြုခြင်းကို ရှောင်ရှားသင့်သည်။
*   **Timing Functions ကို ထိန်းချုပ်ခြင်း:** လိုအပ်သော အရှိန်အဟုန်ကို ရွေးချယ်အသုံးပြုခြင်းဖြင့် လှုပ်ရှားမှုပုံစံကို ထိန်းချုပ်နိုင်သည်။
*   **Performance Test:** ရှုပ်ထွေးသော Animation များ ဖန်တီးသည့်အခါ Browser Developer Tools ဖြင့် Performance ကို စစ်ဆေးသင့်သည်။

### Final Project: Animated Landing Page ဖန်တီးခြင်း

ယခု သင်ယူခဲ့သော Transition နှင့် Animation နည်းပညာများကို အသုံးပြု၍ အသက်ဝင်သော Landing Page တစ်ခုကို စုပေါင်းဖန်တီးကြည့်ကြရအောင်။ ဥပမာအားဖြင့်၊ Navigation Bar တွင် Mouse ကိုတင်လိုက်သည့်အခါ Background Color ကို ချောမွေ့စွာ ပြောင်းလဲခြင်း (Transition)၊ Hero section တွင် Background ကို အချိန်နှင့်အမျှ အစဉ်လိုက် ပြောင်းလဲခြင်း (Animation)၊ နှင့် Button များပေါ်တွင် Click လုပ်လိုက်သည့်အခါ အနည်းငယ် ကျုံ့သွားခြင်း (Transition) တို့ကို ပေါင်းစပ်ဖန်တီးခြင်းဖြင့် အသုံးပြုသူများ၏ အာရုံကို ဆွဲဆောင်နိုင်သော ဝဘ်စာမျက်နှာတစ်ခုကို ဖန်တီးနိုင်မည် ဖြစ်သည်။

CSS Animation နှင့် Transition တို့သည် သင့်ရဲ့ ဝဘ်ဒီဇိုင်းများကို ပုံသေအဖြစ် မထားဘဲ၊ အချိန်နှင့်အမျှ အသက်ဝင်နေသော၊ လှုပ်ရှားနေသော အတွေ့အကြုံများ ပေးစွမ်းနိုင်သည့် စွမ်းရည်ကို ပေးစွမ်းပါသည်။ ဤနည်းပညာများကို ကျွမ်းကျင်စွာ အသုံးပြုခြင်းဖြင့် သင်သည် ပိုမိုဆွဲဆောင်မှုရှိသော နှင့် အတွေ့အကြုံကောင်းသော ဝဘ်ဆိုက်များကို တည်ဆောက်နိုင်မည် ဖြစ်ပါသည်။
