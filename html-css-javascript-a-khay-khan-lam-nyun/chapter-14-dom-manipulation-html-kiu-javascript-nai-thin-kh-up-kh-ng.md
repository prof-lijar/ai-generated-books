# Chapter 14: DOM Manipulation — HTML ကို JavaScript နဲ့ ထိန်းချုပ်ခြင်း

သင်ရေးထားတဲ့ HTML စာမျက်နှာလေးတွေဟာ ရိုးရိုးစာသားတွေသာ မဟုတ်ပါဘူး။ JavaScript ကို အသုံးပြုပြီး ဒီစာမျက်နှာတွေကို အသက်ဝင်လာအောင်၊ အသုံးပြုသူရဲ့ အပြုအမူအပေါ် မူတည်ပြီး ပြောင်းလဲနိုင်အောင် ထိန်းချုပ်နိုင်တယ်ဆိုတာကို ဒီအခန်းမှာ သင်ယူကြပါမယ်။ HTML နဲ့ CSS ဟာ ဝဘ်စာမျက်နှာရဲ့ ပုံသဏ္ဍာန်နဲ့ အလှအပကို ပေးတာဖြစ်ပြီး၊ JavaScript ဟာ အဲဒီ HTML တွေကို 'လုပ်ဆောင်နိုင်စွမ်း' ပေးတဲ့ စွမ်းအား ဖြစ်ပါတယ်။ ဒီစွမ်းအားကို အသုံးချဖို့အတွက် ကျွန်ုပ်တို့သည် DOM Manipulation ကို လေ့လာကြရမှာ ဖြစ်ပါတယ်။ DOM ဆိုတာ အဲဒီ HTML စာမျက်နှာရဲ့ အချက်အလက်တွေကို JavaScript က ဘယ်လို နားလည်ပြီး ကိုင်တွယ်နိုင်အောင် Browser က ပေးထားတဲ့ မူဘောင် (Model) ဖြစ်ပါတယ်။

## 1. DOM (Document Object Model) ဆိုတာ ဘာလဲ? — အခြေခံနားလည်မှု

DOM ဆိုတာ Document Object Model ဖြစ်ပြီး၊ HTML စာမျက်နှာတစ်ခုလုံးကို JavaScript က လည်ပတ်နိုင်မယ့် အရာဝတ္ထု (Objects) အဖြစ် ပြောင်းလဲပေးလိုက်တာ ဖြစ်ပါတယ်။ အခြေခံအားဖြင့်၊ Browser က HTML ကို စာသားအစုအဝေးတစ်ခုအဖြစ်သာ သိရှိသည်။ သို့သော် JavaScript က ထိုစာသားအစုအဝေးကို အခြေခံပြီး အချက်အလက်တွေ (ဥပမာ- အစိတ်အပိုင်းများ၊ စာသားများ၊ ဂုဏ်သတ္တိများ) ကို JavaScript ရဲ့ စည်းမျဉ်းတွေနဲ့ အလွယ်တကူ ကိုင်တွယ်နိုင်အောင် Browser က ပေးထားတဲ့ အဆက်အစပ်ဆိုင်ရာ မူဘောင် (Model) ဖြစ်ပါတယ်။

### Browser ရဲ့ ရှုထောင့်

Browser က HTML ကို ဘယ်လို နားလည်သလဲဆိုတော့ HTML ကို အခြေခံကျသော အစိတ်အပိုင်းများ (Element, Attribute, Text) အဖြစ် စုစည်းထားပုံကို နားလည်ပါတယ်။ ဤအရာအားလုံးသည် တစ်ခုနှင့်တစ်ခု ဆက်စပ်နေပြီး တစ်ခုနှင့်တစ်ခု အရေးပါသော ဆက်စပ်မှုများဖြင့် ပုံစံတူအဆင် (Tree Structure) တစ်ခုကို ဖြစ်ပေါ်စေသည်။ ဤအဆက်အစပ်ဆိုင်ရာ ဆက်စပ်မှုများကို DOM Tree ဟုခေါ်သည်။ ဤ Tree Structure တွင် HTML Element များသည် မိဘ (Parent) နှင့် သား (Child) ဆက်ဆံရေးဖြင့် စုစည်းထားပြီး၊ JavaScript သည် ဤ Tree ကို လိုက်လံရှာဖွေပြီး အဆိုပါ အရာဝတ္ထုများကို ပြောင်းလဲနိုင်သည့် စွမ်းရည်ကို ပေးသည်။

### ဘာကြောင့် DOM ကို လိုအပ်တာလဲ?

Static HTML သည် စာမျက်နှာပေါ်တွင် တည်ရှိနေသော အချက်အလက်များသာ ဖြစ်သည်။ ၎င်းသည် အချိန်နှင့်အမျှ ပြောင်းလဲနိုင်သော အကြောင်းအရာများ (ဥပမာ- အသုံးပြုသူ၏ စိတ်အခြေအနေ၊ အချိန်၊ တာဝန်အလိုက် ပြောင်းလဲမှုများ) ကို ကိုင်တွယ်ရန် မသင့်တော်ပါ။ ထို့ကြောင့် Static HTML မှ Dynamic (အသက်ဝင်သော) Web Application များဆီသို့ ကူးပြောင်းနိုင်ရန်အတွက် JavaScript သည် DOM ကို အသုံးပြု၍ HTML ၏ အကြောင်းအရာ၊ ပုံစံနှင့် အပြုအမူများကို အချိန်နှင့်တပြေးညီ ပြောင်းလဲနိုင်သည့် စွမ်းရည်ကို ရရှိလာသည်။

***Transition Note:*** DOM ကို နားလည်ပြီဆိုတော့၊ အဲဒီ DOM ထဲက အရေးကြီးတဲ့ အစိတ်အပိုင်းတွေကို JavaScript နဲ့ ဘယ်လို ရွေးချယ်ပြီး ထိန်းချုပ်နိုင်မလဲဆိုတာ လေ့လာကြရအောင်။

## 2. HTML Element တွေကို JavaScript နဲ့ ရွေးချယ်ခြင်း (Selecting Elements)

JavaScript သည် HTML စာမျက်နှာရှိ မည်သည့် အစိတ်အပိုင်းကို ပြောင်းလဲမည်ကို သိရှိရန် အရင်ဆုံး ထိုအစိတ်အပိုင်းကို ရှာဖွေတွေ့ရှိရန် လိုအပ်ပါသည်။ ထို့ကြောင့် Element များကို ရှာဖွေခြင်းသည် DOM Manipulation ၏ အခြေခံအုတ်မြစ် ဖြစ်ပါသည်။

### ID ကို အသုံးပြု၍ ရွေးချယ်ခြင်း (The Specific Way)

Element တစ်ခုတည်းကို အတိအကျ ရွေးချယ်ရန်အတွက် `document.getElementById()` ဆိုသော မက်သဒေါဒ် (Method) ကို အသုံးပြုသည်။ ဤနည်းလမ်းသည် အလွန်တိကျသော အကျိုးကျေးဇူးရှိသည်။ ဤမက်သဒေါဒ်သည် HTML တွင် `id` Attribute ကို သတ်မှတ်ထားသော Element ကိုသာ ပြန်ပေးသည်။ ဥပမာအားဖြင့်၊ ကျွန်ုပ်တို့သည် ID ကို အသုံးပြု၍ ခေါင်းစဉ် (Heading) တစ်ခုကို ရွေးချယ်ပြီး ထိုခေါင်းစဉ်၏ စာသားကို ပြောင်းလဲနိုင်ပါသည်။ ဤနည်းလမ်းသည် တစ်ခုတည်းသော အရာကို ရှာဖွေရာတွင် အထိရောက်ဆုံးနှင့် အလျင်မြန်ဆုံး နည်းလမ်းဖြစ်သည်။

### CSS Selector များကို အသုံးပြု၍ ရွေးချယ်ခြင်း (The Flexible Way)

တစ်ခုတည်းသော Element မဟုတ်ဘဲ၊ CSS Selector များအသုံးပြု၍ အများအပြား Element များကို ရွေးချယ်ရန်အတွက် ပိုမိုပြောင်းလွယ်ပြင်လွယ်ရှိသော နည်းလမ်းများ ရှိသည်။

`document.querySelector()` မက်သဒေါဒ်ကို အသုံးပြု၍ CSS Selector (ဥပမာ- ID, Class, Tag name) ကို ထည့်သွင်းပေးလိုက်ခြင်းဖြင့် ထို Selector နှင့် ကိုက်ညီသော ပထမဆုံး Element ကို ပြန်ပေးသည်။ ၎င်းသည် အလွန်ပြောင်းလွယ်ပြင်လွယ်ရှိသည်။

ထို့အပြင်၊ `document.querySelectorAll()` မက်သဒေါဒ်ကို အသုံးပြုခြင်းဖြင့် CSS Selector တစ်ခုကို ထည့်သွင်းပေးလိုက်သည့်အခါ ထို Selector နှင့် ကိုက်ညီသော Element များအားလုံးကို စုစည်းထားသည့် NodeList အဖြစ် ပြန်ပေးသည်။ ဤနည်းလမ်းသည် တစ်ပြိုင်နက်တည်း အများအပြား Element များကို ရွေးချယ်ရန်အတွက် အလွန်အသုံးဝင်သည်။ `getElementById` သည် တစ်ခုတည်းကိုသာ ရှာဖွေပေးပြီး၊ `querySelector` နှင့် `querySelectorAll` တို့သည် ပိုမိုကျယ်ပြန့်သော ရွေးချယ်မှုများကို ခွင့်ပြုသည်။

***Transition Note:*** Element တွေကို ရွေးချယ်နိုင်ပြီဆိုတော့၊ အဲဒီ Element တွေရဲ့ အထဲက အချက်အလက်တွေကို ပြောင်းလဲတာ၊ ဖန်တီးတာတွေ လုပ်လို့ရပြီ။

## 3. Element များ၏ အကြောင်းအရာ (Content), ပုံစံ (Style) နှင့် ဂုဏ်သတ္တိ (Attribute) များ ပြောင်းလဲခြင်း

Element များကို ရွေးချယ်နိုင်ပြီဖြစ်သောအခါ၊ ကျွန်ုပ်တို့သည် ထို Element များ၏ အတွင်းရှိ အချက်အလက်များ (Content)၊ အသွင်အပြင် (Style) နှင့် ဂုဏ်သတ္တိများ (Attribute) ကို JavaScript မှတစ်ဆင့် ပြောင်းလဲနိုင်ပါသည်။

### Element Content ပြောင်းလဲခြင်း

Element များ၏ စာသားအကြောင်းအရာကို ပြောင်းလဲရာတွင် `.textContent` နှင့် `.innerHTML` တို့သည် အရေးကြီးသော ကွာခြားချက်များရှိပါသည်။

`.textContent` သည် Element အတွင်းရှိ စာသား (Text Content) ကိုသာ ပြောင်းလဲပေးသည်။ ၎င်းသည် HTML Tag များ မပါဘဲ စာသားများကိုသာ ကိုင်တွယ်သည်။ ဥပမာအားဖြင့်၊ `<p>Hello</p>` ရှိ `.textContent` ကို ပြောင်းလဲလိုက်ပါက စာသားသည် "New Text" ဖြစ်သွားမည်ဖြစ်သည်။

ယင်းနှင့် ဆန့်ကျင်ဘက်အားဖြင့် `.innerHTML` သည် Element အတွင်းရှိ HTML အပိုင်းအစများ (HTML Content) အားလုံးကို ပြောင်းလဲပေးသည်။ ဤနည်းလမ်းကို အသုံးပြုသောအခါ၊ JavaScript မှ HTML Tag အသစ်များကို ထည့်သွင်းခြင်း သို့မဟုတ် ဖျက်ခြင်းများ ပြုလုပ်နိုင်သည်။ ဤအချက်ကို သတိပြုရမည်မှာ `.innerHTML` ကို အလွန်အမင်း အသုံးပြုရာတွင် မတော်တဆ JavaScript Injection ဖြင့် လုံခြုံရေးဆိုင်ရာ ပြဿနာများ ဖြစ်ပေါ်လာနိုင်သောကြောင့် သတိထားရမည်။

### Style (CSS) ပြောင်းလဲခြင်း

Element များ၏ ပုံစံ (Style) ကို JavaScript မှတစ်ဆင့် ထိန်းချုပ်ရန်အတွက် `.style` Property ကို အသုံးပြုသည်။ ဤ Property သည် Element ၏ Inline CSS ကို တိုက်ရိုက် ထိန်းချုပ်ပေးသည်။ ဥပမာအားဖြင့်၊ Element တစ်ခု၏ နောက်ခံအရောင် (background-color) ကို ပြောင်းလဲလိုပါက `element.style.backgroundColor = 'red';` ဟု ရေးသားနိုင်သည်။ ဤနည်းလမ်းသည် CSS စတိုင်များကို Dynamic ဖြစ်အောင် ထိန်းချုပ်ရာတွင် အလွန်အသုံးဝင်သည်။

### Attribute များ ထိန်းချုပ်ခြင်း

Element များ၏ ဂုဏ်သတ္တိများ (Attributes)၊ ဥပမာ- `src` (ပုံ၏လမ်းကြောင်း), `href` (လင့်ခ်လမ်းကြောင်း), `class` စသည့် အချက်အလက်များကို ထိန်းချုပ်ရန်အတွက် `.setAttribute()` နှင့် `.getAttribute()` မက်သဒေါဒ်များကို အသုံးပြုသည်။

`.setAttribute(name, value)` သည် Element တစ်ခု၏ Attribute အမည် (name) နှင့် တန်ဖိုး (value) ကို သတ်မှတ်ပေးသည်။ ဥပမာအားဖြင့်၊ ပုံတစ်ပုံ၏ `src` ကို ပြောင်းလဲလိုပါက `imageElement.setAttribute('src', 'new_image.jpg');` ဟု ရေးသားနိုင်သည်။ ထို Attribute များ၏ လက်ရှိတန်ဖိုးကို ဖတ်လိုပါက `element.getAttribute(name)` ကို အသုံးပြုရမည် ဖြစ်သည်။

***Transition Note:*** အကြောင်းအရာနဲ့ ပုံစံတွေကို ပြောင်းလဲတာက အခြေခံပါ။ အခုတော့ JavaScript ကို အသုံးပြုပြီး Element အသစ်တွေ ဖန်တီးပြီး စာမျက်နှာပေါ်မှာ ထည့်သွင်းတာကို လေ့လာကြရအောင်။

## 4. Element အသစ်များ ဖန်တီးခြင်းနှင့် DOM ထဲသို့ ထည့်သွင်းခြင်း (Creating and Manipulating Elements)

အခြေခံအားဖြင့်၊ ကျွန်ုပ်တို့သည် ရှိပြီးသား Element များကို ပြောင်းလဲခြင်းအပြင် JavaScript ကို အသုံးပြု၍ လိုအပ်သည့် Element အသစ်များကို ဖန်တီးကာ DOM ထဲသို့ ထည့်သွင်းနိုင်စွမ်းလည်း ရှိသည်။ ဤသည်မှာ Dynamic Web Application များအတွက် အဓိကကျသော စွမ်းဆောင်ရည်တစ်ခု ဖြစ်သည်။

### Element အသစ် ဖန်တီးခြင်း (Creating New Elements)

JavaScript ကို အသုံးပြု၍ HTML Tag အသစ်တစ်ခုကို Memory ထဲတွင် ဖန်တီးရန်အတွက် `document.createElement()` မက်သဒေါဒ်ကို အသုံးပြုသည်။ ဤမက်သဒေါဒ်သည် HTML Tag အသစ်တစ်ခုကို JavaScript မှတစ်ဆင့် ဖန်တီးပေးပြီး Memory ထဲတွင် သိမ်းဆည်းပေးသည်။ ဖန်တီးပြီးသော Element ကို Class များ၊ ID များ နှင့် စာသားများ ထည့်သွင်းနိုင်သော အခွင့်အလမ်းများ ရှိသည်။ ဥပမာအားဖြင့်၊ `document.createElement('li')` ဖြင့် List Item အသစ်တစ်ခုကို ဖန်တီးပြီး၊ ထို့နောက် `newItem.textContent = 'New Item';` ဟု ရေးသား၍ စာသားထည့်နိုင်သည်။

### DOM ထဲသို့ ထည့်သွင်းခြင်း (Inserting into the DOM)

Element အသစ်များကို ဖန်တီးပြီးနောက်၊ ၎င်းတို့ကို HTML စာမျက်နှာပေါ်တွင် မြင်ရစေရန်အတွက် DOM Tree ထဲသို့ ထည့်သွင်းရန် လိုအပ်သည်။ ဤလုပ်ငန်းစဉ်အတွက် အဓိကအားဖြင့် `.appendChild()` မက်သဒေါဒ်ကို အသုံးပြုသည်။ ဤမက်သဒေါဒ်သည် ဖန်တီးထားသော Element ကို ထို Element ၏ Parent Element ထဲသို့ ထည့်သွင်းပေးသည်။

ထို့အပြင်၊ `.prepend()` နှင့် `.append()` တို့သည် Element များကို Parent Element ၏ အစ (prepend) သို့မဟုတ် အဆုံး (append) သို့ ထည့်သွင်းရန် အသုံးပြုသည်။ ဤမက်သဒေါဒ်များသည် Element များအား ထည့်သွင်းရာတွင် ပိုမိုတိကျသော ထိန်းချုပ်မှုကို ပေးသည်။ ထို့အပြင်၊ Element များကို ဖျက်ပစ်ရန်အတွက် `.remove()` မက်သဒေါဒ်ကိုလည်း အသုံးပြုနိုင်သည်။ ၎င်းသည် သတ်မှတ်ထားသော Element ကို DOM မှ ဖယ်ရှားပစ်သည်။ တစ်ခါတစ်ရံတွင် `.removeChild()` ကိုလည်း အသုံးပြု၍ မိဘ Element မှ သား Element ကို ဖယ်ရှားခြင်းဖြင့်လည်း အလားတူ ရလဒ်ကို ရရှိနိုင်သည်။

## 5. လက်တွေ့ လေ့ကျင့်ခန်း (Practical Application) — Dynamic Shopping List ဖန်တီးခြင်း

အထက်ပါ အခြေခံ နည်းလမ်းများကို လက်တွေ့အသုံးချနိုင်ရန်အတွက် ကျွန်ုပ်တို့သည် Dynamic Shopping List တစ်ခုကို JavaScript ဖြင့် ဖန်တီးကြည့်ကြပါမည်။ အသုံးပြုသူက စာရင်းထဲသို့ အသစ်တစ်ခု ထည့်လိုက်တိုင်း၊ ထိုအချက်အလက်များသည် အလိုအလျောက် List ထဲသို့ ပေါ်လာပြီး၊ လိုအပ်ပါက ဖျက်ပစ်နိုင်သည့် စွမ်းဆောင်ရည်ကို ရရှိမည် ဖြစ်သည်။

### Project Overview

ဤလက်တွေ့လေ့ကျင့်ခန်း၏ ရည်ရွယ်ချက်မှာ အသုံးပြုသူမှ Input Box တစ်ခုမှ ပစ္စည်းအမည်ကို ထည့်သွင်းပြီး Button တစ်ခုကို နှိပ်လိုက်သည့်အခါ၊ ထိုအချက်အလက်ကို List ထဲသို့ အလိုအလျောက် ဖန်တီး၍ ထည့်သွင်းခြင်း ဖြစ်သည်။ ဤလုပ်ငန်းစဉ်တွင် Event Listeners ကို အသုံးပြု၍ အသုံးပြုသူ၏ အပြုအမူကို စောင့်ဆိုင်းပြီး၊ DOM Manipulation များကို အသုံးပြု၍ List ကို အချိန်နှင့်တပြေးညီ ပြောင်းလဲပေးရမည် ဖြစ်သည်။

### Step-by-Step Implementation

**၁။ HTML Structure ကို ပြင်ဆင်ခြင်း**
ပထမအဆင့်အနေဖြင့်၊ Shopping List ကို ထည့်သွင်းရန် Input Box နှင့် List များအတွက် HTML Structure ကို ပြင်ဆင်ရမည်။ အဓိကအားဖြင့်၊ ပစ္စည်းထည့်ရန် Input Box တစ်ခုနှင့် ထိုပစ္စည်းများ စုစည်းမည့် List Container တစ်ခုကို ဖန်တီးရမည်။

**၂။ JavaScript တွင် Function များရေးဆွဲခြင်း**
ဒုတိယအဆင့်အနေဖြင့်၊ JavaScript တွင် လိုအပ်သော Function များကို ရေးဆွဲရမည်။ အဓိကအားဖြင့်၊ အသုံးပြုသူ၏ Input ကို စောင့်ဆိုင်းရန် Event Listeners များကို ထည့်သွင်းရမည်။ ထို့နောက်၊ Input မှ အချက်အလက်ကို ရယူပြီး DOM Manipulation လုပ်ဆောင်ရန်အတွက် Function တစ်ခုကို ရေးဆွဲရမည်။ ဤ Function သည် Element အသစ်များကို ဖန်တီးပြီး List Container ထဲသို့ ထည့်သွင်းရန် လိုအပ်သည်။

**၃။ အသုံးပြုသူ Input ကို စောင့်ဆိုင်းပြီး List Item အသစ်များအား DOM ထဲသို့ ထည့်သွင်းခြင်း**
တတိယအဆင့်အနေဖြင့်၊ အသုံးပြုသူ Input ကို ရိုက်ထည့်ပြီး Submit လုပ်သည့်အခါ၊ JavaScript သည် ထိုအချက်အလက်ကို စောင့်ဆိုင်းပြီး List Container ၏ နောက်ဆုံးသို့ Element အသစ်တစ်ခု (ဥပမာ- `<li>`) ကို ဖန်တီးရမည်။ ထို Element ထဲသို့ အသုံးပြုသူ ထည့်သွင်းထားသော စာသားကို ထည့်သွင်းရမည်။ ထို့နောက်၊ ဖန်တီးထားသော List Item ကို List Container ထဲသို့ `.appendChild()` မက်သဒေါဒ်ကို အသုံးပြု၍ ထည့်သွင်းရမည်။ ဤလုပ်ငန်းစဉ်ကို အသုံးပြုသူက ထပ်မံထည့်သွင်းသည့် အချက်အလက်များအတွက် ထပ်မံလုပ်ဆောင်ရမည် ဖြစ်သည်။

**၄။ အသုံးပြုသူ ဖျက်ခိုင်းသည့်အခါ Element များကို မှန်ကန်စွာ ဖျက်ဆီးခြင်း**
စတိုင်တစ်ခုအနေဖြင့်၊ အသုံးပြုသူမှ စာရင်းမှ ပစ္စည်းတစ်ခုကို ဖျက်လိုသည့်အခါ၊ ထို List Item ကို DOM မှ ဖယ်ရှားရန်အတွက် `.remove()` မက်သဒေါဒ်ကို အသုံးပြုရမည်။ ဤလုပ်ဆောင်ချက်သည် ထို Element နှင့် ၎င်း၏ မိဘ Element အားလုံးကို DOM မှ ဖယ်ရှားပစ်ပေးသည်။ ဤသည်မှာ Dynamic List ၏ အစိတ်အပိုင်းများကို ထိန်းချုပ်ရာတွင် အရေးကြီးသော အခန်းကဏ္ဍမှ ပါဝင်သည်။

### Review

ဤလက်တွေ့လေ့ကျင့်ခန်းမှ ရရှိသော Dynamic List သည် Static HTML ထက် အဆပေါင်းများစွာ ပိုမိုကောင်းမွန်သည်။ အသုံးပြုသူ၏ အပြုအမူအပေါ် မူတည်၍ စာရင်းကို အချိန်နှင့်တပြေးညီ ပြောင်းလဲနိုင်ခြင်း၊ အချက်အလက်အသစ်များကို အလွယ်တကူ ထည့်သွင်းနိုင်ခြင်းနှင့် လိုအပ်သည့် အချက်အလက်များကို ဖယ်ရှားပစ်နိုင်ခြင်းတို့ကြောင့် Web Application များ၏ အတွေ့အကြုံကို သိသိသာသာ မြှင့်တင်ပေးနိုင်ပါသည်။

## 📝 အခန်းအနှစ်ချုပ် (Chapter Summary & Conclusion)

DOM Manipulation သည် HTML ကို JavaScript မှတစ်ဆင့် လွှမ်းမိုးထိန်းချုပ်နိုင်သည့် အင်ဂျင်နီယာပိုင်းဆိုင်ရာ အခြေခံကျကျ ဖြစ်သည်။ ကျွန်ုပ်တို့သည် DOM ဆိုသည်မှာ HTML စာမျက်နှာတစ်ခုလုံးကို JavaScript က လည်ပတ်နိုင်မည့် အရာဝတ္ထုများအဖြစ် ပြောင်းလဲပေးလိုက်သော မူဘောင်ဖြစ်ကြောင်း နားလည်ခဲ့ကြပြီ ဖြစ်သည်။

### Key Takeaways Recap

ကျွန်ုပ်တို့ လေ့လာခဲ့သော အဓိက အချက်များမှာ Element များကို `getElementById`၊ `querySelector`၊ `querySelectorAll` တို့ဖြင့် ရှာဖွေခြင်း၊ `.textContent` နှင့် `.innerHTML` ဖြင့် အကြောင်းအရာများကို ပြောင်းလဲခြင်း၊ `.style` နှင့် `.setAttribute()` ဖြင့် ပုံစံနှင့် ဂုဏ်သတ္တိများကို ထိန်းချုပ်ခြင်း၊ ထို့အပြင် `createElement` နှင့် `appendChild` တို့ဖြင့် Element အသစ်များ ဖန်တီးခြင်းနှင့် DOM ထဲသို့ ထည့်သွင်းခြင်း၊ ထို့အပြင် `.remove()` ဖြင့် Element များကို ဖျက်ခြင်းတို့၏ အခြေခံ နည်းလမ်းများကို အနှစ်ချုပ်ဖော်ပြခဲ့သည်။

### Skill Summary

ဤအခန်းမှတစ်ဆင့် သင်သည် HTML ၏ အခြေခံဖွဲ့စည်းပုံကို နားလည်ပြီး JavaScript ကို အသုံးပြု၍ ထိုဖွဲ့စည်းပုံကို Dynamic ဖြစ်အောင် ထိန်းချုပ်နိုင်သော အခြေခံစွမ်းရည်များကို ရရှိခဲ့ပြီ ဖြစ်သည်။ Element များကို ရှာဖွေခြင်း၊ ပြောင်းလဲခြင်း၊ ဖန်တီးခြင်းနှင့် ဖျက်ခြင်းတို့၏ အခြေခံ နည်းလမ်းများကို ကျွမ်းကျင်စွာ အသုံးချနိုင်ပြီ ဖြစ်သည်။

DOM Manipulation ကို ကျွမ်းကျင်လာပြီဆိုလျှင် နောက်အဆင့်အနေဖြင့် ပိုမိုရှုပ်ထွေးသော Web Application များ ဖန်တီးနိုင်ရန်အတွက် Asynchronous Programming (ဥပမာ- Fetch API ကို အသုံးပြု၍ Server မှ အချက်အလက်များ ရယူခြင်း) နှင့် Event Handling (အသုံးပြုသူ၏ ကလစ်၊ တိုက်ရိုက်ထည့်သွင်းမှုများအား တုံ့ပြန်ခြင်း) တို့ကို လေ့လာရန် အကြံပြုအပ်ပါသည်။

### Final Thought

DOM Manipulation ဟာ Web Development ရဲ့ အင်ဂျင်နီယာပိုင်းဆိုင်ရာ အခြေခံကျကျ ဖြစ်သည်။ ဤကျွမ်းကျင်မှုကို အသုံးပြုပြီး သင်စိတ်ကူးထားတဲ့ Web Application တွေကို လက်တွေ့အသက်ဝင်လာအောင် ဖန်တီးနိုင်ပြီဖြစ်ပါသည်။ သင်သည် HTML၊ CSS နှင့် JavaScript တို့၏ အင်အားကို ပေါင်းစပ်အသုံးပြု၍ အသက်ဝင်သော၊ အပြန်အလှန်တုံ့ပြန်နိုင်သော ဝဘ်စာမျက်နှာများကို တည်ဆောက်နိုင်မည့် အခွင့်အလမ်းကို ရရှိခဲ့ပြီ ဖြစ်ပါသည်။
