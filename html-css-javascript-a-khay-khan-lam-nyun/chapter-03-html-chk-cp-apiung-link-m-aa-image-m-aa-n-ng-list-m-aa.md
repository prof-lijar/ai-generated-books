# Chapter 3: HTML ဆက်စပ်အပိုင်း — Link များ၊ Image များ နှင့် List များ

HTML ရဲ့ အခြေခံအုတ်မြစ်တွေ၊ Tag များနဲ့ အခြေခံဖွဲ့စည်းပုံတွေကို ကျွမ်းကျွမ်းကျင်ကျင် တည်ဆောက်ပြီးပြီဆိုတော့၊ အခုအချိန်မှာ ကျွန်တော်တို့ရဲ့ ဝဘ်စာမျက်နှာတွေဟာ တစ်ခုတည်း မဟုတ်ဘဲ အပြန်အလှန် ဆက်စပ်နေပုံကို လေ့လာကြရအောင်။ ဝဘ်ဆိုဒ်တစ်ခုဟာ ဆက်သွယ်ရေးလမ်းကြောင်းတွေ၊ မြင်သာထင်ရှားတဲ့ အကြောင်းအရာတွေနဲ့ စနစ်တကျ စုစည်းထားတဲ့ အချက်အလက်တွေ ပေါင်းစပ်မှုဖြစ်ပါတယ်။ ဒီအခန်းမှာ ကျွန်တော်တို့ဟာ စာမျက်နှာတွေကြား ချိတ်ဆက်တာ၊ ပုံတွေ ထည့်တာနဲ့ အချက်အလက်တွေကို စနစ်တကျ စုစည်းတာတွေကို လေ့လာပြီး၊ ကျွန်တော်တို့ရဲ့ ဝဘ်စာမျက်နှာတွေကို ပိုမို ဆွဲဆောင်မှုရှိအောင် ဖန်တီးနိုင်သွားမှာ ဖြစ်ပါတယ်။ ဒီသင်ခန်းစာတွေဟာ သင်ဟာ အခြေခံ HTML ကို နားလည်ပြီးနောက်မှာ၊ အကြောင်းအရာတွေကို ထိရောက်စွာ တင်ပြနိုင်မယ့် အခြေခံကျွမ်းကျင်မှုတွေ ဖြစ်လာမှာပါ။

**သင်ယူရမည့် အဓိက ရည်မှန်းချက်များ**

ဒီအခန်းကို အဆုံးသတ်တဲ့အခါမှာ ကျွန်တော်တို့ဟာ အောက်ပါအချက်တွေကို ကျွမ်းကျင်စွာ လုပ်ဆောင်နိုင်မှာ ဖြစ်ပါတယ်-

*   **Hyperlinks (Link များ) ဖြင့် ဝဘ်စာမျက်နှာများ ချိတ်ဆက်ပုံကို နားလည်ခြင်း:** အသုံးပြုသူတွေ တစ်နေရာကနေ နောက်တစ်နေရာကို ဘယ်လို လမ်းကြောင်းပြနိုင်မလဲဆိုတာ သိရှိခြင်း။
*   **ပုံရိပ်များ (Images) ကို ထည့်သွင်းခြင်းနှင့် Attribute များ (src, alt) ၏ အရေးပါပုံကို သိရှိခြင်း:** စာမျက်နှာကို မြင်သာစေပြီး အချက်အလက်တွေကို ထိရောက်စွာ ပေးပို့နိုင်ပုံကို သိရှိခြင်း။
*   **အချက်အလက်များကို စနစ်တကျ စုစည်းရန် List များ (<ol>, <ul>) ကို အသုံးပြုပုံ:** ရှုပ်ထွေးတဲ့ အချက်အလက်တွေကို ဖတ်ရှုသူအတွက် လွယ်ကူအောင် စီစဉ်တင်ပြနိုင်ပုံကို သိရှိခြင်း။
*   **HTML Comment များ အသုံးပြုပုံ:** ကုဒ်ကို မှတ်သားထားဖို့နဲ့ အနာဂတ်မှာ ပြန်လည်ပြင်ဆင်ဖို့အတွက် အရေးကြီးတဲ့ နည်းလမ်းကို သိရှိခြင်း။

## အပိုင်း ၁: Hyperlinks (Link များ) — ဝဘ်စာမျက်နှာများ ချိတ်ဆက်ခြင်း

ဝဘ်ဆိုက်တစ်ခုရဲ့ အဓိကအချက်တွေထဲက တစ်ခုကတော့ အကြောင်းအရာတွေကို အပြန်အလှန် ဆက်စပ်ပေးနိုင်ဖို့ ဖြစ်ပါတယ်။ ဒီဆက်စပ်မှုကို ဖန်တီးပေးတဲ့ အခြေခံအရာကတော့ **Hyperlinks** သို့မဟုတ် Link များပဲ ဖြစ်ပါတယ်။ Link များဟာ ကျွန်တော်တို့ရဲ့ HTML စာမျက်နှာတွေကို အပြင်က အခြားစာမျက်နှာတွေနဲ့ ချိတ်ဆက်ပေးပြီး အသုံးပြုသူတွေ အလွယ်တကူ လှည့်ပတ်သွားနိုင်အောင် လမ်းကြောင်းပေးတာ ဖြစ်ပါတယ်။

### Anchor Tag (`<a>`) ကို မိတ်ဆက်ခြင်း

Link များ ဖန်တီးဖို့အတွက် ကျွန်တော်တို့ အသုံးပြုရမယ့် အဓိက Tag ဟာ **Anchor Tag** ဖြစ်တဲ့ `<a`> Tag ဖြစ်ပါတယ်။ ဒီ Tag ဟာ ဘယ်နေရာမဆို စာသား ဒါမှမဟုတ် အခြား HTML Element တွေကို နောက်ကွယ်မှာ ဆွဲထားတဲ့ (Anchor) အမှတ်အသားတစ်ခုအဖြစ် လုပ်ဆောင်ပေးပါတယ်။ ဒီ Anchor ကို ဘယ်ကို ရောက်မလဲဆိုတာကို သတ်မှတ်ဖို့အတွက် **`href` Attribute** ကို အသုံးပြုရပါမယ်။

`<a href="URL">Link Text</a>` ဆိုတဲ့ ပုံစံနဲ့ Link တစ်ခုကို ဖန်တီးနိုင်ပါတယ်။ `href` ဆိုတဲ့ Attribute ဟာ Link ကို ဘယ်စာမျက်နှာကို ရောက်စေမလဲဆိုတဲ့ လိပ်စာ (URL) ကို သတ်မှတ်ပေးပါတယ်။

### Link များ ဖန်တီးခြင်းနှင့် Path များ

Link များ ဖန်တီးရာမှာ အရေးကြီးဆုံးကတော့ `href` ထဲမှာ ထည့်မယ့် လိပ်စာ (Path) အမျိုးအစားကို နားလည်ဖို့ပါပဲ။

**Relative Path များ (ဆက်စပ်ဖိုင်များ)**
Relative Path ဆိုတာက လက်ရှိဖိုင်နဲ့ ဆက်စပ်နေတဲ့ ဖိုင်တွေကို ရည်ညွှန်းတဲ့ လမ်းကြောင်းမျိုး ဖြစ်ပါတယ်။ ဥပမာအားဖြင့်၊ ကျွန်တော်တို့ရဲ့ ဝဘ်ဆိုက်မှာ `index.html` ဆိုတဲ့ စာမျက်နှာနဲ့ `about.html` ဆိုတဲ့ စာမျက်နှာတွေ ရှိတယ်ဆိုပါစို့။ `index.html` ထဲကနေ `about.html` ကို ချိတ်ဆက်ချင်ရင်၊ `href="about.html"` လို့ ရေးလိုက်ရုံပါပဲ။ ဒါဟာ အလွယ်ဆုံးနဲ့ အသုံးအများဆုံး နည်းလမ်းဖြစ်ပြီး၊ ဝဘ်ဆိုက်တစ်ခုတည်းအတွင်းမှာ စာမျက်နှာတွေကြား ချိတ်ဆက်ရာမှာ အလွန်အသုံးဝင်ပါတယ်။

**Absolute Path များ (အပြင်က ဝဘ်ဆိုက်များ)**
Absolute Path ဆိုတာက အခြားသော ဝဘ်ဆိုက်တစ်ခုရဲ့ လိပ်စာကို ရည်ညွှန်းတာ ဖြစ်ပါတယ်။ ဥပမာ၊ Google ရဲ့ ဝဘ်ဆိုက်ကို ချိတ်ဆက်ချင်ရင် `href="https://www.google.com"` လို့ ရေးရမှာ ဖြစ်ပါတယ်။ ဒါဟာ အပြင်က အရင်းအမြစ်တွေကို ရည်ညွှန်းတဲ့အခါ အသုံးပြုရပါတယ်။

**Link အမျိုးအစားများ (Target Attribute)**
Link တွေကို ဘယ်လိုပုံစံနဲ့ ဖွင့်မလဲဆိုတာကို ထိန်းချုပ်ဖို့အတွက် `target` Attribute ကို အသုံးပြုနိုင်ပါတယ်။ အထူးသဖြင့်၊ Link ကို အခြား Browser Tab အသစ်မှာ ဖွင့်ပြချင်တဲ့အခါ `target="_blank"` ကို ထည့်သွင်းအသုံးပြုကြပါတယ်။ ဥပမာ၊ `<a href="page2.html" target="_blank">နောက်စာမျက်နှာသို့ (အသစ် Tab တွင် ဖွင့်ရန်)</a>` ဆိုတာက Link ကို နှိပ်လိုက်တဲ့အခါ လက်ရှိစာမျက်နှာကို မပိတ်ဘဲ နောက်စာမျက်နှာကို အသစ် Tab မှာ ဖွင့်ပြပေးမှာ ဖြစ်ပါတယ်။

### လက်တွေ့လေ့ကျင့်ခန်း (Link များ)

ဒီအချက်တွေကို လက်တွေ့ကျကျ လေ့ကျင့်ဖို့အတွက် ကျွန်တော်တို့ ကိုယ်ပိုင် Navigation စာမျက်နှာတစ်ခုကို ဖန်တီးကြည့်ရအောင်။ ကျွန်တော်တို့မှာ `index.html` နဲ့ `about.html` ဆိုတဲ့ ဖိုင်နှစ်ခု ရှိတယ်လို့ ယူဆပြီး၊ ဒီနှစ်ခုကို Link များနဲ့ ချိတ်ဆက်ပြီး Navigation စာမျက်နှာ (Sitemap) တစ်ခုကို ဖန်တီးကြည့်ရအောင်။ ဒီလေ့ကျင့်ခန်းကနေတစ်ဆင့် Relative Path တွေရဲ့ အရေးပါပုံကို နားလည်သွားမှာ ဖြစ်ပါတယ်။

**Transition Note:** Link များက ဝဘ်ဆိုက်ရဲ့ လမ်းကြောင်းကို ဖန်တီးပေးပါတယ်။ အခုအခါမှာ၊ စာမျက်နှာထဲမှာ ပုံတွေ ထည့်ပြီး အကြောင်းအရာကို ပိုမို ဆွဲဆောင်မှုရှိအောင် ဘယ်လိုလုပ်မလဲဆိုတာ ဆက်လေ့လာကြရအောင်။ ပုံတွေက အကြောင်းအရာကို မြင်သာစေတဲ့ အကောင်းဆုံး နည်းလမ်းတစ်ခု ဖြစ်ပါတယ်။

## အပိုင်း ၂: Images (ပုံရိပ်များ) — ဝဘ်စာမျက်နှာတွင် ပုံများ ထည့်သွင်းခြင်း

ဝဘ်စာမျက်နှာတွေဟာ စာသားတွေတင်မက ပုံတွေ၊ ဂရပ်ဖစ်တွေနဲ့ တွဲဖက်မှသာ အသုံးပြုသူတွေရဲ့ စိတ်ကို ဆွဲဆောင်နိုင်မှာ ဖြစ်ပါတယ်။ ဒီအပိုင်းမှာ ကျွန်တော်တို့ဟာ ပုံရိပ်တွေကို HTML ထဲမှာ ဘယ်လို ထည့်သွင်းမလဲ၊ အဲဒီပုံတွေရဲ့ အချက်အလက်တွေကို ဘယ်လို ထိန်းချုပ်မလဲဆိုတာ လေ့လာကြပါမယ်။

### ပုံရိပ် ထည့်သွင်းခြင်းနှင့် Attribute များ၏ အရေးပါပုံ

ပုံရိပ်တွေကို HTML ထဲမှာ ထည့်သွင်းဖို့အတွက် **`<img>` Tag** ကို အသုံးပြုရပါတယ်။ ဒီ Tag ဟာ ပုံရိပ်ကို စာမျက်နှာပေါ်မှာ ပြသပေးတဲ့ အဓိက ကိရိယာပဲ ဖြစ်ပါတယ်။ `<img src="path/to/image.jpg" alt="Image Description">` ဆိုတဲ့ ပုံစံနဲ့ ရေးသားရမှာ ဖြစ်ပါတယ်။

**အဓိက Attribute များ**

1.  **`src` (Source):** ဒီ Attribute ဟာ ပုံရိပ်ရဲ့ အရင်းအမြစ် (Source) ကို သတ်မှတ်ပေးပါတယ်။ ဆိုလိုတာက ပုံဖိုင်ဟာ ဘယ်နေရာမှာ ရှိနေသလဲဆိုတဲ့ လမ်းကြောင်း (Path) ကို ထည့်ပေးရမှာ ဖြစ်ပါတယ်။ ဒီ Path ဟာ Relative Path (ဆက်စပ်ဖိုင်) ဒါမှမဟုတ် Absolute Path (အပြင်က လိပ်စာ) ဖြစ်နိုင်ပါတယ်။
2.  **`alt` (Alternative Text):** `alt` Attribute ဟာ ပုံရိပ်ရဲ့ အရေးအကြီးဆုံး အစိတ်အပိုင်းတစ်ခု ဖြစ်ပါတယ်။ ပုံဖိုင်ကို Browser က ဖတ်လို့မရတဲ့အခါ (ဥပမာ- Internet Connection မကောင်းတာ၊ ဖိုင်ပျက်တာ) ဒါမှမဟုတ် Screen Reader (မျက်စိမမြင်သူတွေအတွက် အသုံးပြုတဲ့ Software) တွေအတွက် ပုံရဲ့ အကြောင်းအရာကို ဖော်ပြပေးတဲ့ စာသားကို သတ်မှတ်ပေးပါတယ်။ `alt` ကို မှန်ကန်စွာ ထည့်သွင်းခြင်းဟာ **Accessibility** (ရရှိနိုင်မှု) အတွက် အလွန်အရေးကြီးပါတယ်။

**ပုံရိပ် အရွယ်အစား ထိန်းချုပ်ခြင်း**

ပုံရိပ်တွေရဲ့ အရွယ်အစားကို ထိန်းချုပ်ဖို့အတွက် `width` နဲ့ `height` Attribute တွေကို အသုံးပြုပါတယ်။ ဥပမာအားဖြင့်၊ `<img src="photo.jpg" alt="My Photo" width="300" height="200">` လို့ ရေးလိုက်တဲ့အခါ ပုံရဲ့ အကျယ် (width) ကို ၃၀၀ ပစ်ဆယ်နဲ့ အမြင့် (height) ကို ၂၀၀ ပစ်ဆယ်အထိ သတ်မှတ်ပေးလိုက်တာမျိုး ဖြစ်ပါတယ်။ ဒီ Attribute တွေကို သတ်မှတ်ပေးခြင်းအားဖြင့် ပုံတွေဟာ စာမျက်နှာရဲ့ Layout နဲ့ လိုက်လျောညီထွေဖြစ်အောင် ထိန်းချုပ်နိုင်ပါတယ်။

**ပုံရိပ်ကို စီမံခန့်ခွဲခြင်း**

ပုံရိပ်တွေကို မှန်ကန်စွာ ထည့်သွင်းဖို့အတွက် ဖိုင်အမည်နဲ့ လမ်းကြောင်း (Path) ကို မှန်ကန်စွာ စီမံခန့်ခွဲဖို့ လိုအပ်ပါတယ်။ ကျွန်တော်တို့ရဲ့ HTML ဖိုင်တွေနဲ့ ပုံဖိုင်တွေဟာ တစ်နေရာတည်းမှာ ရှိနေဖို့ ဒါမှမဟုတ် မှန်ကန်တဲ့ လမ်းကြောင်းနဲ့ ဆက်စပ်နေဖို့ အရေးကြီးပါတယ်။ ပုံဖိုင်တွေကို သိမ်းဆည်းတဲ့အခါ စနစ်တကျ ဖိုင်အမည်ပေးပြီး လမ်းကြောင်းကို မှန်ကန်စွာ ရေးသားဖို့ လိုအပ်ပါတယ်။

### လက်တွေ့လေ့ကျင့်ခန်း (Images)

ဒီအချက်တွေကို လက်တွေ့ကျကျ လေ့ကျင့်ဖို့အတွက် ကျွန်တော်တို့ ကိုယ်ရေးအကျဉ်းစာမျက်နှာ (Bio Page) တစ်ခုကို ဖန်တီးကြည့်ရအောင်။ ကျွန်တော်တို့မှာ ကိုယ်ရေးအကျဉ်းနဲ့ ပတ်သက်တဲ့ ပုံအနည်းငယ် ရှိတယ်လို့ ယူဆပြီး၊ ဒီပုံတွေကို HTML ထဲမှာ ထည့်သွင်းပြီး `src` နဲ့ `alt` ကို မှန်ကန်စွာ သတ်မှတ်ကြည့်ရအောင်။ `width` နဲ့ `height` ကိုလည်း ထည့်သွင်းပြီး ပုံတွေရဲ့ အရွယ်အစားကို ထိန်းချုပ်ကြည့်ပါ။

---

**Transition Note:** ပုံတွေ ထည့်သွင်းပြီးပြီဆိုတော့၊ အချက်အလက်တွေ (ဥပမာ- ဆက်စပ်အချက်အလက်များ၊ စာရင်းများ) ကို စနစ်တကျ စုစည်းပြီး ပိုမိုဖတ်ရလွယ်ကူအောင် ဘယ်လိုပုံစံမျိုးနဲ့ ဖန်တီးမလဲဆိုတာ လေ့လာကြရအောင်။ အချက်အလက်တွေကို စုစည်းမှုရှိဖို့က အရေးကြီးပါတယ်။

---

## အပိုင်း ၃: Lists (စာရင်းများ) — အချက်အလက်များကို စုစည်းခြင်း

အချက်အလက်တွေ အများကြီးကို တစ်ခုတည်းနဲ့ ရေးသားထားတာထက်၊ စနစ်တကျ စုစည်းထားတာက ပိုပြီး ဖတ်ရှုရလွယ်ကူပါတယ်။ HTML မှာ အချက်အလက်တွေကို စနစ်တကျ စုစည်းဖို့အတွက် **List** အမျိုးအစားတွေကို အသုံးပြုကြပါတယ်။ List များဟာ အချက်အလက်တွေကို အမှတ်အသားနဲ့ အစီအစဉ်နဲ့ တင်ပြဖို့ အထူးကောင်းမွန်တဲ့ နည်းလမ်းတွေ ဖြစ်ပါတယ်။

### Unordered List (`<ul>`)

**Unordered List** ဆိုတာက အမှတ်အသားမရှိဘဲ စာရင်းတွေကို ဖန်တီးဖို့ အသုံးပြုတာ ဖြစ်ပါတယ်။ ဒါဟာ အရာဝတ္ထုအမျိုးမျိုးကို စုစည်းတဲ့အခါ သို့မဟုတ် အစီအစဉ်မရှိတဲ့ အချက်အလက်တွေကို ဖော်ပြတဲ့အခါ အလွန်သင့်တော်ပါတယ်။ Unordered List ကို ဖန်တီးဖို့အတွက် **`<ul>` Tag** ကို အသုံးပြုရပြီး၊ စာရင်းထဲက တစ်ခုချင်းစီကို **`<li>` (List Item) Tag** နဲ့ ထည့်သွင်းရမှာ ဖြစ်ပါတယ်။

ဥပမာအားဖြင့်၊ ကျွန်တော်တို့ရဲ့ ဝါသနာတွေ စာရင်းကို ဖော်ပြတဲ့အခါ အမှတ်အသားမရှိဘဲ စာရင်းပြုစုတာမျိုးကို `<ul>` နဲ့ `<li>` ကို အသုံးပြုပြီး ဖန်တီးနိုင်ပါတယ်။

### Ordered List (`<ol>`)

**Ordered List** ဆိုတာကတော့ အစီအစဉ်လိုက် အမှတ်စဉ်ပေးပြီး စာရင်းတွေကို ဖန်တီးဖို့ အသုံးပြုတာ ဖြစ်ပါတယ်။ အဆင့်လိုက် လုပ်ဆောင်ရမယ့် အရာတွေ၊ အစဉ်လိုက် ဖြစ်နေတဲ့ အချက်အလက်တွေ (ဥပမာ- မှတ်တမ်းများ၊ ညွှန်ကြားချက်များ) ကို ဖော်ပြတဲ့အခါ Ordered List ကို အသုံးပြုသင့်ပါတယ်။ Ordered List ကို ဖန်တီးဖို့အတွက် **`<ol>` Tag** ကို အသုံးပြုရပြီး၊ `<li>` Tag တွေကို အထဲမှာ ထည့်သွင်းရပါတယ်။

Ordered List တွေမှာ အမှတ်စဉ်တွေကို ထိန်းချုပ်ဖို့အတွက် `type` Attribute ကို အသုံးပြုနိုင်ပါတယ်။ အဓိက `type` အမျိုးအစားတွေကတော့ `1` (အမှတ် ၁၊ ၂၊ ၃...)၊ `a` (အက္ခရာ a, b, c...)၊ `A` (အက္ခရာ A, B, C...) စတာတွေ ဖြစ်ပါတယ်။

### List များ၏ အရေးပါပုံ

List များရဲ့ အဓိက အရေးပါပုံကတော့ အချက်အလက်တွေကို စနစ်တကျ၊ လွယ်ကူစွာ ဖတ်ရှုနိုင်စေခြင်းပဲ ဖြစ်ပါတယ်။ အချက်အလက်တွေဟာ ရှုပ်ထွေးနေရင် အလွယ်တကူ နားမလည်နိုင်ဘဲ ဖြစ်သွားတတ်ပါတယ်။ List တွေက အချက်အလက်တွေကို အုပ်စုလိုက် ခွဲခြားပေးပြီး အမြင်ပိုင်းဆိုင်ရာမှာ ပိုမိုရှင်းလင်းစေပါတယ်။

### လက်တွေ့လေ့ကျင့်ခန်း (Lists)

ဒီအချက်တွေကို လက်တွေ့ကျကျ လေ့ကျင့်ဖို့အတွက် ကျွန်တော်တို့ ကိုယ်ရေးအကျဉ်း (သို့မဟုတ်) အတွေ့အကြုံတွေနဲ့ ပတ်သက်တဲ့ အချက်အလက်တွေကို `<ul>` နဲ့ `<ol>` ကို အသုံးပြုပြီး စနစ်တကျ စာရင်းပြုစုကြည့်ရအောင်။ ဥပမာ၊ မိမိရဲ့ ဝါသနာများ၊ အတွေ့အကြုံများ၊ လိုအပ်ချက်များကို စုစည်းပြီး အစီအစဉ်လိုက်နဲ့ အမှတ်အသားမရှိဘဲ ဖော်ပြကြည့်ရအောင်။

---

## အခန်းနိဂုံးနှင့် အနှစ်ချုပ် (Conclusion & Summary)

ဒီအခန်းမှာ ကျွန်တော်တို့ဟာ HTML ရဲ့ အခြေခံအုတ်မြစ်တွေအပြင်၊ ဝဘ်စာမျက်နှာတွေမှာ အရေးပါတဲ့ အစိတ်အပိုင်းသုံးမျိုးဖြစ်တဲ့ Hyperlinks များ၊ ပုံရိပ်များနဲ့ List များအကြောင်းကို အသေးစိတ် လေ့လာခဲ့ကြပါတယ်။

**အနှစ်ချုပ် အချက်များ**

*   **Hyperlinks များ:** Hyperlinks များသည် ဝဘ်ဆိုက်ရဲ့ အဓိက ဆက်သွယ်ရေးလမ်းကြောင်း (Navigation) ဖြစ်ပြီး `<a href="...">` ကို အသုံးပြုပြီး အခြားစာမျက်နှာတွေနဲ့ ချိတ်ဆက်ပေးနိုင်ပါတယ်။ Relative Path နဲ့ Absolute Path တွေရဲ့ ကွာခြားချက်ကို သိရှိထားဖို့ အရေးကြီးပါတယ်။
*   **Images များ:** ပုံရိပ်တွေကို `<img src="..." alt="...">` ကို အသုံးပြုပြီး ထည့်သွင်းပါတယ်။ `src` နဲ့ `alt` Attribute တွေဟာ ပုံရဲ့ အကြောင်းအရာကို ဖော်ပြရုံသာမက Accessibility အတွက်လည်း အလွန်အရေးကြီးပါတယ်။ ပုံရဲ့ အရွယ်အစားကို `width` နဲ့ `height` နဲ့ ထိန်းချုပ်နိုင်ပါတယ်။
*   **Lists များ:** အချက်အလက်တွေကို စနစ်တကျ စုစည်းဖို့အတွက် Unordered List (`<ul>`) နဲ့ Ordered List (`<ol>`) ကို အသုံးပြုပါတယ်။ `<ul>` က အမှတ်အသားမရှိဘဲ စာရင်းပြုစုရန်၊ `<ol>` က အစီအစဉ်လိုက် အမှတ်စဉ်ပေးရန် အသုံးပြုပါတယ်။
*   **HTML Comment များ:** ကုဒ်ကို ဖတ်ရှုသူများအတွက် မှတ်သားထားဖို့နဲ့ နောင်တွင် ပြန်လည်ပြင်ဆင်ရန်အတွက် HTML Comment များ (`<!-- ... -->`) ကို အသုံးပြုခြင်းဟာ ကောင်းမွန်တဲ့ Coding Practice တစ်ခု ဖြစ်ပါတယ်။

**နောက်ဆုံး အတွေးအမြင်**

ဒီအခန်းမှာ သင်ယူခဲ့တဲ့ Link များ၊ ပုံများနှင့် List များနဲ့ အခြေခံ HTML ဖွဲ့စည်းပုံတွေဟာ နောက်ပိုင်းမှာ CSS (Cascading Style Sheets) နဲ့ JavaScript ကို ပေါင်းစပ်တဲ့အခါ ပိုမိုကောင်းမွန်ပြီး ဆွဲဆောင်မှုရှိတဲ့ ဝဘ်ဆိုက်တွေကို တည်ဆောက်ဖို့အတွက် မရှိမဖြစ်လိုအပ်တဲ့ အခြေခံတွေ ဖြစ်ပါတယ်။ ဒီအခြေခံတွေကို ကျွမ်းကျင်စွာ အသုံးပြုနိုင်ပြီဆိုရင်၊ သင်ဟာ အကြောင်းအရာတွေကို အချက်အလက်အဖြစ်သာမက မြင်သာအောင်၊ စနစ်တကျ စုစည်းပြီး ဆက်သွယ်နိုင်တဲ့ ဝဘ်ဆိုက်တွေကို ဖန်တီးနိုင်မှာ ဖြစ်ပါတယ်။ လက်တွေ့လေ့ကျင့်ခန်းတွေကို အသုံးပြုပြီး ကိုယ်ပိုင် Project တစ်ခုကို အောင်မြင်စွာ ဖန်တီးနိုင်ခဲ့ပြီလို့ ဂုဏ်ယူကြပါစို့။

**နောက်ဆက်တွဲ လေ့လာမည့် အကြောင်းအရာ**

HTML ရဲ့ အခြေခံဖွဲ့စည်းပုံတွေကို အပြည့်အဝ နားလည်ပြီးပြီဆိုတော့၊ နောက်အခန်းမှာ ကျွန်တော်တို့ဟာ ဒီ HTML အခြေခံတွေကို လှပအောင် ဒီဇိုင်းဆွဲပုံ၊ အရောင်၊ စာလုံးပုံစံနဲ့ Layout ကို ထိန်းချုပ်နိုင်မယ့် **CSS (Cascading Style Sheets)** ကို လေ့လာပြီး ဒီ HTML အခြေခံတွေကို လက်တွေ့မှာ ဆွဲဆောင်မှုရှိအောင် ပြောင်းလဲပုံကို ဆက်လက်သင်ယူကြပါမည်။
