# Chapter 15: Event Handling — အသုံးပြုသူနဲ့ အပြန်အလှန်ဆက်သွယ်ခြင်း

Web page တစ်ခုကို ကြည့်လိုက်တဲ့အခါ၊ ကျွန်ုပ်တို့ဟာ ရိုးရိုးစာသားတွေ၊ ပုံတွေနဲ့ အရောင်တွေ ပေါ်နေတဲ့ စာမျက်နှာတစ်ခုကို မြင်ရတတ်ကြပါတယ်။ ဒါပေမယ့် အမှန်တကယ် အသုံးဝင်တဲ့ ဝဘ်အက်ပလီကေးရှင်းတွေဟာ static ဖြစ်နေတာထက် အသုံးပြုသူတွေရဲ့ အပြုအမူ (Actions) နဲ့ အပြန်အလှန်တုံ့ပြန်နိုင်မှ အမှန်တကယ် အသုံးဝင်လာပါတယ်။ ကျွန်ုပ်တို့ဟာ ကွန်ပျူတာကို အမိန့်ပေးတာမျိုး မဟုတ်ဘဲ၊ အသုံးပြုသူတွေက ဘာလုပ်မလဲဆိုတာကို နားလည်ပြီး အဲ့ဒီအပေါ်မူတည်ပြီး စာမျက်နှာကို ပြောင်းလဲပြောင်းလဲနေတဲ့ အတွေ့အကြုံကို ဖန်တီးပေးနိုင်မှသာ အမှန်တကယ် ဆွဲဆောင်မှုရှိတဲ့ ဝဘ်ဆိုဒ်တွေ ဖြစ်လာတာ ဖြစ်ပါတယ်။

ဒီအခန်းမှာ ကျွန်ုပ်တို့ဟာ ဒီအရေးကြီးတဲ့ စွမ်းရည်ဖြစ်တဲ့ **Event Handling** ကို လေ့လာသွားမှာ ဖြစ်ပါတယ်။ Event ဆိုတာ ဘာလဲ၊ ဘယ်လိုအမျိုးအစားတွေ ရှိသလဲ၊ ပြီးတော့ JavaScript ကို အသုံးပြုပြီး အသုံးပြုသူရဲ့ လုပ်ဆောင်မှုတိုင်းကို ဘယ်လိုတုံ့ပြန်အောင် လုပ်ဆောင်မလဲဆိုတာကို အသေးစိတ် လေ့လာသွားမှာ ဖြစ်ပါတယ်။

ဒီအခန်းရဲ့ အဓိက လေ့လာမှုတွေကတော့ Events ဆိုတာ ဘာလဲဆိုတာ နားလည်ခြင်း၊ JavaScript နဲ့ Event Listener တွေကို ဘယ်လို တပ်ဆင်မလဲ၊ Event ဖြစ်ပေါ်လာတဲ့အခါ ဘယ်လိုအချက်အလက်တွေကို ရရှိမလဲ၊ ဘယ်လိုလုပ်ဆောင်ချက်တွေကို ထိန်းချုပ်မလဲ၊ နောက်ဆုံးမှာ Event Delegation လိုမျိုး ပိုမိုရှုပ်ထွေးတဲ့ နည်းစနစ်တွေကို ဘယ်လိုအသုံးချမလဲ၊ ပြီးတော့ အားလုံးကို ပေါင်းစပ်ပြီး လက်တွေ့မှာ ဘယ်လိုအသုံးချမလဲဆိုတာကို လေ့လာမှာ ဖြစ်ပါတယ်။

## Section 1: Events ဆိုတာ ဘာလဲ? (Understanding Events)

Web browser တစ်ခုထဲမှာ ဖြစ်ပေါ်လာတဲ့ အဖြစ်အပျက်တွေအားလုံးကို JavaScript က သိရှိပြီး တုံ့ပြန်နိုင်တဲ့ စနစ်တစ်ခု ရှိပါတယ်။ ဒီဖြစ်ပေါ်လာတဲ့ အဖြစ်အပျက်တွေကိုပဲ **Events** လို့ ခေါ်ပါတယ်။ Events ဆိုတာ အသုံးပြုသူရဲ့ အပြုအမူတိုင်းကို ဆိုလိုပါတယ်။ ဥပမာအားဖြင့်၊ မောက်ကြောင်းနဲ့ နှိပ်လိုက်တဲ့အခါ (Mouse click)၊ ကီးဘုတ်ပေါ်က စာလုံးတစ်လုံးကို နှိပ်လိုက်တဲ့အခါ (Keyboard press)၊ ဖောင်တစ်ခုကို တင်သွင်းလိုက်တဲ့အခါ (Form submission)၊ ဒါမှမဟုတ် စာမျက်နှာကို စတင်တင်လိုက်တဲ့အခါ (Page load) စတဲ့ လုပ်ဆောင်မှုတွေအားလုံးဟာ Events ဖြစ်ပါတယ်။

Events အမျိုးအစားတွေက အများကြီးရှိပေမယ့် အခြေခံအားဖြင့်တော့ အဓိကကျတဲ့ အမျိုးအစားတွေကို သိဖို့ လိုပါတယ်။

### Event အမျိုးအစားများ (Types of Events)

ကျွန်ုပ်တို့ အသုံးများတဲ့ အခြေခံ Event အမျိုးအစားတွေကို အောက်မှာ ဖော်ပြပေးလိုက်ပါတယ်။

**Mouse Events:** မောက်ကြောင်းနဲ့ အပြန်အလှန် ဆက်သွယ်မှုတွေဟာ Web မှာ အသုံးအများဆုံး ဖြစ်ပါတယ်။
*   **`click`**: မောက်ကြောင်းကို နှိပ်လိုက်တဲ့အခါ ဖြစ်ပေါ်လာတဲ့ Event ဖြစ်ပါတယ်။
*   **`mouseover`**: မောက်ကြောင်းကို အရာတစ်ခုပေါ် တင်ထားလိုက်တဲ့အခါ ဖြစ်ပေါ်လာတဲ့ Event ဖြစ်ပါတယ်။
*   **`mouseout`**: မောက်ကြောင်းကို အရာတစ်ခုကနေ ထုတ်လိုက်တဲ့အခါ ဖြစ်ပေါ်လာတဲ့ Event ဖြစ်ပါတယ်။

**Keyboard Events:** ကီးဘုတ်အသုံးပြုမှုတွေကလည်း အရေးကြီးပါတယ်။
*   **`keydown`**: ကီးဘုတ်ပေါ်က စာလုံးတစ်ခုကို ဖိထားလိုက်တဲ့အခါ ဖြစ်ပေါ်လာတဲ့ Event ဖြစ်ပါတယ်။
*   **`keyup`**: အဲ့ဒီစာလုံးကို လွှတ်လိုက်တဲ့အခါ ဖြစ်ပေါ်လာတဲ့ Event ဖြစ်ပါတယ်။

**Form Events:** အသုံးပြုသူတွေက ဖောင်တွေမှာ အချက်အလက်တွေ ဖြည့်ပြီး တင်သွင်းတဲ့အခါ ဖြစ်ပေါ်လာတဲ့ Event တွေ ဖြစ်ပါတယ်။
*   **`submit`**: ဖောင်ထဲက အချက်အလက်တွေကို ပို့လိုက်တဲ့အခါ ဖြစ်ပေါ်လာတဲ့ Event ဖြစ်ပါတယ်။

**DOM Events:** Document Object Model (DOM) နဲ့ သက်ဆိုင်တဲ့ Events တွေကတော့ HTML အစိတ်အပိုင်းတွေရဲ့ အခြေအနေ ပြောင်းလဲမှုတွေနဲ့ ဆက်စပ်ပါတယ်။
*   **`load`**: စာမျက်နှာနဲ့ အဲ့ဒီစာမျက်နှာပေါ်က မီဒီယာတွေ (ဥပမာ- ပုံတွေ) အကုန်လုံး တင်ပြီးသွားတဲ့အခါ ဖြစ်ပေါ်လာပါတယ်။
*   **`change`**: Input field တစ်ခုခုရဲ့ တန်ဖိုး (value) ပြောင်းလဲသွားတဲ့အခါ ဖြစ်ပေါ်လာပါတယ်။

ဒီလိုမျိုး Event အမျိုးမျိုးကို နားလည်သွားပြီဆိုတော့၊ အဲ့ဒီ Event တစ်ခု ဖြစ်ပေါ်လာတဲ့အခါမှာ ကျွန်ုပ်တို့ရဲ့ JavaScript က ဘယ်လိုတုံ့ပြန်မလဲဆိုတာကို လေ့လာရပါမယ်။

## Section 2: Event Listener တပ်ဆင်ခြင်း (Attaching Event Listeners)

Events တွေဟာ ဖြစ်ပေါ်လာတာကို သိပြီဆိုတော့၊ နောက်တစ်ဆင့်အနေနဲ့ JavaScript ကို အသုံးပြုပြီး အဲ့ဒီ Event တွေ ဖြစ်ပေါ်လာတဲ့အခါမှာ ကိုယ့်ရဲ့ Code ကို ဘယ်လို အလုပ်လုပ်မလဲဆိုတာ သတ်မှတ်ပေးဖို့ လိုအပ်ပါတယ်။ ဒီအတွက် ကျွန်ုပ်တို့ဟာ **Event Listener** တွေကို HTML Element တွေပေါ်မှာ တပ်ဆင်ရမှာ ဖြစ်ပါတယ်။

### DOM နဲ့ Events ဆက်စပ်ပုံ

JavaScript ဟာ HTML အစိတ်အပိုင်းတွေနဲ့ ဆက်သွယ်နေတဲ့ စနစ်တစ်ခု ဖြစ်ပါတယ်။ HTML Element တွေဟာ DOM (Document Object Model) ရဲ့ အစိတ်အပိုင်းတွေ ဖြစ်ပြီး၊ JavaScript ဟာ ဒီ DOM ကို ထိန်းချုပ်ဖို့အတွက် အသုံးပြုပါတယ်။ Event Handling ဆိုတာ ဒီ DOM မှာ ဖြစ်ပေါ်လာတဲ့ အဖြစ်အပျက်တွေကို JavaScript က သိရှိပြီး ကိုယ်တိုင် ထိန်းချုပ်နိုင်အောင် လုပ်ဆောင်ပေးတဲ့ နည်းလမ်းပဲ ဖြစ်ပါတယ်။

### `addEventListener()` Method ကို အသုံးပြုခြင်း

JavaScript မှာ Event Listener တွေကို တပ်ဆင်ဖို့အတွက် အဓိက အသုံးပြုရမယ့် နည်းလမ်းကတော့ `addEventListener()` Method ဖြစ်ပါတယ်။ ဒီ Method ဟာ HTML Element တစ်ခုပေါ်မှာ သတ်မှတ်ထားတဲ့ Event တစ်ခု ဖြစ်ပေါ်လာတဲ့အခါမှာ JavaScript Function တစ်ခုကို ခေါ်ယူပြီး အဲ့ဒီ Function ကို အလိုအလျောက် အလုပ်လုပ်စေပါတယ်။

`addEventListener()` ရဲ့ အခြေခံ Syntax ဟာ အောက်ပါအတိုင်း ဖြစ်ပါတယ်-

```javascript
element.addEventListener(event_type, function_to_execute);
```

ဒီ Method မှာ အဓိက ပါဝင်တဲ့ အချက်တွေကတော့:
1.  **`element`**: Event ကို နားထောင်မယ့် HTML Element (ဥပမာ- Button, Div စသည်) ဖြစ်ပါတယ်။
2.  **`event_type`**: ဘယ်လို Event ကို နားထောင်မလဲဆိုတာ သတ်မှတ်ပေးရပါမယ် (ဥပမာ- `'click'`, `'keydown'`, `'load'` စသည်)။
3.  **`function_to_execute`**: Event ဖြစ်ပေါ်လာတဲ့အခါ JavaScript က ဘာလုပ်မလဲဆိုတဲ့ Code ဖြစ်ပါတယ်။

### Event Listener ဆိုတာ ဘာလဲ?

Event Listener ဆိုတာ Event တစ်ခု ဖြစ်ပေါ်လာတဲ့အခါ JavaScript က ဘာလုပ်မလဲဆိုတဲ့ အစီအစဉ်ကို သိရှိပြီး အဲ့ဒီအချိန်မှာ သတ်မှတ်ထားတဲ့ လုပ်ဆောင်ချက် (Function) ကို အလိုအလျောက် ခေါ်ယူပြီး လုပ်ဆောင်ပေးတဲ့ အရာကို ဆိုလိုပါတယ်။

ဥပမာအနေနဲ့၊ ကျွန်ုပ်တို့ရဲ့ HTML မှာရှိတဲ့ Button တစ်ခုကို Click လုပ်လိုက်တဲ့အခါ JavaScript က အဲ့ဒီ Click ကို နားထောင်ပြီး၊ Click ဖြစ်တာနဲ့ "Hello World" ဆိုတဲ့ စာကို Console မှာ ပြသဖို့အတွက် `addEventListener()` ကို အသုံးပြုနိုင်ပါတယ်။

```javascript
// HTML မှာ <button id="myButton">Click Me</button> ရှိတယ်လို့ ယူဆပါ
const button = document.getElementById('myButton');

button.addEventListener('click', function() {
    alert('Button ကို နှိပ်လိုက်ပြီ!');
});
```

ဒီဥပမာမှာ `button` ဆိုတဲ့ Element ပေါ်မှာ `'click'` ဆိုတဲ့ Event ကို နားထောင်ပြီး၊ Click ဖြစ်လာတိုင်း `alert()` ဆိုတဲ့ Function ကို အလိုအလျောက် ခေါ်ယူပြီး မက်ဆေ့ချ်ပြသတာကို မြင်ရမှာ ဖြစ်ပါတယ်။

## Section 3: Event Object နှင့် Event.preventDefault() (Event Object and Controlling Flow)

Event Listener ကို တပ်ဆင်လိုက်တဲ့အခါ JavaScript ဟာ Event ဖြစ်ပေါ်လာတဲ့အချိန်မှာ အချက်အလက်အချို့ကို ရရှိပါတယ်။ ဒီအချက်အလက်တွေကို **Event Object** လို့ ခေါ်ပါတယ်။

### Event Object ဆိုတာ ဘာလဲ?

Event Object ဆိုတာ Event ဖြစ်ပေါ်လာတဲ့အချိန်မှာ JavaScript က အဲ့ဒီ Event နဲ့ ပတ်သက်တဲ့ အချက်အလက်အားလုံးကို စုစည်းထားတဲ့ Object တစ်ခု ဖြစ်ပါတယ်။ ဥပမာ- ဘယ်နေရာကနေ Event ဖြစ်လာတာလဲ၊ ဘယ် Element ပေါ်မှာ ဖြစ်တာလဲ စတဲ့ အချက်အလက်တွေ အားလုံးကို Event Object ထဲမှာ သိမ်းထားပါတယ်။

Event Object မှာ အသုံးဝင်တဲ့ အဓိက Property နှစ်ခုကတော့:
*   **`event.target`**: Event ဟာ တိတိကျကျ ဘယ် HTML Element ပေါ်မှာ ဖြစ်ပေါ်လာတာလဲဆိုတာကို ရည်ညွှန်းပေးပါတယ်။
*   **`event.currentTarget`**: Event Listener ကို တပ်ဆင်ထားတဲ့ Element ကို ရည်ညွှန်းပေးပါတယ်။

ဒီအချက်အလက်တွေက Event ကို ပိုမိုတိကျစွာ ကိုင်တွယ်ဖို့ အထောက်အကူပြုပါတယ်။

### `event.preventDefault()` ၏ အရေးပါပုံ

Event Object မှာ ပါဝင်တဲ့ အရေးကြီးတဲ့ Method တစ်ခုကတော့ **`event.preventDefault()`** ဖြစ်ပါတယ်။ ဒီ Method ရဲ့ အဓိက အရေးပါပုံကတော့ Browser ရဲ့ Default လုပ်ဆောင်ချက်တွေကို တားဆီးပေးနိုင်ခြင်းပဲ ဖြစ်ပါတယ်။

အထူးသဖြင့် Form တွေမှာ `submit` Event ကို ကိုင်တွယ်တဲ့အခါ၊ ဖောင်ကို Submit လုပ်လိုက်တဲ့အခါ Browser ဟာ အလိုအလျောက် စာမျက်နှာကို ပြန်လည်တင်တာ (Page Reload) ကို လုပ်ဆောင်လေ့ရှိပါတယ်။ ကျွန်ုပ်တို့ရဲ့ JavaScript က အဲ့ဒီ Page Reload ကို မလိုချင်ဘူးဆိုရင်၊ `event.preventDefault()` ကို အသုံးပြုပြီး Browser ရဲ့ Default လုပ်ဆောင်ချက်ကို ရပ်တန့်စေနိုင်ပါတယ်။

ဥပမာအနေနဲ့၊ ဖောင်တစ်ခုကို Submit လုပ်တဲ့အခါ၊ ကျွန်ုပ်တို့ရဲ့ JavaScript က အဖြေတွေကို စစ်ဆေးပြီးမှသာ နောက်စာမျက်နှာကို သွားခွင့်ပေးချင်တယ်ဆိုပါစို့။ Submit လုပ်လိုက်တာနဲ့ ချက်ချင်း Page Reload မဖြစ်စေချင်တဲ့အခါမှာ၊ `event.preventDefault()` ကို အသုံးပြုပြီး Page Reload ကို တားဆီးနိုင်ပါတယ်။

```javascript
// Form submit ဖြစ်တဲ့အခါ Page Reload မဖြစ်အောင် တားဆီးခြင်း
const form = document.querySelector('form');

form.addEventListener('submit', function(event) {
    event.preventDefault(); // Browser ရဲ့ Default form submission ကို တားဆီးခြင်း
    console.log('Form submission ကို ထိန်းချုပ်လိုက်ပြီ။');
    // ဒီနေရာမှာမှသာ အဖြေတွေကို စစ်ဆေးပြီး နောက်စာမျက်နှာသို့ သွားမည့် Code များကို ထည့်သွင်းနိုင်သည်
});
```

ဒီနည်းလမ်းက Form Handling တွေမှာ အလွန်အရေးကြီးပြီး၊ အသုံးပြုသူရဲ့ လုပ်ဆောင်မှုတွေကို ပိုမိုထိန်းချုပ်နိုင်ဖို့ အထောက်အကူပြုပါတယ်။

## Section 4: Event Delegation (Event Delegation)

Event Handling ကို တစ်ခုချင်းစီ Element အများအပြားပေါ်မှာ `addEventListener()` နဲ့ တပ်ဆင်မယ့်အစား၊ ရှုပ်ထွေးတဲ့ Layout တွေမှာ ပိုမိုထိရောက်စွာ စီမံခန့်ခွဲဖို့ နည်းလမ်းတစ်ခု ရှိပါတယ်။ အဲ့ဒါကတော့ **Event Delegation** ဖြစ်ပါတယ်။

### Event Delegation ဆိုတာ ဘာလဲ?

Event Delegation ဆိုတာ Child Element တစ်ခုရဲ့ Event ကို Parent Element တစ်ခုပေါ်မှာ နားထောင်ပြီး၊ အဲ့ဒီ Event ကို Parent Element ကနေ Child Element တွေဆီကို ပြန်လွှဲပေးတဲ့ နည်းစနစ် ဖြစ်ပါတယ်။ အခြေခံအားဖြင့်၊ Parent Element တစ်ခုတည်းပေါ်မှာ Event Listener တစ်ခုတည်းကို တပ်ဆင်လိုက်ရုံနဲ့၊ အဲ့ဒီ Parent Element အောက်က အခြား Child Element အားလုံးပေါ်မှာ ဖြစ်ပေါ်လာတဲ့ Event တွေကို အလိုအလျောက် သိရှိပြီး ကိုင်တွယ်နိုင်ပါတယ်။

### Why Use Event Delegation? (အကျိုးကျေးဇူးများ)

Event Delegation ကို အသုံးပြုရတဲ့ အဓိက အကျိုးကျေးဇူးတွေကတော့-
*   **Performance မြှင့်တင်ခြင်း**: Element တစ်ခုချင်းစီပေါ်မှာ Event Listener တွေ ထောင်ချီပြီး တပ်ဆင်မယ့်အစား Parent Element တစ်ခုတည်းပေါ်မှာပဲ တပ်ဆင်ရတာမို့ JavaScript ရဲ့ အလုပ်လုပ်ပုံကို ပိုမိုမြန်ဆန်စေပါတယ်။
*   **Code ကို လျှော့ချခြင်း**: Element အများအပြားအတွက် သီးခြား Listener တွေ ရေးသားစရာမလိုတော့ဘဲ အခြေခံ Listener တစ်ခုတည်းနဲ့ အလုပ်လုပ်နိုင်တာကြောင့် Code ကို ပိုမိုရှင်းလင်းပြီး ထိန်းသိမ်းရလွယ်ကူစေပါတယ်။

### `event.delegationListener` ကို အသုံးပြုပုံနှင့် Event Bubbling

Event Delegation ကို နားလည်ဖို့အတွက် **Event Bubbling** ဆိုတဲ့ သဘောတရားကို သိဖို့ လိုပါတယ်။ Event Bubbling ဆိုတာ အသုံးပြုသူက အနိမ့်ဆုံး Level မှာ (ဥပမာ- `<li class="item">` တစ်ခုပေါ်မှာ) Click လုပ်လိုက်တဲ့အခါ အဲ့ဒီ Event ဟာ အဲ့ဒီ Element ရဲ့ Parent Element (ဥပမာ- `<ul>`) ကို အပေါ်သို့ တက်သွားပြီး အဲ့ဒီ Parent Element အပေါ်မှာပါ Event ဖြစ်ပေါ်လာစေတဲ့ ဖြစ်စဉ် ဖြစ်ပါတယ်။

Event Delegation မှာ ဒီ Bubbling ကို အသုံးချပါတယ်။ Parent Element (ဥပမာ- `<ul>`) ပေါ်မှာ Event Listener တစ်ခုတည်းကို တပ်ဆင်လိုက်တဲ့အခါ၊ အဲ့ဒီ `<ul>` အတွင်းက အခြား `<li>` Element တွေပေါ်မှာ ဖြစ်ပေါ်လာတဲ့ Event တွေဟာ အလိုအလျောက် အဲ့ဒီ Parent Listener ကို ရောက်ရှိလာပါတယ်။

### Practical Example: List များပေါ်တွင် Event Delegation

ဥပမာအနေနဲ့၊ List တစ်ခု (<ul>) ထဲမှာရှိတဲ့ အကြောင်းအရာတစ်ခုချင်းစီ (<li>) ပေါ်မှာ Click လုပ်လိုက်တဲ့အခါ၊ အဲ့ဒီ Click ကို JavaScript နဲ့ ကိုင်တွယ်ချင်တယ်ဆိုပါစို့။

**HTML Structure:**
```html
<ul id="myList">
    <li class="item">Item 1</li>
    <li class="item">Item 2</li>
    <li class="item">Item 3</li>
</ul>
```

**JavaScript Implementation:**
```javascript
const myList = document.getElementById('myList');

myList.addEventListener('click', function(event) {
    // event.target ကို အသုံးပြုပြီး ဘယ် Element ကို Click လိုက်လဲဆိုတာ စစ်ဆေးခြင်း
    if (event.target.classList.contains('item')) {
        console.log('Item တစ်ခုကို Click လုပ်လိုက်ပါပြီ:', event.target.textContent);
        // ဤနေရာတွင် Item တစ်ခုချင်းစီအတွက် သင့်လျော်သော လုပ်ဆောင်ချက်များကို ထည့်သွင်းနိုင်သည်
    }
});
```

ဒီဥပမာမှာ၊ ကျွန်ုပ်တို့ဟာ `<ul>` ပေါ်မှာပဲ Listener တစ်ခု တပ်ဆင်လိုက်ရုံနဲ့၊ `<li>` တစ်ခုခုပေါ်မှာ Click ဖြစ်လာတဲ့အခါ အဲ့ဒီ Click ကို `event.target` ကနေ ဘယ် Element ကို Click လိုက်လဲဆိုတာကို စစ်ဆေးပြီး ကိုင်တွယ်နိုင်သွားပါတယ်။ ဒါဟာ Element အများအပြားကို ကိုင်တွယ်ရာမှာ အလွန်ထိရောက်တဲ့ နည်းလမ်းပဲ ဖြစ်ပါတယ်။

## Section 5: လက်တွေ့ လေ့ကျင့်ခန်းနှင့် ပရောဂျက် (Practical Exercises and Project)

အခုချိန်ထိ Event Handling ရဲ့ အခြေခံသဘောတရားတွေ၊ `addEventListener()` ကို အသုံးပြုပုံ၊ Event Object နဲ့ `preventDefault()` ကို ထိန်းချုပ်ပုံ၊ ပြီးတော့ Event Delegation ကို နားလည်သွားပြီ ဖြစ်ပါတယ်။ အခု အဲ့ဒီအသိပညာတွေကို လက်တွေ့မှာ အသုံးချပြီး ကျွမ်းကျင်မှုကို မြှင့်တင်ဖို့အတွက် အပြန်အလှန် ဆက်သွယ်နိုင်တဲ့ **Interactive Quiz Application** တစ်ခုကို ဖန်တီးကြည့်ကြရအောင်။

### Practical Focus: Interactive Quiz Application ဖန်တီးခြင်း

ဒီပရောဂျက်မှာ အသုံးပြုသူရဲ့ အဖြေတွေကို JavaScript က စစ်ဆေးပြီး၊ အဖြေမှန်/မှား ကို ပြသပေးမယ့်အပြင်၊ နောက်ဆုံးမှာ Form Submission ကို ထိန်းချုပ်ပြီး အဖြေမှန်မှား စစ်ဆေးပြီးမှသာ နောက်စာမျက်နှာသို့ သွားခွင့်ပေးမယ့် စနစ်ကို တည်ဆောက်မှာ ဖြစ်ပါတယ်။

**Step 1: HTML Structure တည်ဆောက်ခြင်း**
မေးခွန်းတွေ၊ ရွေးချယ်စရာတွေနဲ့ အဖြေတွေအတွက် HTML ကို စနစ်တကျ တည်ဆောက်ရပါမယ်။ Form တစ်ခုနဲ့ မေးခွန်းအလိုက် အဖြေတွေကို သိမ်းဆည်းဖို့ လိုအပ်ပါတယ်။

**Step 2: CSS ဖြင့် အခြေခံ ဒီဇိုင်းဆွဲခြင်း**
အသုံးပြုရလွယ်ကူပြီး ဆွဲဆောင်မှုရှိတဲ့ ဒီဇိုင်းကို CSS ကို အသုံးပြုပြီး ဖန်တီးရပါမယ်။

**Step 3: JavaScript ဖြင့် Event Handling ကို အသုံးပြုခြင်း**
ဒီအဆင့်မှာ အဓိကကျတဲ့ Event Handling တွေကို အသုံးပြုရပါမယ်။
*   **အသုံးပြုသူရဲ့ ရွေးချယ်မှု (Click) ကို နားထောင်ခြင်း**: အသုံးပြုသူက ရွေးချယ်တဲ့ အဖြေကို Click လုပ်လိုက်တဲ့အခါ JavaScript က အဲ့ဒီအဖြေကို သိရှိရပါမယ်။
*   **အဖြေမှန်/မှား ကို ပြသခြင်း**: အသုံးပြုသူရဲ့ အဖြေနဲ့ မှန်ကန်တဲ့အဖြေကို နှိုင်းယှဉ်ပြီး မှန်ကန်ကြောင်း သို့မဟုတ် မှားကြောင်းကို Screen ပေါ်မှာ ပြသရပါမယ်။
*   **Form Submission ကို ထိန်းချုပ်ခြင်း**: အသုံးပြုသူက "Submit" ခလုတ်ကို နှိပ်လိုက်တဲ့အခါ၊ JavaScript က အဖြေအားလုံးကို စစ်ဆေးရပါမယ်။ အဖြေမှန်မှားကို စစ်ဆေးပြီးမှသာ နောက်စာမျက်နှာသို့ သွားခွင့်ပေးမယ့် Logic ကို အသုံးပြုရပါမယ်။ ဒီနေရာမှာ `event.preventDefault()` ကို အသုံးပြုပြီး Browser ရဲ့ Default လုပ်ဆောင်ချက်ကို ထိန်းချုပ်ဖို့ အရေးကြီးပါတယ်။

**Step 4: Review**
အဆုံးမှာ ဖန်တီးထားတဲ့ Quiz Application ကို ပြန်လည်သုံးသပ်ပြီး၊ အသုံးပြုသူတွေအတွက် ပိုမိုကောင်းမွန်တဲ့ အတွေ့အကြုံရရှိအောင် Code တွေကို ပြင်ဆင်ပြင်ဆင်ရပါမယ်။

## 📝 Chapter Summary (အခန်းအကျဉ်းချုပ်)

ဒီအခန်းမှာ ကျွန်ုပ်တို့ဟာ Web Page တွေမှာ အသုံးပြုသူတွေနဲ့ အပြန်အလှန် ဆက်သွယ်နိုင်ဖို့အတွက် အခြေခံကျတဲ့ စွမ်းရည်ဖြစ်တဲ့ Event Handling ကို အသေးစိတ် လေ့လာခဲ့ကြပါတယ်။

ကျွန်ုပ်တို့ လေ့လာခဲ့တဲ့ အဓိက အချက်တွေကတော့- Events ဆိုတာ ဘာလဲဆိုတာ နားလည်ခြင်း၊ `addEventListener()` ကို အသုံးပြုပြီး Event Listener တွေကို တပ်ဆင်ပုံ၊ Event ဖြစ်ပေါ်လာတဲ့အခါ JavaScript က ဘယ်လိုအချက်အလက်တွေ (Event Object) ကို ရရှိမလဲ၊ Browser ရဲ့ Default လုပ်ဆောင်ချက်တွေကို `event.preventDefault()` နဲ့ ဘယ်လို ထိန်းချုပ်မလဲ၊ ပြီးတော့ ရှုပ်ထွေးတဲ့ Layout တွေမှာ Event ကို ထိရောက်စွာ စီမံခန့်ခွဲမယ့် နည်းလမ်းဖြစ်တဲ့ Event Delegation ကို လေ့လာခဲ့ပါတယ်။

**Key Takeaway:** Event Handling ဆိုတာ Web Application တွေမှာ အသုံးပြုသူရဲ့ Input ကို သိရှိပြီး Dynamic ဖြစ်အောင် ပြောင်းလဲပေးနိုင်ဖို့ အခြေခံကျတဲ့ စွမ်းရည် ဖြစ်ကြောင်း အလေးပေးပြောကြားလိုပါတယ်။ အသုံးပြုသူရဲ့ လုပ်ဆောင်မှုတိုင်းကို JavaScript က နားထောင်ပြီး၊ အဲ့ဒီအပေါ်မူတည်ပြီး စာမျက်နှာကို အချိန်နဲ့အမျှ ပြောင်းလဲပေးနိုင်တဲ့ စွမ်းရည်ဟာ ခေတ်မီ Web Development ရဲ့ အခြေခံအုတ်မြစ် ဖြစ်ပါတယ်။

နောက်အခန်းမှာတော့ ဒီ Event Handling ကို ပိုမိုနက်ရှိုင်းစွာ အသုံးပြုပြီး HTML Element တွေရဲ့ အကြောင်းအရာတွေကို JavaScript နဲ့ တိုက်ရိုက် ပြောင်းလဲပစ်နိုင်တဲ့ **DOM Manipulation** အကြောင်းကို လေ့လာသွားမှာ ဖြစ်ပါတယ်။
