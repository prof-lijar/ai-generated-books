# Chapter 13: JavaScript Function များနှင့် Array များ

## 1. Introduction: JavaScript ၏ အခြေခံအုတ်မြစ် (The Foundation of JavaScript)

မင်္ဂလာပါ! HTML နှင့် CSS ဖြင့် ကျွန်ုပ်တို့သည် ဝက်ဘ်စာမျက်နှာများ၏ အသွင်အပြင်နှင့် ပုံစံကို ဖန်တီးနိုင်ခဲ့ပြီ ဖြစ်သည်။ ယခုအခါ ကျွန်ုပ်တို့သည် JavaScript ၏ အနှစ်သာရကို နားလည်ရန် အရေးကြီးသော အဆင့်သို့ ရောက်ရှိလာခဲ့ပြီ ဖြစ်သည်။ JavaScript သည် ဝက်ဘ်စာမျက်နှာများကို "အသက်ဝင်စေသော" ဘာသာစကားဖြစ်ပြီး၊ ၎င်းကို အသုံးပြု၍ စာမျက်နှာများကို အပြန်အလှန်တုံ့ပြန်နိုင်သော (Interactive) အရာများအဖြစ် ပြောင်းလဲပေးနိုင်ပါသည်။

ဤအခန်းတွင် ကျွန်ုပ်တို့သည် JavaScript ၏ အခြေခံအုတ်မြစ်ဖြစ်သော **Functions (လုပ်ဆောင်ချက်များ)** နှင့် **Arrays (အချက်အလက် စုစည်းမှုများ)** တို့ကို အဓိကထား လေ့လာသွားမည် ဖြစ်သည်။ ဤအရာများသည် ကျွန်ုပ်တို့၏ ကုဒ်များကို ပိုမိုထိရောက်စွာ ရေးသားနိုင်ရန်နှင့် ရှုပ်ထွေးသော ဒေတာများကို စီမံခန့်ခွဲနိုင်ရန်အတွက် အလွန်အရေးကြီးပါသည်။

### 1.1 Function နှင့် Array များ၏ အရေးပါပုံ (The Importance of Functions and Arrays)

ကုဒ်ရေးသားရာတွင် အချိန်ကြာလာသည်နှင့်အမျှ ကျွန်ုပ်တို့သည် ပိုမိုရှုပ်ထွေးလာသည်။ ထိုအချိန်တွင် Functions နှင့် Arrays တို့သည် ကျွန်ုပ်တို့၏ဘဝကို အလွယ်တကူ ပိုမိုကောင်းမွန်စေမည့် အင်တယာပစ္စည်းများ ဖြစ်လာပါသည်။

**Functions (လုပ်ဆောင်ချက်များ)** ကို ကျွန်ုပ်တို့ ဘာကြောင့် လိုအပ်သလဲ။ Functions များသည် ကုဒ်ကို **ပြန်သုံးနိုင်စေခြင်း (Code Reusability)** နှင့် **အပိုင်းလိုက် ခွဲခြားနိုင်ခြင်း (Modularity)** ကို ပေးစွမ်းသည်။ ဆိုလိုသည်မှာ ကျွန်ုပ်တို့သည် အလားတူလုပ်ဆောင်ချက်များကို ထပ်ခါထပ်ခါ ရေးသားစရာမလိုဘဲ၊ တစ်ကြိမ်ရေးပြီးသား Function တစ်ခုကို လိုအပ်သည့်အခါတိုင်း ပြန်ခေါ်သုံးနိုင်ခြင်း ဖြစ်သည်။ ဤသည်မှာ **DRY (Don't Repeat Yourself)** ဟူသော ကောင်းမွန်သော ပရမတ်မူကို အကောင်အထည်ဖော်ခြင်းပင် ဖြစ်သည်။

ထို့အပြင် **Arrays (အချက်အလက် စုစည်းမှုများ)** ကို ဘာကြောင့် လိုအပ်သလဲ။ Arrays များသည် ကျွန်ုပ်တို့၏ ဒေတာများကို စနစ်တကျ၊ အစီအစဉ်တကျ စုစည်းထားရန် ကူညီပေးသည်။ ဥပမာအားဖြင့်၊ လူတစ်စု၏ အမည်များ၊ စာရင်းများ သို့မဟုတ် ဂိမ်းပွဲများ၏ ရမှတ်များကဲ့သို့သော အချက်အလက်များစွာကို တစ်နေရာတည်းတွင် ထိန်းသိမ်းထားရန် Arrays များသည် အကောင်းဆုံး နည်းလမ်းပင် ဖြစ်သည်။

Functions များသည် **လုပ်ဆောင်ချက် (Action)** ကို ကိုယ်စားပြုပြီး Arrays များသည် **ဒေတာ (Data)** ကို ကိုယ်စားပြုသည်။ ဤနှစ်ခုကို ပေါင်းစပ်လိုက်သောအခါ ကျွန်ုပ်တို့သည် ပိုမိုကျယ်ပြန့်သောနှင့် အကျိုးရှိသော အက်ပလီကေးရှင်းများကို ဖန်တီးနိုင်လာပါသည်။

### 1.2 Chapter Overview (ဘာတွေလေ့လာမလဲ)

ဤအခန်းတွင် ကျွန်ုပ်တို့သည် အောက်ပါအဆင့်များအတိုင်း လေ့လာသွားပါမည်။

ပထမအဆင့်အနေဖြင့် **JavaScript Functions** များကို အသေးစိတ်လေ့လာပြီး ကုဒ်ကို မည်သို့ စနစ်တကျ စီစဉ်ပုံကို နားလည်ကြမည်။ ဒုတိယအဆင့်အနေဖြင့် **JavaScript Arrays** များကို လေ့လာပြီး ဒေတာများကို မည်သို့ စုစည်းကာ ထိန်းသိမ်းပုံကို သိရှိကြမည်။ နောက်ဆုံးအဆင့်အနေဖြင့် ဤအရာနှစ်ခုကို ပေါင်းစပ်၍ **လက်တွေ့ To-Do List အက်ပလီကေးရှင်း** တစ်ခု၏ နောက်ကွယ်မှ Logic ကို တည်ဆောက်ကြည့်ရှုလေ့လာကြမည် ဖြစ်သည်။

ဤအစီအစဉ်အတိုင်း လေ့လာသွားပါက သင်သည် JavaScript ၏ အခြေခံများကို ခိုင်မာစွာ နားလည်ပြီး ကိုယ်ပိုင် အက်ပလီကေးရှင်းများကို ကိုယ်တိုင်တည်ဆောက်နိုင်မည့် စွမ်းရည်ကို ရရှိမည် ဖြစ်ပါသည်။

## 2. JavaScript Functions: ကုဒ်ကို ပြန်သုံးနိုင်အောင် ဖွဲ့စည်းခြင်း (Organizing Code for Reusability)

ကုဒ်ရေးသားရာတွင် အကောင်းဆုံးနည်းလမ်းမှာ အပိုင်းငယ်များအဖြစ် ခွဲ၍ စီစဉ်ခြင်း ဖြစ်သည်။ ဤအပိုင်းတွင် ကျွန်ုပ်တို့သည် Functions များ မည်သို့ အလုပ်လုပ်ပုံနှင့် မည်သို့ရေးသားပုံကို အာရုံစိုက်ပါမည်။

### 2.1 Function ဆိုတာ ဘာလဲ? (What is a Function?)

JavaScript တွင် **Function** ဆိုသည်မှာ သတ်မှတ်ထားသော လုပ်ဆောင်ချက်တစ်ခုကို အုပ်စုလိုက် စုစည်းထားသော ကုဒ်အပိုင်းအစ (Code Block) တစ်ခု ဖြစ်သည်။ ၎င်း၏ အဓိကရည်ရွယ်ချက်မှာ အထပ်ထပ် ရေးသားရန် လိုအပ်သည့် ကုဒ်များကို တစ်နေရာတည်းတွင် သိမ်းဆည်းပြီး၊ လိုအပ်သည့်အခါတိုင်း ထို Function ကို ခေါ်ဆိုခြင်းဖြင့် ထိုလုပ်ဆောင်ချက်ကို အလိုအလျောက် လုပ်ဆောင်စေရန် ဖြစ်သည်။

Function များ၏ အဓိက အကျိုးကျေးဇူးမှာ **Reusability (ပြန်သုံးနိုင်ခြင်း)** ဖြစ်သည်။ ကျွန်ုပ်တို့သည် အခက်အခဲရှိသော ကုဒ်များကို ထပ်ခါထပ်ခါ ရေးသားနေရမည့်အစား၊ Function တစ်ခုကိုသာ ရေးပြီး လိုအပ်သည့်နေရာတိုင်းတွင် ထို Function ကို ခေါ်ဆိုရုံဖြင့် အချိန်ကုန်သက်သာပြီး ကုဒ်ကို ပိုမိုရှင်းလင်းစေသည်။ ၎င်းကို **DRY (Don't Repeat Yourself)** ဟု ခေါ်သော ပရမတ်မူနှင့် တိုက်ရိုက်ဆက်စပ်နေပါသည်။

### 2.2 Function ပုံစံများ (Types of Functions)

JavaScript တွင် Function များအား ရေးသားရန် နည်းလမ်းများစွာ ရှိသော်လည်း အဓိကအားဖြင့် သုံးမျိုးကို သိထားရန် လိုအပ်ပါသည်။

**Function Declaration (Function ကြေညာခြင်း):**
ဤပုံစံသည် JavaScript ၏ ရှေးကျသော နည်းလမ်းဖြစ်သည်။ Function ကို စာသားအဖြစ် ကြေညာပြီး၊ ၎င်းကို အမည်ပေးကာ သတ်မှတ်ထားသော နေရာတွင် အသုံးပြုနိုင်သည်။

```javascript
function greet() {
    console.log("မင်္ဂလာပါ!");
}

greet(); // အလုပ်လုပ်သည်
```

**Function Expression (Function ဖော်ပြချက်):**
ဤပုံစံတွင် Function ကို တန်ဖိုးတစ်ခုအဖြစ် သတ်မှတ်ပြီး အမည်ပေးသော Variable တစ်ခုနှင့် ချိတ်ဆက်ထားသည်။ ၎င်းသည် Function ကို အခြားကုဒ်များနှင့် ပိုမိုပေါင်းစပ်ရန် အသုံးဝင်သည်။

```javascript
const sayHello = function() {
    console.log("မင်္ဂလာပါ!");
};

sayHello(); // အလုပ်လုပ်သည်
```

**Arrow Functions (=>):**
Arrow Functions သည် JavaScript ၏ ခေတ်သစ်နှင့် အလွန်ရေပန်းစားသော ရေးသားပုံဖြစ်သည်။ ၎င်းသည် Function ကို ပိုမိုတိုတောင်းပြီး ရေးသားရလွယ်ကူစေသည်။ အထူးသဖြင့်၊ ၎င်းသည် ပိုမိုရှုပ်ထွေးသော Context များတွင် အလွန်အသုံးဝင်သည်။

```javascript
const add = (a, b) => {
    return a + b;
};

console.log(add(5, 3)); // အထွက်: 8
```

### 2.3 Function များ၏ အစိတ်အပိုင်းများ (Components of Functions)

မည်သည့် Function ကိုမဆို အောက်ပါ အစိတ်အပိုင်းသုံးခုကို နားလည်ထားရန် လိုအပ်သည်။

**Parameters (အချက်အလက် လက်ခံခြင်း):**
Parameters ဆိုသည်မှာ Function ကို ခေါ်ဆိုသည့်အခါ အတွင်းသို့ ထည့်သွင်းပေးမည့် **Input** များ ဖြစ်သည်။ ၎င်းတို့ကို Function ကို ရေးသားသည့်အခါ ခေါ်ဆိုမှုများအတွက် နေရာလွတ် (placeholders) အဖြစ် သတ်မှတ်ပေးသည်။

**Arguments (အမှန်တကယ် တန်ဖိုးများ):**
Arguments ဆိုသည်မှာ Function ကို တကယ်ခေါ်ဆိုသည့်အခါ Parameter များထဲသို့ ထည့်သွင်းပေးလိုက်သော **Actual Values** များ ဖြစ်သည်။

**Return Value (အကျိုးရလဒ် ပြန်ပေးခြင်း):**
Return Value ဆိုသည်မှာ Function တစ်ခု၏ လုပ်ဆောင်မှုပြီးဆုံးသောအခါ ပြန်ပေးလိုက်သော **Output** သို့မဟုတ် **Result** ဖြစ်သည်။ Function တစ်ခုသည် အချက်အလက်ကို ပြန်ပေးရန် (return) သို့မဟုတ် တန်ဖိုးတစ်ခုကို ပြန်ပေးရန် လိုအပ်သည့်အခါ အသုံးပြုသည်။

**ဥပမာ:**

```javascript
function calculateArea(length, width) { // length နှင့် width တို့သည် Parameters
    let area = length * width;
    return area; // area ကို Return Value အဖြစ် ပြန်ပေးသည်
}

// Function ကို ခေါ်ဆိုရာတွင် Arguments များ ပေးပို့သည်
const result = calculateArea(10, 5); // 10 နှင့် 5 တို့သည် Arguments များ
console.log("ဧရိယာမှာ:", result); // အထွက်: 50
```

### 2.4 လက်တွေ့လေ့ကျင့်ခန်း (Practical Exercise)

Function တစ်ခုကို အသုံးပြု၍ ရိုးရှင်းသော ဂဏန်းတွက်ချက်မှုတစ်ခုကို လုပ်ဆောင်ကြည့်ရအောင်။

```javascript
// Area Calculation Function
function calculateArea(length, width) {
    // Length နှင့် width ကို မြှောက်ပြီး အကျိုးရလဒ်ကို ပြန်ပေးသည်
    return length * width;
}

// Function ကို ခေါ်ဆိုပြီး ရလဒ်ကို သိမ်းဆည်းခြင်း
const roomArea = calculateArea(12, 8);

console.log("အလျား ၁၂ နှင့် အနံ ၈ ၏ ဧရိယာမှာ:", roomArea);
// အထွက်: အလျား ၁၂ နှင့် အနံ ၈ ၏ ဧရိယာမှာ: 96
```

ဤဥပမာမှ သင်သည် Function တစ်ခုသည် မည်သို့ Input များ လက်ခံကာ Process လုပ်ပြီး Output တစ်ခုကို ပြန်ပေးနိုင်သည်ကို မြင်တွေ့ရမည် ဖြစ်သည်။

## 3. JavaScript Arrays: အချက်အလက် စုစည်းမှုများ (Managing Collections of Data)

Functions များက လုပ်ဆောင်ချက်များကို စီမံခန့်ခွဲပေးနိုင်သကဲ့သို့၊ Arrays များသည် ကျွန်ုပ်တို့၏ ဒေတာများကို စုစည်းထိန်းသိမ်းရန်အတွက် အလွန်အရေးကြီးပါသည်။

### 3.1 Array ဆိုတာ ဘာလဲ? (What is an Array?)

Array ဆိုသည်မှာ **အစီအစဉ်တကျ စီစဉ်ထားသော၊ Index များဖြင့် အမှတ်အသားပြုထားသော တန်ဖိုးများ၏ စုစည်းမှု** ဖြစ်သည်။ ဆိုလိုသည်မှာ Array တစ်ခုသည် တန်ဖိုးများ (ဥပမာ- နံပါတ်များ၊ စာသားများ၊ အခြား Array များ) ကို တစ်နေရာတည်းတွင် စုစည်းထားသည့် ကွန်တိန်နာ (Container) တစ်ခု ဖြစ်သည်။

Array များသည် JavaScript တွင် အလွန်အသုံးများပြီး ဒေတာများကို စီမံခန့်ခွဲရာတွင် အလွန်ထိရောက်သည်။ ၎င်းတို့သည် အစီအစဉ်တကျရှိသော အချက်အလက်များကို သိမ်းဆည်းရန်အတွက် အကောင်းဆုံး နည်းလမ်း ဖြစ်သည်။

**Array များ ဖန်တီးခြင်း:**
Array တစ်ခုကို ဖန်တီးရန်အတွက် စက်ဝိုင်းပုံတန်းအကွင်းများ (`[]`) ကို အသုံးပြုရသည်။

```javascript
// အလွတ် Array တစ်ခု ဖန်တီးခြင်း
let numbers = [];

// တန်ဖိုးများ ထည့်သွင်းခြင်း
numbers.push(10);
numbers.push(20);
numbers.push(30);

console.log(numbers); // အထွက်: [10, 20, 30]
```

### 3.2 Array ဝင်ရောက်မှုများနှင့် အခြေခံ Indexing (Accessing and Indexing)

Array များ၏ အဓိကအချက်မှာ အချက်အလက်များကို Index များဖြင့် မှတ်သားထားခြင်းပင် ဖြစ်သည်။ JavaScript တွင် Index များသည် **Zero-based Indexing** ကို အသုံးပြုသည်။ ဆိုလိုသည်မှာ ပထမဆုံး အချက်အလက်သည် Index 0 မှ စတင်သည်။

**Index ဖြင့် ဝင်ရောက်ခြင်း:**
Array အတွင်းရှိ တန်ဖိုးတစ်ခုကို ရယူရန်အတွက် Square Brackets (`[]`) ကို အသုံးပြုပြီး Index ကို ထည့်ပေးရသည်။

```javascript
let fruits = ["Apple", "Banana", "Cherry", "Date"];

// Index 0 မှ Apple ကို ရယူခြင်း
console.log(fruits[0]); // အထွက်: Apple

// Index 2 မှ Cherry ကို ရယူခြင်း
console.log(fruits[2]); // အထွက်: Cherry
```

**`.length` Property:**
Array တစ်ခုတွင် မည်မျှသော အချက်အလက်များ စုစည်းထားသည်ကို သိရှိရန်အတွက် `.length` property ကို အသုံးပြုသည်။

```javascript
console.log(fruits.length); // အထွက်: 4
```

### 3.3 Array Manipulation Methods (Array Methods)

Array များသည် ဒေတာများကို စီမံခန့်ခွဲရန်အတွက် အလွန်အစွမ်းထက်သော Built-in Methods များ ပိုင်ဆိုင်ထားသည်။ ဤ Methods များသည် ဒေတာများကို ပြောင်းလဲခြင်း (Mutation) သို့မဟုတ် အသစ်သော Array တစ်ခုကို ပြန်လည်ဖန်တီးခြင်း (Immutability) များဖြင့် လုပ်ဆောင်သည်။

**Element များ ထည့်ခြင်းနှင့် ဖယ်ရှားခြင်း (Adding/Removing Elements):**

*   **`.push()`:** Array ၏ အဆုံးသို့ အသစ်သော Element တစ်ခု သို့မဟုတ် ထို့ထက်မကကို ထည့်သွင်းသည်။
*   **`.pop()`:** Array ၏ အဆုံးမှ Element တစ်ခုကို ဖယ်ရှားပြီး ထို Element ကို ပြန်ပေးသည်။
*   **`.shift()`:** Array ၏ အစမှ Element တစ်ခုကို ဖယ်ရှားပြီး ထို Element ကို ပြန်ပေးသည်။
*   **`.unshift()`:** Array ၏ အစသို့ အသစ်သော Element တစ်ခု သို့မဟုတ် ထို့ထက်မကကို ထည့်သွင်းသည်။

```javascript
let colors = ["Red", "Green"];

// .push() - အဆုံးသို့ ထည့်ခြင်း
colors.push("Blue");
console.log(colors); // အထွက်: ["Red", "Green", "Blue"]

// .shift() - အစမှ ဖယ်ရှားခြင်း
let firstColor = colors.shift();
console.log("ဖယ်ရှားလိုက်သော အရောင်:", firstColor); // အထွက်: Red
console.log(colors); // အထွက်: ["Green", "Blue"]
```

**Data Transformation နှင့် Iteration (Iterating/Transforming Data):**

ဤ Method များသည် Array တစ်ခုရှိ အချက်အလက်အားလုံးကို တစ်ခုချင်းစီ စစ်ဆေးပြီး ပြောင်းလဲပေးခြင်း သို့မဟုတ် အသစ်သော Array တစ်ခုကို ပြန်လည်ပေးအပ်ခြင်းများအတွက် အလွန်အသုံးဝင်သည်။

*   **`.forEach()`:** Array ရှိ Element တစ်ခုချင်းစီအတွက် သတ်မှတ်ထားသော လုပ်ဆောင်ချက်ကို ထပ်ခါထပ်ခါ လုပ်ဆောင်စေသည်။
*   **`.map()`:** Array ရှိ Element တစ်ခုချင်းစီကို ပြောင်းလဲပြီး **အသစ်သော Array တစ်ခု** ကို ပြန်ပေးသည်။
*   **`.filter()`:** Array ရှိ Element များထဲမှ သတ်မှတ်ချက် (Condition) နှင့် ကိုက်ညီသော Element များ **သာ ပါဝင်သည့် အသစ်သော Array တစ်ခု** ကို ပြန်ပေးသည်။

```javascript
let scores = [85, 92, 78, 95, 88];

// .forEach() - အချက်အလက်အားလုံးကို ပြသခြင်း
console.log("Score များ:");
scores.forEach(score => {
    console.log(score);
});
// အထွက်: 85, 92, 78, 95, 88

// .map() - Score များကို 10 မှတ်အထိ ပြောင်းလဲခြင်း
let mappedScores = scores.map(score => score + 10);
console.log("အဆင့်မြှင့်ပြီးသော စကောများ:", mappedScores); // အထွက်: [95, 102, 88, 105, 98]

// .filter() - 90 မှ အထက်ရှိသော စကောများကိုသာ ထုတ်選ခြင်း
let passingScores = scores.filter(score => score >= 90);
console.log("အောင်မှတ်များ:", passingScores); // အထွက်: [92, 95]
```

## 4. လက်တွေ့ပေါင်းစပ်ခြင်း: To-Do List Application Logic (Putting It Together: The Logic of a To-Do List)

ယခုအခါ ကျွန်ုပ်တို့သည် Functions နှင့် Arrays တို့ကို ပေါင်းစပ်အသုံးပြု၍ အမှန်တကယ် အကျိုးရှိသော အက်ပလီကေးရှင်းတစ်ခုကို မည်သို့တည်ဆောက်ရမည်ကို လေ့လာကြမည် ဖြစ်သည်။ ကျွန်ုပ်တို့၏ ပန်းတိုင်မှာ **To-Do List** တစ်ခုကို စီမံခန့်ခွဲနိုင်သော Logic ကို ရေးသားရန် ဖြစ်သည်။

### 4.1 Project Setup and Goal Definition

To-Do List Application ၏ ရည်ရွယ်ချက်မှာ အသုံးပြုသူက လုပ်ဆောင်ရန် အလုပ်များကို စာရင်းအဖြစ် သိမ်းဆည်းခြင်း၊ ထပ်ထည့်ခြင်း၊ ပြသခြင်းနှင့် ဖယ်ရှားခြင်းတို့ကို လုပ်ဆောင်နိုင်ရန် ဖြစ်သည်။

ကျွန်ုပ်တို့အတွက် လိုအပ်သော အချက်အလက်များမှာ-
၁။ လုပ်ဆောင်ရန် အလုပ်များ (Tasks) ကို စုစည်းထားမည့် Array တစ်ခု။
၂။ အလုပ်အသစ်ကို ထည့်ရန်အတွက် Function တစ်ခု။
၃။ စာရင်းအား ပြသရန်အတွက် Function တစ်ခု။
၄။ အလုပ်တစ်ခုကို ဖယ်ရှားရန်အတွက် Function တစ်ခု။

အစတွင် ကျွန်ုပ်တို့သည် အလုပ်များအား သိမ်းဆည်းရန်အတွက် **အလွတ် Array တစ်ခု** ကို စတင်ဖန်တီးရမည်။

```javascript
// Step 1: Initial Structure Setup
let todoList = []; // လုပ်ဆောင်ရန် အလုပ်များအတွက် Array ကို စတင်ဖန်တီးခြင်း
```

### 4.2 Implementing Core Functionality (Function Implementation)

ယခု ကျွန်ုပ်တို့သည် အခြေခံလုပ်ဆောင်ချက်များအတွက် Functions များကို ရေးသားကြပါမည်။

**A. Task ထည့်ရန် Function (`addTask`)**
ဤ Function သည် အသစ်သော အလုပ်တစ်ခုကို Array ထဲသို့ ထည့်ရန်အတွက် အသုံးပြုမည်။

```javascript
function addTask(taskDescription) {
    // အသစ်သော Task object ကို ဖန်တီးခြင်း (အနာဂတ်တွင် Object များဖြင့် ပိုကောင်းလာမည်)
    const newTask = {
        description: taskDescription,
        completed: false // အလုပ်သည် ပြီးစီးခြင်း ရှိ/မရှိကို မှတ်သားရန်
    };
    todoList.push(newTask);
    console.log(`အလုပ်အသစ် ထည့်သွင်းပြီးပြီ: ${taskDescription}`);
}
```

**B. Task များ ပြသရန် Function (`displayTasks`)**
ဤ Function သည် လက်ရှိရှိနေသော Task များအားလုံးကို Console တွင် ပြသရန် အသုံးပြုမည်။

```javascript
function displayTasks() {
    console.log("\n--- လက်ရှိ To-Do List ---");
    if (todoList.length === 0) {
        console.log("စာရင်းတွင် လုပ်ဆောင်ရန် အလုပ်မရှိသေးပါ။");
        return;
    }
    
    todoList.forEach((task, index) => {
        console.log(`${index + 1}. [${task.completed ? '✅' : '⏳'}] ${task.description}`);
    });
    console.log("----------------------\n");
}
```

**C. Task တစ်ခုကို ဖယ်ရှားရန် Function (`deleteTask`)**
ဤ Function သည် Index ကို အသုံးပြု၍ Array မှ အချက်အလက်ကို ဖယ်ရှားရန်အတွက် အသုံးပြုမည်။

```javascript
function deleteTask(taskIndex) {
    // Index မှ Task ကို ဖယ်ရှားရန်အတွက် filter() method ကို အသုံးပြု၍ အသစ်သော Array ကို ပြန်ရယူခြင်း
    const initialLength = todoList.length;
    
    // ဖယ်ရှားမည့် Index မဟုတ်သော Element များဖြင့် Array အသစ်ကို ဖန်တီးခြင်း
    todoList = todoList.filter((_, index) => index !== taskIndex);
    
    if (todoList.length < initialLength) {
        console.log(`Index ${taskIndex} ရှိသော အလုပ်ကို ဖယ်ရှားပြီးပါပြီ။`);
    } else {
        console.log("အမှား: ထို Index တွင် အလုပ်မရှိပါ။");
    }
}
```

### 4.3 Managing the Task List (Array Manipulation in Action)

ယခုအခါ ကျွန်ုပ်တို့သည် Functions များဖြင့် Array Methods များကို ပေါင်းစပ်အသုံးပြု၍ အလုပ်များကို စီမံခန့်ခွဲပုံကို လေ့လာကြပါမည်။

**အလုပ်များ ထည့်သွင်းခြင်း:**
အသုံးပြုသူမှ အလုပ်အသစ်တစ်ခုကို ထည့်လိုက်သောအခါ `addTask()` Function ကို ခေါ်ဆိုလိုက်ရမည်။

```javascript
addTask("စာအုပ်ဖတ်ရန်");
addTask("JavaScript သင်ခန်းစာ လေ့လာရန်");
addTask("ကျောင်းစာ ပြန်လည်ဖြေဆိုရန်");
displayTasks();
```

**အလုပ်များ ပြသခြင်း:**
အသုံးပြုသူသည် စာရင်းကို ကြည့်လိုသောအခါ `displayTasks()` Function ကို ခေါ်ဆိုရမည်။

**အလုပ်တစ်ခုကို ဖယ်ရှားခြင်း:**
အသုံးပြုသူက စာရင်းမှ အလုပ်တစ်ခုကို ဖယ်ရှားလိုပါက၊ ထိုအလုပ်၏ Index ကို သိရှိပြီး `deleteTask()` Function ကို ခေါ်ဆိုရမည်။

ဥပမာအားဖြင့်၊ ဒုတိယအလုပ် (Index 1) ကို ဖယ်ရှားလိုလျှင်:

```javascript
// Index 1 ရှိသော အလုပ်ကို ဖယ်ရှားရန်
deleteTask(1); 

displayTasks();
```

### 4.4 Full Logic Flow Review

ဤလုပ်ဆောင်ချက်များအားလုံးကို ပေါင်းစပ်ကြည့်လျှင်-

၁။ **စတင်ခြင်း:** `todoList = []` ဖြင့် Array ကို စတင်သည်။
၂။ **ထည့်သွင်းခြင်း:** `addTask()` Function ကို ခေါ်ဆိုပြီး Array ထဲသို့ Task အသစ်များကို `.push()` ဖြင့် ထည့်သွင်းသည်။
၃။ **ပြသခြင်း:** `displayTasks()` Function ကို ခေါ်ဆိုပြီး Array ထဲရှိ ဒေတာအားလုံးကို `.forEach()` ဖြင့် တစ်ခုချင်းစီ စစ်ဆေးကာ ပြသသည်။
၎။ **ဖယ်ရှားခြင်း:** `deleteTask(index)` Function ကို ခေါ်ဆိုပြီး `.filter()` Method ကို အသုံးပြုကာ ထို Index ရှိသော အချက်အလက်ကို Array မှ ဖယ်ရှားပြီး Array ကို ပြန်လည်ပြင်ဆင်သည်။

ဤနည်းလမ်းဖြင့် ကျွန်ုပ်တို့သည် Functions များက **လုပ်ဆောင်ချက် (Behavior)** ကို ပေးပြီး Arrays များက **ဒေတာ (Data)** ကို စုစည်းထိန်းသိမ်းပေးကာ၊ ၎င်းတို့နှစ်ခု ပေါင်းစပ်ခြင်းဖြင့် Dynamic ဖြစ်သော အက်ပလီကေးရှင်းတစ်ခုကို တည်ဆောက်နိုင်ခဲ့ပါသည်။

## 5. Chapter Summary နှင့် နောက်ဆက်တွဲ (Chapter Summary and Next Steps)

### 5.1 Key Takeaways (အဓိက အနှစ်ချုပ်)

ဤအခန်းတွင် ကျွန်ုပ်တို့သည် JavaScript ၏ အခြေခံကျသော အုတ်မြစ်နှစ်ခုကို အဓိကထား လေ့လာခဲ့ကြသည်-

*   **Functions (လုပ်ဆောင်ချက်များ):** ကုဒ်ကို ပြန်သုံးနိုင်စေရန် (Reusability) နှင့် ပိုင်းခြားနိုင်ရန် (Modularity) အဓိကဖြစ်သည်။ Function များသည် Parameters နှင့် Arguments မှ Input များ လက်ခံပြီး Return Value အဖြစ် Output များ ပြန်ပေးသည်။ Arrow Functions သည် ခေတ်မီပြီး တိုတောင်းသော ရေးသားပုံဖြစ်သည်။
*   **Arrays (အချက်အလက် စုစည်းမှုများ):** ဒေတာများကို အစီအစဉ်တကျ စုစည်းထားရန် အသုံးပြုသည်။ Indexing ဖြင့် တန်ဖိုးများကို ဝင်ရောက်ကြည့်ရှုနိုင်ပြီး `.push()`, `.pop()`, `.shift()`, `.unshift()` ကဲ့သို့သော Method များဖြင့် ဒေတာများကို လျင်မြန်စွာ စီမံခန့်ခွဲနိုင်သည်။ ထို့အပြင် `.map()` နှင့် `.filter()` ကဲ့သို့သော Method များဖြင့် ဒေတာများကို အလွန်ထိရောက်စွာ ပြောင်းလဲဖန်တီးနိုင်သည်။
*   **The Synergy:** Functions များသည် အလုပ်လုပ်ပုံ (How) ကို သတ်မှတ်ပေးပြီး Arrays များသည် ထိုအလုပ်များအတွက် လိုအပ်သော ဒေတာများကို သိမ်းဆည်းပေးသည်။ ဤနှစ်ခုကို ပေါင်းစပ်လိုက်သောအခါ ကျွန်ုပ်တို့သည် အချက်အလက်များကို အခြေခံ၍ တုံ့ပြန်နိုင်သော (Responsive) အက်ပလီကေးရှင်းများကို တည်ဆောက်နိုင်လာသည်။

### 5.2 Common Pitfalls (အဖြစ်များသော အမှားများ)

*   **Scope Issues:** Function များ၏ Scope (အကျယ်အဝန်း) ကို နားမလည်ပါက Variable များအား မှားယွင်းစွာ အသုံးပြုမိနိုင်သည်။ အထူးသဖြင့် `let`, `const` နှင့် `var` တို့၏ ကွာခြားချက်ကို သိထားရန် အရေးကြီးသည်။
*   **Array Mutation vs. Immutability:** `.filter()` သို့မဟုတ် `.map()` ကဲ့သို့သော Method များသည် **အသစ်သော Array တစ်ခုကို ပြန်ပေးသည်** (Immutable)။ Array ကို တိုက်ရိုက်ပြောင်းလဲရန် (`push`, `pop` ကဲ့သို့) နှင့် အသစ်သော Array ကို ဖန်တီးရန် ကွာခြားမှုကို နားလည်ထားရန် လိုအပ်သည်။

### 5.3 Next Steps (နောက်ဆက်တွဲ)

ဤအခန်းတွင် ကျွန်ုပ်တို့သည် အခြေခံများကို ခိုင်မာစွာ တည်ဆောက်ပြီးပြီ ဖြစ်သည်။ နောက်အခန်းများတွင် ကျွန်ုပ်တို့သည် ဤအခြေခံများကို အသုံးပြု၍ ပိုမိုရှုပ်ထွေးသော ဒေတာများကို ကိုင်တွယ်နိုင်မည့် **Objects (အရာဝတ္ထုများ)** ကို လေ့လာသွားမည် ဖြစ်သည်။ ထို့နောက်တွင် ထို Object များအား အသုံးပြု၍ အချက်အလက်များကို ပိုမိုရှုပ်ထွေးစွာ စုစည်းနိုင်မည့် နည်းလမ်းများကို လေ့လာကြမည် ဖြစ်သည်။ နောက်ဆုံးတွင် အသုံးပြုသူထံမှ Input များ လက်ခံခြင်းနှင့် DOM (HTML ဖွဲ့စည်းပုံ) နှင့် အပြန်အလှန်တုံ့ပြန်မှုများ ပြုလုပ်ခြင်းတို့ကို လေ့လာသွားမည် ဖြစ်ပါသည်။
