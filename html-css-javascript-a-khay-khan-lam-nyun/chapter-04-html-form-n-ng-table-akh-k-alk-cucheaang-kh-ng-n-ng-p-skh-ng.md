# Chapter 4: HTML Form နှင့် Table — အချက်အလက်စုဆောင်းခြင်းနှင့် ပြသခြင်း (HTML Forms and Tables — Collecting and Displaying Information)

ဒီနေ့ သင်ခန်းစာမှာ ကျွန်ုပ်တို့ဟာ Website တွေရဲ့ အရေးကြီးတဲ့ အစိတ်အပိုင်းနှစ်ခုဖြစ်တဲ့ **အချက်အလက်တွေကို စုဆောင်းပြီး ပြသတဲ့ နည်းလမ်း** (HTML Forms နှင့် Tables) ကို လေ့လာသွားကြမှာ ဖြစ်ပါတယ်။ Website တစ်ခုဟာ အချက်အလက်တွေ ထည့်သွင်းပြီး ပြန်ပြသတဲ့ ပုံစံမျိုး ဖြစ်ပါတယ်။ ဒီနေ့ သင်ယူရမှာက အဲဒီအချက်အလက်တွေကို ဘယ်လိုပုံစံနဲ့ စနစ်တကျ ဖန်တီးမလဲ ဆိုတာကို လက်တွေ့လုပ်ဆောင်ပြီး လေ့လာသွားမှာ ဖြစ်ပါတယ်။ အချက်အလက်တွေကို စနစ်တကျ စုဆောင်းနိုင်ပြီး အဲဒီအချက်အလက်တွေကို ဇယားနဲ့ ပုံစံတကျ ပြသနိုင်တဲ့ HTML စွမ်းရည်ကို ပိုင်ဆိုင်လာပါလိမ့်မယ်။

## 📚 Section 1: အချက်အလက်များကို စနစ်တကျ ပြသခြင်း (Displaying Information Systematically)

Website များတွင် အချက်အလက်များကို စနစ်တကျ စီစဉ်ပြီး ဖော်ပြရန်အတွက် HTML Table (ဇယား) သည် အလွန်အသုံးဝင်သော ကိရိယာတစ်ခု ဖြစ်ပါသည်။

### 1.1 HTML Table ၏ အခြေခံဖွဲ့စည်းပုံ (Basics of HTML Table Structure)

HTML Table ကို ဖန်တီးရာတွင် အဓိကအားဖြင့် လေးမျိုးသော Tag များကို အသုံးပြုရမည် ဖြစ်သည်။ ပထမဆုံးအနေဖြင့် ဇယားတစ်ခုလုံးကို စတင်ဖန်တီးရန်အတွက် `<table>` Tag ကို အသုံးပြုရပါသည်။ ဤ Tag သည် ဇယား၏ အပြင်ဘက် အ কাঠামো ကို သတ်မှတ်ပေးပါသည်။

ဇယားအတွင်းရှိ အချက်အလက်များကို စာကြောင်းများ (Rows) အဖြစ် ခွဲခြားရန်အတွက် `<tr>` (Table Row) Tag ကို အသုံးပြုပါသည်။ ဤ `<tr>` တစ်ခုချင်းစီသည် ဇယားရှိ စာကြောင်းတစ်ခုကို ကိုယ်စားပြုပါသည်။

အရေးအကြီးဆုံး အစိတ်အပိုင်းများမှာ ခေါင်းစဉ်များနှင့် အချက်အလက်ဆဲလ်များ ဖြစ်သည်။ ဇယား၏ ခေါင်းစဉ် (Header) များအတွက် `<th>` (Table Header) Tag ကို အသုံးပြုပြီး၊ အချက်အလက်များ (Data) အတွက် `<td` (Table Data) Tag များကို အသုံးပြုပါသည်။ `<th>` Tag များသည် စာကြောင်းများ၏ ခေါင်းစဉ်အဖြစ် အထူးသဖြင့် အရေးကြီးသော အချက်အလက်များကို ဖော်ပြရန် အသုံးပြုပါသည်။

ဥပမာအားဖြင့်၊ အောက်ပါအတိုင်း ဇယားတစ်ခုကို ဖန်တီးပုံကို ကြည့်ရှုနိုင်ပါသည်။

```html
<table>
  <tr>
    <th>နာမည်</th>
    <th>အမှတ်</th>
    <th>စာမေးပွဲရမှတ်</th>
  </tr>
  <tr>
    <td>မောင်မောင်</td>
    <td>၁၀</td>
    <td>၈၅</td>
  </tr>
  <tr>
    <td>မမမ</td>
    <td>၁၂</td>
    <td>၉၂</td>
  </tr>
</table>
```

ဤဥပမာတွင်၊ ပထမဆုံး `<table>` သည် ဇယားတစ်ခုလုံးကို စတင်သည်။ ထို့နောက် `<tr>` များဖြင့် စာကြောင်းများကို ခွဲခြားသည်။ ပထမဆုံး `<tr>` တွင် ခေါင်းစဉ်များဖြစ်သော "နာမည်"၊ "အမှတ်" နှင့် "စာမေးပွဲရမှတ်" တို့ကို `<th>` အဖြစ် သတ်မှတ်သည်။ ထို့နောက် ဒုတိယနှင့် တတိယ `<tr>` များတွင် အချက်အလက်များကို `<td` အဖြစ် ထည့်သွင်းဖော်ပြထားပါသည်။

### 1.2 ဇယားကို ပိုမိုလှပအောင် စီစဉ်ခြင်း (Organizing the Table)

ဇယားကို ဖန်တီးပြီးနောက် ၎င်းကို ပိုမိုကောင်းမွန်စွာ စီစဉ်ရန် လိုအပ်ပါသည်။ ဇယား၏ ခေါင်းစဉ်များ (Header) ကို သတ်မှတ်ခြင်းသည် အလွန်အရေးကြီးပါသည်။ ဤခေါင်းစဉ်များသည် ဇယားအတွင်းရှိ အချက်အလက်များ၏ အဓိပ္ပာယ်ကို ဖော်ပြပေးရမည် ဖြစ်သည်။

ဇယား၏ ခေါင်းစဉ်များကို ဖော်ပြရာတွင် HTML တွင် `<th>` Tag ကို အသုံးပြုရုံမက၊ ၎င်းတို့ကို ပုံစံတကျ ဖော်ပြရန်အတွက် CSS (Cascading Style Sheets) နှင့် ချိတ်ဆက်အသုံးပြုရန် အထူးလိုအပ်ပါသည်။ CSS ကို အသုံးပြုခြင်းဖြင့် ဇယား၏ ခေါင်းစဉ်များကို စာလုံးအရွယ်အစား၊ အရောင်၊ စာတန်း (background) စသည်တို့ကို လိုသလို ပြင်ဆင်နိုင်ပြီး အချက်အလက်များကို ပိုမိုဆွဲဆောင်မှုရှိစွာ ဖော်ပြနိုင်ပါသည်။

ဇယားအတွင်းရှိ အချက်အလက်များကို ထည့်သွင်းရာတွင်လည်း စနစ်တကျ ထည့်သွင်းရန် လိုအပ်ပါသည်။ တစ်ကြောင်းချင်းစီတွင် အချက်အလက်များကို `<td` Tag များဖြင့် ခွဲခြား၍ ထည့်သွင်းရမည် ဖြစ်သည်။ ဤနည်းလမ်းဖြင့် အချက်အလက်များသည် စာကြောင်းနှင့် ကော်လံများအလိုက် မှန်ကန်စွာ စီစဉ်နေမည် ဖြစ်သည်။

### 1.3 လက်တွေ့လေ့ကျင့်ခန်း (Practical Exercise)

ယခု သင်ယူခဲ့သော အခြေခံများကို လက်တွေ့အသုံးချကြည့်ကြရအောင်။ ကျောင်းသားတစ်ဦး၏ အချက်အလက်များကို ဇယားပုံစံဖြင့် ဖန်တီးကြည့်ရအောင်။

**လေ့ကျင့်ခန်း:** ကျောင်းသားအမည်၊ အမှတ်၊ စာမေးပွဲရမှတ် စသည့် အချက်အလက်များကို Table ပုံစံဖြင့် ဖန်တီးခြင်း။

ဤလေ့ကျင့်ခန်းအတွက် အောက်ပါ HTML ကုဒ်ကို အသုံးပြုနိုင်ပါသည်။

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Student Data Table</title>
</head>
<body>
    <h1>ကျောင်းသားအချက်အလက်များ</h1>
    <table>
      <thead>
        <tr>
          <th>နာမည်</th>
          <th>အမှတ်</th>
          <th>စာမေးပွဲရမှတ်</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td>မောင်မောင်</td>
          <td>၁၀</td>
          <td>၈၅</td>
        </tr>
        <tr>
          <td>မမမ</td>
          <td>၁၂</td>
          <td>၉၂</td>
        </tr>
      </tbody>
    </table>
</body>
</html>
```

ဤကုဒ်ကို Browser တွင် ဖွင့်ကြည့်လိုက်ပါက သင်သည် အချက်အလက်များကို စနစ်တကျ စုစည်းထားသော ဇယားတစ်ခုကို မြင်တွေ့ရမည် ဖြစ်ပါသည်။ ဤသည်မှာ HTML Table ကို အသုံးပြု၍ အချက်အလက်များကို ပြသနိုင်ပုံကို ပြသနေပါသည်။

အချက်အလက်တွေကို ဇယားပုံစံနဲ့ စုစည်းပြသနိုင်ပြီဆိုတော့၊ နောက်တစ်ဆင့်အနေနဲ့ အသုံးပြုသူတွေဆီက အချက်အလက်တွေကို လက်ခံဖို့ ဘယ်လိုလုပ်မလဲဆိုတာကို လေ့လာရပါမယ်။ အဲဒါကတော့ HTML Form တွေနဲ့ ဆက်စပ်နေပါတယ်။

## 📚 Section 2: အချက်အလက်စုဆောင်းခြင်းအတွက် HTML Form (HTML Forms for Data Collection)

အချက်အလက်များကို ပြသနိုင်ပြီဆိုတော့၊ နောက်တစ်ဆင့်အနေဖြင့် အသုံးပြုသူများထံမှ အချက်အလက်များကို လက်ခံမည့် နည်းလမ်းကို လေ့လာရပါမည်။ ထိုနည်းလမ်းမှာ HTML Form များ ဖြစ်သည်။

### 2.1 HTML Form ၏ အခြေခံ (Basics of HTML Form)

HTML Form သည် အသုံးပြုသူများထံမှ အချက်အလက်များကို လက်ခံရန်အတွက် အသုံးပြုသော အထူး Tag တစ်ခု ဖြစ်သည်။ ဤဖောင်တစ်ခုကို စတင်ဖန်တီးရန်အတွက် `<form>` Tag ကို အသုံးပြုရပါသည်။ ဖောင်တစ်ခုကို စတင်ဖန်တီးသောအခါ ၎င်းနှင့်အတူ အရေးကြီးသော Attribute များ ထည့်သွင်းပေးရမည်ဖြစ်ပြီး ၎င်းတို့သည် ဖောင်၏ အလုပ်လုပ်ပုံနှင့် ပတ်သက်၍ အရေးကြီးပါသည်။

ဖောင်တစ်ခု၏ အဓိက အလုပ်လုပ်ပုံမှာ အသုံးပြုသူများ ထည့်သွင်းသော အချက်အလက်များကို စုစည်းပြီး Server သို့မဟုတ် နောက်ထပ် Processing လုပ်ဆောင်မှုတစ်ခုသို့ ပေးပို့ခြင်း (Submission Process) ဖြစ်သည်။ ဤလုပ်ငန်းစဉ်သည် HTML တစ်ခုတည်းဖြင့် မပြီးနိုင်ဘဲ နောက်ပိုင်းတွင် JavaScript သို့မဟုတ် Server-side scripting များဖြင့် ဆက်စပ်၍ အလုပ်လုပ်ကြသည်။

### 2.2 အခြေခံ Input အမျိုးအစားများ (Fundamental Input Types)

Form များတွင် အသုံးပြုသူထံမှ မည်သည့်အချက်အလက်မျိုးကို လိုချင်သလဲဆိုသည့်အပေါ် မူတည်၍ ကွဲပြားသော Input အမျိုးအစားများကို အသုံးပြုရမည် ဖြစ်သည်။ အခြေခံကျသော Input အမျိုးအစားများမှာ အောက်ပါအတိုင်း ဖြစ်သည်။

စာသားများ လက်ခံရန်အတွက် `<input type="text">` ကို အသုံးပြုသည်။ ၎င်းသည် အသုံးပြုသူများက စာသားများကို ရိုက်ထည့်နိုင်သည့် Box တစ်ခုကို ဖန်တီးပေးသည်။

အီးမေးလ်လိပ်စာများ စုဆောင်းရန်အတွက် `<input type="email">` ကို အသုံးပြုသည်။ ဤ Input အမျိုးအစားသည် အီးမေးလ်ပုံစံ မှန်ကန်မှု ရှိမရှိကို Browser မှ အနည်းငယ် စစ်ဆေးပေးနိုင်သော အထူးအကျိုးကျေးဇူးရှိသည်။

လျှို့ဝှက်စာသားများ (Password) ကဲ့သို့သော လုံခြုံမှုလိုအပ်သော အချက်အလက်များ ထည့်သွင်းရန်အတွက် `<input type="password">` ကို အသုံးပြုသည်။ ဤ Input သည် ရိုက်ထည့်သည့် စာသားများကို လျှို့ဝှက်သော အမှတ်အသားများ (ဥပမာ- \*\*\*\*) ဖြင့် ဖုံးကွယ်ပေးထားသည်။

ဂဏန်းများ သို့မဟုတ် ကိန်းဂဏန်းများသာ ထည့်သွင်းရန် လိုအပ်သော အချက်အလက်များအတွက် `<input type="number">` ကို အသုံးပြုသည်။ ဤ Input သည် ဂဏန်းများကိုသာ လက်ခံစေပြီး စာသားများ ထည့်သွင်းခြင်းကို ကန့်သတ်ထားသည်။

### 2.3 ရွေးချယ်စရာများနှင့် အထူး Input များ (Selections and Special Inputs)

အသုံးပြုသူများအား ရွေးချယ်စရာများ ပေးရန်အတွက်လည်း HTML Form များတွင် အထူး Input အမျိုးအစားများကို အသုံးပြုကြသည်။

ရွေးချယ်စရာများ အများအပြား ရွေးနိုင်ရန်အတွက် `<input type="checkbox">` ကို အသုံးပြုသည်။ ဤ Checkbox တစ်ခုစီသည် ကိုယ်ပိုင် ရွေးချယ်စရာတစ်ခုကို ကိုယ်စားပြုပြီး အသုံးပြုသူများသည် လိုအပ်သလို ရွေးချယ်နိုင်ပါသည်။

ရွေးချယ်စရာများ တစ်ခုတည်းကိုသာ ရွေးချယ်ရန်အတွက် `<input type="radio">` ကို အသုံးပြုသည်။ Radio Button များသည် အုပ်စုလိုက်ရှိသော ရွေးချယ်စရာများတွင် တစ်ခုတည်းကိုသာ ရွေးချယ်နိုင်စေရန် အသုံးပြုသည်။ ဤ Radio Button များသည် အတူတကွ ဆက်စပ်နေသော ရွေးချယ်စရာများကို ဖော်ပြရန် အထူးသင့်လျော်သည်။

ဖောင်ကို တင်သွင်းရန်အတွက် အသုံးပြုသူများအတွက် ခလုတ်တစ်ခု ပေးရန်အတွက် `<input type="submit">` ကို အသုံးပြုသည်။ ဤ Submit ခလုတ်ကို နှိပ်လိုက်သောအခါ Form အတွင်းရှိ အချက်အလက်အားလုံးကို Server သို့ ပေးပို့ရန် အချက်ပေးသည်။

### 2.4 အချက်အလက်ကို ပိုမိုကောင်းမွန်အောင် ထိန်းချုပ်ခြင်း (Controlling Input with Labels and Attributes)

Input fields များကို အသုံးပြုသောအခါ ၎င်းတို့၏ အသုံးပြုရလွယ်ကူမှုနှင့် မှန်ကန်မှုကို မြှင့်တင်ရန်အတွက် အခြား HTML Element များကို အသုံးပြုရသည်။

`<label>` Tag ကို အသုံးပြုခြင်းဖြင့် Input Field နှင့် ၎င်းနှင့် သက်ဆိုင်သော စာသား (Label) တို့ကို မှန်ကန်စွာ ချိတ်ဆက်ပေးရမည်။ ဤချိတ်ဆက်မှုသည် အသုံးပြုသူများအနေဖြင့် မည်သည့် Input Field ကို မည်သည့် အချက်အလက်အတွက် ဖြည့်ရမည်ကို ရှင်းလင်းစွာ နားလည်စေသည်။

`<label>` နှင့် Input ကို ချိတ်ဆက်ရာတွင် `for` Attribute ကို အသုံးပြုပြီး `<label>` ၏ `for` Attribute သည် Input ၏ `id` Attribute နှင့် တူညီရမည်။ ဤနည်းလမ်းသည် Accessibility (လက်လှမ်းမီနိုင်မှု) ကို မြှင့်တင်ပေးသည်။

Input Box အတွင်းတွင် အသုံးပြုသူများအတွက် ညွှန်ကြားချက်ပေးရန်အတွက် `placeholder` Attribute ကို အသုံးပြုနိုင်သည်။ ၎င်းသည် Input Box အတွင်း၌ လျှို့ဝှက်ထားသော ညွှန်ကြားချက်တစ်ခုကို ပြသပေးသည်။

အချက်အလက်များ မဖြစ်မနေ ဖြည့်ရမည့် အချက်များအတွက် `required` Attribute ကို အသုံးပြုနိုင်သည်။ ဤ Attribute ကို Input Field များတွင် ထည့်သွင်းခြင်းဖြင့် အသုံးပြုသူသည် ထို Input Field ကို မဖြည့်ဘဲ Form ကို တင်သွင်း၍မရအောင် ထိန်းချုပ်နိုင်ပါသည်။

## 📚 Section 3: ရှုပ်ထွေးသော အချက်အလက်များ ထည့်သွင်းခြင်း (Handling Complex Data Entry)

အခြေခံ Input အမျိုးအစားများဖြင့် အချက်အလက်များ စုဆောင်းနိုင်ပြီဆိုသော်လည်း၊ အကြောင်းအရာများ အများကြီး ထည့်သွင်းရန် သို့မဟုတ် ရွေးချယ်စရာများ ပိုမိုရှုပ်ထွေးစွာ ဖန်တီးရန်အတွက် အခြား Tag များ လိုအပ်ပါသည်။

### 3.1 စာသားအများကြီး ထည့်သွင်းခြင်း (Handling Large Text Entry)

အသုံးပြုသူများထံမှ အကြောင်းအရာများ၊ မှတ်ချက်များ သို့မဟုတ် ရှည်လျားသော စာသားများကို ထည့်သွင်းရန်အတွက် `<textarea>` Tag ကို အသုံးပြုရသည်။ ဤ Tag သည် Input Box များထက် ပိုမိုကျယ်ဝန်းသော စာသားများကို ထည့်သွင်းနိုင်သော နေရာကို ဖန်တီးပေးသည်။

`<textarea>` ကို အသုံးပြုသောအခါ ၎င်း၏ အမြင့် (rows) နှင့် အနံ (cols) ကို သတ်မှတ်ပေးရမည် ဖြစ်သည်။ ဤ Attribute များသည် စာသားများ ထည့်သွင်းနိုင်မည့် နေရာ၏ အရွယ်အစားကို ထိန်းချုပ်ပေးသည်။

### 3.2 ရွေးချယ်စရာများ ဖန်တီးခြင်း (Creating Selection Options)

အသုံးပြုသူများအတွက် ရွေးချယ်စရာများ (Dropdown List) ပုံစံဖြင့် ရွေးချယ်စေလိုသောအခါ `<select>` Tag ကို အသုံးပြုရသည်။ ဤ Tag သည် Dropdown List တစ်ခုကို ဖန်တီးပေးသည်။

`<select>` Tag အတွင်း၌ ရွေးချယ်စရာများကို သတ်မှတ်ရန်အတွက် `<option>` Tag များကို အသုံးပြုရသည်။ `<option>` Tag များသည် Dropdown List ထဲရှိ တစ်ခုချင်းစီ၏ ရွေးချယ်စရာများကို ကိုယ်စားပြုသည်။ ဤနည်းလမ်းဖြင့် အသုံးပြုသူများသည် စာရင်းထဲမှ လိုအပ်သော ရွေးချယ်စရာကို ရွေးချယ်နိုင်မည် ဖြစ်သည်။

`<select>` နှင့် `<option>` ကို အသုံးပြုခြင်း၏ အကျိုးကျေးဇူးမှာ ၎င်းသည် အချက်အလက်များကို စနစ်တကျ စုစည်းထားပြီး အသုံးပြုရလွယ်ကူသော Dropdown List တစ်ခုကို ဖန်တီးပေးနိုင်ခြင်း ဖြစ်သည်။

### 3.3 Form တစ်ခုလုံးကို စုစည်းခြင်း (Assembling the Complete Form)

အထက်ပါ အချက်အလက်များ (Input အမျိုးအစားများ၊ Label များ၊ Textarea နှင့် Select များ) အားလုံးကို မှန်ကန်စွာ စုစည်းပြီး တစ်ခုတည်းသော Form တစ်ခုအဖြစ် ဖန်တီးရမည် ဖြစ်သည်။ ဤအရာအားလုံးကို HTML ၏ `<body>` အတွင်း၌ ထည့်သွင်းရမည် ဖြစ်ပြီး၊ Form တစ်ခုလုံးသည် မှန်ကန်စွာ အလုပ်လုပ်ရန်အတွက် အထူးသဖြင့် `<form>` Tag ကို အသုံးပြု၍ စတင်ဖန်တီးရမည် ဖြစ်သည်။

ဤအဆင့်တွင် အသုံးပြုသူများထံမှ အချက်အလက်များကို စုဆောင်းရန် လိုအပ်သော အားလုံးသော အစိတ်အပိုင်းများကို မှန်ကန်စွာ ပေါင်းစပ်ထည့်သွင်းရမည် ဖြစ်သည်။

## 🚀 Section 4: လက်တွေ့အသုံးချခြင်း - မှတ်ပုံတင်ဖောင် ဖန်တီးခြင်း (Practical Application: Creating a Registration Form)

ယခုအချိန်အထိ ကျွန်ုပ်တို့သည် အချက်အလက်များကို ပြသခြင်း (Table) နှင့် စုဆောင်းခြင်း (Form) တို့ကို HTML ဖြင့် မည်သို့ပြုလုပ်ရမည်ကို လေ့လာခဲ့ပြီး ဖြစ်သည်။ ယခုအခါတွင် ဤအသိပညာများကို ပေါင်းစပ်၍ လက်တွေ့ကျသော မှတ်ပုံတင်ဖောင်တစ်ခုကို ဖန်တီးကြည့်ကြရအောင်။

### 4.1 ပရောဂျက် အစီအစဉ် (Project Outline)

ကျွန်ုပ်တို့၏ ပရောဂျက်သည် အသုံးပြုသူတစ်ဦး၏ မှတ်ပုံတင်အချက်အလက်များ (ဥပမာ- အမည်၊ အီးမေးလ်၊ မွေးသက္ကရာဇ်၊ လိင်၊ စိတ်အခြေအနေ စသည်) ကို စုဆောင်းသည့် ဖောင်တစ်ခုကို ဖန်တီးရန် ဖြစ်သည်။

အစတွင် လိုအပ်သော အချက်အလက်များကို ဆုံးဖြတ်ရမည်။ ထို့နောက် ထိုအချက်အလက်များအတွက် မည်သည့် Input အမျိုးအစား (ဥပမာ- စာသားအတွက် text၊ အီးမေးလ်အတွက် email၊ လိင်အတွက် radio၊ နှစ်ခုမရွေးဘဲ ရွေးရန်အတွက် checkbox) ကို အသုံးပြုမည်ကို စီစဉ်ရမည်။ ဤအစီအစဉ်သည် ဖောင်၏ ရည်ရွယ်ချက်နှင့် ကိုက်ညီမှုရှိစေရန် အရေးကြီးသည်။

### 4.2 Code ရေးသားခြင်း (Writing the Code)

ယခု အစီအစဉ်အတိုင်း HTML ကုဒ်များကို စတင်ရေးသားရမည် ဖြစ်သည်။ ပထမဆုံးအနေဖြင့် Form တစ်ခုလုံးကို စတင်ဖန်တီးပြီး၊ ထိုဖောင်အတွင်း၌ လိုအပ်သော Label များ၊ Input fields များ၊ Radio buttons၊ Checkboxes နှင့် Textarea များကို မှန်ကန်စွာ ထည့်သွင်းရမည် ဖြစ်သည်။ ဤအချက်အလက်များအားလုံးကို HTML ၏ `<body>` အတွင်း၌ စနစ်တကျ ထည့်သွင်းရမည် ဖြစ်သည်။

ဥပမာအားဖြင့်၊ အောက်ပါအတိုင်း မှတ်ပုံတင်ဖောင်၏ အခြေခံကို ရေးသားပုံကို ကြည့်ရှုနိုင်ပါသည်။

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Registration Form</title>
</head>
<body>
    <h1>အသုံးပြုသူ မှတ်ပုံတင်ခြင်း</h1>
    <form action="/submit_form" method="post">
      
      <!-- အမည် ထည့်သွင်းရန် -->
      <label for="name">အမည်:</label>

      <!-- အီးမေးလ် ထည့်သွင်းရန် -->

      <!-- လိင် ရွေးချယ်ရန် (Radio Buttons) -->

ဤကုဒ်တွင် ကျွန်ုပ်တို့သည် အမည်နှင့် အီးမေးလ်အတွက် text နှင့် email input များကို အသုံးပြုထားသည်။ လိင်အတွက် radio buttons များကို အသုံးပြုထားပြီး၊ စိတ်အခြေအနေအတွက် checkbox များကို အသုံးပြုထားသည်။ ထို့အပြင် အခြားမှတ်ချက်များ ထည့်သွင်းရန်အတွက် textarea ကို အသုံးပြုထားသည်။ `required` attribute ကို အသုံးပြု၍ အမည်နှင့် အီးမေးလ်ကို မဖြစ်မနေ ဖြည့်ရန် သတ်မှတ်ထားသည်။ အဆုံးတွင် `<input type="submit">` ခလုတ်ကို ထည့်သွင်းထားသည်။

### 4.3 စစ်ဆေးခြင်းနှင့် ပြင်ဆင်ခြင်း (Review and Refinement)

ဖန်တီးထားသော Form တစ်ခုလုံး မှန်ကန်စွာ အလုပ်လုပ်ပုံကို စမ်းသပ်ကြည့်ရမည် ဖြစ်သည်။ ဤကုဒ်ကို Browser တွင် ဖွင့်ကြည့်လိုက်ပါက Input fields များသည် မှန်ကန်စွာ ပေါ်လာပြီး၊ Radio buttons နှင့် Checkboxes များသည် မှန်ကန်စွာ အလုပ်လုပ်ပုံကို မြင်တွေ့ရမည် ဖြစ်သည်။

အထူးသဖြင့် `required` attribute များသည် အသုံးပြုသူက မည်သည့် Input Field ကိုမဆို ဖြည့်ခြင်းမရှိဘဲ Submit ခလုတ်ကို နှိပ်ပါက Browser မှ မှားယွင်းသော မက်ဆေ့ချ်ကို ပြသမည်ကို စမ်းသပ်ကြည့်ရမည် ဖြစ်သည်။ ဤအဆင့်သည် အသုံးပြုသူများထံမှ မှန်ကန်သော အချက်အလက်များ စုဆောင်းနိုင်ရန် အရေးကြီးပါသည်။

### 4.4 အခန်း၏ အနှစ်ချုပ် (Chapter Summary)

ဤအခန်းတွင် ကျွန်ုပ်တို့သည် HTML Table နှင့် HTML Form များ၏ အဓိက ကွာခြားချက်နှင့် အသုံးဝင်ပုံများကို လေ့လာခဲ့ကြသည်။ HTML Table များသည် အချက်အလက်များကို **ပြသရန်** (Display) အကောင်းဆုံးဖြစ်ပြီး၊ HTML Form များသည် အချက်အလက်များကို **စုဆောင်းရန်** (Collect) အကောင်းဆုံးဖြစ်သည်။ Table များသည် စာရင်းများကို စနစ်တကျ စီစဉ်ရန်နှင့် ပြသရန်အတွက် အသုံးဝင်ပြီး၊ Form များသည် အသုံးပြုသူများထံမှ အချက်အလက်များကို လက်ခံရန်အတွက် အဓိကအသုံးပြုပါသည်။

**Next Steps:** ယခုအချိန်တွင် သင်ဟာ အချက်အလက်များကို Website ပေါ်မှာ ကိုယ်တိုင် ထိန်းချုပ်နိုင်တဲ့ စွမ်းရည်ကို ရရှိသွားပြီ ဖြစ်သည်။ နောက်ထပ် လေ့လာမှုများတွင် ဤအခြေခံများကို အခြေခံပြီး ပိုမိုရှုပ်ထွေးသော ဒီဇိုင်းနှင့် အပြန်အလှန် ဆက်သွယ်မှုများအတွက် CSS နှင့် JavaScript တို့၏ အရေးပါပုံကို လေ့လာသွားကြပါမည်။

**Final Thought:** သင်ဟာ အချက်အလက်တွေကို Website ပေါ်မှာ ကိုယ်တိုင် ထိန်းချုပ်နိုင်တဲ့ စွမ်းရည်ကို ရရှိသွားပြီ။ နောက်ထပ် လေ့လာမှုတွေမှာ ဒီအခြေခံတွေကို အခြေခံပြီး ပိုမိုရှုပ်ထွေးတဲ့ Web Application တွေကို တည်ဆောက်နိုင်မှာ ဖြစ်ပါတယ်။
